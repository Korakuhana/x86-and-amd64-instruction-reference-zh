BNDLDX — 使用地址转换加载扩展边界

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
NP 0F 1A /r BNDLDX bnd, mib RM V/V MPX 使用 mib 的基址进行地址转换，将存储在边界表项（BTE）中的边界加载到 bnd 中，前提是 mib 的索引与 BTE 中的指针值匹配。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3
RM ModRM:reg (w) SIB.base (r): 指针地址 SIB.index(r) N/A

描述¶

BNDLDX 使用由内存操作数（mib）的 SIB 寻址形式的基址寄存器和偏移量构造的线性地址来执行地址转换，以访问边界表项，并在条件满足时将 BTE 中的边界加载到目的地。如果 mib 的索引寄存器的内容与存储在 BTE 中的指针值匹配，则用 BTE 中的边界更新目标寄存器。

如果指针值比较失败，则用 INIT 边界（lb = 0x0, ub = 0x0）更新目的地（注：如前所述，上界使用反码表示，因此上界的 0x0 值允许访问整个内存）。

此指令不会导致对 mib 线性地址或基址引用的有效地址进行内存访问，也不会读取或写入任何标志位。

段超越适用于使用 mib 基址的线性地址计算，并在地址转换期间用于生成边界表项的地址。默认情况下，BTE 的地址假定为线性地址。不会对 mib 的基址执行任何段检查。

由于不访问内存，因此不会检查 mib 的基址是否违反规范地址。

任何未指定基址或索引寄存器的此指令编码都将这些寄存器视为零（常量）。此指令的寄存器-寄存器（reg-reg）形式仍为 NOP。

SIB 字节的比例（scale）字段对这些指令没有影响，并被忽略。

在发生内存故障时，边界寄存器可能会被部分更新。内存操作数的加载顺序取决于具体实现。

操作¶

base := mib.SIB.base ? mib.SIB.base + Disp: 0;
ptr_value := mib.SIB.index ? mib.SIB.index : 0;

Outside 64-bit Mode¶

A_BDE[31:0] := (Zero_extend32(base[31:12] << 2) + (BNDCFG[31:12] << 12 );
A_BT[31:0] := LoadFrom(A_BDE );
IF A_BT[0] equal 0 Then
    BNDSTATUS := A_BDE | 02H;
    #BR;
FI;
A_BTE[31:0] := (Zero_extend32(base[11:2] << 4) + (A_BT[31:2] << 2 );
Temp_lb[31:0] := LoadFrom(A_BTE);
Temp_ub[31:0] := LoadFrom(A_BTE + 4);
Temp_ptr[31:0] := LoadFrom(A_BTE + 8);
IF Temp_ptr equal ptr_value Then
    BND.LB := Temp_lb;
    BND.UB := Temp_ub;
ELSE
    BND.LB := 0;
    BND.UB := 0;
FI;

In 64-bit Mode¶

A_BDE[63:0] := (Zero_extend64(base[47+MAWA:20] << 3) + (BNDCFG[63:12] << 12 );1
A_BT[63:0] := LoadFrom(A_BDE);
IF A_BT[0] equal 0 Then
    BNDSTATUS := A_BDE | 02H;
    #BR;
FI;
A_BTE[63:0] := (Zero_extend64(base[19:3] << 5) + (A_BT[63:3] << 3 );
Temp_lb[63:0] := LoadFrom(A_BTE);
Temp_ub[63:0] := LoadFrom(A_BTE + 8);
Temp_ptr[63:0] := LoadFrom(A_BTE + 16);
IF Temp_ptr equal ptr_value Then
    BND.LB := Temp_lb;
    BND.UB := Temp_ub;
ELSE
    BND.LB := 0;
    BND.UB := 0;
FI;

Intel C/C++ 编译器等价内部函数¶

BNDLDX: Generated by compiler as needed.

受影响的标志位¶

无。

保护模式异常¶

#BR 如果边界目录项无效。
#UD 如果使用了 LOCK 前缀。
如果启用 Intel MPX 时 ModRM.r/m 编码为 BND4-BND7。
如果未使用 67H 前缀且 CS.D=0。
如果使用了 67H 前缀且 CS.D=1。
#GP(0) 如果边界表项的目标有效地址超出 DS 段限制。
如果 DS 寄存器包含空段选择子。
#PF(错误码) 如果发生缺页异常。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。
如果启用 Intel MPX 时 ModRM.r/m 编码为 BND4-BND7。
如果使用了 16 位寻址。
#GP(0) 如果边界表项的目标有效地址超出 DS 段限制。

1. 如果 CPL < 3，则使用管理员 MAWA (MAWAS)；该值为 0。如果 CPL = 3，则使用用户 MAWA (MAWAU)；该值在 CPUID.(EAX=07H,ECX=0H):ECX.MAWAU[bits 21:17] 中枚举。请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷的附录 E.3.1。

虚拟-8086 模式异常¶

#UD 如果使用了 LOCK 前缀。
如果启用 Intel MPX 时 ModRM.r/m 编码为 BND4-BND7。
如果使用了 16 位寻址。
#GP(0) 如果边界表项的目标有效地址超出 DS 段限制。
#PF(错误码) 如果发生缺页异常。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#BR 如果边界目录项无效。
#UD 如果 ModRM 是 RIP 相对的。
如果使用了 LOCK 前缀。
如果启用 Intel MPX 时 ModRM.r/m 和 REX 编码为 BND4-BND15。
#GP(0) 如果内存地址（A_BDE 或 A_BTE）处于非规范形式。
#PF(错误码) 如果发生缺页异常。
