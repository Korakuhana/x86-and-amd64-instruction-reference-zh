DIV — 无符号除法

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
F6 /6 DIV r/m8 M Valid Valid 无符号除法，用 AX 除以 r/m8，结果存储在 AL := 商，AH := 余数。
REX + F6 /6 DIV r/m81 M Valid N.E. 无符号除法，用 AX 除以 r/m8，结果存储在 AL := 商，AH := 余数。
F7 /6 DIV r/m16 M Valid Valid 无符号除法，用 DX:AX 除以 r/m16，结果存储在 AX := 商，DX := 余数。
F7 /6 DIV r/m32 M Valid Valid 无符号除法，用 EDX:EAX 除以 r/m32，结果存储在 EAX := 商，EDX := 余数。
REX.W + F7 /6 DIV r/m64 M Valid N.E. 无符号除法，用 RDX:RAX 除以 r/m64，结果存储在 RAX := 商，RDX := 余数。

1. 在 64 位模式下，如果使用 REX 前缀，则不能将 r/m8 编码为访问以下字节寄存器：AH、BH、CH、DH。

指令操作数编码 ¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (w) N/A N/A N/A

描述 ¶

对 AX、DX:AX、EDX:EAX 或 RDX:RAX 寄存器中的无符号值（被除数）除以源操作数（除数），并将结果存储在 AX (AH:AL)、DX:AX、EDX:EAX 或 RDX:RAX 寄存器中。源操作数可以是通用寄存器或内存地址。此指令的行为取决于操作数大小（被除数/除数）。使用 64 位操作数的除法仅在 64 位模式下可用。

非整数结果向 0 截断。余数的绝对值总是小于除数的绝对值。溢出通过 #DE（除法错误）异常来指示，而不是通过 CF 标志。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 前缀可将操作提升至 64 位。在 64 位模式下，当应用 REX.W 时，该指令用 RDX:RAX 中的无符号值除以源操作数，并将商存储在 RAX 中，余数存储在 RDX 中。

请参阅本节开头的汇总图表以获取编码数据和限制。请参见表 3-15。

操作 ¶ 

IF SRC = 0
    THEN #DE; FI; (* Divide Error *)
IF OperandSize = 8 (* Word/Byte Operation *)
    THEN
        temp := AX / SRC;
        IF temp > FFH
            THEN #DE; (* Divide error *)
            ELSE
                AL := temp;
                AH := AX MOD SRC;
        FI;
    ELSE IF OperandSize = 16 (* Doubleword/word operation *)
        THEN
            temp := DX:AX / SRC;
            IF temp > FFFFH
                THEN #DE; (* Divide error *)
            ELSE
                AX := temp;
                DX := DX:AX MOD SRC;
            FI;
        FI;
    ELSE IF Operandsize = 32 (* Quadword/doubleword operation *)
        THEN
            temp := EDX:EAX / SRC;
            IF temp > FFFFFFFFH
                THEN #DE; (* Divide error *)
            ELSE
                EAX := temp;
                EDX := EDX:EAX MOD SRC;
            FI;
        FI;
    ELSE IF 64-Bit Mode and Operandsize = 64 (* Doublequadword/quadword operation *)
        THEN
            temp := RDX:RAX / SRC;
            IF temp > FFFFFFFFFFFFFFFFH
                THEN #DE; (* Divide error *)
            ELSE
                RAX := temp;
                RDX := RDX:RAX MOD SRC;
            FI;
        FI;
FI;

受影响的标志位 ¶

CF、OF、SF、ZF、AF 和 PF 标志位未定义。

保护模式异常 ¶

#DE 如果源操作数（除数）为 0。
如果商过大，无法存入指定寄存器。
#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
