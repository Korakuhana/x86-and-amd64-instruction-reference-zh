描述¶

该指令有条件地预取最多十六个 32 位或八个 64 位整型字节数据元素。这些元素是通过 VSIB 指定的（即索引寄存器是一个包含打包索引的 zmm 寄存器）。只有当元素的相应掩码位为 1 时，才会预取该元素。

预取的缓存行被加载到由局部性暗示 (T0) 指定的缓存层次结构中的位置：

  • T0（时间数据）——将数据预取到一级缓存中。

[PS 数据] 对于 dword 索引，指令将预取十六个内存位置。对于 qword 索引，指令将预取八个值。

[PD 数据] 对于 dword 和 qword 索引，指令将预取八个内存位置。

请注意：

(1) 预取可以按任何顺序发生（或者根本不发生）。该指令是一个暗示。

(2) 掩码保持不变。

(3) 对于 16 位有效地址无效。将导致 #UD 故障。

(4) 此指令不会产生任何浮点或内存故障。

(5) 预取不处理缓存行跨越（splits）。

(6) 如果内存操作数的编码中没有 SIB 字节，将发出 #UD 信号。

操作¶

BASE_ADDR stands for the memory operand base address (a GPR); may not exist.
VINDEX stands for the memory operand vector of indices (a vector register).
SCALE stands for the memory operand scalar (1, 2, 4 or 8).
DISP is the optional 1, 2 or 4 byte displacement.
PREFETCH(mem, Level, State) Prefetches a byte memory location pointed by ‘mem’ into the cache level specified by ‘Level’; a request
for exclusive/ownership is done if ‘State’ is 1. Note that the memory location ignore cache line splits. This operation is considered a
hint for the processor and may be skipped depending on implementation.

VGATHERPF0DPS (EVEX 编码版本)¶

(KL, VL) = (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j]
        Prefetch( [BASE_ADDR + SignExtend(VINDEX[i+31:i]) * SCALE + DISP], Level=0, RFO = 0)
    FI;
ENDFOR

VGATHERPF0DPD (EVEX 编码版本)¶

(KL, VL) = (8, 512)
FOR j := 0 TO KL-1
    i := j * 64
    k := j * 32
    IF k1[j]
        Prefetch( [BASE_ADDR + SignExtend(VINDEX[k+31:k]) * SCALE + DISP], Level=0, RFO = 0)
    FI;
ENDFOR
