描述¶

此指令对源操作数（第二操作数）中的打包二进制编码 FP16 值执行归约转换，并在写掩码 k1 控制下，将归约后的二进制浮点格式结果存储在目的操作数（第一操作数）中。

归约转换从二进制浮点源值中减去整数部分和前导 M 个尾数位，其中 M 是由 imm8[7:4] 指定的无符号整数。具体而言，归约转换可以表示为：

dest = src − (ROUND(2M * src)) * 2−M

其中 ROUND() 将 src、2M 以及它们的乘积视为具有规范化有效位数和偏置指数的二进制浮点数。

归约结果的幅度可以通过令 src = 2p * man2 来表示，其中 'man2' 是规范化的有效位数，'p' 是无偏指数。

如果 RC=RNE：0 ≤ |归约结果| ≤ 2−M−1。

如果 RC ≠ RNE：0 ≤ |归约结果| < 2−M。

此指令可能会导致精度异常置位。然而，在设置了 SPE（即精度异常抑制，即 imm8[3]=1）的情况下，将不报告精度异常。

此指令可能会产生微小的非零结果。如果产生该结果，即使未屏蔽下溢异常（MXCSR 寄存器中的 UM 标志为 0），它也不会报告下溢异常。

特殊情况请参阅 表 5-30。

1. Round(.) 函数使用由 (imm8[2]? MXCSR.RC : imm8[1:0]) 指定的舍入控制。

操作¶

def reduce_fp16(src, imm8):
    nan := (src.exp = 0x1F) and (src.fraction != 0)
    if nan:
        return QNAN(src)
    m := imm8[7:4]
    rc := imm8[1:0]
    rc_source := imm8[2]
    spe := imm[3] // suppress precision exception
    tmp := 2^(-m) * ROUND(2^m * src, spe, rc_source, rc)
    tmp := src - tmp // using same RC, SPE controls
    return tmp

VL = 128, 256 or 512
KL := VL/16
FOR i := 0 to KL-1:
    IF k1[i] or *no writemask*:
        IF SRC is memory and (EVEX.b = 1):
            tsrc := src.fp16[0]
        ELSE:
            tsrc := src.fp16[i]
        DEST.fp16[i] := reduce_fp16(tsrc, imm8)
    ELSE IF *zeroing*:
        DEST.fp16[i] := 0
    //else DEST.fp16[i] remains unchanged
