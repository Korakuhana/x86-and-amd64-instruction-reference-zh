CVTDQ2PS — 将打包双字整数转换为打包单精度浮点值

操作码/指令 Op / En 64/32位模式支持 CPUID 特征标志 描述

VEX.128.0F.WIG 5B /r VCVTDQ2PS xmm1, xmm2/m128 A V/V AVX 将四个来自 xmm2/mem 的打包有符号双字整数转换为 xmm1 中的四个打包单精度浮点值。

EVEX.512.0F.W0 5B /r VCVTDQ2PS zmm1 {k1}{z}, zmm2/m512/m32bcst{er} B V/V AVX512F 在写入掩码 k1 下，将十六个来自 zmm2/m512/m32bcst 的打包有符号双字整数转换为 zmm1 中的十六个打包单精度浮点值。

描述¶

将源操作数中的四个、八个或十六个打包有符号双字整数转换为目标操作数中的四个、八个或十六个打包单精度浮点值。

EVEX 编码版本：源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置或从 32 位内存位置广播的 512/256/128 位向量。目标操作数是 ZMM/YMM/XMM 寄存器，在写入掩码 k1 下有条件地更新。

VEX.256 编码版本：源操作数是 YMM 寄存器或 256 位内存位置。目标操作数是 YMM 寄存器。目标寄存器相应的高位（MAXVL-1:256）清零。

VEX.128 编码版本：源操作数是 XMM 寄存器或 128 位内存位置。目标操作数是 XMM 寄存器。目标寄存器相应的高位（MAXVL-1:128）清零。

128 位传统 SSE 版本：源操作数是 XMM 寄存器或 128 位内存位置。目标操作数是 XMM 寄存器。目标寄存器相应的高位（MAXVL-1:128）保持不变。

VEX.vvvv 和 EVEX.vvvv 是保留的，必须为 1111b，否则指令将触发 #UD 异常。

操作¶

VCVTDQ2PS (EVEX Encoded Versions) When SRC Operand is a Register¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
IF (VL = 512) AND (EVEX.b = 1)
	THEN
		SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(EVEX.RC); ; refer to Table 15-4 in the Intel® 64 and IA-32 Architectures
Software Developer’s Manual, Volume 1
	ELSE
		SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(MXCSR.RC); ; refer to Table 15-4 in the Intel® 64 and IA-32 Architectures
Software Developer’s Manual, Volume 1
FI;
FOR j := 0 TO KL-1
	i := j * 32
	IF k1[j] OR *no writemask*
		THEN DEST[i+31:i] :=
			Convert_Integer_To_Single_Precision_Floating_Point(SRC[i+31:i])
		ELSE
			IF *merging-masking* ; merging-masking
				THEN *DEST[i+31:i] remains unchanged*
				ELSE ; zeroing-masking
					DEST[i+31:i] := 0
			FI
	FI;
ENDFOR
DEST[MAXVL-1:VL] := 0

VCVTDQ2PS (EVEX Encoded Versions) When SRC Operand is a Memory Source¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
FOR j := 0 TO KL-1
	i := j * 32
	IF k1[j] OR *no writemask*
		THEN
			IF (EVEX.b = 1)
				THEN
					DEST[i+31:i] :=
			Convert_Integer_To_Single_Precision_Floating_Point(SRC[31:0])
				ELSE
					DEST[i+31:i] :=
			Convert_Integer_To_Single_Precision_Floating_Point(SRC[i+31:i])
			FI;
		ELSE
			IF *merging-masking* ; merging-masking
				THEN *DEST[i+31:i] remains unchanged*
				ELSE ; zeroing-masking
					DEST[i+31:i] := 0
			FI
	FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
