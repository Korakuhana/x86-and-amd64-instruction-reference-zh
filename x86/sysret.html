SYSRET — 快速系统调用返回

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 07 SYSRET ZO Valid Invalid 从快速系统调用返回到兼容模式。
REX.W + 0F 07 SYSRET ZO Valid Invalid 从快速系统调用返回到 64 位模式。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

SYSRET 是 SYSCALL 指令的配套指令。It returns from an OS system-call handler to user code at privilege level 3. It does so by loading RIP from RCX and loading RFLAGS from R11.1如果使用 64 位操作数大小，SYSRET 保持在 64 位模式下；否则，它进入兼容模式，仅加载寄存器的低 32 位。

SYSRET 用派生自 IA32_STAR MSR 的第 63:48 位的值加载 CS 和 SS 选择子。然而，CS 和 SS 描述符缓存不会从那些选择子引用的描述符（在 GDT 或 LDT 中）加载。相反，描述符缓存被加载为固定值。详情请参阅操作（Operation）部分。操作系统软件有责任确保这些选择子值引用的描述符（在 GDT 或 LDT 中）与加载到描述符缓存中的固定值相对应；SYSRET 指令并不保证这种对应关系。

SYSRET 指令不修改栈指针（ESP 或 RSP）。因此，软件需要切换到用户栈。操作系统可在执行 SYSRET 之前加载用户栈指针（如果在 SYSCALL 之后已保存）；或者，用户代码在从 SYSRET 接收到控制权之后加载栈指针（如果在 SYSCALL 之前已保存）。

如果操作系统在执行 SYSRET 之前加载栈指针，它必须确保在恢复栈指针与成功执行 SYSRET 之间传递的任何中断或异常处理程序不会配合用户栈被调用。它可以采用类似以下的方法来实现这一点：

  • 外部中断。操作系统可以通过在加载用户栈指针之前清除 EFLAGS.IF 来防止外部中断的传递。
  • 不可屏蔽中断（NMI）。操作系统可以通过在 IDT 中对中断门 2 (NMI) 使用中断栈表（IST）机制来确保 NMI 处理程序以正确的栈被调用（请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3A 卷第 6.14.5 节“中断栈表”）。
  • 通用保护异常（#GP）。如果 RCX 的值不是规范形式，SYSRET 指令将产生 #GP(0)。操作系统可以使用以下一个或多个方法来处理这种可能性：
    • 在执行 SYSRET 之前确认 RCX 的值是规范形式。
    • 使用分页来确保 SYSCALL 指令永远不会将非规范值保存到 RCX 中。
    • 在 IDT 中对中断门 13 (#GP) 使用 IST 机制。

当在特权级 3 启用阴影栈时，该指令用来自 IA32_PL3_SSP MSR 的值加载 SSP。有关额外的 CET 详细信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 6 章“过程调用、中断和异常”和第 17 章“控制流强制技术（CET）”。

1. 不论 R11 的值如何，执行 SYSRET 后 RFLAGS 中的 RF 和 VM 标志始终为 0。此外，RFLAGS 中的所有保留位均保留其固定值。

指令顺序。 在早先指令完成执行之前，可能会从内存中获取 SYSRET 之后的指令，但是在 SYSRET 之前的所有指令都完成执行之前，它们将不会执行（即使是猜测性执行）（后续指令可能会在早先指令存储的数据变得全局可见之前执行）。

操作¶

IF (CS.L ≠ 1 ) or (IA32_EFER.LMA ≠ 1) or (IA32_EFER.SCE ≠ 1)
(* Not in 64-Bit Mode or SYSCALL/SYSRET not enabled in IA32_EFER *)
    THEN #UD; FI;
IF (CPL ≠ 0) THEN #GP(0); FI;
IF (operand size is 64-bit)
    THEN (* Return to 64-Bit Mode *)
        IF (RCX is not canonical) THEN #GP(0);
        RIP := RCX;
    ELSE (* Return to Compatibility Mode *)
        RIP := ECX;
FI;
RFLAGS := (R11 & 3C7FD7H) | 2; (*
                Clear RF, VM, reserved bits; set bit 1 *)
IF (operand size is 64-bit)
    THEN CS.Selector := IA32_STAR[63:48]+16;
    ELSE CS.Selector := IA32_STAR[63:48];
FI;
CS.Selector := CS.Selector OR 3;
            (* RPL forced to 3 *)
(* Set rest of CS to a fixed value *)
CS.Base := 0;
            (* Flat segment *)
CS.Limit := FFFFFH;
            (* With 4-KByte granularity, implies a 4-GByte limit *)
CS.Type := 11;
            (* Execute/read code, accessed *)
CS.S := 1;
CS.DPL := 3;
CS.P := 1;
IF (operand size is 64-bit)
    THEN (* Return to 64-Bit Mode *)
        CS.L := 1;
            (* 64-bit code segment *)
        CS.D := 0;
            (* Required if CS.L = 1 *)
    ELSE (* Return to Compatibility Mode *)
        CS.L := 0;
            (* Compatibility mode *)
        CS.D := 1;
            (* 32-bit code segment *)
FI;
CS.G := 1;
            (* 4-KByte granularity *)
CPL := 3;
IF ShadowStackEnabled(CPL)
    SSP := IA32_PL3_SSP;
FI;
SS.Selector := (IA32_STAR[63:48]+8) OR 3;
            (* RPL forced to 3 *)
(* Set rest of SS to a fixed value *)
SS.Base := 0;
            (* Flat segment *)
SS.Limit := FFFFFH;
            (* With 4-KByte granularity, implies a 4-GByte limit *)
SS.Type := 3;
            (* Read/write data, accessed *)
SS.S := 1;
SS.DPL := 3;
SS.P := 1;
SS.B := 1;
            (* 32-bit stack segment*)
SS.G := 1;
            (* 4-KByte granularity *)

受影响的标志位¶

所有标志位。

保护模式异常¶

#UD SYSRET 指令在保护模式下无法识别。

实地址模式异常¶

#UD SYSRET 指令在实地址模式下无法识别。

虚拟-8086 模式异常¶

#UD SYSRET 指令在虚拟-8086 模式下无法识别。

兼容模式异常¶

#UD SYSRET 指令在兼容模式下无法识别。

64位模式异常¶

#UD 如果 IA32_EFER.SCE = 0。
如果使用了 LOCK 前缀。
#GP(0) 如果 CPL != 0。
如果返回到 64 位模式且 RCX 包含非规范形式的地址。
