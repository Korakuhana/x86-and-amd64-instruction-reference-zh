POPF/POPFD/POPFQ — 出栈到 EFLAGS 寄存器

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
9D POPF ZO Valid Valid 将栈顶弹出到 EFLAGS 的低 16 位中。
9D POPFD ZO N.E. Valid 将栈顶弹出到 EFLAGS 中。
9D POPFQ ZO Valid N.E. 将栈顶弹出并零扩展到 RFLAGS 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

从栈顶弹出一个双字（POPFD）（如果当前操作数大小属性为 32）并将该值存储在 EFLAGS 寄存器中；或者从栈顶弹出一个字（如果操作数大小属性为 16）并将该值存储在 EFLAGS 寄存器的低 16 位（即 FLAGS 寄存器）中。这些指令的作用与 PUSHF/PUSHFD/PUSHFQ 指令相反。

助记符 POPF（弹出标志）和 POPFD（弹出双字标志）指向相同的操作码。POPF 指令旨在在操作数大小属性为 16 时使用；POPFD 指令旨在在操作数大小属性为 32 时使用。某些汇编器可能会在下述情况下强制确定操作数大小：POPF 为 16，POPFD 为 32。其他汇编器可能将这些助记符视为同义词（POPF/POPFD），并使用操作数大小属性的设置来确定要从栈中弹出的值的大小。

POPF/POPFD 对 EFLAGS 寄存器的影响因操作模式而异。有关详细信息，请参阅表 4-16及下方的说明。

在特权级别为 0 的保护模式、兼容模式或 64 位模式下运行（或在实地址模式下，等效于特权级别 0）时，可以修改 EFLAGS 寄存器中除 RF1、VIP、VIF 和 VM 之外的所有非保留标志。VIP、VIF 和 VM 保持不受影响。

当在保护模式、兼容模式或 64 位模式下运行，特权级别大于 0 但小于或等于 IOPL 时，可以修改除 IOPL 字段和 RF、IF、VIP、VIF、VM 之外的所有标志；这些保持不受影响。仅当操作数大小属性为 32 时，才能修改 AC 和 ID 标志。只有在至少与 IOPL 具有相同特权的级别执行时，才会更改中断标志（IF）。如果在特权不足的情况下执行 POPF/POPFD 指令，则不会发生异常，但特权位不会改变。

在没有虚拟 8086 模式扩展（CR4.VME = 0）的虚拟 8086 模式（EFLAGS.VM = 1）下运行时，只有在 IOPL = 3 时才能使用 POPF/POPFD 指令；否则，会发生一般保护性异常（#GP）。如果启用了虚拟 8086 模式扩展（CR4.VME = 1），则可以在 IOPL < 3 的虚拟 8086 模式下执行 POPF（但不能执行 POPFD）。

（通过设置 CR4.PVI 启用的保护模式虚拟中断功能以与虚拟 8086 模式扩展相同的方式影响 CLI 和 STI 指令。然而，POPF 不受 CR4.PVI 的影响。）

在 64 位模式下，分配的助记符是 POPFQ（请注意，32 位操作数无法编码）。POPFQ 从栈中弹出 64 位。RFLAGS 的保留位（包括 RFLAGS 的高 32 位）不受影响。

有关 EFLAGS 寄存器的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷的第 3 章。

1. 执行 POPF 后，RF 始终为零。这是因为 POPF 与所有指令一样，在开始执行时会清除 RF。

1. #GP 故障 - 无标志更新

2. 如果 EFLAGS 寄存器中的 VIP=1 且栈上 FLAGS 值中的 IF=1，则 #GP 故障，且无标志更新

3. 如果栈上 FLAGS 值中的 TF=1，则 #GP 故障，且无标志更新

说明键
S 从栈中更新
SV 从栈上 FLAGS 值中的 IF（第 9 位）进行更新
N 数值无变化
X 不更新 EFLAGS
0 数值被清除

操作¶

IF EFLAGS.VM = 0 (* Not in Virtual-8086 Mode *)
    THEN IF CPL = 0 OR CR0.PE = 0
        THEN
            IF OperandSize = 32;
                THEN
                    EFLAGS := Pop(); (* 32-bit pop *)
                    (* All non-reserved flags except RF, VIP, VIF, and VM can be modified;
                    VIP, VIF, VM, and all reserved bits are unaffected. RF is cleared. *)
                ELSE IF (Operandsize = 64)
                    RFLAGS = Pop(); (* 64-bit pop *)
                    (* All non-reserved flags except RF, VIP, VIF, and VM can be modified;
                    VIP, VIF, VM, and all reserved bits are unaffected. RF is cleared. *)
                ELSE (* OperandSize = 16 *)
                    EFLAGS[15:0] := Pop(); (* 16-bit pop *)
                    (* All non-reserved flags can be modified. *)
            FI;
        ELSE (* CPL > 0 *)
            IF OperandSize = 32
                THEN
                    IF CPL > IOPL
                        THEN
                            EFLAGS := Pop(); (* 32-bit pop *)
                            (* All non-reserved bits except IF, IOPL, VIP, VIF, VM, and RF can be modified;
                            IF, IOPL, VIP, VIF, VM, and all reserved bits are unaffected; RF is cleared. *)
                        ELSE
                            EFLAGS := Pop(); (* 32-bit pop *)
                            (* All non-reserved bits except IOPL, VIP, VIF, VM, and RF can be modified;
                            IOPL, VIP, VIF, VM, and all reserved bits are unaffected; RF is cleared. *)
                    FI;
                ELSE IF (Operandsize = 64)
                    IF CPL > IOPL
                        THEN
                            RFLAGS := Pop(); (* 64-bit pop *)
                            (* All non-reserved bits except IF, IOPL, VIP, VIF, VM, and RF can be modified;
                            IF, IOPL, VIP, VIF, VM, and all reserved bits are unaffected; RF is cleared. *)
                        ELSE
                            RFLAGS := Pop(); (* 64-bit pop *)
                            (* All non-reserved bits except IOPL, VIP, VIF, VM, and RF can be modified;
                            IOPL, VIP, VIF, VM, and all reserved bits are unaffected; RF is cleared. *)
                    FI;
                ELSE (* OperandSize = 16 *)
                    EFLAGS[15:0] := Pop(); (* 16-bit pop *)
                    (* All non-reserved bits except IOPL can be modified; IOPL and all
                    reserved bits are unaffected. *)
            FI;
        FI;
    ELSE (* In virtual-8086 mode *)
        IF IOPL = 3
            THEN
                IF OperandSize = 32
                    THEN
                        EFLAGS := Pop();
                        (* All non-reserved bits except IOPL, VIP, VIF, VM, and RF can be modified;
                        VIP, VIF, VM, IOPL, and all reserved bits are unaffected. RF is cleared. *)
                    ELSE
                        EFLAGS[15:0] := Pop(); FI;
                        (* All non-reserved bits except IOPL can be modified; IOPL and all reserved bits are unaffected. *)
                FI;
            ELSE (* IOPL < 3 *)
                IF (Operandsize = 32) OR (CR4.VME = 0)
                    THEN #GP(0); (* Trap to virtual-8086 monitor. *)
                    ELSE (* Operandsize = 16 and CR4.VME = 1 *)
                        tempFLAGS := Pop();
                        IF (EFLAGS.VIP = 1 AND tempFLAGS[9] = 1) OR tempFLAGS[8] = 1
                            THEN #GP(0);
                            ELSE
                                EFLAGS.VIF := tempFLAGS[9];
                                EFLAGS[15:0] := tempFLAGS;
                                (* All non-reserved bits except IOPL and IF can be modified;
                                IOPL, IF, and all reserved bits are unaffected. *)
                        FI;
                FI;
        FI;
FI;

影响的标志位¶

所有标志位都可能受影响；有关详细信息，请参阅“操作”部分。

保护模式异常¶

#SS(0) 如果栈顶不在栈段边界内。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且在 CPL = 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#SS 如果栈顶不在栈段边界内。
#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#GP(0) 如果 IOPL < 3 并且未启用 VME。
如果 IOPL < 3 并且使用了 32 位操作数大小。
如果 IOPL < 3，EFLAGS.VIP = 1，并且栈上的 FLAGS 值中设置了第 9 位（IF）。
如果 IOPL < 3 并且栈上的 FLAGS 值中设置了第 8 位（TF）。
如果尝试使用操作数大小覆盖前缀执行 POPF/POPFD 指令。
#SS(0) 如果栈顶不在栈段边界内。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果栈地址是非规范形式。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
