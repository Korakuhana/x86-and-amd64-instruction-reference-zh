FILD — 加载整数

操作码 指令 64位模式 兼容/传统模式 描述
DF /0 FILD m16int 有效 有效 将 m16int 压入 FPU 寄存器栈。
DB /0 FILD m32int 有效 有效 将 m32int 压入 FPU 寄存器栈。
DF /5 FILD m64int 有效 有效 将 m64int 压入 FPU 寄存器栈。

描述¶

将有符号整数源操作数转换为双精度扩展浮点格式，并将该值压入 FPU 寄存器栈。源操作数可以是字、双字或四字整数。加载过程不会产生舍入误差。源操作数的符号得以保留。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

TOP := TOP − 1;
ST(0) := ConvertToDoubleExtendedPrecisionFP(SRC);

影响的 FPU 标志寄存器¶

C1 如果发生栈溢出，则设置为 1；否则设置为 0。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈溢出。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含空段选择器。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段 of 内存地址为非规范形式。
#GP(0) 如果内存地址为非规范形式。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在未决的 x87 FPU 异常。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
