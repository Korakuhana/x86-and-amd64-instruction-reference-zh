描述¶

将标量双精度浮点值从源操作数（第二个操作数）移动到目的操作数（第一个操作数）。源操作数和目的操作数可以是 XMM 寄存器或 64 位内存位置。此指令可用于在 XMM 寄存器的低四字与 64 位内存位置之间传输双精度浮点值，或在两个 XMM 寄存器的低四字之间移动双精度浮点值。该指令不能用于在内存位置之间传送数据。

传统版本：当源和目的操作数都是 XMM 寄存器时，目的操作数的位 MAXVL:64 保持不变。当源操作数是内存位置且目的操作数是 XMM 寄存器时，目的操作数 127:64 位处的四字被清零，目的操作数的位 MAXVL:128 保持不变。

VEX 和 EVEX 编码的寄存器-寄存器语法：将标量双精度浮点值从第二个源操作数（第三个操作数）移动到目的操作数（第一个操作数）的低四字元素。目的操作数的 127:64 位复制自第一个源操作数（第二个操作数）。对应目的寄存器的位 (MAXVL-1:128) 被清零。

VEX 和 EVEX 编码的内存存储与加载语法：当源操作数是内存位置且目的操作数是 XMM 寄存器时，目的操作数的位 MAXVL:64 被清零。

EVEX 编码版本：目的操作数的低四字根据写掩码进行更新。

注意：对于 VMOVSD（内存存储和加载形式），VEX.vvvv 和 EVEX.vvvv 是保留的，必须为 1111b，否则指令将产生 #UD 异常。

操作¶

VMOVSD (EVEX.LLIG.F2.0F 10 /r: VMOVSD xmm1, m64 With Support for 32 Registers)¶

IF k1[0] or *no writemask*
    THEN DEST[63:0] := SRC[63:0]
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[63:0] remains unchanged*
            ELSE ; zeroing-masking
                THEN DEST[63:0] := 0
        FI;
FI;
DEST[MAXVL-1:64] := 0

VMOVSD (EVEX.LLIG.F2.0F 11 /r: VMOVSD m64, xmm1 With Support for 32 Registers)¶

IF k1[0] or *no writemask*
    THEN DEST[63:0] := SRC[63:0]
    ELSE *DEST[63:0] remains unchanged* ; merging-masking
FI;

VMOVSD (EVEX.LLIG.F2.0F 11 /r: VMOVSD xmm1, xmm2, xmm3)¶

IF k1[0] or *no writemask*
    THEN DEST[63:0] := SRC2[63:0]
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[63:0] remains unchanged*
            ELSE ; zeroing-masking
                THEN DEST[63:0] := 0
        FI;
FI;
DEST[127:64] := SRC1[127:64]
DEST[MAXVL-1:128] := 0
