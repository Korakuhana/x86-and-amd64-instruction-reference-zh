RDPMC — 读取性能监控计数器

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 33 RDPMC ZO Valid Valid 将 ECX 指定的性能监控计数器读取到 EDX:EAX 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将 ECX 寄存器中指定的性能监控计数器 (PMC) 的内容读取到寄存器 EDX:EAX 中。（在支持 Intel 64 架构的处理器上，RCX 的高 32 位被忽略。）EDX 寄存器加载 PMC 的高 32 位，EAX 寄存器加载低 32 位。（在支持 Intel 64 架构的处理器上，RAX 和 RDX 各自的高 32 位均被清零。）如果所读取的 PMC 实现的位数少于 64 位，则返回给 EDX:EAX 的未实现位的值将为零。

在支持架构性能监控 (CPUID.0AH:EAX[7:0] ̸= 0) 的处理器上，PMC 的宽度由 CPUID.0AH:EAX[23:16] 报告。在不支持架构性能监控 (CPUID.0AH:EAX[7:0]=0) 的处理器上，通用性能 PMC 的宽度为 40 位，而特殊用途 PMC 的宽度是具体实现特定的。

使用 ECX 来指定 PMC 取决于处理器是否支持架构性能监控：

  • 如果处理器不支持架构性能监控 (CPUID.0AH:EAX[7:0]=0)，则 ECX[30:0] 指定要读取的 PMC 的索引。如果支持，设置 ECX[31] 将选择“快速”读取模式。在这种模式下，RDPMC 在 EAX 中返回 PMC 的位 31:0，同时将 EDX 清零。
  • 如果处理器支持架构性能监控 (CPUID.0AH:EAX[7:0] ̸= 0)，则 ECX[31:16] 指定 PMC 的类型，而 ECX[15:0] 指定在该类型中要读取的 PMC 的索引。目前定义了以下 PMC 类型：
    • 通用计数器使用类型 0。索引 x（读取 IA32_PMCx）必须小于 CPUID.0AH.EAX[15:8] 枚举的值（因此 ECX[15:8] 必须为零）。
    • 通用计数器使用类型 0。索引 x（读取 IA32_PMCx）必须小于 CPUID.0AH.EAX[15:8] 枚举的值（因此 ECX[15:8] 必须为零）。
    • 固定功能计数器使用类型 4000H。如果 CPUID.0AH.EDX[4:0] > x 或 CPUID.0AH.ECX[x] = 1（因此 ECX[15:5] 必须为 0），则可以使用索引 x（以读取 IA32_FIXED_CTRx）。
    • 固定功能计数器使用类型 4000H。如果 CPUID.0AH.EDX[4:0] > x 或 CPUID.0AH.ECX[x] = 1（因此 ECX[15:5] 必须为 0），则可以使用索引 x（以读取 IA32_FIXED_CTRx）。
    • 性能度量使用类型 2000H。此类型仅当 IA32_PERF_CAPABILITIES.PERF_METRICS_AVAILABLE[位 15]=1 时才可以使用。对于此类型，ECX[15:0] 中的索引是具体实现特定的。
    • 性能度量使用类型 2000H。此类型仅当 IA32_PERF_CAPABILITIES.PERF_METRICS_AVAILABLE[位 15]=1 时才可以使用。对于此类型，ECX[15:0] 中的索引是具体实现特定的。

指定不受支持的 PMC 编码将导致通用保护异常 #GP(0)。有关 PMC 的详细信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3B 卷第 20 章“性能监控”。

在保护模式或虚拟 8086 模式下，寄存器 CR4 中的**启用性能监控计数器** (PCE) 标志限制了 RDPMC 指令的使用。当设置了 PCE 标志时，RDPMC 指令可以在任何特权级执行；当清除该标志时，该指令只能在特权级 0 执行。（在实地址模式下，RDPMC 指令总是启用的。）在特权级 0 执行时，也可以使用 RDMSR 指令读取 PMC。

RDPMC 指令不是序列化指令；即它并不意味着前面的指令所引起的所有事件都已完成，或者后面的指令所引起的事件尚未开始。如果需要精确的事件计数，软件必须在 RDPMC 指令之前和/或之后插入序列化指令（例如 CPUID 指令）。

执行连续的快速读取不能保证是单调的。为了保证连续读取的单调性，必须在两条 RDPMC 指令之间放置一条序列化指令。

RDPMC 指令可以在 16 位寻址模式或虚拟 8086 模式下执行；但是，ECX 寄存器的全部内容被用于选择 PMC，并且事件计数存储在完整的 EAX 和 EDX 寄存器中。RDPMC 指令是在 Pentium Pro 处理器以及带 MMX 技术的 Pentium 处理器中引入到 IA-32 架构中的。较早的 Pentium 处理器具有 PMC，但必须使用 RDMSR 指令进行读取。

操作¶

MSCB = Most Significant Counter Bit (* 模型特定 *)
IF (((CR4.PCE = 1) or (CPL = 0) or (CR0.PE = 0)) and (ECX indicates a supported counter))
    THEN
        EAX := counter[31:0];
        EDX := ZeroExtend(counter[MSCB:32]);
    ELSE (* ECX 无效，或者 CR4.PCE 为 0 且 CPL 为 1、2 或 3 且 CR0.PE 为 1 *)
        #GP(0);
FI;

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果当前特权级不为 0 且 CR4 寄存器中的 PCE 标志被清除。
如果指定了无效的性能计数器索引。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果指定了无效的性能计数器索引。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果 CR4 寄存器中的 PCE 标志被清除。
如果指定了无效的性能计数器索引。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#GP(0) 如果当前特权级不为 0 且 CR4 寄存器中的 PCE 标志被清除。
如果指定了无效的性能计数器索引。
#UD 如果使用了 LOCK 前缀。
