VTESTPD/VTESTPS — 打包位测试

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
VEX.128.66.0F38.W0 0E /r VTESTPS xmm1, xmm2/m128 RM V/V AVX 根据打包单精度浮点源的符号位 AND 和 ANDN，设置 ZF 和 CF。
VEX.256.66.0F38.W0 0E /r VTESTPS ymm1, ymm2/m256 RM V/V AVX 根据打包单精度浮点源的符号位 AND 和 ANDN，设置 ZF 和 CF。
VEX.128.66.0F38.W0 0F /r VTESTPD xmm1, xmm2/m128 RM V/V AVX 根据打包双精度浮点源的符号位 AND 和 ANDN，设置 ZF 和 CF。
VEX.256.66.0F38.W0 0F /r VTESTPD ymm1, ymm2/m256 RM V/V AVX 根据打包双精度浮点源的符号位 AND 和 ANDN，设置 ZF 和 CF。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) N/A N/A

描述¶

VTESTPS 对第一源操作数中打包单精度元素的所有符号位与第二源操作数中对应的符号位进行按位比较。如果源符号位与目的符号位的 AND 结果为全零，则设置 ZF，否则清除 ZF。如果源符号位与目的符号位取反后的 AND 结果为全零，则设置 CF，否则清除 CF。试图执行带有 VEX.W=1 的 VTESTPS 会导致 #UD。

VTESTPD 对第一源操作数中打包双精度元素的所有符号位与第二源操作数中对应的符号位进行按位比较。如果源符号位与目的符号位的 AND 结果为全零，则设置 ZF，否则清除 ZF。如果源符号位与目的符号位取反后的 AND 结果为全零，则设置 CF，否则清除 CF。试图执行带有 VEX.W=1 的 VTESTPD 会导致 #UD。

第一源寄存器由 ModR/M reg 字段指定。

128 位版本：第一源寄存器是 XMM 寄存器。第二源寄存器可以是 XMM 寄存器或 128 位内存位置。目的寄存器不被修改。

VEX.256 编码版本：第一源寄存器是 YMM 寄存器。第二源寄存器可以是 YMM 寄存器或 256 位内存位置。目的寄存器不被修改。

注意：在 VEX 编码版本中，VEX.vvvv 是保留的，必须为 1111b，否则指令将触发 #UD 故障。

操作¶

VTESTPS (128位版本)¶

TEMP[127:0] := SRC[127:0] AND DEST[127:0]
IF (TEMP[31] = TEMP[63] = TEMP[95] = TEMP[127] = 0)
    THEN ZF := 1;
    ELSE ZF := 0;
TEMP[127:0] := SRC[127:0] AND NOT DEST[127:0]
IF (TEMP[31] = TEMP[63] = TEMP[95] = TEMP[127] = 0)
    THEN CF := 1;
    ELSE CF := 0;
DEST (unmodified)
AF := OF := PF := SF := 0;

受影响的标志位¶

OF, AF, PF, SF 标志被清零，ZF, CF 标志根据操作进行设置。

SIMD 浮点异常¶

无。

其他异常¶

请参阅表 2-21，“第 4 类异常条件”。

此外：

#UD 如果 VEX.vvvv ≠ 1111B。
如果对于 VTESTPS 或 VTESTPD，VEX.W = 1。
