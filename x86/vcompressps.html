VCOMPRESSPS — 将稀疏打包单精度浮点值存储到紧凑内存中

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
EVEX.128.66.0F38.W0 8A /r VCOMPRESSPS xmm1/m128 {k1}{z}, xmm2 A V/V AVX512VL AVX512F 在写掩码 k1 的控制下，将打包的单精度浮点值从 xmm2 压缩存储到 xmm1/m128 中。
EVEX.256.66.0F38.W0 8A /r VCOMPRESSPS ymm1/m256 {k1}{z}, ymm2 A V/V AVX512VL AVX512F 在写掩码 k1 的控制下，将打包的单精度浮点值从 ymm2 压缩存储到 ymm1/m256 中。
EVEX.512.66.0F38.W0 8A /r VCOMPRESSPS zmm1/m512 {k1}{z}, zmm2 A V/V AVX512F 在控制掩码 k1 的控制下，将打包的单精度浮点值从 zmm2 压缩存储到 zmm1/m512 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Tuple1 Scalar ModRM:r/m (w) ModRM:reg (r) N/A N/A

描述¶

将源操作数（第二个操作数）中最多 16 个单精度浮点值压缩（存储）到目的操作数（第一个操作数）。源操作数是 ZMM/YMM/XMM 寄存器，目的操作数可以是 ZMM/YMM/XMM 寄存器或 512/256/128 位内存位置。

操作掩码寄存器 k1 从源操作数中选择活动元素（如果活动元素少于 16 个，则为部分向量或可能是非连续的），以压缩为一个连续的向量。该连续向量从目的操作数的低位元素开始写入目的操作数。

内存目的版本：仅将该连续向量写入目的内存位置。EVEX.z 必须为 0。

寄存器目的版本：如果该连续向量的向量长度小于源操作数中输入向量的长度，当未设置 EVEX.z 时，目的寄存器的高位部分保持不变；否则高位部分被清零。

EVEX.vvvv 保留，必须为 1111b，否则指令会导致 #UD。

请注意，压缩偏移量假定预缩放比例 (N) 对应于单个元素的大小，而不是整个向量的大小。

操作¶

VCOMPRESSPS（EVEX 编码版本）存储形式¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
SIZE := 32
k := 0
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN
            DEST[k+SIZE-1:k] := SRC[i+31:i]
            k := k + SIZE
    FI;
ENDFOR;

VCOMPRESSPS（EVEX 编码版本）寄存器-寄存器形式¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
SIZE := 32
k := 0
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN
            DEST[k+SIZE-1:k] := SRC[i+31:i]
            k := k + SIZE
    FI;
ENDFOR
IF *merging-masking*
    THEN *DEST[VL-1:k] remains unchanged*
    ELSE DEST[VL-1:k] := 0
FI
DEST[MAXVL-1:VL] := 0

SIMD 浮点异常¶

无。

此外：

#UD 如果 EVEX.vvvv != 1111B。
