RSTORSSP — 恢复保存的影子栈指针

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
F3 0F 01 /5 (mod!=11, /5, memory only) RSTORSSP m64 M V/V CET_SS 恢复 SSP。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r, w) N/A N/A N/A

描述¶

从 m64 指向的影子栈恢复（shadow-stack-restore）令牌中恢复 SSP。如果 SSP 恢复成功，则该指令将影子栈恢复令牌替换为 previous-ssp 令牌。该指令设置 CF 标志，以指示处理的影子栈恢复令牌中记录的 SSP 地址是否为 4 字节对齐，即在此影子栈上压入恢复影子栈令牌时是否创建了对齐空洞（alignment hole）。

在 RSTORSSP 之后，如果需要将恢复影子栈令牌保存到前一个影子栈，请使用 SAVEPREVSSP 指令。

如果不需要在前一个影子栈上压入恢复影子栈令牌，可以使用 INCSSPQ 指令弹出 previous-ssp 令牌。如果设置了 CF 标志以指示存在对齐空洞，则需要额外的 INCSSPD 指令来使 SSP 越过对齐空洞。

操作¶

IF CPL = 3
        IF (CR4.CET & IA32_U_CET.SH_STK_EN) = 0
            THEN #UD; FI;
ELSE
        IF (CR4.CET & IA32_S_CET.SH_STK_EN) = 0
            THEN #UD; FI;
FI;
SSP_LA = Linear_Address(mem operand)
IF SSP_LA not aligned to 8 bytes
        THEN #GP(0); FI;
previous_ssp_token = SSP | (IA32_EFER.LMA AND CS.L) | 0x02
Start Atomic Execution
restore_ssp_token = Locked shadow_stack_load 8 bytes from SSP_LA
fault = 0
IF ((restore_ssp_token & 0x03) != (IA32_EFER.LMA & CS.L))
        THEN fault = 1; FI; (* 如果令牌中的 L 标志与 IA32_EFER.LMA & CS.L 不匹配，或位 1 不为 0 *)
IF ((IA32_EFER.LMA AND CS.L) = 0 AND restore_ssp_token[63:32] != 0)
        THEN fault = 1; FI; (* 如果为兼容/传统模式且要恢复的 SSP 不低于 4G *)
TMP = restore_ssp_token & ~0x01
TMP = (TMP - 8)
TMP = TMP & ~0x07
IF TMP != SSP_LA
        THEN fault = 1; FI; (* 如果令牌中的地址与请求的栈顶不匹配 *)
TMP = (fault == 0) ? previous_ssp_token : restore_ssp_token
shadow_stack_store 8 bytes of TMP to SSP_LA and release lock
End Atomic Execution
IF fault == 1
    THEN #CP(RSTORSSP); FI;
SSP = SSP_LA
// 如果恢复令牌中的 SSP 是 4 字节对齐的（即存在对齐空洞），则设置 CF
RFLAGS.CF = (restore_ssp_token & 0x04) ? 1 : 0;
RFLAGS.ZF,PF,AF,OF,SF := 0;

受影响的标志位¶

设置 CF 以指示恢复令牌中的影子栈指针是否为 4 字节对齐，否则清除。ZF、PF、AF、OF 和 SF 被清除。

C/C++ 编译器等价内部函数¶

RSTORSSP void _rstorssp(void *);

保护模式异常¶

#UD 如果使用了 LOCK 前缀。
如果 CR4.CET = 0。
如果 CPL = 3 且 IA32_U_CET.SH_STK_EN = 0。
如果 CPL < 3 且 IA32_S_CET.SH_STK_EN = 0。
#GP(0) 如果内存操作数的线性地址未 8 字节对齐。
如果内存操作数的有效地址超出了 CS、DS、ES、FS 或 GS 段的界限。
如果目标操作数位于不可写的段中。
如果使用 DS、ES、FS 或 GS 寄存器来访问内存，且它包含一个 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出了 SS 段的界限。
#CP(rstorssp) 如果令牌中的 L 位与 (IA32_EFER.LMA & CS.L) 不匹配。
如果令牌中的地址与内存操作数的线性地址不匹配。
如果处于 32 位或兼容模式，且令牌中的地址不低于 4G。
#PF(错误码) 如果发生缺页异常。

实地址模式异常¶

#UD 在实地址模式下无法识别 RSTORSSP 指令。

虚拟-8086 模式异常¶

#UD 在虚拟-8086 模式下无法识别 RSTORSSP 指令。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#UD 如果使用了 LOCK 前缀。
如果 CR4.CET = 0。
如果 CPL = 3 且 IA32_U_CET.SH_STK_EN = 0。
如果 CPL < 3 且 IA32_S_CET.SH_STK_EN = 0。
#GP(0) 如果内存操作数的线性地址未 8 字节对齐。
如果内存地址为非规范形式。
