描述¶

VPSHUFBITQMB 指令以第二源操作数作为控制，以第一源操作数作为数据，执行位收集选择。每一位使用 6 个控制位（第二源操作数）来选择要收集的数据位（第一源操作数）。给定的位只能访问 64 个不同的数据位（前 64 个目的位可以访问前 64 个数据位，后 64 个目的位可以访问后 64 个数据位，以此类推）。

每个输出位的控制数据存储在 SRC2 的 8 位元素中，但仅使用每个元素的 6 个最低有效位。

此指令使用写掩码（仅清零）。此指令支持内存故障抑制。

第一源操作数是 ZMM 寄存器。第二源操作数是 ZMM 寄存器或内存位置。目的操作数是掩码寄存器。

操作¶

VPSHUFBITQMB DEST, SRC1, SRC2¶

(KL, VL) = (16,128), (32,256), (64, 512)
FOR i := 0 TO KL/8-1: //Qword
    FOR j := 0 to 7: // Byte
        IF k2[i*8+j] or *no writemask*:
            m := SRC2.qword[i].byte[j] & 0x3F
            k1[i*8+j] := SRC1.qword[i].bit[m]
        ELSE:
            k1[i*8+j] := 0
k1[MAX_KL-1:KL] := 0
