CLTS — 清除 CR0 中的任务切换 (TS) 标志位

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 06 CLTS ZO Valid Valid 清除 CR0 中的 TS 标志位。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

清除 CR0 寄存器中的任务切换（TS）标志位。此指令旨在用于操作系统例程。它是一条特权指令，只能在 CPL 为 0 时执行。它被允许在实地址模式下执行，以允许对保护模式进行初始化。

处理器在每次发生任务切换时都会设置 TS 标志位。该标志位用于在多任务应用程序中同步保存 FPU 上下文。有关此标志的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3A 卷第 2 章中标题为“控制寄存器”一节中关于 TS 标志的描述。

CLTS 指令在非 64 位模式和 64 位模式下的操作相同。

有关此指令在 VMX 非根操作中行为的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3C 卷第 26 章“VMX 非根操作”。

操作¶

CR0.TS[bit 3] := 0;

受影响的标志位¶

清除 CR0 寄存器中的 TS 标志位。

保护模式异常¶

#GP(0) 如果当前特权级别不为 0。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#GP(0) 在虚拟 8086 模式下无法识别 CLTS 指令。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#GP(0) 如果 CPL 大于 0。
#UD 如果使用了 LOCK 前缀。
