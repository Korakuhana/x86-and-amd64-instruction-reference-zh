描述¶

VPMOVDB 使用截断将源操作数（第二操作数）中的 32 位整数元素向下转换为打包字节。VPMOVSDB 使用有符号饱和度将有符号 32 位整数转换为打包的有符号字节。VPMOVUSDB 使用无符号饱和度将无符号双字值转换为无符号字节值。

源操作数是 ZMM/YMM/XMM 寄存器。目的操作数是 XMM 寄存器或 128/64/32 位内存位置。

向下转换的字节元素从最低有效字节写入目的操作数（第一操作数）。目的操作数的字节元素根据写掩码进行更新。寄存器目的地的 bits (MAXVL-1:128/64/32) 清零。

EVEX.vvvv 保留且必须为 1111b，否则指令将导致 #UD。

操作¶

当目的地是寄存器时 VPMOVDB 指令 (EVEX 编码版本)¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
FOR j := 0 TO KL-1
    i := j * 8
    m := j * 32
    IF k1[j] OR *no writemask*
        THEN DEST[i+7:i] := TruncateDoubleWordToByte (SRC[m+31:m])
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+7:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+7:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL/4] := 0;
