SYSEXIT — 从快速系统调用快速返回

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 35 SYSEXIT ZO Valid Valid 快速返回到特权级 3 用户代码。
REX.W + 0F 35 SYSEXIT ZO Valid Valid 快速返回到 64 位模式特权级 3 用户代码。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

执行对特权级 3 用户代码的快速返回。SYSEXIT 是 SYSENTER 指令的伴侣指令。该指令经过优化，旨在为从特权级 0 运行的系统程序到特权级 3 运行的用户程序的返回提供最大性能。它必须从特权级 0 执行的代码中执行。

当操作数大小为 64 位时，SYSEXIT 保持在 64 位模式；否则，它要么进入兼容模式（如果逻辑处理器在 IA-32e 模式下），要么保持在保护模式（如果不在该模式下）。

在执行 SYSEXIT 之前，软件必须通过向以下 MSR 和通用寄存器写入值来指定特权级 3 代码段 and 代码入口点，以及特权级 3 栈段 and 栈指针：

  • IA32_SYSENTER_CS（MSR 地址 174H）— 包含一个 32 位值，用于确定特权级 3 代码段和栈段的段选择子（见“操作”部分）。
  • RDX — 此寄存器中的规范地址被加载到 RIP 中（因此，此值引用用户代码中要执行的第一条指令）。如果返回目标不是 64 位模式，则仅加载 31:0 位。
  • ECX — 此寄存器中的规范地址被加载到 RSP 中（因此，此值包含特权级 3 栈的栈指针）。如果返回目标不是 64 位模式，则仅加载 31:0 位。

可以使用 RDMSR 和 WRMSR 读写 IA32_SYSENTER_CS MSR。

虽然 SYSEXIT 使用从 IA32_SYSENTER_CS MSR 派生的值加载 CS 和 SS 选择子，但 CS 和 SS 描述符缓存并不是从这些选择子引用的描述符（在 GDT 或 LDT 中）加载的。相反，描述符缓存被加载为固定值。详情请参阅“操作”部分。操作系统软件有责任确保这些选择子值引用的描述符（在 GDT 或 LDT 中）与加载到描述符缓存中的固定值相对应；SYSEXIT 指令不保证这种对应关系。

SYSEXIT 指令可以从除实地址模式和虚拟-8086 模式之外的所有运行模式下调用。

SYSENTER 和 SYSEXIT 指令是在 Pentium II 处理器中引入到 IA-32 架构中的。处理器上这些指令的可用性通过 CPUID 指令返回到 EDX 寄存器的 SYSENTER/SYSEXIT 存在 (SEP) 特征标志来指示。验证 SEP 标志的操作系统还必须验证处理器系列和型号，以确保 SYSENTER/SYSEXIT 指令确实存在。例如：

IF CPUID SEP bit is set

THEN IF (Family = 6) and (Model < 3) and (Stepping < 3)

THEN

SYSENTER/SYSEXIT_Not_Supported; FI;

ELSE

SYSENTER/SYSEXIT_Supported; FI;

FI;

当在 Pentium Pro 处理器（型号 1）上执行 CPUID 指令时，处理器返回的 SEP 标志为已设置，但不支持 SYSENTER/SYSEXIT 指令。

当在特权级 3 启用影子栈时，该指令将使用来自 IA32_PL3_SSP MSR 的值加载 SSP。有关额外的 CET 详细信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 6 章“中断和异常处理”以及第 17 章“控制流执行技术 (CET)”。

指令顺序。紧跟在 SYSEXIT 之后的指令可能会在较早的指令完成执行之前从内存中提取，但在 SYSEXIT 之前的所有指令完成执行之前，它们不会执行（即使是猜测性执行）（后面的指令可能会在较早指令存储的数据全局可见之前执行）。

操作¶

IF IA32_SYSENTER_CS[15:2] = 0 OR CR0.PE = 0 OR CPL ≠ 0 THEN #GP(0); FI;
IF operand size is 64-bit
    THEN (* 返回到 64 位模式 *)
        RSP := RCX;
        RIP := RDX;
    ELSE (* 返回到保护模式或兼容模式 *)
        RSP := ECX;
        RIP := EDX;
FI;
IF operand size is 64-bit (* 操作系统提供 CS；RPL 强制为 3 *)
    THEN CS.Selector := IA32_SYSENTER_CS[15:0] + 32;
    ELSE CS.Selector := IA32_SYSENTER_CS[15:0] + 16;
FI;
CS.Selector := CS.Selector OR 3;
            (* RPL 强制为 3 *)
(* 将 CS 的其余部分设置为固定值 *)
CS.Base := 0;
            (* 扁平段 *)
CS.Limit := FFFFFH;
            (* 具有 4KB 粒度，意味着 4GB 界限 *)
CS.Type := 11;
            (* 执行/读取代码，已访问 *)
CS.S := 1;
CS.DPL := 3;
CS.P := 1;
IF operand size is 64-bit
    THEN (* 返回到 64 位模式 *)
        CS.L := 1;
            (* 64位代码段 *)
        CS.D := 0;
    ELSE (* 返回到保护模式或兼容模式 *)
        CS.L := 0;
        CS.D := 1;
            (* 32位代码段 *)
FI;
CS.G := 1;
            (* 4KB 粒度 *)
CPL := 3;
IF ShadowStackEnabled(CPL)
    THEN SSP := IA32_PL3_SSP;
FI;
SS.Selector := CS.Selector + 8;
            (* SS 紧随 CS 之后 *)
(* 将 SS 的其余部分设置为固定值 *)
SS.Base := 0;
            (* 扁平段 *)
SS.Limit := FFFFFH;
            (* 具有 4KB 粒度，意味着 4GB 界限 *)
SS.Type := 3;
            (* 读取/写入数据，已访问 *)
SS.S := 1;
SS.DPL := 3;
SS.P := 1;
SS.B := 1;
            (* 32位栈段 *)
SS.G := 1; (* 4KB 粒度 *)

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果 IA32_SYSENTER_CS[15:2] = 0。
如果 CPL != 0。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP SYSEXIT 指令在实地址模式下无法识别。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) SYSEXIT 指令在虚拟-8086 模式下无法识别。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#GP(0) 如果 IA32_SYSENTER_CS = 0。
如果 CPL != 0。
如果 RCX 或 RDX 包含非规范地址。
#UD 如果使用了 LOCK 前缀。
