LSL — 加载段界限

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 03 /r LSL r16, r16/m16 RM 有效 有效 加载：r16 := 段界限，选择子为 r16/m16。
0F 03 /r LSL r32, r32/m161 RM 有效 有效 加载：r32 := 段界限，选择子为 r32/m16。
REX.W + 0F 03 /r LSL r64, r32/m161 RM 有效 有效 加载：r64 := 段界限，选择子为 r32/m16。

1. 对于所有加载（无论目的操作数的大小如何），仅使用位 16-0。其他位被忽略。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

将第二个操作数（源操作数）指定的段描述符中重组的段界限加载到第一个操作数（目的操作数）中，并设置 EFLAGS 寄存器中的 ZF 标志。源操作数（可以是寄存器或内存位置）包含要访问的段描述符的段选择子。目的操作数是通用寄存器。

作为加载过程的一部分，处理器执行访问检查。一旦加载到目的寄存器中，软件就可以将段界限与指针的偏移量进行比较。

段界限是一个 20 位的值，包含在段描述符的字节 0 和 1 以及字节 6 的前 4 位中。如果描述符具有字节精细度段界限（粒度标志设为 0），则目的操作数将被加载字节精细度值（字节界限）。如果描述符具有页精细度段界限（粒度标志设为 1），则 LSL 指令在将页精细度界限（页界限）加载到目的操作数之前，会将其转换为字节界限。该转换是通过将 20 位“原始”界限左移 12 位，并将低 12 位填充为 1 来完成的。

当操作数大小为 32 位时，32 位字节界限将存储在目的操作数中。当操作数大小为 16 位时，会计算出一个有效的 32 位界限；但高 16 位被截断，只有低 16 位被加载到目的操作数中。

此指令在将段界限加载到目的寄存器之前执行以下检查：

  • 检查段选择子是否不为 NULL。
  • 检查段选择子是否指向在要访问的 GDT 或 LDT 限制内的描述符。
  • 检查该描述符类型对于此指令是否有效。所有代码和数据段描述符对于 LSL 指令均有效（可使用 LSL 指令访问）。合法的特殊段和门描述符类型在下表中给出。
  • 如果段不是一致代码段，该指令会检查指定的段描述符在 CPL 处是否可见（即，如果段选择子的 CPL 和 RPL 小于或等于段选择子的 DPL）。

如果段描述符无法访问或对于该指令是无效类型，则清除 ZF 标志，且目的操作数中不会加载任何值。

1. 在这种情况下，描述符由 16 字节组成；高 4 字节的第 12:8 位必须为 0。

IF SRC(Offset) > descriptor table limit
    THEN ZF := 0; FI;
Read segment descriptor;
IF SegmentDescriptor(Type) ≠ conforming code segment

or Segment type is not valid for instruction

受影响的标志位¶

如果成功加载段界限，则将 ZF 标志设置为 1；否则，将其设置为 0。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存且它包含一个 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出了 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时内存操作数有效地址未对齐。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD LSL 指令在实地址模式下无法执行。

虚拟-8086 模式异常¶

#UD LSL 指令在虚拟-8086 模式下无法执行。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果指向 SS 段的内存操作数有效地址为非规范形式。
#GP(0) 如果内存操作数有效地址为非规范形式。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时内存操作数有效地址未对齐。
#UD 如果使用了 LOCK 前缀。
