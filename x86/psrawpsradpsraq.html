描述¶

将目的操作数（第一个操作数）中各个数据元素（字、双字或四字）的位向右移位，位移量由计数操作数（第二个操作数）指定的位数决定。当数据元素中的位向右移位时，空出的高位被填充为该数据元素符号位的初始值。如果计数操作数指定的值大于 15（对于字）、31（对于双字）或 63（对于四字），则目的操作数的每个数据元素都将被填充为该元素符号位的初始值。（图 4-18 给出了在 64 位操作数中移位字的示例。）

注意，仅检查 128 位移位计数操作数的前 64 位来计算移位计数。如果第二个源操作数是内存地址，则加载 128 位。

(V)PSRAW 指令将目的操作数中的每个字右移计数操作数指定的位数，(V)PSRAD 指令将目的操作数中的每个双字右移。

在 64 位模式下且未在 VEX/EVEX 中编码时，使用 REX.R 形式的 REX 前缀允许此指令访问其他寄存器（XMM8-XMM15）。

传统 SSE 指令 64 位操作数：目的操作数是 MMX 技术寄存器；计数操作数可以是 MMX 技术寄存器或 64 位内存位置。

128位传统 SSE 版本：目的操作数和第一个源操作数是 XMM 寄存器。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。计数操作数可以是 XMM 寄存器、128 位内存位置或 8 位立即数。如果计数操作数是内存地址，则加载 128 位，但高 64 位被忽略。

VEX.128 编码版本：目的操作数和第一个源操作数是 XMM 寄存器。目的 YMM 寄存器的位（MAXVL-1:128）清零。计数操作数可以是 XMM 寄存器、128 位内存位置或 8 位立即数。如果计数操作数是内存地址，则加载 128 位，但高 64 位被忽略。

VEX.256 编码版本：目的操作数是 YMM 寄存器。源操作数是 YMM 寄存器或内存位置。计数操作数可以来自 XMM 寄存器、内存位置或 8 位立即数。对应 ZMM 寄存器的位（MAXVL-1:256）清零。

EVEX 编码版本：目的操作数是根据写掩码更新的 ZMM 寄存器。计数操作数可以是 8 位立即数（立即数计数版本）或来自 XMM 寄存器或内存位置的 8 位值（变量计数版本）。对于立即数计数版本，源操作数（第二个操作数）可以是 ZMM 寄存器、512 位内存位置，或从 32/64 位内存位置广播的 512 位向量。对于变量计数版本，第一个源操作数（第二个操作数）是 ZMM 寄存器，第二个源操作数（第三个操作数，8 位变量计数）可以是 XMM 寄存器或内存位置。

注意：在带有立即数计数的 VEX/EVEX 编码移位版本中，VEX/EVEX 的 vvvv 编码目的寄存器，而 VEX.B/EVEX.B + ModRM.r/m 编码源寄存器。

注意：对于带有立即数计数的移位（VEX.128.66.0F 71-73 /4，EVEX.128.66.0F 71-73 /4），VEX.vvvv/EVEX.vvvv 编码目的寄存器。

VPSRAW（EVEX 版本，xmm/m128）¶

(KL, VL) = (8, 128), (16, 256), (32, 512)
IF VL = 128
    TMP_DEST[127:0] := ARITHMETIC_RIGHT_SHIFT_WORDS_128b(SRC1[127:0], SRC2)
FI;
IF VL = 256
    TMP_DEST[255:0] := ARITHMETIC_RIGHT_SHIFT_WORDS_256b(SRC1[255:0], SRC2)
FI;
IF VL = 512
    TMP_DEST[255:0] := ARITHMETIC_RIGHT_SHIFT_WORDS_256b(SRC1[255:0], SRC2)
    TMP_DEST[511:256] := ARITHMETIC_RIGHT_SHIFT_WORDS_256b(SRC1[511:256], SRC2)
FI;
FOR j := 0 TO KL-1
    i := j * 16
    IF k1[j] OR *no writemask*
        THEN DEST[i+15:i] := TMP_DEST[i+15:i]
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+15:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+15:i] = 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
