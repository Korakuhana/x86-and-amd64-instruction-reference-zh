FICOM/FICOMP — 比较整数

操作码 指令 64位模式 兼容/传统模式 描述
DE /2 FICOM m16int 有效 有效 将 ST(0) 与 m16int 进行比较。
DA /2 FICOM m32int 有效 有效 将 ST(0) 与 m32int 进行比较。
DE /3 FICOMP m16int 有效 有效 将 ST(0) 与 m16int 进行比较并弹出寄存器栈。
DA /3 FICOMP m32int 有效 有效 将 ST(0) 与 m32int 进行比较并弹出寄存器栈。

描述¶

将 ST(0) 中的值与整数源操作数进行比较，并根据结果设置 FPU 状态字中的条件码标志 C0、C2 和 C3（见下表）。在进行比较之前，整数值会被转换为双精度扩展浮点格式。

这些指令执行“无序比较”。无序比较还会检查正在比较的数字类别（请参阅本章中的“FXAM—检查浮点”）。如果任一操作数是 NaN 或处于未定义格式，则条件标志会被设置为“无序”。

零的符号被忽略，因此 –0.0 := +0.0。

FICOMP 指令在比较后弹出寄存器栈。为了弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

CASE (relation of operands) OF
    ST(0) > SRC:
            C3, C2, C0 := 000;
    ST(0) < SRC:
            C3, C2, C0 := 001;
    ST(0) = SRC:
            C3, C2, C0 := 100;
    Unordered:
            C3, C2, C0 := 111;
ESAC;
IF Instruction = FICOMP
    THEN
        PopRegisterStack;
FI;

影响的 FPU 标志寄存器¶

C1 设置为 0。
C0, C2, C3 见前页表格。

浮点异常¶

#IS 发生栈下溢。
#IA 一个或两个操作数是 NaN 值，或具有不受支持的格式。
#D 一个或两个操作数是渐进下溢（非正规化）值。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含空段选择器。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

#GP(0) 如果内存地址为非规范形式。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在未决的 x87 FPU 异常。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
