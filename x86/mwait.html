MWAIT — 监控等待

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 01 C9 MWAIT ZO 有效 有效 一个提示，允许处理器停止指令执行并进入与实现相关的优化状态，直到发生某类事件。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

MWAIT 指令提供提示以允许处理器进入与实现相关的优化状态。主要有两个目标用途：地址范围监控和高级电源管理。MWAIT 的这两种用途都需要使用 MONITOR 指令。

CPUID.01H:ECX.MONITOR[位 3] 指示处理器中 MONITOR 和 MWAIT 的可用性。当置位时，MWAIT 只能在特权级 0 执行（在任何其他特权级使用会导致无效操作码异常）。操作系统或系统 BIOS 可以通过使用 IA32_MISC_ENABLE MSR 禁用此指令；禁用 MWAIT 会清除 CPUID 特征标志并导致执行产生无效操作码异常。

此指令在非64位模式和64位模式下的操作相同。

ECX 指定 MWAIT 指令的可选扩展。EAX 可以包含诸如处理器应进入的首选优化状态等提示。首批实现 MWAIT 的处理器仅支持 EAX 和 ECX 为零值。后来的处理器允许设置 ECX[0] 以将屏蔽的中断启用为 MWAIT 的唤醒事件（见下文）。软件可以使用 CPUID 指令来确定处理器支持的扩展和提示。

MWAIT 用于地址范围监控¶

对于地址范围监控，MWAIT 指令与 MONITOR 指令配合操作。这两个指令允许定义等待的地址（MONITOR）和在等待地址处开始的与实现相关的优化操作（MWAIT）。执行 MWAIT 是对处理器的提示，提示它可以在等待事件或对 MONITOR 设防的地址范围进行存储操作时，进入与实现相关的优化状态。

以下情况会导致处理器退出与实现相关的优化状态：对 MONITOR 指令设防的地址范围进行存储、NMI 或 SMI、调试异常、机器检查异常、BINIT# 信号、INIT# 信号和 RESET# 信号。其他与实现相关的事件也可能导致处理器退出与实现相关的优化状态。

此外，在以下情况下，外部中断也会导致处理器退出与实现相关的优化状态：(1) 如果中断将递交给软件（例如，就像执行了 HLT 而不是 MWAIT 一样）；或 (2) 如果 ECX[0] = 1。只有在 CPUID.05H:ECX[位 1] = 1 时，软件才能在 ECX[0] = 1 的情况下执行 MWAIT。（特定于实现的条件可能导致即使屏蔽了中断且 ECX[0] = 0，中断仍会导致处理器退出与实现相关的优化状态。）

退出与实现相关的优化状态后，控制权转移到 MWAIT 指令之后的指令。在执行该指令之前，可能会递交未屏蔽的挂起中断（包括 NMI 或 SMI）。与 HLT 指令不同，MWAIT 指令不支持在处理 SMI 后在 MWAIT 指令处重新开始。

如果前面的 MONITOR 指令没有成功对地址范围设防，或者在执行 MWAIT 之前没有执行 MONITOR 指令，则处理器不会进入与实现相关的优化状态。执行将在 MWAIT 之后的指令处恢复。

MWAIT 用于电源管理¶

MWAIT 接受对处理器的提示和可选扩展，使其在等待事件或对 MONITOR 设防的地址范围进行存储操作时，可以进入指定的目标 C 状态。对用于电源管理的 MWAIT 扩展的支持由 CPUID.05H:ECX[位 0] 报告为 1 来指示。

EAX 和 ECX 用于向 MWAIT 指令传递附加信息，例如处理器应进入的优化状态类型。ECX 指定 MWAIT 指令的可选扩展。EAX 可以包含诸如处理器首选的优化状态等提示。特定于实现的条件可能导致处理器忽略该提示并进入不同的优化状态。未来的处理器实现可能会实现多个优化的“等待”状态，并将根据提示参数在这些状态之间进行选择。

注意，如果使用 MWAIT 进入数值上大于 C1 的任何 C 状态，则只有当存储是由其他处理器代理发起时，对 MONITOR 指令设防的地址范围的存储才会导致处理器退出 MWAIT。在这种情况下，来自非处理器代理的存储可能不会导致处理器退出 MWAIT。

示例¶

MONITOR/MWAIT 指令对必须编写在同一个循环中，因为执行 MWAIT 指令会触发监控硬件。仅执行一次 MONITOR 然后在循环中执行 MWAIT 是不正确的用法。仅设置 MONITOR 而不执行 MWAIT 没有不良影响。

通常，MONITOR/MWAIT 对按以下顺序使用：

EAX = Logical Address(Trigger)

ECX = 0 (*Hints *)

EDX = 0 (* Hints *)

IF ( !trigger_store_happened) {

MONITOR EAX, ECX, EDX

IF ( !trigger_store_happened ) {

MWAIT EAX, ECX

}

}

上述代码序列确保在第一次检查触发条件和执行监控指令之间不会发生触发性存储。如果没有第二次检查，该触发性存储将不会被察觉。MONITOR 和 MWAIT 的典型用法是将上述代码序列置于循环中。

数值异常¶

无。

保护模式异常¶

#GP(0) 如果 ECX[31:1] ≠ 0。
如果 ECX[0] = 1 且 CPUID.05H:ECX[位 1] = 0。
#UD 如果 CPUID.01H:ECX.MONITOR[位 3] = 0。
如果当前特权级不为 0。

实地址模式异常¶

#GP 如果 ECX[31:1] ≠ 0。
如果 ECX[0] = 1 且 CPUID.05H:ECX[位 1] = 0。
#UD 如果 CPUID.01H:ECX.MONITOR[位 3] = 0。

虚拟-8086 模式异常¶

#UD 在虚拟-8086模式下无法识别 MWAIT 指令（即使 CPUID.01H:ECX.MONITOR[位 3] = 1）。

兼容模式异常¶

与保护模式下的异常相同。

#GP(0) 如果 RCX[63:1] ≠ 0。
如果 RCX[0] = 1 且 CPUID.05H:ECX[位 1] = 0。
#UD 如果当前特权级不为 0。
如果 CPUID.01H:ECX.MONITOR[位 3] = 0。
