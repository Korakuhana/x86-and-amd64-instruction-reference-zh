CVTPD2DQ — 将打包双精度浮点值转换为打包双字整数

操作码/指令 Op / En 64/32位模式支持 CPUID 特征标志 描述
F2 0F E6 /r CVTPD2DQ xmm1, xmm2/m128 A V/V SSE2 将两个来自 xmm2/mem 的打包双精度浮点值转换为 xmm1 中的两个有符号双字整数。
VEX.128.F2.0F.WIG E6 /r VCVTPD2DQ xmm1, xmm2/m128 A V/V AVX 将两个来自 xmm2/mem 的打包双精度浮点值转换为 xmm1 中的两个有符号双字整数。
VEX.256.F2.0F.WIG E6 /r VCVTPD2DQ xmm1, ymm2/m256 A V/V AVX 将四个来自 ymm2/mem 的打包双精度浮点值转换为 xmm1 中的四个有符号双字整数。
EVEX.128.F2.0F.W1 E6 /r VCVTPD2DQ xmm1 {k1}{z}, xmm2/m128/m64bcst B V/V AVX512VL AVX512F 在写入掩码 k1 下，将两个来自 xmm2/m128/m64bcst 的打包双精度浮点值转换为 xmm1 中的两个有符号双字整数。
EVEX.256.F2.0F.W1 E6 /r VCVTPD2DQ xmm1 {k1}{z}, ymm2/m256/m64bcst B V/V AVX512VL AVX512F 在写入掩码 k1 下，将四个来自 ymm2/m256/m64bcst 的打包双精度浮点值转换为 xmm1 中的四个有符号双字整数。
EVEX.512.F2.0F.W1 E6 /r VCVTPD2DQ ymm1 {k1}{z}, zmm2/m512/m64bcst{er} B V/V AVX512F 在写入掩码 k1 下，将八个来自 zmm2/m512/m64bcst 的打包双精度浮点值转换为 ymm1 中的八个有符号双字整数。

描述¶

将源操作数（第二操作数）中的打包双精度浮点值转换为目标操作数（第一操作数）中的打包有符号双字整数。

当转换不精确时，返回的值将根据 MXCSR 寄存器中的舍入控制位或嵌入式舍入控制位进行舍入。如果转换后的结果无法以目标格式表示，则会引发浮点无效异常，并且如果该异常被屏蔽，则返回不确定整数值（2w-1，其中 w 表示目标格式的位数）。

EVEX 编码版本：源操作数是 ZMM/YMM/XMM 寄存器、512 位内存位置或从 64 位内存位置广播的 512 位向量。目标操作数是 ZMM/YMM/XMM 寄存器，在写入掩码 k1 下有条件地更新。相应目标的高位（MAXVL-1:256/128/64）清零。

VEX.256 编码版本：源操作数是 YMM 寄存器或 256 位内存位置。目标操作数是 XMM 寄存器。目标 ZMM 寄存器的相应高位（MAXVL-1:128）清零。

VEX.128 编码版本：源操作数是 XMM 寄存器或 128 位内存位置。目标操作数是 XMM 寄存器。目标 ZMM 寄存器的相应高位（MAXVL-1:64）清零。

128 位传统 SSE 版本：源操作数是 XMM 寄存器或 128 位内存位置。目标操作数是 XMM 寄存器。目标 XMM 寄存器的 Bits[127:64] 清零。然而，目标 ZMM 寄存器的相应高位（MAXVL-1:128）保持不变。

VEX.vvvv 和 EVEX.vvvv 是保留的，必须为 1111b，否则指令将触发 #UD 异常。

操作¶

VCVTPD2DQ (EVEX Encoded Versions) When SRC Operand is a Register¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
IF (VL = 512) AND (EVEX.b = 1)
	THEN
		SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(EVEX.RC);
	ELSE
		SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(MXCSR.RC);
FI;
FOR j := 0 TO KL-1
	i := j * 32
	k := j * 64
	IF k1[j] OR *no writemask*
		THEN DEST[i+31:i] :=
			Convert_Double_Precision_Floating_Point_To_Integer(SRC[k+63:k])
		ELSE
			IF *merging-masking* ; merging-masking
				THEN *DEST[i+31:i] remains unchanged*
				ELSE
						; zeroing-masking
					DEST[i+31:i] := 0
			FI
	FI;
ENDFOR
DEST[MAXVL-1:VL/2] := 0

VCVTPD2DQ (EVEX Encoded Versions) When SRC Operand is a Memory Source¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
	i := j * 32
	k := j * 64
	IF k1[j] OR *no writemask*
		THEN
			IF (EVEX.b = 1)
				THEN
					DEST[i+31:i] :=
			Convert_Double_Precision_Floating_Point_To_Integer(SRC[63:0])
				ELSE
					DEST[i+31:i] :=
			Convert_Double_Precision_Floating_Point_To_Integer(SRC[k+63:k])
			FI;
		ELSE
			IF *merging-masking* ; merging-masking
				THEN *DEST[i+31:i] remains unchanged*
				ELSE ; zeroing-masking
					DEST[i+31:i] := 0
			FI
	FI;
ENDFOR
DEST[MAXVL-1:VL/2] := 0

VCVTPD2DQ (VEX.256 Encoded Version)¶

DEST[31:0] := Convert_Double_Precision_Floating_Point_To_Integer(SRC[63:0])
DEST[63:32] := Convert_Double_Precision_Floating_Point_To_Integer(SRC[127:64])
DEST[95:64] := Convert_Double_Precision_Floating_Point_To_Integer(SRC[191:128])
DEST[127:96] := Convert_Double_Precision_Floating_Point_To_Integer(SRC[255:192)
DEST[MAXVL-1:128] := 0

VCVTPD2DQ (VEX.128 Encoded Version)¶

DEST[31:0] := Convert_Double_Precision_Floating_Point_To_Integer(SRC[63:0])
DEST[63:32] := Convert_Double_Precision_Floating_Point_To_Integer(SRC[127:64])
DEST[MAXVL-1:64] := 0

CVTPD2DQ (128-bit Legacy SSE Version)¶

DEST[31:0] := Convert_Double_Precision_Floating_Point_To_Integer(SRC[63:0])
DEST[63:32] := Convert_Double_Precision_Floating_Point_To_Integer(SRC[127:64])
DEST[127:64] := 0
DEST[MAXVL-1:128] (unmodified)
