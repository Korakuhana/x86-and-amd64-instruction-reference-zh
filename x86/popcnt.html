POPCNT — 返回设置为 1 的位数

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
F3 0F B8 /r POPCNT r16, r/m16 RM 有效 有效 计算 r/m16 中设置为 1 的位数
F3 0F B8 /r POPCNT r32, r/m32 RM 有效 有效 计算 r/m32 中设置为 1 的位数
F3 REX.W 0F B8 /r POPCNT r64, r/m64 RM 有效 无效 计算 r/m64 中设置为 1 的位数

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

此指令计算第二个操作数（源操作数）中设置为 1 的位数，并在第一个操作数（目的寄存器）中返回该计数。

操作¶

Count = 0;
For (i=0; i < OperandSize; i++)
{ IF (SRC[ i] = 1) // i’th bit
    THEN Count++; FI;
}
DEST := Count;

受影响的标志位¶

OF, SF, ZF, AF, CF, PF 全部被清零。如果 SRC = 0，则设置 ZF，否则清除 ZF。

Intel C/C++ 编译器等价内部函数¶

POPCNT int _mm_popcnt_u32(unsigned int a);
POPCNT int64_t _mm_popcnt_u64(unsigned __int64 a);

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段的限制。
#SS(0) 如果内存操作数有效地址超出 SS 段的限制。
#PF （错误码）如果是缺页异常。
#AC(0) 如果在当前特权级为 3 且启用了对齐检查时进行了未对齐的内存引用。
#UD 如果 CPUID.01H:ECX.POPCNT [位 23] = 0。
如果使用了 LOCK 前缀。

#GP(0) 如果操作数的任何部分超出从 0 到 0FFFFH 的有效地址空间。
#SS(0) 如果内存操作数有效地址超出 SS 段的限制。
#UD 如果 CPUID.01H:ECX.POPCNT [位 23] = 0。
如果使用了 LOCK 前缀。
