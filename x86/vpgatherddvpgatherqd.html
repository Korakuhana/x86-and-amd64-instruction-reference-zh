描述¶

该指令使用双字索引，有条件地从内存操作数（第二操作数）指定的内存地址加载最多 4 或 8 个双字值。内存操作数使用 SIB 字节的 VSIB 形式，来指定一个通用寄存器操作数作为通用基址，指定一个向量寄存器来存放相对于该基址的索引数组，以及一个常数比例因子。

掩码操作数（第三操作数）指定从每个内存地址的条件加载操作以及目的地操作数（第一操作数）中每个数据元素的相应更新。条件性由掩码寄存器中每个数据元素的最显著位指定。如果某元素的掩码位未设置，则目的地寄存器的对应元素保持不变。目的地寄存器和掩码寄存器中的数据元素宽度是相同的。除非指令引发异常，否则此指令会将整个掩码寄存器设置为 0。

使用四字索引，该指令有条件地从 VSIB 寻址内存操作数中加载最多 2 或 4 个四字值，并更新目的地寄存器的下半部分。在四字索引下，目的寄存器的高 128 或 256 位清零。

如果至少已经收集了一个元素（即，如果异常是由其掩码位置位的最右侧元素以外的元素触发的），则该指令可以被异常暂停。当发生这种情况时，目的地寄存器和掩码操作数会被部分更新；那些已经收集的元素被放入目的寄存器中，且它们的掩码位被清零。如果来自已收集元素的任何陷阱或中断处于挂起状态，它们将代替异常被递送；在这种情况下，EFLAG.RF 被设置为 1，以便在指令继续执行时不会重新触发指令断点。

如果数据大小和索引大小不同，目的地寄存器的一部分和掩码寄存器的一部分不对应于正在收集的任何元素。此指令将这些部分设置为 0。即使指令触发异常，甚至在收集任何元素之前触发异常，它也可能对这两个寄存器中的一个或两个执行此操作。

VEX.128 版本：对于双字索引，该指令将收集四个双字值。对于四字索引，该指令将收集两个值，并将目的地的上 64 位清零。

VEX.256 版本：对于双字索引，该指令将收集八个双字值。对于四字索引，该指令将收集四个值，并将目的地的上 128 位清零。

注意：

  • 如果索引、掩码或目的地寄存器中任何两个寄存器相同，此指令会导致 UD 故障。
  • 可以以任何顺序从内存中读取这些值。与其他指令的内存排序遵循 Intel-64 memory-ordering 模型。
  • 故障以从右到左的方式递送。也就是说，如果一个元素触发了故障并递送，所有更靠近目的地 LSB 的元素都将被完成（且无故障）。更靠近 MSB 的个别元素可能完成，也可能未完成。如果给定元素触发多个故障，它们将按常规顺序递送。
  • 元素可以以任何顺序收集，但故障必须按从右到左的顺序递送；因此，在递送故障之前，可能会收集故障元素左侧的元素。此指令的给定实现是可重复的——在相同的输入值和架构状态下，将收集故障元素左侧的同一组元素。
  • 此指令不执行 AC 检查，因此永远不会递送 AC 故障。
  • 如果地址大小属性为 16 位，此指令将导致 #UD。
  • 如果内存操作数的编码没有 SIB 字节，此指令将导致 #UD。
  • 此指令不应该用于访问内存映射 I/O，因为它所执行的单个加载的排序是特定于实现的，某些实现可能会使用大于数据元素大小的加载，或者加载元素不确定的次数。
  • 与处理器使用的地址位相比，缩放后的索引可能需要更多位来表示（例如，在 32 位模式下，如果比例因子大于 1）。在这种情况下，超出地址位数的最显著位将被忽略。

操作¶

DEST := SRC1;
BASE_ADDR: base register encoded in VSIB addressing;
VINDEX: the vector index register encoded by VSIB addressing;
SCALE: scale factor encoded by SIB:[7:6];
DISP: optional 1, 4 byte displacement;
MASK := SRC3;

VPGATHERDD (VEX.128 版本)¶

MASK[MAXVL-1:128] := 0;
FOR j := 0 to 3
    i := j * 32;
    IF MASK[31+i] THEN
        MASK[i +31:i] := FFFFFFFFH; // extend from most significant bit
    ELSE
        MASK[i +31:i] := 0;
    FI;
ENDFOR
FOR j := 0 to 3
    i := j * 32;
    DATA_ADDR := BASE_ADDR + (SignExtend(VINDEX[i+31:i])*SCALE + DISP;
    IF MASK[31+i] THEN
        DEST[i +31:i] := FETCH_32BITS(DATA_ADDR); // a fault exits the instruction
    FI;
    MASK[i +31:i] := 0;
ENDFOR
DEST[MAXVL-1:128] := 0;
