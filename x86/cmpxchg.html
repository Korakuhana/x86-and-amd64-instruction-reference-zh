CMPXCHG — 比较并交换

操作码/指令 Op/En 64位模式 兼容/传统模式 描述
0F B0/r CMPXCHG r/m8, r8 MR Valid Valid* 将 AL 与 r/m8 进行比较。如果相等，则设置 ZF 并将 r8 加载到 r/m8 中。否则，清除 ZF 并将 r/m8 加载到 AL。
REX + 0F B0/r CMPXCHG r/m8**,r8 MR Valid N.E. 将 AL 与 r/m8 进行比较。如果相等，则设置 ZF 并将 r8 加载到 r/m8 中。否则，清除 ZF 并将 r/m8 加载到 AL。
0F B1/r CMPXCHG r/m16, r16 MR Valid Valid* 将 AX 与 r/m16 进行比较。如果相等，则设置 ZF 并将 r16 加载到 r/m16 中。否则，清除 ZF 并将 r/m16 加载到 AX。
0F B1/r CMPXCHG r/m32, r32 MR Valid Valid* 将 EAX 与 r/m32 进行比较。如果相等，则设置 ZF 并将 r32 加载到 r/m32 中。否则，清除 ZF 并将 r/m32 加载到 EAX。
REX.W + 0F B1/r CMPXCHG r/m64, r64 MR Valid N.E. 将 RAX 与 r/m64 进行比较。如果相等，则设置 ZF 并将 r64 加载到 r/m64 中。否则，清除 ZF 并将 r/m64 加载到 RAX。

* 请参阅下文的 IA-32 架构兼容性部分。

** 在 64 位模式下，如果使用 REX 前缀，则无法将 r/m8 编码为访问以下字节寄存器：AH、BH、CH、DH。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
MR ModRM:r/m (r, w) ModRM:reg (r) N/A N/A

描述¶

将 AL, AX, EAX 或 RAX 寄存器中的值与第一个操作数（目的操作数）进行比较。如果这两个值相等，则将第二个操作数（源操作数）加载到目的操作数中。否则，将目的操作数加载到 AL, AX, EAX 或 RAX 寄存器中。RAX 寄存器仅在 64 位模式下可用。

此指令可以配合 LOCK 前缀使用，以允许指令原子地执行。为了简化与处理器总线的接口，不论比较结果如何，目的操作数都会接收一个写周期。如果比较失败，则将目的操作数写回；否则，将源操作数写入目的操作数。（处理器从不产生只有锁定读而没有锁定写的情况。）

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 前缀可将操作提升至 64 位。请参阅本节开头的汇总图表以获取编码数据和限制。

IA-32 架构兼容性¶

早于 Intel486 处理器的 Intel 处理器不支持此指令。

操作¶

(* Accumulator = AL, AX, EAX, or RAX depending on whether a byte, word, doubleword, or quadword comparison is being performed *)
TEMP := DEST
IF accumulator = TEMP
    THEN
        ZF := 1;
        DEST := SRC;
    ELSE
        ZF := 0;
        accumulator := TEMP;
        DEST := TEMP;
FI;

受影响的标志位¶

如果目的操作数中的值与 AL、AX 或 EAX 寄存器中的值相等，则设置 ZF 标志；否则清除该标志。CF、PF、AF、SF 和 OF 标志根据比较操作的结果进行设置。

保护模式异常¶

#GP(0) 如果目的地位于不可写的段中。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀，但目的地不是内存操作数。
