描述 ¶

将第一个操作数（目的操作数）的位向左或向右移位（循环移位）第二个操作数（计数操作数）所指定的位数，并将结果存储在目的操作数中。目的操作数可以是寄存器或内存地址；计数操作数是一个无符号整数，可以是立即数或 CL 寄存器中的值。计数被屏蔽（掩码）为 5 位（如果在 64 位模式下且 REX.W = 1，则为 6 位）。

循环左移 (ROL) 和带进位循环左移 (RCL) 指令将所有位向更高有效位位置移位，但最高有效位除外，它会被循环移动到最低有效位位置。循环右移 (ROR) 和带进位循环右移 (RCR) 指令将所有位向更低有效位位置移位，但最低有效位除外，它会被循环移动到最高有效位位置。

RCL 和 RCR 指令在循环移位中包含 CF 标志位。RCL 指令将 CF 标志位移入最低有效位，并将最高有效位移入 CF 标志位。RCR 指令将 CF 标志位移入最高有效位，并将最低有效位移入 CF 标志位。对于 ROL 和 ROR 指令，CF 标志的原始值不是结果的一部分，但 CF 标志会接收从一端移位到另一端的位的副本。

OF 标志仅对 1 位循环移位有定义；在所有其他情况下均未定义（仅 RCL 和 RCR 指令除外：0 位循环移位不执行任何操作，即不影响任何标志）。对于左循环移位，OF 标志设置为 CF 位（旋转后）与结果的最高有效位的异或（XOR）。对于右循环移位，OF 标志设置为结果的最高两个有效位的异或（XOR）。

在 64 位模式下，使用 REX.R 形式的 REX 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 将第一个操作数提升至 64 位，并使计数操作数变为 6 位计数器。

IA-32 架构兼容性 ¶

8086 不会屏蔽循环移位计数。然而，所有其他 IA-32 处理器（从 Intel 286 处理器开始）都会将循环移位计数屏蔽为 5 位，从而实现最大计数 31。这种屏蔽操作在所有工作模式下进行（包括虚拟-8086 模式），以减少指令的最大执行时间。

操作 ¶

(* RCL and RCR Instructions *) ¶ 

SIZE := OperandSize;
CASE (determine count) OF
    SIZE := 8:
        tempCOUNT := (COUNT AND 1FH) MOD 9;
    SIZE := 16:
        tempCOUNT := (COUNT AND 1FH) MOD 17;
    SIZE := 32:
        tempCOUNT := COUNT AND 1FH;
    SIZE := 64:
        tempCOUNT := COUNT AND 3FH;
esac;
IF OperandSize = 64
    THEN COUNTMASK = 3FH;
    ELSE COUNTMASK = 1FH;
FI;

(* RCL Instruction Operation *) ¶ 

WHILE (tempCOUNT ≠ 0)
    DO
        tempCF := MSB(DEST);
        DEST := (DEST ∗ 2) + CF;
        CF := tempCF;
        tempCOUNT := tempCOUNT – 1;
    OD;
ELIHW;
IF (COUNT & COUNTMASK) = 1
    THEN OF := MSB(DEST) XOR CF;
    ELSE OF is undefined;
FI;

(* RCR Instruction Operation *) ¶ 

IF (COUNT & COUNTMASK) = 1
    THEN OF := MSB(DEST) XOR CF;
    ELSE OF is undefined;
FI;
WHILE (tempCOUNT ≠ 0)
    DO
        tempCF := LSB(SRC);
        DEST := (DEST / 2) + (CF * 2SIZE);
        CF := tempCF;
        tempCOUNT := tempCOUNT – 1;
    OD;

(* ROL Instruction Operation *) ¶ 

tempCOUNT := (COUNT & COUNTMASK) MOD SIZE
WHILE (tempCOUNT ≠ 0)
    DO
        tempCF := MSB(DEST);
        DEST := (DEST ∗ 2) + tempCF;
        tempCOUNT := tempCOUNT – 1;
    OD;
ELIHW;
IF (COUNT & COUNTMASK) ≠ 0
    THEN CF := LSB(DEST);
FI;
IF (COUNT & COUNTMASK) = 1
    THEN OF := MSB(DEST) XOR CF;
    ELSE OF is undefined;
FI;

(* ROR Instruction Operation *) ¶ 

tempCOUNT := (COUNT & COUNTMASK) MOD SIZE
WHILE (tempCOUNT ≠ 0)
    DO
        tempCF := LSB(SRC);
        DEST := (DEST / 2) + (tempCF ∗ 2SIZE);
        tempCOUNT := tempCOUNT – 1;
    OD;
ELIHW;
IF (COUNT & COUNTMASK) ≠ 0
    THEN CF := MSB(DEST);
FI;
IF (COUNT & COUNTMASK) = 1
    THEN OF := MSB(DEST) XOR MSB − 1(DEST);
    ELSE OF is undefined;
FI;

受影响的标志位 ¶

对于 RCL 和 RCR 指令，0 位循环移位不执行任何操作，即不影响任何标志。对于 ROL 和 ROR 指令，如果屏蔽后的计数为 0，则标志不受影响。如果屏蔽后的计数为 1，则 OF 标志受到影响，否则（屏蔽后的计数大于 1）OF 标志未定义。

对于所有指令，当屏蔽后的计数非零时，CF 标志受到影响。SF、ZF、AF 和 PF 标志始终不受影响。
