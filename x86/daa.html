DAA — 加法后的十进制调整 AL

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
27 DAA ZO Invalid Valid 加法后的十进制调整 AL。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

调整两个组合 BCD 值的和以产生组合 BCD 结果。AL 寄存器是隐式源和目的操作数。仅当 DAA 指令跟在将两个两位数组合 BCD 值相加（二进制加法）并将字节结果存储在 AL 寄存器中的 ADD 指令之后时才有用。然后，DAA 指令调整 AL 寄存器的内容以包含正确的两位数组合 BCD 结果。如果检测到十进制进位，则相应地设置 CF 和 AF 标志。

该指令在兼容模式和传统模式下如上所述执行。在 64 位模式下无效。

操作¶

IF 64-Bit Mode
    THEN
        #UD;
    ELSE
        old_AL := AL;
        old_CF := CF;
        CF := 0;
        IF (((AL AND 0FH) > 9) or AF = 1)
                THEN
                    AL := AL + 6;
                    CF := old_CF or (Carry from AL := AL + 6);
                    AF := 1;
                ELSE
                    AF := 0;
        FI;
        IF ((old_AL > 99H) or (old_CF = 1))
            THEN
                    AL := AL + 60H;
                    CF := 1;
            ELSE
                    CF := 0;
        FI;
FI;

示例¶

ADD AL, BL 执行前：AL=79H BL=35H EFLAGS(OSZAPC)=XXXXXX

执行后：AL=AEH BL=35H EFLAGS(0SZAPC)=110000

DAA 执行前：AL=AEH BL=35H EFLAGS(OSZAPC)=110000

执行后：AL=14H BL=35H EFLAGS(0SZAPC)=X00111

DAA 执行前：AL=2EH BL=35H EFLAGS(OSZAPC)=110000

执行后：AL=34H BL=35H EFLAGS(0SZAPC)=X00101

受影响的标志位¶

如果值的调整导致结果的任何一位中产生十进制进位，则设置 CF 和 AF 标志（参见上面的“操作”部分）。SF、ZF 和 PF 标志根据结果进行设置。OF 标志是未定义的。

保护模式异常¶

#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

#UD 如果使用了 LOCK 前缀。

64位模式异常¶

#UD 如果处于 64 位模式。
