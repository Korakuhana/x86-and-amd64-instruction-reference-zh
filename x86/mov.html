1. 在 64 位模式下，如果使用了 REX 前缀，则不能通过编码 r/m8 来访问以下字节寄存器：AH、BH、CH、DH。

2. 在 32 位模式下，汇编器可能会在此指令中插入 16 位操作数大小前缀（有关更多信息，请参见以下“描述”部分）。

3. moffs8、moffs16、moffs32 和 moffs64 操作数指定相对于段基址的简单偏移量，其中 8、16、32 和 64 指的是数据大小。该指令的地址大小属性决定偏移量的大小，可以是 16 位、32 位或 64 位。

描述 ¶

将第二操作数（源操作数）复制到第一操作数（目的操作数）。源操作数可以是一个立即数、通用寄存器、段寄存器或内存位置；目的寄存器可以是一个通用寄存器、段寄存器或内存位置。两个操作数必须具有相同的大小，该大小可以是一个字节、一个字、一个双字或一个四字。

MOV 指令不能用于加载 CS 寄存器。尝试这样做会导致无效操作码异常（#UD）。要加载 CS 寄存器，请使用远跳转 JMP、CALL 或 RET 指令。

如果目的操作数是段寄存器（DS、ES、FS、GS 或 SS），则源操作数必须是有效的段选择子。在保护模式下，将段选择子移入段寄存器会自动将与该段选择子关联的段描述符信息加载到段寄存器的隐藏（阴影）部分。在加载此信息时，会对段选择子和段描述符信息进行验证（参见下文的“操作”算法）。段描述符数据是从指定段选择子的 GDT 或 LDT 项中获取的。

可以将 NULL 段选择子（值 0000-0003）加载到 DS、ES、FS 和 GS 寄存器中，而不会引发保护异常。但是，随后任何试图引用其相应段寄存器加载了 NULL 值的段的操作都会引发通用保护异常（#GP），并且不会发生内存引用。

使用 MOV 指令加载 SS 寄存器会抑制或禁止某些调试异常，并禁止在下一条指令边界上的中断。（该禁止在引发异常或执行下一条指令后结束。）这种行为允许在发生事件之前通过下一条指令（MOV ESP，栈指针值）将栈指针加载到 ESP 寄存器中。参见 Intel® 64 和 IA-32 架构软件开发人员手册第 3A 卷第 6.8.3 节“切换栈时屏蔽异常和中断”。Intel 建议软件使用 LSS 指令同时加载 SS 寄存器和 ESP。

当执行 MOV Reg, Sreg 时，处理器将 Sreg 的内容复制到通用寄存器的 16 个最低有效位。大多数 IA-32 处理器（Pentium Pro 处理器及更高版本）和所有 Intel 64 处理器的目的寄存器高位都为零，但在 Intel Quark X1000 处理器、Pentium 及更早版本的处理器中，位 31:16 是未定义的。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 前缀可将操作提升至 64 位。请参阅本节开头的汇总图表以获取编码数据 and 限制。

操作 ¶ 

DEST := SRC;
Loading a segment register while in protected mode results in special checks and actions, as described in the following listing. These
checks are performed on the segment selector and the segment descriptor to which it points.
IF SS is loaded
    THEN
        IF segment selector is NULL
            THEN #GP(0); FI;
        IF segment selector index is outside descriptor table limits
        OR segment selector's RPL ≠ CPL
        OR segment is not a writable data segment
        OR DPL ≠ CPL
            THEN #GP(selector); FI;
        IF segment not marked present
            THEN #SS(selector);
            ELSE
                SS := segment selector;
                SS := segment descriptor; FI;
FI;
IF DS, ES, FS, or GS is loaded with non-NULL selector
THEN
    IF segment selector index is outside descriptor table limits
    OR segment is not a data or readable code segment
    OR ((segment is a data or nonconforming code segment) AND ((RPL > DPL) or (CPL > DPL)))
        THEN #GP(selector); FI;
    IF segment not marked present
        THEN #NP(selector);
        ELSE
            SegmentRegister := segment selector;
            SegmentRegister := segment descriptor; FI;
FI;
IF DS, ES, FS, or GS is loaded with NULL selector
    THEN
        SegmentRegister := segment selector;
        SegmentRegister := segment descriptor;
FI;

保护模式异常 ¶

#GP(0) 如果试图使用 NULL 段选择子加载 SS 寄存器。
如果目的操作数处于不可写的段中。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段界限。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#GP(选择子) 如果段选择子索引超出描述符表限制。
如果正在加载 SS 寄存器，并且段选择子的 RPL 和段描述符的 DPL 不等于 CPL。
如果正在加载 SS 寄存器，并且指向的段是不可写的数据段。
如果正在加载 DS、ES、FS 或 GS 寄存器，并且指向的段不是数据或可读代码段。
如果正在加载 DS、ES、FS 或 GS 寄存器，且指向的段是数据或非一致性代码段，并且 RPL 或 CPL 大于 DPL。
#SS(0) 如果内存操作数的有效地址超出 SS 段界限。
#SS(选择子) 如果正在加载 SS 寄存器，且指向的段被标记为不存在。
#NP 如果正在加载 DS、ES、FS 或 GS 寄存器，且指向的段被标记为不存在。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果试图加载 CS 寄存器。
如果使用了 LOCK 前缀。
