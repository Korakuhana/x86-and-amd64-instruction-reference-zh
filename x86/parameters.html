GETSEC[PARAMETERS] — 报告 SMX 参数

操作码 指令 描述
NP 0F 37 (EAX=6) GETSEC[PARAMETERS] 报告 SMX 参数。参数索引在 EBX 中输入，结果在 EAX、EBX 和 ECX 中返回。

描述¶

GETSEC[PARAMETERS] 指令返回处理器支持的 SMX 功能的特定参数信息。参数信息在 EAX、EBX 和 ECX 中返回，输入参数使用 EBX 进行选择。

软件通过从 0 开始搜索 EBX 的输入索引来检索参数信息，然后读取在 EAX、EBX 和 ECX 中返回的结果。EAX[4:0] 指定用于返回参数类型字段，表示参数是否可用以及其类型。如果返回的 EAX[4:0] 为 0，这表示空参数，表示没有更多参数可用。

表 7-7 定义了当前和未来实现中支持的参数类型。

对于支持 SMX 的特定处理器，可以通过类型 1 参数确定支持的 AC 模块版本（由 AC 模块 HeaderVersion 字段定义）。使用 EBX 对由 GETSEC[PARAMETERS] 报告的可用于每个为类型 1 返回的唯一参数集的参数进行索引，软件可以确定支持的 AC 模块版本的完整列表。

对于每个参数集，EBX 返回比较掩码，而 ECX 在目标 HeaderVersion 与比较掩码进行逻辑与之后，返回所支持的可用 HeaderVersion 字段值。软件然后可以通过遵循下面给出的伪代码搜索程序来确定是否支持特定的 AC 模块版本：

parameter_search_index= 0 do { EBX= parameter_search_index++ EAX= 6 GETSEC if (EAX[4:0] = 1) { if ((version_query & EBX) = ECX) { version_is_supported= 1 break } }

} while (EAX[4:0] ≠ 0)

如果处理器仅支持 HeaderVersion 为 0 的 AC 模块，则仅返回类型 1 的一个参数集，如下所示：EAX = 00000001H，EBX = FFFFFFFFH 且 ECX = 00000000H。

处理器支持的已认证代码执行区域的最大容量用参数类型 2 报告。以字节为单位的最大支持大小是通过将 EAX[31:5] 中返回的大小乘以 32 来确定的。因此，对于最大支持的已认证 RAM 大小为 32KB，EAX 返回 00008002H。

参数类型 3 报告在已认证代码执行区域之外映射的内存的可支持内存类型。当 AC 模式（通过 GETSEC 函数 SENTER 和 ENTERACCS 启动并由 EXITAC 终止）处于活动状态时，对系统内存的其余部分允许使用哪些内存类型存在限制。系统软件有责任初始化内存类型范围寄存器 (MTRR) MSR 和/或页属性表 (PAT)，以仅映射与此参数报告一致的内存类型。外部内存的可支持内存类型的报告使用 EAX[31:8] 中返回的位图表示。这些位位置对应于为 MTRR MSR 和 PAT 编程定义的内存类型编码。参见表 7-9。

参数类型 4 用于枚举选择性 GETSEC[SENTER] 功能禁用控制的可用性。如果在返回的参数 EAX 的 14:8 位中报告了 1，则表示 SENTER 对特定功能存在禁用控制功能。14:8 位中的枚举字段对应于 SENTER 的 EDX 输入参数的 6:0 位的使用。如果枚举字段位设置为 1，则 EDX 的对应 EDX 输入参数位可以设置为 1 以禁用该指定功能。如果枚举字段位为 0 或未报告此参数，则 SENTER 的对应 EDX 输入参数不存在禁用功能，并且 EDX 位必须清零以启用 SENTER 的执行。如果枚举中不存在 SENTER 的选择性禁用功能，则如果设置了 MSR 的 SENTER 全局启用位 15，IA32_FEATURE_CONTROL MSR 的对应 14:8 位也必须编程为 1。这是为了支持将来对于 SENTER 选择性禁用功能的扩展（相对于潜在的独立软件初始化 MSR）。

如果给定的支持 SMX 的处理器不存在 GETSEC[PARAMETERS] 叶或特定参数，则应假定默认参数值。这些在表 7-10 中定义。

操作¶

(* 仅支持 0.0 HeaderVersion、32K ACRAM 大小、内存类型 UC 和 WC 的处理器的示例 *)
IF (CR4.SMXE=0)
    THEN #UD;
ELSE IF (in VMX non-root operation)
    THEN VM Exit (reason=”GETSEC instruction”);
ELSE IF (GETSEC leaf unsupported)
    THEN #UD;
    (* 支持 0.0 HeaderVersion 的处理器的示例 *)
IF (EBX=0) THEN
    EAX := 00000001h;
    EBX := FFFFFFFFh;
    ECX := 00000000h;
ELSE IF (EBX=1)
    (* 支持 32K ACRAM 大小的处理器的示例 *)
    THEN EAX := 00008002h;
ESE IF (EBX= 2)
    (* 支持外部内存类型 UC 和 WC 的处理器的示例 *)
    THEN EAX := 00000303h;
ESE IF (EBX= other value(s) less than unsupported index value)
    (* EAX 值会有所不同。请参阅表 7-7 和表 *)
ELSE (* 不受支持的索引 *)
    EAX := 00000000h;
END;

受影响的标志位¶

无。

前缀的使用¶

LOCK 前缀导致 #UD。

REP* 前缀导致 #UD（包括 REPNE/REPNZ 和 REP/REPE/REPZ）。

操作数大小前缀导致 #UD。

不允许 66/F2/F3 前缀。

段超越前缀被忽略。

地址大小前缀被忽略。

REX 前缀被忽略。

保护模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[PARAMETERS] 未被 GETSEC[CAPABILITIES] 报告为受支持。

实地址模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[PARAMETERS] 未被 GETSEC[CAPABILITIES] 报告为受支持。

虚拟-8086 模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[PARAMETERS] 未被 GETSEC[CAPABILITIES] 报告为受支持。

兼容模式异常¶

所有保护模式异常均适用。

64位模式异常¶

所有保护模式异常均适用。

VM-Exit 条件¶

原因 (GETSEC) 如果处于 VMX 非根操作中。
