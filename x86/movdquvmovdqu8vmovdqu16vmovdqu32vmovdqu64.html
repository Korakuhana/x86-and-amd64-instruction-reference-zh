描述¶

注意：VEX.vvvv 和 EVEX.vvvv 是保留的，必须为 1111b，否则指令将产生 #UD 异常。

EVEX 编码版本：

将 128、256 或 512 位打包的字节/字/双字/四字整数值从源操作数（第二个操作数）移动到目的操作数（第一个操作数）。此指令可用于从内存位置加载向量寄存器，将向量寄存器的内容存储到内存位置中，或者在两个向量寄存器之间移动数据。

目的操作数根据写掩码以 8 位 (VMOVDQU8)、16 位 (VMOVDQU16)、32 位 (VMOVDQU32) 或 64 位 (VMOVDQU64) 粒度更新。

VEX.256 编码版本：

将 256 位打包整数值从源操作数（第二个操作数）移动到目的操作数（第一个操作数）。此指令可用于从 256 位内存位置加载 YMM 寄存器，将 YMM 寄存器的内容存储到 256 位内存位置中，或者在两个 YMM 寄存器之间移动数据。目的寄存器的高位 (MAXVL-1:256) 被清零。

128 位版本：

将 128 位打包整数值从源操作数（第二个操作数）移动到目的操作数（第一个操作数）。此指令可用于从 128 位内存位置加载 XMM 寄存器，将 XMM 寄存器的内容存储到 128 位内存位置中，或者在两个 XMM 寄存器之间移动数据。

128 位传统 SSE 版本：对应目的寄存器的位 (MAXVL-1:128) 保持不变。

当源或目的操作数是内存操作数时，操作数可以未对齐到任何边界，而不会导致产生一般保护性异常 (#GP)。

VEX.128 编码版本：目的寄存器的位 (MAXVL-1:128) 被清零。

操作¶

VMOVDQU8 (EVEX Encoded Versions, Register-Copy Form)¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
FOR j := 0 TO KL-1
    i := j * 8
    IF k1[j] OR *no writemask*
        THEN DEST[i+7:i] := SRC[i+7:i]
        ELSE
            IF *merging-masking*
                THEN *DEST[i+7:i] remains unchanged*
                ELSE DEST[i+7:i] := 0 ; zeroing-masking
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
