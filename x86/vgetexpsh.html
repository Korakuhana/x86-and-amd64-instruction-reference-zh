VGETEXPSH — 将标量 FP16 值的指数转换为 FP16 值

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.66.MAP6.W0 43 /r VGETEXPSH xmm1{k1}{z}, xmm2, xmm3/m16 {sae} A V/V AVX512-FP16 将源操作数低位字中 FP16 值的指数转换为表示无偏整数指数的 FP16 结果，并将结果存储在受写掩码 k1 限制的目的寄存器的低位字中。xmm2 的第 127:16 位复制到 xmm1[127:16] 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4

描述¶

此指令从源操作数（第二操作数）低位字元素的规范化 FP16 表示中提取有偏指数作为无偏有符号整数值，或者将输入数据的非规范表示转换为无偏负整数值。无偏指数的整数值被转换为 FP16 值，并作为 FP16 数写入目的操作数（第一操作数）的低位字元素。

目的操作数的第 127:16 位复制自第一源操作数中的对应位。目的操作数的第 MAXVL-1:128 位清零。目的操作数的低位 FP16 元素根据写掩码更新。

每个 GETEXP 操作都会将指数值转换为浮点数（允许非规范表示 of 输入值）。输入值的特殊情况在 表 5-16 中列出。

公式为：

GETEXP(x) = floor(log2(|x|))

记号 floor(x) 表示不超过实数 x 的最大整数。

VGETEXPxx 和 VGETMANTxx 指令的软件使用通常结合了 GETEXP 操作和 GETMANT 操作（参见 VGETMANTSH）。因此，VGETEXPSH 指令不需要软件来处理 SIMD 浮点异常。

操作¶

VGETEXPSH dest{k1}, src1, src2¶

IF k1[0] or *no writemask*:
    DEST.fp16[0] := getexp_fp16(src2.fp16[0]) // see VGETEXPPH
ELSE IF *zeroing*:
    DEST.fp16[0] := 0
//else DEST.fp16[0] remains unchanged
DEST[127:16] := src1[127:16]
DEST[MAXVL-1:128] := 0
