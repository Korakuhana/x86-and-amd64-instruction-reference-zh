MOVHPS — 移动高位打包单精度浮点值

描述¶

此指令不能用于寄存器到寄存器或内存到内存的移动。

128 位传统 SSE 加载：

将两个打包单精度浮点值从源 64 位内存操作数移动并存储在目的 XMM 寄存器的高 64 位中。XMM 寄存器的低 64 位保持不变。对应目的寄存器的位 (MAXVL-1:128) 保持不变。

VEX.128 和 EVEX 编码的加载：

从源 64 位内存操作数（第三个操作数）加载两个单精度浮点值并将其存储在目的 XMM 寄存器（第一个操作数）的高 64 位中。第一个源操作数（第二个操作数）的低 64 位复制到目的地的低 64 位。对应目的寄存器的位 (MAXVL-1:128) 被清零。

128 位存储：

将来自 XMM 寄存器源（第二个操作数）高 64 位的两个打包单精度浮点值存储到 64 位内存位置（第一个操作数）。

注意：VMOVHPS（存储）(VEX.128.0F 17 /r) 是合法的，行为与现有的 0F 17 存储相同。对于 VMOVHPS（存储），VEX.vvvv 和 EVEX.vvvv 是保留的，必须为 1111b，否则指令将产生 #UD 异常。

如果 VMOVHPS 用 VEX.L 或 EVEX.L'L= 1 编码，尝试执行用 VEX.L 或 EVEX.L'L= 1 编码的指令将导致 #UD 异常。

操作¶

MOVHPS (128-bit Legacy SSE Load)¶

DEST[63:0] (Unmodified)
DEST[127:64] := SRC[63:0]
DEST[MAXVL-1:128] (Unmodified)

VMOVHPS (VEX.128 and EVEX Encoded Load)¶

DEST[63:0] := SRC1[63:0]
DEST[127:64] := SRC2[63:0]
DEST[MAXVL-1:128] := 0

VMOVHPS (Store)¶

DEST[63:0] := SRC[127:64]

MOVHPS __m128 _mm_loadh_pi ( __m128 a, __m64 *p)

MOVHPS void _mm_storeh_pi (__m64 *p, __m128 a)

SIMD 浮点异常¶

无。
