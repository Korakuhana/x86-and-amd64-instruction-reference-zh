描述¶

VINSERTF128/VINSERTF32x4 和 VINSERTF64x2 以 128 位粒度的偏移量（乘以 imm8[0] (256位) 或 imm8[1:0]）将来自第二源操作数（第三操作数）的 128 位打包浮点值插入目的操作数（第一操作数）。目的操作数的其余部分复制自第一源操作数（第二操作数）的对应字段。第二源操作数可以是一个 XMM 寄存器或一个 128 位内存位置。目的操作数和第一源操作数是向量寄存器。

VINSERTF32x4：目的操作数是 ZMM/YMM 寄存器，并根据写掩码以 32 位粒度进行更新。立即数的高 6/7 位被忽略。

VINSERTF64x2：目的操作数是 ZMM/YMM 寄存器，并根据写掩码以 64 位粒度进行更新。立即数的高 6/7 位被忽略。

VINSERTF32x8 和 VINSERTF64x4 以 256 位粒度的偏移量（乘以 imm8[0]）将来自第二源操作数（第三操作数）的 256 位打包浮点值插入目的操作数（第一操作数）。目的操作数的其余部分复制自第一源操作数（第二操作数）的对应字段。第二源操作数可以是一个 YMM 寄存器或一个 256 位内存位置。立即数的高 7 位被忽略。目的操作数是 ZMM 寄存器，并根据写掩码以 32/64 位粒度进行更新。

操作¶

VINSERTF32x4 (EVEX 编码版本)¶

(KL, VL) = (8, 256), (16, 512)
TEMP_DEST[VL-1:0] := SRC1[VL-1:0]
IF VL = 256
    CASE (imm8[0]) OF
        0: TMP_DEST[127:0] := SRC2[127:0]
        1: TMP_DEST[255:128] := SRC2[127:0]
    ESAC.
FI;
IF VL = 512
    CASE (imm8[1:0]) OF
        00: TMP_DEST[127:0]:=SRC2[127:0]
        01: TMP_DEST[255:128]:=SRC2[127:0]
        10: TMP_DEST[383:256]:=SRC2[127:0]
        11: TMP_DEST[511:384]:=SRC2[127:0]
    ESAC.
FI;
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN DEST[i+31:i] := TMP_DEST[i+31:i]
        ELSE
            IF *merging-masking*
                THEN *DEST[i+31:i] remains unchanged*
                ELSE ; zeroing-masking
                    DEST[i+31:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0

VINSERTF128 (VEX 编码版本)¶

TEMP[255:0] := SRC1[255:0]

    0: TEMP[127:0] := SRC2[127:0]
