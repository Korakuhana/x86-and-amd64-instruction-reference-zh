SCAS/SCASB/SCASW/SCASD — 扫描字符串

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
AE SCAS m8 ZO Valid Valid 将 AL 与 ES:(E)DI 或 RDI 处的字节进行比较，然后设置状态标志位。1
AF SCAS m16 ZO Valid Valid 将 AX 与 ES:(E)DI 或 RDI 处的字进行比较，然后设置状态标志位。1
AF SCAS m32 ZO Valid Valid 将 EAX 与 ES:(E)DI 或 RDI 处的双字进行比较，然后设置状态标志位。1
REX.W + AF SCAS m64 ZO Valid N.E. 将 RAX 与 RDI 或 EDI 处的四字进行比较，然后设置状态标志位。
AE SCASB ZO Valid Valid 将 AL 与 ES:(E)DI 或 RDI 处的字节进行比较，然后设置状态标志位。1
AF SCASW ZO Valid Valid 将 AX 与 ES:(E)DI 或 RDI 处的字进行比较，然后设置状态标志位。1
AF SCASD ZO Valid Valid 将 EAX 与 ES:(E)DI 或 RDI 处的双字进行比较，然后设置状态标志位。1
REX.W + AF SCASQ ZO Valid N.E. 将 RAX 与 RDI 或 EDI 处的四字进行比较，然后设置状态标志位。

1. 在 64 位模式下，仅支持 64 位 (RDI) 和 32 位 (EDI) 地址大小。在非 64 位模式下，仅支持 32 位 (EDI) 和 16 位 (DI) 地址大小。

描述¶

在非 64 位模式和默认的 64 位模式下：此指令将内存操作数指定的字节、字、双字或四字与 AL、AX 或 EAX 中的值进行比较。然后它在 EFLAGS 中设置状态标志位以记录结果。内存操作数地址从 ES:(E)DI 寄存器读取（取决于指令的地址大小属性和当前操作模式）。请注意，不能使用段覆盖前缀覆盖 ES。

在汇编代码级别，此指令允许两种形式。显式操作数形式和无操作数形式。显式操作数形式（使用 SCAS 助记符指定）允许显式指定内存操作数。内存操作数必须是指示操作数大小和位置的符号。然后自动选择寄存器操作数以匹配内存操作数的大小（AL 寄存器用于字节比较，AX 用于字比较，EAX 用于双字比较）。提供显式操作数形式是为了便于编写文档。请注意，此形式提供的文档可能会误导。也就是说，内存操作数符号必须指定操作数的正确类型（大小）（字节、字或双字），但不必指定正确的位置。位置始终由 ES:(E)DI 指定。

指令的无操作数形式使用 SCAS 的简短形式。同样，假定 ES:(E)DI 是内存操作数，AL、AX 或 EAX 是寄存器操作数。操作数的大小由助记符选择：SCASB（字节比较）、SCASW（字比较）或 SCASD（双字比较）。

比较后，(E)DI 寄存器将根据 EFLAGS 寄存器中 DF 标志的设置自动递增或递减。如果 DF 标志为 0，则 (E)DI 寄存器递增；如果 DF 标志为 1，则 (E)DI 寄存器递减。字节操作递增或递减 1，字操作递增或递减 2，双字操作递增或递减 4。

SCAS、SCASB、SCASW、SCASD 和 SCASQ 指令之前可以带有 REP 前缀，用于对 ECX 个字节、字、双字或四字进行块比较。然而，这些指令通常在根据状态标志位的设置采取行动的 LOOP 结构中使用。有关 REP 前缀的描述，请参阅本章中的“REP/REPE/REPZ /REPNE/REPNZ — 重复字符串操作前缀”。

In 64-bit mode, the instruction’s default address size is 64-bits, 32-bit address size is supported using the prefix 67H. Using a REX prefix in the form of REX.W promotes operation on doubleword operand to 64 bits. The 64-bit no-operand mnemonic is SCASQ. Address of the memory operand is specified in either RDI or EDI, and AL/AX/EAX/RAX may be used as the register operand. After a comparison, the destination register is incremented or decremented by the current operand size (depending on the value of the DF flag). See the summary chart at the beginning of this section for encoding data and limits.

操作¶

非 64 位模式：¶

IF (Byte comparison)
    THEN
        temp := AL − SRC;
        SetStatusFlags(temp);
            THEN IF DF = 0
                THEN (E)DI := (E)DI + 1;
                ELSE (E)DI := (E)DI – 1; FI;
    ELSE IF (Word comparison)
        THEN
            temp := AX − SRC;
            SetStatusFlags(temp);
            IF DF = 0
                THEN (E)DI := (E)DI + 2;
                ELSE (E)DI := (E)DI – 2; FI;
        FI;
    ELSE IF (Doubleword comparison)
        THEN
            temp := EAX – SRC;
            SetStatusFlags(temp);
            IF DF = 0
                THEN (E)DI := (E)DI + 4;
                ELSE (E)DI := (E)DI – 4; FI;
        FI;
FI;

64 位模式：¶

IF (Byte comparison)
    THEN
        temp := AL − SRC;
        SetStatusFlags(temp);
            THEN IF DF = 0
                THEN (R|E)DI := (R|E)DI + 1;
                ELSE (R|E)DI := (R|E)DI – 1; FI;
    ELSE IF (Word comparison)
        THEN
            temp := AX − SRC;
            SetStatusFlags(temp);
            IF DF = 0
                THEN (R|E)DI := (R|E)DI + 2;
                ELSE (R|E)DI := (R|E)DI – 2; FI;
        FI;
    ELSE IF (Doubleword comparison)
        THEN
            temp := EAX – SRC;
            SetStatusFlags(temp);
            IF DF = 0
                THEN (R|E)DI := (R|E)DI + 4;
                ELSE (R|E)DI := (R|E)DI – 4; FI;
        FI;
    ELSE IF (Quadword comparison using REX.W )
        THEN
            temp := RAX − SRC;
            SetStatusFlags(temp);
            IF DF = 0
                THEN (R|E)DI := (R|E)DI + 8;
                ELSE (R|E)DI := (R|E)DI – 8;
            FI;
    FI;
FI;

影响的标志位¶

根据比较的临时结果设置 OF、SF、ZF、AF、PF 和 CF 标志。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 ES 段的限制。
如果 ES 寄存器包含 NULL 段选择器。
如果给出了 ES 段中非法的内存操作数有效地址。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀。
