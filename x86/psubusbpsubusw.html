描述¶

从目的操作数（第一个操作数）中的打包无符号整数中，SIMD 减去源操作数（第二个操作数）中的打包无符号整数，并将打包无符号整数结果存储在目的操作数中。有关 SIMD 操作的图解，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷中的图 9-4。溢出将通过无符号饱和处理，如下文所述。

这些指令可以操作 64 位或 128 位操作数。

(V)PSUBUSB 指令减去打包无符号字节整数。当单个字节结果小于零时，向目的操作数写入饱和值 00H。

(V)PSUBUSW 指令减去打包无符号字整数。当单个字结果小于零时，向目的操作数写入饱和值 0000H。

在 64 位模式下且未在 VEX/EVEX 中编码时，使用 REX.R 形式的 REX 前缀允许此指令访问其他寄存器（XMM8-XMM15）。

传统 SSE 版本 64 位操作数：目的操作数必须是 MMX 技术寄存器，源操作数可以是 MMX 技术寄存器或 64 位内存位置。

128位传统 SSE 版本：第二个源操作数是 XMM 寄存器或 128 位内存位置。第一个源操作数和目的操作数是 XMM 寄存器。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：第二个源操作数是 XMM 寄存器或 128 位内存位置。第一个源操作数和目的操作数是 XMM 寄存器。目的 YMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：第二个源操作数是 YMM 寄存器或 256 位内存位置。第一个源操作数和目的操作数是 YMM 寄存器。对应 ZMM 寄存器的位（MAXVL-1:256）清零。

EVEX 编码版本：第二个源操作数是 ZMM/YMM/XMM 寄存器或 512/256/128 位内存位置。第一个源操作数和目的操作数是 ZMM/YMM/XMM 寄存器。目的操作数在写掩码 k1 下有条件地更新。

VPSUBUSB（EVEX 编码版本）¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
FOR j := 0 TO KL-1
    i := j * 8;
    IF k1[j] OR *no writemask*
        THEN DEST[i+7:i] := SaturateToUnsignedByte (SRC1[i+7:i] - SRC2[i+7:i])
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+7:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+7:i] := 0;
            FI
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0;
