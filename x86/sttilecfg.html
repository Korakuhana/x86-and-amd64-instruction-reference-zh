STTILECFG — 存储 Tile 配置

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
VEX.128.66.0F38.W0 49 !(11):000:bbb STTILECFG m512 A V/N.E. AMX-TILE 将 tile 配置存储在 m512 中。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2 操作数 3 操作数 4
A N/A ModRM:r/m (w) N/A N/A N/A

描述¶

STTILECFG 指令接受一个指向 64 字节内存位置（在“LDTILECFG—加载 tile 配置”条目中的表 3-10 中描述）的指针，该位置在成功执行此指令后将包含已配置的 tile 的描述。为了配置 tile，必须设置 CPUID 中的 AMX-TILE 位，且操作系统必须已启用 tile 架构。

如果未配置 tile，则 STTILECFG 会将 64 字节的零存储到指定的内存位置。

在 Intel TSX 事务中尝试执行 STTILECFG 指令都会导致事务中止。

操作¶

STTILECFG mem¶

if TILES_CONFIGURED == 0:
    //write 64 bytes of zeros at mem pointer
    buf[0..63] := 0
    write_memory(mem, 64, buf)
else:
    buf.byte[0] := tilecfg.palette_id
    buf.byte[1] := tilecfg.start_row
    buf.byte[2..15] := 0
    p := 16
    for n in 0 ... palette_table[tilecfg.palette_id].max_names-1:
        buf.word[p/2] := tilecfg.t[n].colsb
        p := p + 2
    if p < 47:
        buf.byte[p..47] := 0
    p := 48
    for n in 0 ... palette_table[tilecfg.palette_id].max_names-1:
        buf.byte[p++] := tilecfg.t[n].rows
    if p < 63:
        buf.byte[p..63] := 0
    write_memory(mem, 64, buf)
