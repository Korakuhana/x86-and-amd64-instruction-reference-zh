PUSHA/PUSHAD — 将所有通用寄存器压入栈

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
60 PUSHA ZO Invalid Valid 压入 AX, CX, DX, BX, 原始 SP, BP, SI, 以及 DI。
60 PUSHAD ZO Invalid Valid 压入 EAX, ECX, EDX, EBX, 原始 ESP, EBP, ESI, 以及 EDI。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将通用寄存器的内容压入栈中。寄存器在栈中的存储顺序如下：EAX、ECX、EDX、EBX、ESP（原始值）、EBP、ESI、EDI（如果当前操作数大小属性为 32）；以及 AX、CX、DX、BX、SP（原始值）、BP、SI、DI（如果操作数大小属性为 16）。这些指令执行与 POPA/POPAD 指令相反的操作。压入的 ESP 或 SP 寄存器的值是压入第一个寄存器之前的初始值（参见下方的“操作”部分）。

助记符 PUSHA（全部压栈）和 PUSHAD（全部双字压栈）指向同一个操作码。PUSHA 指令旨在操作数大小属性为 16 时使用，而 PUSHAD 指令旨在操作数大小属性为 32 时使用。某些汇编器可能会在下述情况下强制确定操作数大小：使用 PUSHA 时为 16，使用 PUSHAD 时为 32。其他汇编器可能将这些助记符视为同义词（PUSHA/PUSHAD），并使用操作数大小属性的当前设置来确定从栈中压入的值的大小，而不管所使用的助记符如何。

在实地址模式下，如果执行 PUSHA/PUSHAD 时 ESP 或 SP 寄存器为 1、3 或 5：将产生 #SS 异常，但不会传送（报告的栈错误会阻止 #SS 传送）。接下来，处理器产生 #DF 异常并进入关机状态，如《Intel® 64 和 IA-32 架构软件开发人员手册》第 3A 卷第 6 章中关于 #DF 的讨论所述。

此指令在兼容模式和传统模式下的执行过程与所述相同。该指令在 64 位模式下无效。

操作¶

IF 64-bit Mode
    THEN #UD
FI;
IF OperandSize = 32 (* PUSHAD instruction *)
    THEN
        Temp := (ESP);
        Push(EAX);
        Push(ECX);
        Push(EDX);
        Push(EBX);
        Push(Temp);
        Push(EBP);
        Push(ESI);
        Push(EDI);
    ELSE (* OperandSize = 16, PUSHA instruction *)
        Temp := (SP);
        Push(AX);
        Push(CX);
        Push(DX);
        Push(BX);
        Push(Temp);
        Push(BP);
        Push(SI);
        Push(DI);
FI;

影响的标志位¶

无。

保护模式异常¶

#SS(0) 如果栈的起始或结束地址超出栈段限制。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果 ESP 或 SP 寄存器包含 7、9、11、13 或 15。
#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#GP(0) 如果 ESP 或 SP 寄存器包含 7、9、11、13 或 15。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#UD 如果处于 64 位模式下。
