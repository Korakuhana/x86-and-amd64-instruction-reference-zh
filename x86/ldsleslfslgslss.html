LDS/LES/LFS/LGS/LSS — 加载远指针

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

从第二个操作数（源操作数）加载远指针（段选择子和偏移量）到段寄存器和第一个操作数（目的操作数）中。源操作数指定内存中的 48 位或 32 位指针，具体取决于操作数大小属性的当前设置（分别为 32 位或 16 位）。指令操作码和目的操作数指定一个段寄存器/通用寄存器对。源操作数中的 16 位段选择子加载到由操作码指定的段寄存器（DS、SS、ES、FS 或 GS）中。32 位或 16 位偏移量加载到由目的操作数指定的寄存器中。

如果在保护模式下执行这些指令之一，则来自源操作数中段选择子所指向的段描述符的附加信息将加载到所选段寄存器的隐藏部分中。

同样在保护模式下，空选择子（数值 0000 至 0003）可以加载到 DS、ES、FS 或 GS 寄存器中，而不会引起保护异常。（随后对相应段寄存器加载了空选择子的段的任何引用，都会导致通用保护异常 (#GP)，并且不会对该段进行内存引用。）

在 64 位模式下，指令的默认操作大小为 32 位。使用 REX.W 形式的 REX 前缀可提升操作以指定引用内存中 80 位指针（16 位选择子，64 位偏移量）的源操作数。使用 REX.R 形式的 REX 前缀允许访问附加寄存器 (R8-R15)。有关编码数据和限制，请参阅本节开头的汇总图表。

操作¶

64-BIT_MODE
    IF SS is loaded
        THEN
            IF SegmentSelector = NULL and ( (RPL = 3) or
                    (RPL ≠ 3 and RPL ≠ CPL) )
                THEN #GP(0);
            ELSE IF descriptor is in non-canonical space
                THEN #GP(selector); FI;
            ELSE IF Segment selector index is not within descriptor table limits
                    or segment selector RPL ≠ CPL
                    or access rights indicate nonwritable data segment
                    or DPL ≠ CPL
                THEN #GP(selector); FI;
            ELSE IF Segment marked not present
                THEN #SS(selector); FI;
            FI;
            SS := SegmentSelector(SRC);
            SS := SegmentDescriptor([SRC]);
    ELSE IF attempt to load DS, or ES
        THEN #UD;
    ELSE IF FS, or GS is loaded with non-NULL segment selector
        THEN IF Segment selector index is not within descriptor table limits
            or access rights indicate segment neither data nor readable code segment
            or segment is data or nonconforming-code segment
            and ( RPL > DPL or CPL > DPL)
                THEN #GP(selector); FI;
            ELSE IF Segment marked not present
                THEN #NP(selector); FI;
            FI;
            SegmentRegister := SegmentSelector(SRC) ;
            SegmentRegister := SegmentDescriptor([SRC]);
        FI;
    ELSE IF FS, or GS is loaded with a NULL selector:
        THEN
            SegmentRegister := NULLSelector;
            SegmentRegister(DescriptorValidBit) := 0; FI; (* Hidden flag;
                not accessible by software *)
    FI;
    DEST := Offset(SRC);
PREOTECTED MODE OR COMPATIBILITY MODE;
    IF SS is loaded
        THEN
            IF SegementSelector = NULL
                THEN #GP(0);
            ELSE IF Segment selector index is not within descriptor table limits
                    or segment selector RPL ≠ CPL
                    or access rights indicate nonwritable data segment
                    or DPL ≠ CPL
                THEN #GP(selector); FI;
            ELSE IF Segment marked not present
                THEN #SS(selector); FI;
            FI;
            SS := SegmentSelector(SRC);
            SS := SegmentDescriptor([SRC]);
    ELSE IF DS, ES, FS, or GS is loaded with non-NULL segment selector
        THEN IF Segment selector index is not within descriptor table limits
            or access rights indicate segment neither data nor readable code segment
            or segment is data or nonconforming-code segment
            and (RPL > DPL or CPL > DPL)
                THEN #GP(selector); FI;
            ELSE IF Segment marked not present
                THEN #NP(selector); FI;
            FI;
            SegmentRegister := SegmentSelector(SRC) AND RPL;
            SegmentRegister := SegmentDescriptor([SRC]);
        FI;
    ELSE IF DS, ES, FS, or GS is loaded with a NULL selector:
        THEN
            SegmentRegister := NULLSelector;
            SegmentRegister(DescriptorValidBit) := 0; FI; (* Hidden flag;
                not accessible by software *)
    FI;
    DEST := Offset(SRC);
Real-Address or Virtual-8086 Mode
    SegmentRegister := SegmentSelector(SRC); FI;
    DEST := Offset(SRC);

受影响的标志位¶

无。

保护模式异常¶

#UD 如果源操作数不是内存位置。
如果使用了 LOCK 前缀。
#GP(0) 如果向 SS 寄存器加载了 NULL 选择子。
如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存且它包含一个 NULL 段选择子。
#GP(译码器/选择子) 如果正在加载 SS 寄存器且以下任一条件成立：段选择子索引不在描述符表限制内、段选择子 RPL 不等于 CPL、该段是不可写的数据段，或 DPL 不等于 CPL。
如果正在使用非空段选择子加载 DS、ES、FS 或 GS 寄存器且以下任一条件成立：段选择子索引不在描述符表限制内、该段既不是数据段也不是可读代码段，或者该段是数据或非一致代码段且 RPL 和 CPL 都大于 DPL。
#SS(0) 如果内存操作数有效地址超出了 SS 段限制。
#SS(译码器/选择子) 如果正在加载 SS 寄存器且该段被标记为不存在。
#NP(译码器/选择子) 如果正在使用非空段选择子加载 DS、ES、FS 或 GS 寄存器且该段被标记为不存在。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。

#GP 如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出了 SS 段限制。
#UD 如果源操作数不是内存位置。
如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

#SS(0) 如果内存操作数有效地址为非规范形式。

#NP(译码器/选择子) 如果正在使用非空段选择子加载 FS 或 GS 寄存器且该段被标记为不存在。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果源操作数不是内存位置。
如果使用了 LOCK 前缀。
