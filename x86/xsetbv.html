XSETBV — 设置扩展控制寄存器

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
NP 0F 01 D1 XSETBV ZO Valid Valid 将 EDX:EAX 中的值写入由 ECX 指定的 XCR。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将寄存器 EDX:EAX 的内容写入到由 ECX 寄存器指定的 64 位扩展控制寄存器 (XCR) 中。（在支持 Intel 64 架构的处理器上，RCX 的高 32 位被忽略。）EDX 寄存器的内容被复制到所选 XCR 的高 32 位，EAX 寄存器的内容被复制到该 XCR 的低 32 位。（在支持 Intel 64 架构的处理器上，RAX 和 RDX 各自的高 32 位被忽略。）XCR 中未定义或保留的位应该设置为以前读取的值。

此指令必须在特权级别 0 或在实地址模式下执行；否则会触发通用保护异常 #GP(0)。在 ECX 中指定保留或未实现的 XCR 也会导致通用保护异常。如果软件尝试向 XCR 中的保留位写入数据，处理器也会触发通用保护异常。

目前仅支持 XCR0。因此，ECX 的所有其他值均被保留，并将导致 #GP(0)。请注意，XCR0 的第 0 位（对应于 x87 状态）必须设置为 1；如果尝试清除此位，该指令将导致 #GP(0)。此外，如果尝试在清除 XCR0[1]（SSE 状态）的同时设置 XCR0[2]（AVX 状态），则该指令会导致 #GP(0)；要使用 AVX 指令，必须同时设置这两个位；请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》卷 1 第 13.3 节“启用 XSAVE 功能集和启用 XSAVE 的功能”。

操作¶

XCR[ECX] := EDX:EAX;

受影响的标志位¶

无。

Intel C/C++ 编译器等价内部函数¶

XSETBV void _xsetbv( unsigned int index, unsigned __int64 val);

保护模式异常¶

#GP(0) 如果当前特权级别不为 0。
如果 ECX 中指定了无效的 XCR。
如果 EDX:EAX 中的值设置了在由 ECX 指定的 XCR 中保留的位。
如果尝试清除 XCR0 的第 0 位。
如果尝试将 XCR0[2:1] 设置为 10b。
#UD 如果 CPUID.01H:ECX.XSAVE[bit 26] = 0。
如果 CR4.OSXSAVE[bit 18] = 0。
如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果 ECX 中指定了无效的 XCR。
如果 EDX:EAX 中的值设置了在由 ECX 指定的 XCR 中保留的位。
如果尝试清除 XCR0 的第 0 位。
如果尝试将 XCR0[2:1] 设置为 10b。
#UD 如果 CPUID.01H:ECX.XSAVE[bit 26] = 0。
如果 CR4.OSXSAVE[bit 18] = 0。
如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) XSETBV 指令在虚拟-8086 模式下不被识别。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

与保护模式下的异常相同。
