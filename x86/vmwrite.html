VMWRITE — 向虚拟机控制结构写入字段

操作码/指令 Op/En 描述
NP 0F 79 VMWRITE r64, r/m64 RM 写入指定的 VMCS 字段（在 64 位模式下）。
NP 0F 79 VMWRITE r32, r/m32 RM 写入指定的 VMCS 字段（在 64 位模式以外）。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) NA NA

描述¶

将主要源操作数（寄存器或内存）的内容写入到 VMCS 中的指定字段。在 VMX 根操作中，该指令写入当前 VMCS。如果在 VMX 非根操作中执行，该指令将写入当前 VMCS 中“VMCS 链接指针”字段所引用的 VMCS。

VMCS 字段由寄存器辅助源操作数中包含的 VMCS 字段编码指定。在 IA-32e 模式外，无论 CS.D 的值如何，辅助源操作数始终为 32 位。在 64 位模式下，辅助源操作数有 64 位。

主要源操作数（可以是寄存器或在内存中）的有效大小在 IA-32e 模式外始终为 32 位（就操作数大小而言，CS.D 的设置将被忽略），而在 64 位模式下为 64 位。如果由辅助源操作数指定的 VMCS 字段短于此有效操作数大小，则忽略主要源操作数的高位。如果 VMCS 字段较长，则该字段的高位被清零为 0。

请注意，访问内存源操作数所导致的任何故障，均在下方的操作部分中确定相关 VMCS 指针有效之后、但在确定是否支持目的 VMCS 字段之前发生。

操作¶

IF (not in VMX operation) or (CR0.PE = 0) or (RFLAGS.VM = 1) or (IA32_EFER.LMA = 1 and CS.L = 0)
    THEN #UD;
ELSIF in VMX non-root operation AND (“VMCS shadowing” is 0 OR secondary source operand sets bits in range 63:15 OR
VMWRITE bit corresponding to bits 14:0 of secondary source operand is 1)1
    THEN VMexit;
ELSIF CPL > 0
    THEN #GP(0);
ELSIF (in VMX root operation AND current-VMCS pointer is not valid) OR
(in VMX non-root operation AND VMCS-link pointer is not valid)
    THEN VMfailInvalid;
ELSIF secondary source operand does not correspond to any VMCS field
    THEN VMfailValid(VMREAD/VMWRITE from/to unsupported VMCS component);
ELSIF VMCS field indexed by secondary source operand is a VM-exit information field AND
processor does not support writing to such fields2
    THEN VMfailValid(VMWRITE to read-only VMCS component);
    ELSE

1. 辅助源操作数的 VMWRITE 位定义如下。设 x 为辅助源操作数第 14:0 位的值，设 addr 为 VMWRITE 位图地址。对应的 VMWRITE 位处于物理地址 addr | (x » 3) 处的字节中的 bit 位置 x & 7。

2. 软件可以通过读取 VMX 能力 MSR IA32_VMX_MISC（参见附录 A.6）来发现这些字段是否可以写入。

        IF in VMX root operation
            THEN field indexed by secondary source operand in current VMCS := primary source operand;
            ELSE field indexed by secondary source operand in VMCS referenced by VMCS link pointer := primary source operand;
    FI;
    VMsucceed;
FI;

受影响的标志位¶

请参阅操作部分和第 31.2 节。

保护模式异常¶

#GP(0) 如果当前特权级别不为 0。
如果内存源操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含不可用的段。
如果源操作数位于仅执行代码段中。
#PF(错误码) 如果访问内存源操作数时发生缺页异常。
#SS(0) 如果内存源操作数有效地址超出 SS 段限制。
如果 SS 寄存器包含不可用的段。
#UD 如果不在 VMX 操作中。

实地址模式异常¶

#UD 在实地址模式下无法识别 VMWRITE 指令。

虚拟-8086 模式异常¶

#UD 在虚拟-8086 模式下无法识别 VMWRITE 指令。

兼容模式异常¶

#UD 在兼容模式下无法识别 VMWRITE 指令。

64位模式异常¶

#GP(0) 如果当前特权级别不为 0。
如果内存源操作数位于 CS、DS、ES、FS 或 GS 段中，且内存地址为非规范形式。
#PF(错误码) 如果访问内存源操作数时发生缺页异常。
#SS(0) 如果内存源操作数位于 SS 段中，且内存地址为非规范形式。
#UD 如果不在 VMX 操作中。
