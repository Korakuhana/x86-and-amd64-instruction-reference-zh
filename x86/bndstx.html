BNDSTX — 使用地址翻译存储扩展边界

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
NP 0F 1B /r BNDSTX mib, bnd MR V/V MPX 使用 mib 的基址进行地址翻译，将 bnd 中的边界以及 mib 的索引寄存器中的指针值存储到边界表项 (BTE) 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3
MR SIB.base (r): 指针 SIB.index(r) 的地址 ModRM:reg (r) N/A

描述¶

BNDSTX 使用由内存操作数 (mib) 的 SIB 寻址形式的偏移量和基址寄存器构建的线性地址，执行地址翻译以存储到边界表项。源操作数 bnd 中的边界被写入 BTE 中的下界 and 上界。mib 的索引寄存器的内容被写入 BTE 中的指针值字段。

此指令不会导致对 mib 的线性地址或由基址引用的有效地址进行内存访问，并且不会读取或写入任何标志位。

段超越适用于使用 mib 基址的线性地址计算，并在地址翻译期间用于生成边界表项的地址。默认情况下，BTE 的地址假定为线性地址。不会对 mib 的基址进行段检查。

不会检查 mib 的基址是否违反规范地址，因为它不访问内存。

此指令的任何未指定基址或索引寄存器的编码都会将这些寄存器视为零（常量）。此指令的寄存器-寄存器 (reg-reg) 形式将保持为 NOP。

SIB 字节的比例 (Scale) 字段对这些指令没有影响，并会被忽略。

在内存故障时，边界寄存器可能会被部分更新。加载内存操作数的顺序是具体实现相关的。

操作¶

base := mib.SIB.base ? mib.SIB.base + Disp: 0;
ptr_value := mib.SIB.index ? mib.SIB.index : 0;

Outside 64-bit Mode¶

A_BDE[31:0] := (Zero_extend32(base[31:12] « 2) + (BNDCFG[31:12] «12 );
A_BT[31:0] := LoadFrom(A_BDE);
IF A_BT[0] equal 0 Then
    BNDSTATUS := A_BDE | 02H;
    #BR;
FI;
A_DEST[31:0] := (Zero_extend32(base[11:2] « 4) + (A_BT[31:2] « 2 ); // address of Bound table entry
A_DEST[8][31:0] := ptr_value;
A_DEST[0][31:0] := BND.LB;
A_DEST[4][31:0] := BND.UB;

In 64-bit Mode¶

A_BDE[63:0] := (Zero_extend64(base[47+MAWA:20] « 3) + (BNDCFG[63:12] «12 );1
A_BT[63:0] := LoadFrom(A_BDE);
IF A_BT[0] equal 0 Then
    BNDSTATUS := A_BDE | 02H;
    #BR;
FI;
A_DEST[63:0] := (Zero_extend64(base[19:3] « 5) + (A_BT[63:3] « 3 ); // address of Bound table entry
A_DEST[16][63:0] := ptr_value;
A_DEST[0][63:0] := BND.LB;
A_DEST[8][63:0] := BND.UB;

Intel C/C++ 编译器等价内部函数¶

BNDSTX: _bnd_store_ptr_bounds(const void **ptr_addr, const void *ptr_val);

受影响的标志位¶

无。

保护模式异常¶

#BR 如果边界目录项无效。
#UD 如果使用了 LOCK 前缀。
如果启用 Intel MPX 时 ModRM.r/m 编码为 BND4-BND7。
如果未使用 67H 前缀且 CS.D=0。
如果使用了 67H 前缀且 CS.D=1。
#GP(0) 如果边界 table 项的目的有效地址超出 DS 段限制。
如果 DS 寄存器包含空段选择子。
如果目的操作数指向不可写的段。
#PF(故障 代码) 如果发生缺页异常。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。
如果启用 Intel MPX 时 ModRM.r/m 编码为 BND4-BND7。
如果使用了 16 位寻址。
#GP(0) 如果边界表项的目的有效地址超出 DS 段限制。

虚拟-8086 模式异常¶

#UD 如果使用了 LOCK 前缀。
如果启用 Intel MPX 时 ModRM.r/m 编码为 BND4-BND7。
如果使用了 16 位寻址。
#GP(0) 如果边界表项的目的有效地址超出 DS 段限制。
#PF(故障 代码) 如果发生缺页异常。

兼容模式异常¶

与保护模式下的异常相同。

1. 如果 CPL < 3，则使用管理模式下的 MAWA (MAWAS)；该值为 0。如果 CPL = 3，则使用用户模式下的 MAWA (MAWAU)；该值在 CPUID.(EAX=07H,ECX=0H):ECX.MAWAU[bits 21:17] 中枚举。参见《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷附录 E.3.1。

64位模式异常¶

#BR 如果边界目录项无效。
#UD 如果 ModRM 为 RIP 相对寻址。
如果使用了 LOCK 前缀。
如果启用 Intel MPX 时 ModRM.r/m 和 REX 编码为 BND4-BND15。
#GP(0) 如果内存地址 (A_BDE 或 A_BTE) 处于非规范形式。
如果目的操作数指向不可写的段。
#PF(故障 代码) 如果发生缺页异常。
