VREDUCESH — 对标量 FP16 值执行归约转换

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.NP.0F3A.W0 57 /r /ib VREDUCESH xmm1{k1}{z}, xmm2, xmm3/m16 {sae}, imm8 A V/V AVX512-FP16 在写掩码 k1 控制下，通过减去 imm8 字段指定的若干尾数位，对 xmm3/m16 中的低二进制编码 FP16 值执行归约转换。将结果存储在 xmm1 中。将 xmm2 的 bits 127:16 复制到 xmm1[127:16]。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2 操作数 3 操作数 4
A Scalar ModRM:reg (w) VEX.vvvv (r) ModRM:r/m (r) imm8 (r)

描述¶

此指令对源操作数（第二操作数）中的低二进制编码 FP16 值执行归约转换，并在写掩码 k1 控制下，将归约后的二进制浮点格式结果存储在目的操作数（第一操作数）的低元素中。有关详细信息，请参阅 VREDUCEPH 的描述。

目的操作数的 bits 127:16 从第一源操作数的对应位复制。目的操作数的 bits MAXVL-1:128 清零。目的操作数的低 FP16 元素根据写掩码进行更新。

此指令可能会导致精度异常置位。然而，在设置了 SPE（即精度异常抑制，即 imm8[3]=1）的情况下，将不报告精度异常。

此指令可能会产生微小的非零结果。如果产生该结果，即使未屏蔽下溢异常（MXCSR 寄存器中的 UM 标志为 0），它也不会报告下溢异常。

特殊情况请参阅 表 5-30。

操作¶

VREDUCESH dest{k1}, src, imm8¶

IF k1[0] or *no writemask*:
    dest.fp16[0] := reduce_fp16(src2.fp16[0], imm8)
        // see VREDUCEPH
ELSE IF *zeroing*:
    dest.fp16[0] := 0
//else dest.fp16[0] remains unchanged
DEST[127:16] := src1[127:16]
DEST[MAXVL-1:128] := 0
