GETSEC[SMCTRL] — SMX 模式控制

操作码 指令 描述
NP 0F 37 (EAX = 7) GETSEC[SMCTRL] 根据输入 EBX 选择执行指定的 SMX 模式控制。

描述¶

GETSEC[SMCTRL] 指令可用于执行某些特定的 SMX 模式控制操作。要执行的操作通过输入寄存器 EBX 选择。目前仅支持 EBX 输入值为 0。所有其他 EBX 设置都将导致触发通用保护违规。

如果 EBX 设置为 0，则使用 SMCTRL 叶函数来重新启用 SMI 事件。SMI 被执行 GETSEC[SENTER] 指令的 ILP 屏蔽（SMI 在响应逻辑处理器中也是作为对 SENTER 会合消息的响应而被屏蔽的）。何时允许此指令以及解除屏蔽的事件取决于处理器上下文（见表 7-11）。为简明起见，当 EBX=0 时使用 SMCTRL 将被称为 GETSEC[SMCTRL(0)]。

作为支持启动度量环境的一部分，SMI、NMI 和 INIT 事件在 GETSEC[SENTER] 之后被屏蔽，并且在退出已认证执行模式后保持屏蔽。解除这些事件的屏蔽应伴随安全地启用这些事件处理程序。这些安全问题在 VMX 操作中可以由 MVMM 处理。

VM 监控程序可以选择两种方法：

  • 在双监控程序方法中，执行软件将建立一个与执行 VMM（即 MVMM）并行的 SMM 监控程序，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3C 卷第 32 章“系统管理模式”。SMM 监控程序专用处理 SMI 事件而不损害 MVMM 的安全性。在度量环境处于活动状态时处理 SMI 的此使用模型不需要使用 GETSEC[SMCTRL(0)]，因为在 VMX 环境启动后，事件重新启用是隐式处理的，并通过单独的基于 VMX 的控制来处理。
  • 如果将不建立专用的 SMM 监控程序，且要在度量环境内处理 SMI，则执行软件可以使用 GETSEC[SMCTRL(0)] 重新启用因 SENTER 而被屏蔽的 SMI。

表 7-11 定义了可以使用 GETSEC[SMCTRL(0)] 的处理器上下文以及将要解除屏蔽的事件。请请注意，解除屏蔽的事件取决于当前运行的处理器上下文。

操作¶

(* 内部标志位 ACMODEFLAG 和 SENTERFLAG 的状态跨指令边界持久化 *)
IF (CR4.SMXE=0)
    THEN #UD;
ELSE IF (in VMX non-root operation)
    THEN VM Exit (reason=”GETSEC instruction”);
ELSE IF (GETSEC leaf unsupported)
    THEN #UD;
ELSE IF ((CR0.PE=0) or (CPL>0) OR (EFLAGS.VM=1))
    THEN #GP(0);
ELSE IF((EBX=0) and (SENTERFLAG=1) and (ACMODEFLAG=0) and (IN_SMM=0) and
        (((in VMX root operation) and (SMM monitor not configured)) or (not in VMX operation)) )
    THEN unmask SMI;
ELSE
    #GP(0);
END

受影响的标志位¶

无。

前缀的使用¶

LOCK 导致 #UD。

REP* 导致 #UD（包括 REPNE/REPNZ 和 REP/REPE/REPZ）。

操作数大小导致 #UD。

不允许使用 NP 66/F2/F3 前缀。

段重写被忽略。

地址大小被忽略。

REX 被忽略。

保护模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[SMCTRL] 未报告支持 GETSEC[SMCTRL]。
#GP(0) 如果 CR0.PE=0 或 CPL>0 或 EFLAGS.VM=1。
如果处于 VMX 根操作中。
如果保护分区尚未激活或处理器当前处于已认证代码模式。
如果处理器处于 SMM 中。
如果未配置 SMM 监控程序。

实地址模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[SMCTRL] 未报告支持 GETSEC[SMCTRL]。
#GP(0) GETSEC[SMCTRL] 在实地址模式下无法识别。

虚拟-8086 模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[SMCTRL] 未报告支持 GETSEC[SMCTRL]。
#GP(0) GETSEC[SMCTRL] 在虚拟-8086 模式下无法识别。

兼容模式异常¶

所有保护模式异常均适用。

64位模式异常¶

所有保护模式异常均适用。

VM-exit 条件¶

原因 (GETSEC) 如果处于 VMX 非根操作中。
