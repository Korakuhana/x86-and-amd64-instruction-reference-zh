LZCNT — 计算前导零位个数

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
F3 0F BD /r LZCNT r16, r/m16 RM 有效/有效 LZCNT 计算 r/m16 中前导零位的个数，并将结果返回到 r16 中。
F3 0F BD /r LZCNT r32, r/m32 RM 有效/有效 LZCNT 计算 r/m32 中前导零位的个数，并将结果返回到 r32 中。
F3 REX.W 0F BD /r LZCNT r64, r/m64 RM 有效/非有效 LZCNT 计算 r/m64 中前导零位的个数，并将结果返回到 r64 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

计算源操作数（第二个操作数）中前导最高有效零位的个数，并将结果返回到目的操作数（第一个操作数）中。

LZCNT 与 BSR 不同。例如，当输入操作数为零时，LZCNT 将产生操作数大小的值。需要注意的是，在不支持 LZCNT 的处理器上，该指令的字节编码将作为 BSR 执行。

在 64 位模式下，64 位操作数大小需要 REX.W=1。

操作¶

temp := OperandSize - 1
DEST := 0
WHILE (temp >= 0) AND (Bit(SRC, temp) = 0)
DO
    temp := temp - 1
    DEST := DEST+ 1
OD
IF DEST = OperandSize
    THEN CF := 1;
    ELSE CF := 0;
FI;
IF DEST = 0
    THEN ZF := 1;
    ELSE ZF := 0;
FI;

受影响的标志位¶

如果输出为零（源的最高有效位被设置），ZF 标志设置为 1，否则设置为 0；如果输入为零，CF 标志设置为 1，否则清除。OF、SF、PF 和 AF 标志未定义。

保护模式异常¶

#GP(0) 如果在 CS、DS、ES、FS 或 GS 段中存在非法的内存操作数有效地址。
如果使用 DS、ES、FS 或 GS 寄存器访问内存且它包含一个空段选择子。
#SS(0) 如果 SS 段中的地址非法。
#PF (错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
