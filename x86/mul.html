MUL — 无符号乘法

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
F6 /4 MUL r/m8 M Valid Valid 无符号乘法 (AX := AL ∗ r/m8)。
REX + F6 /4 MUL r/m81 M Valid N.E. 无符号乘法 (AX := AL ∗ r/m8)。
F7 /4 MUL r/m16 M Valid Valid 无符号乘法 (DX:AX := AX ∗ r/m16)。
F7 /4 MUL r/m32 M Valid Valid 无符号乘法 (EDX:EAX := EAX ∗ r/m32)。
REX.W + F7 /4 MUL r/m64 M Valid N.E. 无符号乘法 (RDX:RAX := RAX ∗ r/m64)。

1. 在 64 位模式下，如果使用 REX 前缀，则不能将 r/m8 编码为访问以下字节寄存器：AH、BH、CH、DH。

描述 ¶

对第一个操作数（目的操作数）和第二个操作数（源操作数）进行无符号乘法，并将结果存储在目的操作数中。目的操作数是一个隐含的操作数，位于寄存器 AL、AX 或 EAX 中（取决于操作数的大小）；源操作数位于通用寄存器或内存地址中。此指令的行为和结果存储位置取决于操作码和操作数大小，如表 4-9 所示。

结果存储在寄存器 AX、寄存器对 DX:AX 或寄存器对 EDX:EAX 中（取决于操作数大小），乘积的高位部分分别包含在寄存器 AH、DX 或 EDX 中。如果乘积的高位部分为 0，则清除 CF 和 OF 标志；否则设置这些标志。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 前缀可将操作提升至 64 位。

请参阅本节开头的汇总图表以获取编码数据和限制。

操作 ¶ 

IF (Byte operation)
    THEN
        AX := AL ∗ SRC;
    ELSE (* Word or doubleword operation *)
        IF OperandSize = 16
            THEN
                DX:AX := AX ∗ SRC;
            ELSE IF OperandSize = 32
                THEN EDX:EAX := EAX ∗ SRC; FI;
            ELSE (* OperandSize = 64 *)
                RDX:RAX := RAX ∗ SRC;
        FI;
FI;

受影响的标志位 ¶

如果结果的高半部分为 0，则 OF 和 CF 标志位被设置为 0；否则被设置为 1。SF、ZF、AF 和 PF 标志位未定义。

保护模式异常 ¶

#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

#SS 如果内存操作数的有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常 ¶

兼容模式异常 ¶
