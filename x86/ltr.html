LTR — 加载任务寄存器

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 00 /3 LTR r/m16 M Valid Valid 将 r/m16 加载到任务寄存器中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r) N/A N/A N/A

描述¶

将源操作数加载到任务寄存器的段选择子字段中。源操作数（通用寄存器或内存位置）包含一个指向任务状态段（TSS）的段选择子。在任务寄存器中加载段选择子之后，处理器使用该段选择子在全局描述符表（GDT）中定位该 TSS 的段描述符。然后，它将 TSS 的段限长和基地址从段描述符加载到任务寄存器中。由任务寄存器指向的任务被标记为繁忙，但不会发生到该任务的切换。

LTR 指令供操作系统软件使用；不应在应用程序中使用。它只能在 CPL 为 0 的保护模式下执行。它通常用于初始化代码中以建立第一个要执行的任务。

操作数大小属性对此指令没有影响。

在 64 位模式下，操作数大小仍固定为 16 位。该指令引用 16 字节的描述符以加载 64 位基地址。

操作¶

IF SRC is a NULL selector
    THEN #GP(0);
IF SRC(Offset) > descriptor table limit OR IF SRC(type) ≠ global
    THEN #GP(segment selector); FI;
Read segment descriptor;
IF segment descriptor is not for an available TSS
    THEN #GP(segment selector); FI;
IF segment descriptor is not present
    THEN #NP(segment selector); FI;
TSSsegmentDescriptor(busy) := 1;
(* Locked read-modify-write operation on the entire descriptor when setting busy flag *)
TaskRegister(SegmentSelector) := SRC;
TaskRegister(SegmentDescriptor) := TSSSegmentDescriptor;

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果当前特权级别不为 0。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果源操作数包含 NULL 段选择子。
如果 DS、ES、FS 或 GS 寄存器用于访问内存且包含 NULL 段选择子。
#GP(选择子) 如果源选择子指向的段不是 TSS，或者指向已处于繁忙状态的任务的 TSS。
如果选择子指向 LDT 或超出 GDT 限制。
#NP(选择子) 如果 TSS 描述符被标记为不存在（not present）。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 在实地址模式下无法识别 LTR 指令。

虚拟-8086 模式异常¶

#UD 在虚拟-8086 模式下无法识别 LTR 指令。

兼容模式异常¶

与保护模式下的异常相同，并增加了以下异常：

#GP(选择子) 如果源选择子指向一个 16 位的 TSS。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果当前特权级别不为 0。
如果内存地址为非规范形式。
如果源操作数包含 NULL 段选择子。
#GP(选择子) 如果源选择子指向的段不是 TSS，或指向 16 位 TSS，或指向已处于繁忙状态的任务的 TSS。
如果选择子指向 LDT 或超出 GDT 限制。
如果 16 字节描述符的高 8 字节描述符类型不为零。
#NP(选择子) 如果 TSS 描述符被标记为不存在（not present）。
#PF(错误码) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。
