VPERMB — 置换打包字节元素

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.128.66.0F38.W0 8D /r VPERMB xmm1 {k1}{z}, xmm2, xmm3/m128 A V/V AVX512VL AVX512_VBMI 使用 xmm2 中的字节索引置换 xmm3/m128 中的字节，在写掩码 k1 下将结果存储在 xmm1 中。
EVEX.256.66.0F38.W0 8D /r VPERMB ymm1 {k1}{z}, ymm2, ymm3/m256 A V/V AVX512VL AVX512_VBMI 使用 ymm2 中的字节索引置换 ymm3/m256 中的字节，在写掩码 k1 下将结果存储在 ymm1 中。
EVEX.512.66.0F38.W0 8D /r VPERMB zmm1 {k1}{z}, zmm2, zmm3/m512 A V/V AVX512_VBMI 使用 zmm2 中的字节索引置换 zmm3/m512 中的字节，在写掩码 k1 下将结果存储在 zmm1 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Full Mem ModRM:reg (w) EVEX.vvvv (r) ModRM:r/m (r) N/A

描述¶

根据第一源操作数（第二操作数）中的字节索引，将字节从第二源操作数（第三操作数）复制到目的操作数（第一操作数）。注意，此指令允许将源操作数中的某个字节复制到目的操作数中的多个位置。

仅使用每个字节索引的低 6(EVEX.512)/5(EVEX.256)/4(EVEX.128) 位来从第二源操作数中选择源字节的位置。

第一源操作数是 ZMM/YMM/XMM 寄存器。第二源操作数可以是 ZMM/YMM/XMM 寄存器或 512/256/128 位内存位置。目的操作数是 ZMM/YMM/XMM 寄存器，在写掩码 k1 下以字节粒度更新。

操作¶

VPERMB (EVEX 编码版本)¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
IF VL = 128:
    n := 3;
ELSE IF VL = 256:
    n := 4;
ELSE IF VL = 512:
    n := 5;
FI;
FOR j := 0 TO KL-1:
    id := SRC1[j*8 + n : j*8] ; // location of the source byte
    IF k1[j] OR *no writemask* THEN
        DEST[j*8 + 7: j*8] := SRC2[id*8 +7: id*8];
    ELSE IF zeroing-masking THEN
        DEST[j*8 + 7: j*8] := 0;
    *ELSE
        DEST[j*8 + 7: j*8] remains unchanged*
    FI
ENDFOR
DEST[MAX_VL-1:VL] := 0;
