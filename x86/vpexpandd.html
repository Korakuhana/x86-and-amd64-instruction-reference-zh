VPEXPANDD — 从紧凑内存/寄存器加载稀疏打包双字整数值

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.128.66.0F38.W0 89 /r VPEXPANDD xmm1 {k1}{z}, xmm2/m128 A V/V AVX512VL AVX512F 使用写掩码 k1，将 xmm2/m128 中的打包双字整数值展开到 xmm1。
EVEX.256.66.0F38.W0 89 /r VPEXPANDD ymm1 {k1}{z}, ymm2/m256 A V/V AVX512VL AVX512F 使用写掩码 k1，将 ymm2/m256 中的打包双字整数值展开到 ymm1。
EVEX.512.66.0F38.W0 89 /r VPEXPANDD zmm1 {k1}{z}, zmm2/m512 A V/V AVX512F 使用写掩码 k1，将 zmm2/m512 中的打包双字整数值展开到 zmm1。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Tuple1 Scalar ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

根据写掩码 k1 的选择，将源操作数（第二操作数）中输入向量的最多 16 个连续双字整数值展开（加载）到目的操作数（第一操作数）中的稀疏元素。目的操作数是 ZMM 寄存器，源操作数可以是 ZMM 寄存器或内存位置。

输入向量从源操作数中的最低元素开始。操作掩码寄存器 k1 选择要由输入向量中递增的元素替换的目的地元素（如果少于 8 个元素，则为部分向量或稀疏元素）。未被写掩码 k1 选择的目的地元素要么保持未修改，要么清零，具体取决于 EVEX.z。

注意：EVEX.vvvv 保留且必须为 1111b，否则指令将导致 #UD。

注意，压缩位移假设的预缩放 (N) 对应于单个元素的大小，而不是整个向量的大小。

操作¶

VPEXPANDD (EVEX 编码版本)¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
k := 0
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN
            DEST[i+31:i] := SRC[k+31:k];
            k := k + 32
        ELSE
            IF *merging-masking*
                        ; merging-masking
                THEN *DEST[i+31:i] remains unchanged*
                ELSE ; zeroing-masking
                    DEST[i+31:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0

SIMD 浮点异常¶

无。

其他异常¶

EVEX 编码指令，请参阅表 2-49，“第 E4 类异常条件”中的异常类型 E4.nb。

此外：

#UD 如果 EVEX.vvvv != 1111B。
