FMUL/FMULP/FIMUL — 乘法

操作码 指令 64位模式 兼容/传统模式 描述
D8 /1 FMUL m32fp 有效 有效 将 ST(0) 乘以 m32fp，并将结果存储在 ST(0) 中。
DC /1 FMUL m64fp 有效 有效 将 ST(0) 乘以 m64fp，并将结果存储在 ST(0) 中。
D8 C8+i FMUL ST(0), ST(i) 有效 有效 将 ST(0) 乘以 ST(i)，并将结果存储在 ST(0) 中。
DC C8+i FMUL ST(i), ST(0) 有效 有效 将 ST(i) 乘以 ST(0)，并将结果存储在 ST(i) 中。
DE C8+i FMULP ST(i), ST(0) 有效 有效 将 ST(i) 乘以 ST(0)，并将结果存储在 ST(i) 中，然后弹出寄存器栈。
DE C9 FMULP 有效 有效 将 ST(1) 乘以 ST(0)，并将结果存储在 ST(1) 中，然后弹出寄存器栈。
DA /1 FIMUL m32int 有效 有效 将 ST(0) 乘以 m32int，并将结果存储在 ST(0) 中。
DE /1 FIMUL m16int 有效 有效 将 ST(0) 乘以 m16int，并将结果存储在 ST(0) 中。

描述¶

将目的操作数和源操作数相乘，并将乘积存储在目的位置。目的操作数始终是 FPU 数据寄存器；源操作数可以是 FPU 数据寄存器或内存位置。内存中的源操作数可以是单精度或双精度浮点格式，也可以是字或双字整数格式。

无操作数版本的指令将 ST(1) 寄存器的内容乘以 ST(0) 寄存器的内容，并将乘积存储在 ST(1) 寄存器中。单操作数版本将 ST(0) 寄存器的内容乘以内存位置的内容（浮点值或整数值），并将乘积存储在 ST(0) 寄存器中。双操作数版本将 ST(0) 寄存器的内容乘以 ST(i) 寄存器的内容，反之亦然，结果存储在第一个操作数（目的操作数）指定的寄存器中。

FMULP 指令在存储乘积后执行弹出 FPU 寄存器栈的附加操作。要弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。无操作数版本的浮点乘法指令始终会导致寄存器栈被弹出。在某些汇编器中，此指令的助记符是 FMUL 而不是 FMULP。

FIMUL 指令在执行乘法之前将整数源操作数转换为双精度扩展浮点格式。

结果的符号始终是源符号的异或（exclusive-OR），即使被乘的一个或多个值是 0 或 ∞。当源操作数是整数 0 时，它被视为 +0。

下表显示了在相乘各类数字时获得的结果，假设既没有发生溢出也没有发生下溢。

F 表示有限浮点值。

I 表示整数。

* 表示无效算术操作数 (#IA) 异常。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

IF Instruction = FIMUL
    THEN
        DEST := DEST ∗ ConvertToDoubleExtendedPrecisionFP(SRC);
    ELSE (* Source operand is floating-point value *)
        DEST := DEST ∗ SRC;
FI;
IF Instruction = FMULP
    THEN
        PopRegisterStack;
FI;

影响的 FPU 标志寄存器¶

C1 如果发生栈下溢，则设置为 0。
如果结果被向上舍入，则设置；否则清除。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈下溢。
#IA 操作数是 SNaN 值或不受支持的格式。
一个操作数是 ±0，另一个是 ±∞。
#D 源操作数是反常（denormal）值。
#U 结果对于目的格式过小。
#O 结果对于目的格式过大。
#P 值无法在目的格式中精确表示。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存，并且它包含空段选择器。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
