VUCOMISH — 无序比较标量 FP16 值并设置 EFLAGS

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.NP.MAP5.W0 2E /r VUCOMISH xmm1, xmm2/m16 {sae} A V/V AVX512-FP16 比较 xmm1 和 xmm2/m16 中的低 FP16 值，并相应地设置 EFLAGS 标志位。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2 操作数 3 操作数 4
A Scalar ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

此指令比较操作数 1（第一操作数）的低字和操作数 2（第二操作数）中的 FP16 值，并根据结果（无序、大于、小于或等于）设置 EFLAGS 寄存器中的 ZF、PF 和 CF 标志。EFLAGS 寄存器中的 OF、SF 和 AF 标志位设置为 0。如果任一源操作数为 NaN（QNaN 或 SNaN），则返回无序结果。

操作数 1 是 XMM 寄存器；操作数 2 可以是 XMM 寄存器或 16 位内存位置。

VUCOMISH 指令与 VCOMISH 指令的不同之处在于，仅当源操作数为 SNaN 时才触发 SIMD 浮点无效操作异常 (#I)。当源操作数是 QNaN 或 SNaN 时，VCOMISH 指令都会触发无效数值异常。

如果产生了未屏蔽的 SIMD 浮点异常，则不更新 EFLAGS 寄存器。EVEX.vvvv 是保留字段，必须为 1111b，否则指令将触发 #UD 故障。

操作¶

VUCOMISH¶

RESULT := UnorderedCompare(SRC1.fp16[0],SRC2.fp16[0])
if RESULT is UNORDERED:
    ZF, PF, CF := 1, 1, 1
else if RESULT is GREATER_THAN:
    ZF, PF, CF := 0, 0, 0
else if RESULT is LESS_THAN:
    ZF, PF, CF := 0, 0, 1
else: // RESULT is EQUALS
    ZF, PF, CF := 1, 0, 0
OF, AF, SF := 0, 0, 0
