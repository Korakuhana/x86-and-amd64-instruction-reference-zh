VDPBF16PS — 累加到打包单精度中的 BF16 对的点积

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
EVEX.128.F3.0F38.W0 52 /r VDPBF16PS xmm1{k1}{z}, xmm2, xmm3/m128/m32bcst A V/V AVX512VL AVX512_BF16 将 xmm2 和 xmm3/m128 中的 BF16 对相乘，并在写掩码 k1 的控制下将产生的打包单精度结果累加到 xmm1 中。
EVEX.256.F3.0F38.W0 52 /r VDPBF16PS ymm1{k1}{z}, ymm2, ymm3/m256/m32bcst A V/V AVX512VL AVX512_BF16 将 ymm2 和 ymm3/m256 中的 BF16 对相乘，并在写掩码 k1 的控制下将产生的打包单精度结果累加到 ymm1 中。
EVEX.512.F3.0F38.W0 52 /r VDPBF16PS zmm1{k1}{z}, zmm2, zmm3/m512/m32bcst A V/V AVX512F AVX512_BF16 将 zmm2 和 zmm3/m512 中的 BF16 对相乘，并在写掩码 k1 的控制下将产生的打包单精度结果累加到 zmm1 中。

描述¶

此指令执行两个 BF16 对的 SIMD 点积，并累加到打包单精度寄存器中。

执行 FMA 的每次累加时均使用“舍入到最接近的偶数”（RNE）舍入模式。输出非正规数总是被清零（FTZ），输入非正规数总是被视为零（DAZ）。不参考也不更新 MXCSR。

NaN 传播优先级在表 5-1 中描述。

操作¶

Define make_fp32(x):
    // The x parameter is bfloat16. Pack it in to upper 16b of a dword. The bit pattern is a legal fp32 value. Return that bit pattern.
    dword := 0
    dword[31:16] := x
    RETURN dword

VDPBF16PS srcdest, src1, src2¶

VL = (128, 256, 512)
KL = VL/32
origdest := srcdest
FOR i := 0 to KL-1:
    IF k1[ i ] or *no writemask*:
        IF src2 is memory and evex.b == 1:
            t := src2.dword[0]
        ELSE:
            t := src2.dword[ i ]
        // FP32 FMA with daz in, ftz out and RNE rounding. MXCSR neither consulted nor updated.
        srcdest.fp32[ i ] += make_fp32(src1.bfloat16[2*i+1]) * make_fp32(t.bfloat[1])
        srcdest.fp32[ i ] += make_fp32(src1.bfloat16[2*i+0]) * make_fp32(t.bfloat[0])
    ELSE IF *zeroing*:
        srcdest.dword[ i ] := 0
    ELSE: // merge masking, dest element unchanged
        srcdest.dword[ i ] := origdest.dword[ i ]
srcdest[MAXVL-1:VL] := 0
