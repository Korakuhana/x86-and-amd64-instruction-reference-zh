描述¶

对源操作数（第二个操作数）和目的操作数（第一个操作数）执行按位逻辑或操作，并将结果存储在目的操作数中。如果第一个和第二个操作数的对应位中任一或两个都为 1，则结果的每个对应位都设置为 1；否则，设置为 0。

在 64 位模式下且未采用 VEX/EVEX 编码时，使用 REX.R 形式的 REX 前缀允许该指令访问其他寄存器（XMM8-XMM15）。

传统 SSE 版本：源操作数可以是 MMX 技术寄存器或 64 位内存位置。目的操作数是 MMX 技术寄存器。

128位传统 SSE 版本：第二个源操作数是 XMM 寄存器或 128 位内存位置。第一个源操作数和目的操作数可以是 XMM 寄存器。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：第二个源操作数是 XMM 寄存器或 128 位内存位置。第一个源操作数和目的操作数可以是 XMM 寄存器。目的 YMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：第二个源操作数是 YMM 寄存器或 256 位内存位置。第一个源操作数和目的操作数可以是 YMM 寄存器。

EVEX 编码版本：第一个源操作数是 ZMM/YMM/XMM 寄存器。第二个源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 32/64 位内存位置广播的 512/256/128 位向量。目的操作数是 ZMM/YMM/XMM 寄存器，在写掩码 k1 下以 32/64 位粒度条件性地更新。

操作¶

POR（64 位操作数）¶

DEST := DEST OR SRC

POR（128位传统 SSE 版本）¶

DEST := DEST OR SRC
DEST[MAXVL-1:128] (Unmodified)

VPOR（VEX.128 编码版本）¶

DEST := SRC1 OR SRC2
DEST[MAXVL-1:128] := 0

VPOR（VEX.256 编码版本）¶

DEST := SRC1 OR SRC2
DEST[MAXVL-1:256] := 0

VPORD（EVEX 编码版本）¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask* THEN
            IF (EVEX.b = 1) AND (SRC2 *is memory*)
                THEN DEST[i+31:i] := SRC1[i+31:i] BITWISE OR SRC2[31:0]
                ELSE DEST[i+31:i] := SRC1[i+31:i] BITWISE OR SRC2[i+31:i]
            FI;
        ELSE
            IF *merging-masking* ; merging-masking
                *DEST[i+31:i] remains unchanged*
                ELSE
                        ; zeroing-masking
                    DEST[i+31:i] := 0
            FI;
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0
