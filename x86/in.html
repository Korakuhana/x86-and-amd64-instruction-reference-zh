IN — 从端口输入

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
E4 ib IN AL, imm8 I Valid Valid 从立即数 imm8 指定的 I/O 端口地址输入字节到 AL。
E5 ib IN AX, imm8 I Valid Valid 从立即数 imm8 指定的 I/O 端口地址输入字到 AX。
E5 ib IN EAX, imm8 I Valid Valid 从立即数 imm8 指定的 I/O 端口地址输入双字到 EAX。
EC IN AL, DX ZO Valid Valid 从 DX 中指定的 I/O 端口输入字节到 AL。
ED IN AX, DX ZO Valid Valid 从 DX 中指定的 I/O 端口输入字到 AX。
ED IN EAX, DX ZO Valid Valid 从 DX 中指定的 I/O 端口输入双字到 EAX。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
I imm8 N/A N/A N/A
ZO N/A N/A N/A N/A

描述¶

将第二个操作数（源操作数）指定的 I/O 端口中的值复制到目标操作数（第一个操作数）。源操作数可以是 8 位立即数或 DX 寄存器；目标操作数可以是 AL、AX 或 EAX 寄存器，具体取决于所访问的端口大小（分别为 8 位、16 位或 32 位）。使用 DX 寄存器作为源操作数允许访问 0 到 65,535 的 I/O 端口地址；使用 8 位立即数允许访问 0 到 255 的 I/O 端口地址。

访问 8 位 I/O 端口时，操作码决定端口大小；访问 16 位和 32 位 I/O 端口时，操作数大小属性决定端口大小。在机器码级别，访问 8 位 I/O 端口时的 I/O 指令更短。此时，端口地址的高 8 位将为 0。

此指令仅用于访问位于处理器 I/O 地址空间中的 I/O 端口。有关在 I/O 地址空间中访问 I/O 端口的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷中的第 19 章“输入/输出”。

此指令在非 64 位模式和 64 位模式下的操作相同。

操作¶

IF ((PE = 1) and ((CPL > IOPL) or (VM = 1)))
    THEN (* Protected mode with CPL > IOPL or virtual-8086 mode *)
        IF (Any I/O Permission Bit for I/O port being accessed = 1)
            THEN (* I/O operation is not allowed *)
                #GP(0);
            ELSE ( * I/O operation is allowed *)
                DEST := SRC; (* Read from selected I/O port *)
        FI;
    ELSE (Real Mode or Protected Mode with CPL ≤ IOPL *)
        DEST := SRC; (* Read from selected I/O port *)
FI;

影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果 CPL 大于（特权级较低）I/O 特权级别（IOPL），并且正在访问的 I/O 端口在 TSS 中对应的任意 I/O 许可位为 1。
#PF(fault-code) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#GP(0) 如果所访问的 I/O 端口在 TSS 中的任意 I/O 许可位为 1。
#PF(fault-code) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#GP(0) 如果 CPL 大于（特权级较低）I/O 特权级别（IOPL），并且正在访问的 I/O 端口在 TSS 中对应的任意 I/O 许可位为 1。
#PF(fault-code) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。
