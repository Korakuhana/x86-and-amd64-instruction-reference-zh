UD — 未定义指令

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F FF /r UD01 r32, r/m32 RM Valid Valid 引发无效操作码异常。
0F B9 /r UD1 r32, r/m32 RM Valid Valid 引发无效操作码异常。
0F 0B UD2 ZO Valid Valid 引发无效操作码异常。

1. 某些处理器在解码 UD0 指令时不带 ModR/M 字节。结果是，当带有 ModR/M 字节（以及任何隐式字节）的指令跨越页面或段边界时，这些处理器将交付无效操作码异常，而不是在指令获取时发生错误。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4

描述¶

产生无效操作码异常。提供此指令用于软件测试，以显式产生无效操作码异常。此指令的操作码为此目的而保留。

除了引发无效操作码异常外，该指令对处理器状态或内存没有影响。

尽管是 UD 指令的执行引起了无效操作码异常，但通过交付异常所保存的指令指针引用的是 UD 指令（而不是其后的指令）。

该指令在非 64 位模式和 64 位模式下的操作相同。

操作¶

#UD (* Generates invalid opcode exception *);

受影响的标志位¶

无。

异常（所有操作模式）¶

#UD 在所有操作模式下引发无效操作码异常。
