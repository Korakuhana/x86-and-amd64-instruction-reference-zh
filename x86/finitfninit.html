FINIT/FNINIT — 初始化浮点单元

操作码 指令 64位模式 兼容/传统模式 描述
9B DB E3 FINIT 有效 有效 在检查是否存在未屏蔽的挂起浮点异常后，初始化 FPU。
DB E3 FNINIT1 有效 有效 在不检查是否存在未屏蔽的挂起浮点异常的情况下，初始化 FPU。

1. 参见下文的“IA-32 架构兼容性”部分。

描述¶

将 FPU 控制、状态、标记（tag）、指令指针和数据指针寄存器设置为其默认状态。FPU 控制字被设置为 037FH（舍入到最接近的数、屏蔽所有异常、64位精度）。状态字被清零（没有设置异常标志，TOP 设置为 0）。寄存器栈中的数据寄存器保持不变，但它们全都被标记为空（11B）。指令指针和数据指针都被清零。

FINIT 指令在执行初始化之前会检查并处理任何未屏蔽的挂起浮点异常；而 FNINIT 指令不会。

汇编器为 FINIT 指令生成两条指令（FWAIT 指令后跟 FNINIT 指令），处理器分别执行这两条指令。如果其中任一指令产生异常，保存的 EIP 将指向引起该异常的指令。

此指令的操作在非 64 位模式和 64 位模式下相同。

IA-32 架构兼容性¶

在 MS-DOS 兼容模式下运行 Pentium 或 Intel486 处理器时，在罕见情况下，FNINIT 指令可能会在执行前被中断以处理挂起的 FPU 异常。有关这些情况的描述，请参阅《Intel® 64 and IA-32 Architectures Software Developer’s Manual, Volume 1》附录 D 中标题为“No-Wait FPU Instructions Can Get FPU Interrupt in Window”的部分。在较新的 Intel 处理器上，除了 Intel Quark™ X1000 处理器外，FNINIT 指令不能以这种方式被中断。

在 Intel387 数学协处理器中，FINIT/FNINIT 指令不清除指令指针和数据指针。

此指令仅影响 x87 FPU。它不影响 XMM 和 MXCSR 寄存器。

操作¶

FPUControlWord := 037FH;
FPUStatusWord := 0;
FPUTagWord := FFFFH;
FPUDataPointer := 0;
FPUInstructionPointer := 0;
FPULastInstructionOpcode := 0;

影响的 FPU 标志寄存器¶

C0, C1, C2, C3 设置为 0。

浮点异常¶

无。

保护模式异常¶

#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在未决 a x87 FPU 异常。
#UD 如果使用了 LOCK 前缀。
