FCOMI/FCOMIP/FUCOMI/FUCOMIP — 比较浮点值并设置 EFLAGS

操作码 指令 64位模式 兼容/传统模式 描述
DB F0+i FCOMI ST, ST(i) 有效 有效 将 ST(0) 与 ST(i) 比较并相应地设置状态标志。
DF F0+i FCOMIP ST, ST(i) 有效 有效 将 ST(0) 与 ST(i) 比较，相应地设置状态标志，并弹出寄存器栈。
DB E8+i FUCOMI ST, ST(i) 有效 有效 将 ST(0) 与 ST(i) 比较，检查是否有序值，并相应地设置状态标志。
DF E8+i FUCOMIP ST, ST(i) 有效 有效 将 ST(0) 与 ST(i) 比较，检查是否有序值，相应地设置状态标志，并弹出寄存器栈。

描述¶

对寄存器 ST(0) 和 ST(i) 的内容执行无序比较，并根据结果在 EFLAGS 寄存器中设置状态标志 ZF、PF 和 CF（见下表）。比较时忽略零的符号，即 -0.0 等于 +0.0。

* 参见下方的 IA-32 架构兼容性部分。

** 如果产生了未屏蔽的无效算术操作数 (#IA) 异常，则不设置标志。

无序比较会检查正在比较的数字类别（参见本章中的“FXAM — 检查浮点数”）。FUCOMI/FUCOMIP 指令执行与 FCOMI/FCOMIP 指令相同的操作。唯一的区别是，FUCOMI/FUCOMIP 指令仅在操作数中的一个或两个是 SNaN 或处于不受支持的格式时引发无效算术操作数异常 (#IA)；QNaN 会导致条件码标志被设置为无序，但不会导致产生异常。FCOMI/FCOMIP 指令在操作数中的一个或两个是任何种类的 NaN 值或处于不受支持的格式时会引发无效操作异常。

如果操作导致引发无效算术操作数异常，则仅在屏蔽该异常时才设置 EFLAGS 寄存器中的状态标志。

FCOMI/FCOMIP 和 FUCOMI/FUCOMIP 指令将 EFLAGS 寄存器中的 OF、SF 和 AF 标志设置为零（无论是否检测到无效操作异常）。

FCOMIP 和 FUCOMIP 指令还在比较操作之后弹出寄存器栈。要弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。

此指令的操作在非 64 位模式和 64 位模式下相同。

IA-32 架构兼容性¶

FCOMI/FCOMIP/FUCOMI/FUCOMIP 指令是在 P6 系列处理器中引入到 IA-32 架构的，在早期的 IA-32 处理器中不可用。

操作¶

CASE (relation of operands) OF
	ST(0) > ST(i):
						ZF, PF, CF := 000;
	ST(0) < ST(i):
						ZF, PF, CF := 001;
	ST(0) = ST(i):
						ZF, PF, CF := 100;
ESAC;
IF Instruction is FCOMI or FCOMIP
	THEN
		IF ST(0) or ST(i) = NaN or unsupported format
			THEN
				#IA
				IF FPUControlWord.IM = 1
						THEN
							ZF, PF, CF := 111;
				FI;
		FI;
FI;
IF Instruction is FUCOMI or FUCOMIP
	THEN
		IF ST(0) or ST(i) = QNaN, but not SNaN or unsupported format
			THEN
				ZF, PF, CF := 111;
			ELSE (* ST(0) or ST(i) is SNaN or unsupported format *)
					#IA;
				IF FPUControlWord.IM = 1
						THEN
							ZF, PF, CF := 111;
				FI;
		FI;
FI;
IF Instruction is FCOMIP or FUCOMIP
	THEN
		PopRegisterStack;
FI;

影响的 FPU 标志寄存器¶

C1 设置为 0。
C0, C2, C3 不受影响。

浮点异常¶

#IS 发生栈下溢。
#IA （FCOMI 或 FCOMIP 指令）一个或两个操作数是 NaN 值或具有不受支持的格式。
（FUCOMI 或 FUCOMIP 指令）一个或两个操作数是 SNaN 值（但不是 QNaN）或具有未定义的格式。检测到 QNaN 值不会引发无效操作数异常。

保护模式异常¶

#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果有挂起的 x87 FPU 异常。
#UD 如果使用了 LOCK 前缀。
