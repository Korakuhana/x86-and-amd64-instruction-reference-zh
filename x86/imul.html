描述 ¶

对两个操作数执行有符号乘法。根据操作数的数量，该指令有三种形式。

  • 单操作数形式 ——此形式与 MUL 指令所使用的形式相同。在这里，源操作数（在通用寄存器或内存位置中）乘以 AL、AX、EAX 或 RAX 寄存器中的值（具体取决于操作数大小），乘积（大小为输入操作数两倍）分别存储在 AX、DX:AX、EDX:EAX 或 RDX:RAX 寄存器中。
  • 双操作数形式 ——在此形式中，目的操作数（第一操作数）乘以源操作数（第二操作数）。目的操作数是一个通用寄存器，源操作数可以是一个立即数、通用寄存器或内存位置。中间乘积（大小为输入操作数两倍）被截断并存储在目的操作数位置。
  • 三操作数形式 ——此形式需要一个目的操作数（第一操作数）和两个源操作数（第二和第三操作数）。在这里，第一源操作数（可以是通用寄存器或内存位置）乘以第二源操作数（一个立即数）。中间乘积（大小为第一源操作数两倍）被截断并存储在目的操作数（通用寄存器）中。

当立即数用作操作数时，它会被符号扩展到目的操作数格式的长度。

当中间乘积的有符号整数值与符号扩展后被截断的操作数大小乘积不同时，CF 和 OF 标志位会被置位；否则，CF 和 OF 标志位会被清除。

IMUL 指令的三种形式是相似的，因为乘积的长度都被计算为操作数长度的两倍。在使用单操作数形式时，乘积精确地存储在目的地中。然而，在使用双操作数和三操作数形式时，结果在存储到目的寄存器之前，会被截断到目的操作数的长度。由于存在这种截断，应该测试 CF 或 OF 标志位以确保没有丢失有效位。

双操作数和三操作数形式也可以与无符号操作数一起使用，因为无论操作数是有符号还是无符号，乘积的下半部分都是相同的。然而，CF 和 OF 标志位不能用于确定结果的上半部分是否为非零。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 前缀可将操作提升至 64 位。使用 REX.W 会对指令的三种形式产生如下修改：

  • 单操作数形式 ——源操作数（在 64 位通用寄存器或内存位置中）乘以 RAX 寄存器中的值，乘积存储在 RDX:RAX 寄存器中。
  • 双操作数形式 ——如果源操作数是寄存器或内存位置，它会被提升到 64 位。目的操作数被提升到 64 位。
  • 三操作数形式 ——第一源操作数（寄存器或内存位置）和目的操作数被提升到 64 位。如果源操作数是立即数，它会被符号扩展到 64 位。

操作 ¶ 

IF (NumberOfOperands = 1)
    THEN IF (OperandSize = 8)
        THEN
            TMP_XP := AL ∗ SRC (* Signed multiplication; TMP_XP is a signed integer at twice the width of the SRC *);
            AX := TMP_XP[15:0];
            IF SignExtend(TMP_XP[7:0]) = TMP_XP
                THEN CF := 0; OF := 0;
                ELSE CF := 1; OF := 1; FI;
        ELSE IF OperandSize = 16
            THEN
                TMP_XP := AX ∗ SRC (* Signed multiplication; TMP_XP is a signed integer at twice the width of the SRC *)
                DX:AX := TMP_XP[31:0];
                IF SignExtend(TMP_XP[15:0]) = TMP_XP
                    THEN CF := 0; OF := 0;
                    ELSE CF := 1; OF := 1; FI;
            ELSE IF OperandSize = 32
                THEN
                    TMP_XP := EAX ∗ SRC (* Signed multiplication; TMP_XP is a signed integer at twice the width of the SRC*)
                    EDX:EAX := TMP_XP[63:0];
                    IF SignExtend(TMP_XP[31:0]) = TMP_XP
                        THEN CF := 0; OF := 0;
                        ELSE CF := 1; OF := 1; FI;
                ELSE (* OperandSize = 64 *)
                    TMP_XP := RAX ∗ SRC (* Signed multiplication; TMP_XP is a signed integer at twice the width of the SRC *)
                    EDX:EAX := TMP_XP[127:0];
                    IF SignExtend(TMP_XP[63:0]) = TMP_XP
                        THEN CF := 0; OF := 0;
                        ELSE CF := 1; OF := 1; FI;
                FI;
        FI;
    ELSE IF (NumberOfOperands = 2)
        THEN
            TMP_XP := DEST ∗ SRC (* Signed multiplication; TMP_XP is a signed integer at twice the width of the SRC *)
            DEST := TruncateToOperandSize(TMP_XP);
            IF SignExtend(DEST) ≠ TMP_XP
                THEN CF := 1; OF := 1;
                ELSE CF := 0; OF := 0; FI;
        ELSE (* NumberOfOperands = 3 *)
            TMP_XP := SRC1 ∗ SRC2 (* Signed multiplication; TMP_XP is a signed integer at twice the width of the SRC1 *)
            DEST := TruncateToOperandSize(TMP_XP);
            IF SignExtend(DEST) ≠ TMP_XP
                THEN CF := 1; OF := 1;
                ELSE CF := 0; OF := 0; FI;
    FI;
FI;

受影响的标志位 ¶

对于指令的单操作数形式，当有效位被带入结果的上半部分时，CF 和 OF 标志位会被置位；当结果完全适合结果的下半部分时，CF 和 OF 标志位会被清除。对于指令的双操作数和三操作数形式，当结果必须被截断以适应目的操作数大小时，CF 和 OF 标志位会被置位；当结果完全适合目的操作数大小时，CF 和 OF 标志位会被清除。SF、ZF、AF 和 PF 标志位未定义。
