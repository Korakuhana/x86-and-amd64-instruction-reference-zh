描述¶

对目的操作数（第一个操作数）和源操作数（第二个操作数）中的打包字节、字或双字有符号整数进行 SIMD 有符号大于比较。如果目的操作数中的某个数据元素大于源操作数中的对应数据元素，则目的操作数中的对应数据元素设置为全 1；否则，设置为全 0。

PCMPGTB 指令比较目的操作数 and 源操作数中的对应有符号字节整数；PCMPGTW 指令比较目的操作数 and 源操作数中的对应有符号字整数；PCMPGTD 指令比较目的操作数 and 源操作数中的对应有符号双字整数。

在 64 位模式下且未使用 VEX/EVEX 编码时，使用 REX.R 形式的 REX 前缀允许此指令访问额外的寄存器（XMM8-XMM15）。

传统 SSE 指令：源操作数可以是 MMX 技术寄存器或 64 位内存位置。目的操作数可以是 MMX 技术寄存器。

128位传统 SSE 版本：第二个源操作数可以是 XMM 寄存器或 128 位内存位置。第一个源操作数和目的操作数是 XMM 寄存器。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：第二个源操作数可以是 XMM 寄存器或 128 位内存位置。第一个源操作数和目的操作数是 XMM 寄存器。对应 YMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：第一个源操作数是 YMM 寄存器。第二个源操作数是 YMM 寄存器或 256 位内存位置。目的操作数是 YMM 寄存器。

EVEX 编码的 VPCMPGTD：第一个源操作数（第二个操作数）是 ZMM/YMM/XMM 寄存器。第二个源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 32 位内存位置广播的 512/256/128 位向量。目的操作数（第一个操作数）是根据写掩码 k2 更新的掩码寄存器。

EVEX 编码的 VPCMPGTB/W：第一个源操作数（第二个操作数）是 ZMM/YMM/XMM 寄存器。第二个源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置。目的操作数（第一个操作数）是根据写掩码 k2 更新的掩码寄存器。

VPCMPGTB（EVEX 编码版本）¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
FOR j := 0 TO KL-1
    i := j * 8
    IF k2[j] OR *no writemask*
        THEN
            /* signed comparison */
            CMP := SRC1[i+7:i] > SRC2[i+7:i];
            IF CMP = TRUE
                THEN DEST[j] := 1;
                ELSE DEST[j] := 0; FI;
        ELSE DEST[j] := 0
                    ; zeroing-masking onlyFI;
    FI;
ENDFOR
DEST[MAX_KL-1:KL] := 0

VPCMPGTD（EVEX 编码版本）¶

(KL, VL) = (4, 128), (8, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k2[j] OR *no writemask*
                THEN
                    /* signed comparison */
                    IF (EVEX.b = 1) AND (SRC2 *is memory*)
                        THEN CMP := SRC1[i+31:i] > SRC2[31:0];
                        ELSE CMP := SRC1[i+31:i] > SRC2[i+31:i];
                    FI;
                    IF CMP = TRUE
                        THEN DEST[j] := 1;
                        ELSE DEST[j] := 0; FI;
                ELSE
                        DEST[j] := 0
                            ; zeroing-masking only
        I
            ;
ENDFOR
DEST[MAX_KL-1:KL] := 0
