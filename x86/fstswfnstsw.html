FSTSW/FNSTSW — 存储 x87 FPU 状态字

操作码 指令 64位模式 兼容/传统模式 描述
9B DD /7 FSTSW m2byte Valid Valid 在检查是否有待处理的未屏蔽浮点异常后，将 FPU 状态字存储到 m2byte。
9B DF E0 FSTSW AX Valid Valid 在检查是否有待处理的未屏蔽浮点异常后，将 FPU 状态字存储到 AX 寄存器。
DD /7 FNSTSW1 m2byte Valid Valid 在不检查是否有待处理的未屏蔽浮点异常的情况下，将 FPU 状态字存储到 m2byte。
DF E0 FNSTSW1 AX Valid Valid 在不检查是否有待处理的未屏蔽浮点异常的情况下，将 FPU 状态字存储到 AX 寄存器。

1. 参见下文的 IA-32 架构兼容性部分。

描述¶

将 x87 FPU 状态字的当前值存储在目的位置。目的操作数可以是一个两字节的内存位置，也可以是 AX 寄存器。FSTSW 指令在存储状态字之前会检查并处理待处理的未屏蔽浮点异常；而 FNSTSW 指令则不会。

该指令的 FNSTSW AX 形式主要用于条件分支（例如，在 FPU 比较指令或 FPREM、FPREM1、FXAM 指令之后），其中分支的方向取决于 FPU 条件码标志的状态。（请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 8 章中标题为“FPU 条件码的分支和条件传送”的部分。）该指令也可用于在不使用中断的环境中调用异常处理程序（通过检查异常标志）。当执行 FNSTSW AX 指令时，AX 寄存器会在处理器执行任何进一步指令之前被更新。因此，保证存储在 AX 寄存器中的状态来自前一个 FPU 指令的完成。

汇编器会为 FSTSW 指令发布两条指令（一条 FWAIT 指令紧跟一条 FNSTSW 指令），处理器会分别执行这两条指令中的每一条。如果这两条指令中的任何一条产生了异常，保存的 EIP 会指向导致该异常的指令。

此指令的操作在非 64 位模式和 64 位模式下相同。

IA-32 架构兼容性¶

在 MS-DOS 兼容模式下运行 Pentium 或 Intel486 处理器时，（在不寻常的情况下）FNSTSW 指令有可能在执行前被中断，以处理待处理的 FPU 异常。有关这些情况的描述，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷附录 D 中标题为“无等待 FPU 指令可在窗口中获取 FPU 中断”的部分。在后来的 Intel 处理器上，FNSTSW 指令不能以此类方式被中断，除了 Intel Quark™ X1000 处理器。

操作¶

DEST := FPUStatusWord;

FPU 标志寄存器影响¶

C0, C1, C2, 和 C3 未定义。

浮点异常¶

无。

#GP(0) 如果目的操作数位于不可写的段中。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段界限。
如果使用 DS、ES、FS 或 GS 寄存器访问内存，且该寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段界限。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
