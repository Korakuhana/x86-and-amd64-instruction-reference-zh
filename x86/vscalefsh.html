VSCALEFSH — 用 FP16 值缩放标量 FP16 值

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.66.MAP6.W0 2D /r VSCALEFSH xmm1{k1}{z}, xmm2, xmm3/m16 {er} A V/V AVX512-FP16 在写掩码 k1 控制下，使用来自 xmm3/m16 的值缩放 xmm2 中的 FP16 值，并将结果存储在 xmm1 中。将 xmm2 中的 bits 127:16 复制到 xmm1[127:16]。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2 操作数 3 操作数 4
A Scalar ModRM:reg (w) VEX.vvvv (r) ModRM:r/m (r) N/A

描述¶

此指令对第一源操作数中的低 FP16 元素进行浮点缩放：将其乘以 2 的“第二源操作数中低 FP16 元素”次幂，并将结果存储在目的操作数的低位元素中。

目的操作数的 bits 127:16 从第一源操作数的对应位复制。目的操作数的 bits MAXVL-1:128 清零。目的操作数的低 FP16 元素在写掩码控制下更新。

此运算的方程式为：

xmm1 := xmm2 * 2floor(xmm3)

Floor(xmm3) 表示 ≤ xmm3 的最大整数值。

如果结果无法用 FP16 表示，则发出适当的溢出响应（对于正缩放操作数）或适当的下溢响应（对于负缩放操作数）。溢出和下溢响应取决于舍入模式（适用于符合 IEEE 标准的舍入），以及 MXCSR 中的其他设置（异常掩码位、FTZ 位）和 SAE 位。

特殊情况输入值的处理列在 表 5-41 和 表 5-42 中。

操作¶

VSCALEFSH dest{k1}, src1, src2¶

IF (EVEX.b = 1) and no memory operand:
    SET_RM(EVEX.RC)
ELSE
    SET_RM(MXCSR.RC)
IF k1[0] or *no writemask*:
    dest.fp16[0] := scale_fp16(src1.fp16[0], src2.fp16[0]) // see VSCALEFPH
ELSE IF *zeroing*:
    dest.fp16[0] := 0
//else DEST.fp16[0] remains unchanged
DEST[127:16] := src1[127:16]
DEST[MAXVL-1:128] := 0

SIMD 浮点异常¶

无效, 下溢, 溢出, 精度, 渐近下溢。

其他异常¶

请参阅表 2-47，“第 E3 类异常条件”。

针对 src1 操作数检查并触发渐近下溢操作数异常 (#D)，但不针对 src2 操作数检查。仅在 src2操作数不是 NaN 的情况下才对 src1 操作数检查渐近下溢操作数异常。如果 src2 操作数是 NaN，则处理器产生 NaN 并且不触发渐近下溢操作数异常，即使 src1 操作数是渐近下溢数。
