FBSTP — 存储 BCD 整数并出栈

操作码 模式 传统模式 描述
DF /6 将 ST(0) 存储在 m80bcd 中并弹出 ST(0)。

描述¶

将 ST(0) 寄存器中的值转换为 18 位打包的 BCD 整数，将结果存储在目的操作数中，并弹出寄存器栈。如果源值是非整数值，则根据 FPU 控制字中的 RC 字段指定的舍入模式，将其舍入为整数值。要弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。

目的操作数指定要存储第一个字节目的值的地址。BCD 值（包括其符号位）在内存中需要 10 字节的空间。

下表显示了在以打包的 BCD 格式存储各类数字时获得的结果。

F 表示有限浮点值。

D 表示打包的 BCD 数字。

* 表示浮点无效操作 (#IA) 异常。

** ±0 或 ±1，取决于舍入模式。

如果转换后的值对于目的格式来说太大，或者如果源操作数是 ∞、SNaN、QNaN，或者是不受支持的格式，则会发出无效算术操作数条件的信号。如果未屏蔽无效操作异常，则会产生无效算术操作数异常 (#IA)，并且不将任何值存储在目的操作数中。如果屏蔽了无效操作异常，则将打包的 BCD 不确定值存储在内存中。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

DEST := BCD(ST(0));
PopRegisterStack;

影响的 FPU 标志寄存器¶

C1 如果发生栈下溢，则设置为 0。
如果结果被向上舍入，则设置；否则清除。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈下溢。
#IA 转换后的值长度超过 18 位 BCD 数位。
源操作数是 SNaN、QNaN、±∞，或处于不受支持的格式。
#P 值无法在目标格式中精确表示。

保护模式异常¶

#GP(0) 如果段寄存器正在加载指向不可写段的段选择子。
如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(错误代码) 如果发生页面错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(错误代码) 如果发生页面错误。
#AC(0) 如果启用了对齐检查，并且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

#MF 如果有挂起的 x87 FPU 异常。
