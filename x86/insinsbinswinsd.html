INS/INSB/INSW/INSD — 从端口输入到字符串

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
6C INS m8, DX ZO 有效 有效 将 DX 中指定的 I/O 端口中的字节输入到 ES:(E)DI 或 RDI 指定的内存位置。1
6D INS m16, DX ZO 有效 有效 将 DX 中指定的 I/O 端口中的字输入到 ES:(E)DI 或 RDI 指定的内存位置。1
6D INS m32, DX ZO 有效 有效 将 DX 中指定的 I/O 端口中的双字输入到 ES:(E)DI 或 RDI 指定的内存位置。1
6C INSB ZO 有效 有效 将 DX 中指定的 I/O 端口中的字节输入到 ES:(E)DI 或 RDI 指定的内存位置。1
6D INSW ZO 有效 有效 将 DX 中指定的 I/O 端口中的字输入到 ES:(E)DI 或 RDI 指定的内存位置。1
6D INSD ZO 有效 有效 将 DX 中指定的 I/O 端口中的双字输入到 ES:(E)DI 或 RDI 指定的内存位置。1

1. 在 64 位模式下，仅支持 64 位 (RDI) 和 32 位 (EDI) 地址大小。在非 64 位模式下，仅支持 32 位 (EDI) 和 16 位 (DI) 地址大小。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将来自源操作数（第二个操作数）指定的 I/O 端口的数据复制到目的操作数（第一个操作数）。源操作数是从 DX 寄存器中读取的 I/O 端口地址（从 0 到 65,535）。目的操作数是一个内存位置，其地址分别从 ES:DI、ES:EDI 或 RDI 寄存器中读取（取决于指令的地址大小属性，分别为 16、32 或 64）。（ES 段不能用段超越前缀来覆盖。）被访问的 I/O 端口大小（即源操作数和目的操作数的大小）由 8 位 I/O 端口的操作码或 16 位或 32 位 I/O 端口的指令操作数大小属性决定。

在汇编代码级别，此指令允许两种形式：“显式操作数”形式和“无操作数”形式。显式操作数形式（使用 INS 助记符指定）允许显式指定源操作数和目的操作数。在这里，源操作数必须为“DX”，目的操作数应当是一个指示 I/O 端口大小和目的地址的符号。提供这种显式操作数形式是为了便于文档化；然而，请注意，这种形式提供的文档说明可能会产生误导。即目的操作数符号必须指定正确的操作数**类型**（大小，即字节、字或双字），但不必指定正确的**位置**。位置始终由 ES:(E)DI 寄存器指定，在执行 INS 指令之前必须正确加载这些寄存器。

无操作数形式提供了 INS 指令的字节、字和双字版本的“简短形式”。在这里，处理器也假定 DX 是源操作数，ES:(E)DI 是目的操作数。I/O 端口的大小通过选择助记符来指定：INSB（字节）、INSW（字）或 INSD（双字）。

在字节、字或双字从 I/O 端口传输到内存位置后，DI/EDI/RDI 寄存器会根据 EFLAGS 寄存器中 DF 标志的设置自动递增或递减。（如果 DF 标志为 0，则递增 (E)DI 寄存器；如果 DF 标志为 1，则递减 (E)DI 寄存器。）对于字节操作，(E)DI 寄存器递增或递减 1；对于字操作，递增或递减 2；对于双字操作，递增或递减 4。

INS、INSB、INSW 和 INSD 指令前面可以加上 REP 前缀，以便对 ECX 个字节、字或双字进行块输入。有关 REP 前缀的描述，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》卷 2B 第 4 章中的“REP/REPE/REPZ/REPNE/REPNZ — 重复字符串操作前缀”。

这些指令仅对访问位于处理器 I/O 地址空间中的 I/O 端口有用。有关访问 I/O 地址空间中 I/O 端口的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》卷 1 第 19 章“输入/输出”。

在 64 位模式下，默认地址大小为 64 位，使用前缀 67H 支持 32 位地址大小。内存目的地址由 RDI 或 EDI 指定。在 64 位模式下不支持 16 位地址大小。操作数大小不会提升。

如果在写入时由于写入操作（例如 #PF）发生异常或 VM 退出，这些指令可能在不写入内存位置的情况下从 I/O 端口读取。如果这会带来问题（例如，因为 I/O 端口读取具有副作用），软件应确保向内存位置的写入不会引起异常或 VM 退出。

操作¶

IF ((PE = 1) and ((CPL > IOPL) or (VM = 1)))
    THEN (* Protected mode with CPL > IOPL or virtual-8086 mode *)
        IF (Any I/O Permission Bit for I/O port being accessed = 1)
            THEN (* I/O operation is not allowed *)
                #GP(0);
            ELSE (* I/O operation is allowed *)
                DEST := SRC; (* Read from I/O port *)
        FI;
    ELSE (Real Mode or Protected Mode with CPL IOPL *)
        DEST := SRC; (* Read from I/O port *)
FI;
Non-64-bit Mode:
IF (Byte transfer)
    THEN IF DF = 0
        THEN (E)DI := (E)DI + 1;
        ELSE (E)DI := (E)DI – 1; FI;
    ELSE IF (Word transfer)
        THENIFDF =0
            THEN (E)DI := (E)DI + 2;
            ELSE (E)DI := (E)DI – 2; FI;
        ELSE (* Doubleword transfer *)
            THEN IF DF = 0
                THEN (E)DI := (E)DI + 4;
                ELSE (E)DI := (E)DI – 4; FI;
        FI;
FI;
FI64-bit Mode:
IF (Byte transfer)
    THEN IF DF = 0
        THEN (E|R)DI := (E|R)DI + 1;
        ELSE (E|R)DI := (E|R)DI – 1; FI;
    ELSE IF (Word transfer)
        THENIFDF =0
            THEN (E)DI := (E)DI + 2;
            ELSE (E)DI := (E)DI – 2; FI;
        ELSE (* Doubleword transfer *)
            THEN IF DF = 0
                THEN (E|R)DI := (E|R)DI + 4;
                ELSE (E|R)DI := (E|R)DI – 4; FI;
        FI;
FI;

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果 CPL 大于（特权级低于）I/O 特权级 (IOPL)，且 TSS 中被访问 I/O 端口的任一对应 I/O 许可位为 1。
如果目标操作数位于不可写的段中。
如果给出了 ES 段中非法的内存操作数有效地址。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出了 SS 段限制。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果 TSS 中被访问 I/O 端口的任一对应 I/O 许可位为 1。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查并且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果 CPL 大于（特权级低于）I/O 特权级 (IOPL)，且 TSS 中被访问 I/O 端口的任一对应 I/O 许可位为 1。
如果内存地址为非规范形式。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
