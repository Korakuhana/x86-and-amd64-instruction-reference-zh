VERR/VERW — 校验段的可读性或可写性

操作码/指令 Op/En 64位模式 兼容/传统模式 描述
0F 00 /4 VERR r/m16 M 有效 有效 如果使用 r/m16 指定的段可读，则设置 ZF=1。
0F 00 /5 VERW r/m16 M 有效 有效 如果使用 r/m16 指定的段可写，则设置 ZF=1。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r) N/A N/A N/A

描述¶

校验由源操作数指定的代码段或数据段从当前特权级（CPL）来看是否可读（VERR）或可写（VERW）。源操作数是一个 16 位寄存器或一个内存位置，其中包含要校验的段的段选择子。如果该段可访问且可读（VERR）或可写（VERW），则设置 ZF 标志；否则，清除 ZF 标志。代码段绝不会被校验为可写。此检查无法在系统段上执行。

要设置 ZF 标志，必须满足以下条件：

  • 段选择子不能为 NULL。
  • 段选择子必须指示描述符表（GDT 或 LDT）范围内的描述符。
  • 段选择子必须指示代码段或数据段的描述符（不能是系统段或门的描述符）。
  • 对于 VERR 指令，段必须是可读的。
  • 对于 VERW 指令，段必须是可写的数据段。
  • 如果该段不是一致性代码段，则段的 DPL 必须大于或等于（即具有较低或相同的特权级）CPL 和段选择子的 RPL。

执行的验证与将段选择子加载到 DS、ES、FS 或 GS 寄存器并执行指示的访问（读或写）时所执行的验证相同。段选择子的值不会导致保护异常，这使软件能够预先判断可能的段访问问题。

此指令的操作在非 64 位模式 and 64 位模式下相同。操作数大小固定为 16 位。

操作¶

IF SRC(Offset) > (GDTR(Limit) or (LDTR(Limit))
    THEN ZF := 0; FI;
Read segment descriptor;
IF SegmentDescriptor(DescriptorType) = 0 (* System segment *)
or (SegmentDescriptor(Type) ≠ conforming code segment)
and (CPL > DPL) or (RPL > DPL)
    THEN
        ZF := 0;
    ELSE
        IF ((Instruction = VERR) and (Segment readable))
        or ((Instruction = VERW) and (Segment writable))
            THEN
                ZF := 1;
            ELSE
                ZF := 0;
        FI;
FI;

受影响的标志位¶

如果段可访问且可读（VERR）或可写（VERW），则将 ZF 标志设置为 1；否则，将其设置为 0。

保护模式异常¶

针对这些指令生成的唯一异常是与源操作数的非法寻址相关的异常。

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存，且它包含 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD VERR 和 VERW 指令在实地址模式下不被识别。
如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#UD VERR 和 VERW 指令在虚拟-8086 模式下不被识别。
如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果指向 SS 段的内存地址为非规范形式。
#GP(0) 如果内存地址为非规范形式。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
