MOVZX — 带零扩展的传送

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F B6 /r MOVZX r16, r/m8 RM Valid Valid 带零扩展将字节传送到字。
0F B6 /r MOVZX r32, r/m8 RM Valid Valid 带零扩展将字节传送到双字。
REX.W + 0F B6 /r MOVZX r64, r/m81 RM Valid N.E. 带零扩展将字节传送到四字。
0F B7 /r MOVZX r32, r/m16 RM Valid Valid 带零扩展将字传送到双字。
REX.W + 0F B7 /r MOVZX r64, r/m16 RM Valid N.E. 带零扩展将字传送到四字。

1. 在 64 位模式下，如果使用 REX 前缀，则不能将 r/m8 编码为访问以下字节寄存器：AH、BH、CH、DH。

指令操作数编码 ¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述 ¶

将源操作数（寄存器或内存地址）的内容复制到目的操作数（寄存器）并对其进行零扩展。转换后的数值大小取决于操作数大小属性。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 前缀可将操作提升至 64 位操作数。请参阅本节开头的汇总图表以获取编码数据和限制。

操作 ¶ 

DEST := ZeroExtend(SRC);

受影响的标志位 ¶

无。

保护模式异常 ¶

#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常 ¶

#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数的有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常 ¶

#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
