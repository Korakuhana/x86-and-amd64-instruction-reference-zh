描述¶

重复执行一个字符串操作指令，重复次数由计数寄存器指定，或者直到状态标志位 ZF 不再满足指示条件为止。REP（重复）、REPE（相等时重复）、REPNE（不相等时重复）、REPZ（为零时重复）和 REPNZ（不为零时重复）是可附加到字符串操作指令之一的前缀。REP 前缀可以添加到 INS、OUTS、MOVS、LODS 和 STOS 指令上，而 REPE、REPNE、REPZ 和 REPNZ 前缀可以添加到 CMPS 和 SCAS 指令上。（REPZ 和 REPNZ 前缀分别是 REPE 和 REPNE 前缀的同义词。）F3H 前缀针对以下指令有定义，对其他指令则未定义：

  • F3H 作为字符串和输入/输出指令的 REP/REPE/REPZ 前缀。
  • F3H 是 POPCNT、LZCNT 和 ADOX 的强制前缀。

REP 前缀一次仅适用于一条字符串指令。要重复执行一个指令块，请使用 LOOP 指令或其他循环结构。所有这些重复前缀都会导致关联的指令重复执行，直到计数寄存器递减到 0。参见表 4-17。

* 默认情况下计数寄存器是 CX、ECX 或 RCX，取决于操作模式的属性。

REPE、REPNE、REPZ 和 REPNZ 前缀还在每次迭代后检查 ZF 标志的状态，并在 ZF 标志不在指定状态时终止重复循环。当测试两个终止条件时，可以通过使用 JECXZ 指令测试计数寄存器，或使用 JZ、JNZ 或 JNE 指令测试 ZF 标志，来确定重复终止的原因。

使用 REPE/REPZ 和 REPNE/REPNZ 前缀时，ZF 标志不需要初始化，因为 CMPS 和 SCAS 指令都会根据它们所做的比较结果来影响 ZF 标志。

重复字符串操作可以被异常或中断挂起。当发生这种情况时，寄存器的状态将被保留，以便在从异常或中断处理程序返回时可以恢复字符串操作。源和目标寄存器指向下一个要操作的字符串元素，EIP 寄存器指向字符串指令，而 ECX 寄存器保留其在指令上一次成功迭代后的值。此机制允许执行长字符串操作，而不会影响系统的中断响应时间。

当在带有 REPE 或 REPNE 前缀的 CMPS 或 SCAS 指令执行期间发生故障时，EFLAGS 值将恢复到该指令执行之前的状态。由于 SCAS 和 CMPS 指令不使用 EFLAGS 作为输入，处理器可以在缺页异常处理程序处理完成后恢复执行该指令。

请谨慎使用 REP INS 和 REP OUTS 指令。并非所有 I/O 端口都能处理这些指令的执行速率。请注意，REP STOS 指令是初始化大内存块的最快方法。

在 64 位模式下，计数寄存器的操作数大小与地址大小属性关联。因此，默认的计数寄存器是 RCX；REX.W 对地址大小和计数寄存器没有影响。在 64 位模式下，如果使用 67H 来覆盖地址大小属性，计数寄存器为 ECX，任何隐式的源/目标操作数都将使用相应的 32 位索引寄存器。有关编码数据和限制，请参阅本节开头的汇总表。

如果在执行写入操作时因异常或 VM 退出而导致写入无法完成（例如 #PF），则 REP INS 可能在未向内存位置写入数据的情况下从 I/O 端口读取。如果这会产生问题（例如因为 I/O 端口读取有副作用），则软件应确保向内存位置的写入操作不会引起异常或 VM 退出。

操作¶

IF AddressSize = 16
    THEN
            Use CX for CountReg;
            Implicit Source/Dest operand for memory use of SI/DI;
    ELSE IF AddressSize = 64
            THEN Use RCX for CountReg;
            Implicit Source/Dest operand for memory use of RSI/RDI;
    ELSE
            Use ECX for CountReg;
            Implicit Source/Dest operand for memory use of ESI/EDI;
FI;
WHILE CountReg ≠ 0
        DO
                Service pending interrupts (if any);
                Execute associated string instruction;
                CountReg := (CountReg – 1);
                IF CountReg = 0
                    THEN exit WHILE loop; FI;
                IF (Repeat prefix is REPZ or REPE) and (ZF = 0)
                or (Repeat prefix is REPNZ or REPNE) and (ZF = 1)
                    THEN exit WHILE loop; FI;
        OD;

影响的标志位¶

无；但是，CMPS 和 SCAS 指令确实会设置 EFLAGS 寄存器中的状态标志。

异常（所有工作模式）¶

与该前缀相关联的指令可能会产生异常。

64位模式异常¶

#GP(0) 如果内存地址是非规范形式。
