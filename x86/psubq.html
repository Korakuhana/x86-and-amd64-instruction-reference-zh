描述¶

从第一个操作数（目的操作数）中减去第二个操作数（源操作数），并将结果存储在目的操作数中。当使用打包四字操作数时，将执行 SIMD 减法。当四字结果太大而无法在 64 位中表示时（溢出），结果会回绕，并且低 64 位将写入目的元素（即借位被忽略）。

注意，(V)PSUBQ 指令可以操作无符号或有符号（补码表示）的整数；但是，它不会设置 EFLAGS 寄存器中的位来指示溢出和/或借位。为了防止未检测到的溢出条件，软件必须控制其操作的值的范围。

在 64 位模式下且未在 VEX/EVEX 中编码时，使用 REX.R 形式的 REX 前缀允许此指令访问其他寄存器（XMM8-XMM15）。

传统 SSE 版本 64 位操作数：源操作数可以是存储在 MMX 技术寄存器中的四字整数，或是 64 位内存位置。

128位传统 SSE 版本：第二个源操作数是 XMM 寄存器或 128 位内存位置。第一个源操作数和目的操作数是 XMM 寄存器。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：第二个源操作数是 XMM 寄存器或 128 位内存位置。第一个源操作数和目的操作数是 XMM 寄存器。目的 YMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：第二个源操作数是 YMM 寄存器或 256 位内存位置。第一个源操作数和目的操作数是 YMM 寄存器。对应 ZMM 寄存器的位（MAXVL-1:256）清零。

EVEX 编码 VPSUBQ：第二个源操作数是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或是从 32/64 位内存位置广播的 512/256/128 位向量。第一个源操作数和目的操作数是 ZMM/YMM/XMM 寄存器。目的操作数在写掩码 k1 下有条件地更新。

操作¶

PSUBQ（带 64位操作数）¶

DEST[63:0] := DEST[63:0] − SRC[63:0];

PSUBQ（带 128位操作数）¶

DEST[63:0] := DEST[63:0] − SRC[63:0];
DEST[127:64] := DEST[127:64] − SRC[127:64];

VPSUBQ（VEX.128 编码版本）¶

DEST[63:0] := SRC1[63:0]-SRC2[63:0]
DEST[127:64] := SRC1[127:64]-SRC2[127:64]
DEST[MAXVL-1:128] := 0

VPSUBQ（VEX.256 编码版本）¶

DEST[63:0] := SRC1[63:0]-SRC2[63:0]
DEST[127:64] := SRC1[127:64]-SRC2[127:64]
DEST[191:128] := SRC1[191:128]-SRC2[191:128]
DEST[255:192] := SRC1[255:192]-SRC2[255:192]
DEST[MAXVL-1:256] := 0

VPSUBQ（EVEX 编码版本）¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no writemask* THEN
            IF (EVEX.b = 1) AND (SRC2 *is memory*)
                THEN DEST[i+63:i] := SRC1[i+63:i] - SRC2[63:0]
                ELSE DEST[i+63:i] := SRC1[i+63:i] - SRC2[i+63:i]
            FI;
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+63:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+63:i] := 0
            FI
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0
