ENTER — 创建过程参数的栈帧

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
C8 iw 00 ENTER imm16, 0 II Valid Valid 为过程创建栈帧。
C8 iw 01 ENTER imm16, 1 II Valid Valid 为过程创建包含一个嵌套指针的栈帧。
C8 iw ib ENTER imm16, imm8 II Valid Valid 为过程创建包含多个嵌套指针的栈帧。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
II iw imm8 N/A N/A

描述¶

为过程创建栈帧（包含用于动态存储的空间和 1-32 个帧指针存储空间）。第一个操作数 (imm16) 指定栈帧中动态存储的大小（即在栈上为该过程动态分配的字节数）。第二个操作数 (imm8) 给出过程的词法嵌套层级（0 到 31）。嵌套层级（imm8 mod 32）和 OperandSize 属性决定了用于帧指针的存储空间大小（以字节为单位）。

嵌套层级决定了从前一个栈帧复制到新栈帧“显示区域（display area）”中的帧指针数量。帧指针的默认大小是 StackAddrSize 属性，但可以使用 66H 前缀进行重写。因此，OperandSize 属性决定了将复制到栈帧中的每个帧指针的大小，以及从 SP/ESP/RSP 寄存器传输到 BP/EBP/RBP 寄存器的数据大小。

提供 ENTER 指令和配套的 LEAVE 指令以支持块结构语言。ENTER 指令（如果使用）通常是过程中的第一条指令，用于为过程设置新的栈帧。然后，在过程结束时（紧接在 RET 指令之前）使用 LEAVE 指令来释放该栈帧。

如果嵌套层级为 0，处理器将 BP/EBP/RBP 寄存器中的帧指针压入栈中，将 SP/ESP/RSP 寄存器中的当前栈指针复制到 BP/EBP/RBP 寄存器中，并用当前栈指针值减去大小操作数中的值来加载 SP/ESP/RSP 寄存器。对于 1 或更大的嵌套层级，处理器在调整栈指针之前将额外的帧指针压入栈中。这些额外的帧指针为被调用过程提供了访问栈上其他嵌套帧的接入点。有关 ENTER 指令行为的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 6 章中的“块结构语言的过程调用”。

每当使用栈指针的最终值（在当前栈段内）进行写入会导致缺页异常时，ENTER 指令就会引起缺页异常。

在 64 位模式下，默认操作大小为 64 位；不能编码 32 位操作大小。使用 66H 前缀将帧指针操作数大小更改为 16 位。

当使用 66H 前缀导致 OperandSize 属性小于 StackAddrSize 时，软件负责以下内容：

  • 配套的 LEAVE 指令也必须使用 66H 前缀。
  • 在执行 “66H ENTER” 之前，RBP/EBP 寄存器中的值必须与当前栈指针 (RSP/ESP) 在同一个 16KByte 区域内，以便 “66H ENTER” 之后的 RBP/EBP 值仍是栈中的有效地址。这可确保 “66H LEAVE” 能从栈中恢复 16 位的数据。

操作¶

AllocSize := imm16;
NestingLevel := imm8 MOD 32;
IF (OperandSize = 64)
    THEN
        Push(RBP); (* RSP decrements by 8 *)
        FrameTemp := RSP;
    ELSE IF OperandSize = 32
        THEN
            Push(EBP); (* (E)SP decrements by 4 *)
            FrameTemp := ESP; FI;
    ELSE (* OperandSize = 16 *)
            Push(BP); (* RSP or (E)SP decrements by 2 *)
            FrameTemp := SP;
FI;
IF NestingLevel = 0
    THEN GOTO CONTINUE;
FI;
IF (NestingLevel > 1)
    THEN FOR i := 1 to (NestingLevel - 1)
        DO
            IF (OperandSize = 64)
                THEN
                    RBP := RBP - 8;
                    Push([RBP]); (* Quadword push *)
                ELSE IF OperandSize = 32
                    THEN
                        IF StackSize = 32
                            EBP := EBP - 4;
                            Push([EBP]); (* Doubleword push *)
                        ELSE (* StackSize = 16 *)
                            BP := BP - 4;
                            Push([BP]); (* Doubleword push *)
                        FI;
                    FI;
                ELSE (* OperandSize = 16 *)
                    IF StackSize = 64
                        THEN
                            RBP := RBP - 2;
                            Push([RBP]); (* Word push *)
                    ELSE IF StackSize = 32
                        THEN
                            EBP := EBP - 2;
                            Push([EBP]); (* Word push *)
                        ELSE (* StackSize = 16 *)
                            BP := BP - 2;
                            Push([BP]); (* Word push *)
                    FI;
                FI;
    OD;
FI;
IF (OperandSize = 64) (* nestinglevel 1 *)
    THEN
        Push(FrameTemp); (* Quadword push and RSP decrements by 8 *)
    ELSE IF OperandSize = 32
        THEN
            Push(FrameTemp); FI; (* Doubleword push and (E)SP decrements by 4 *)
    ELSE (* OperandSize = 16 *)
            Push(FrameTemp); (* Word push and RSP|ESP|SP decrements by 2 *)
FI;
CONTINUE:
IF 64-Bit Mode (StackSize = 64)
    THEN
            RBP := FrameTemp;
            RSP := RSP − AllocSize;
    ELSE IF OperandSize = 32
        THEN
            EBP := FrameTemp;
            ESP := ESP − AllocSize; FI;
    ELSE (* OperandSize = 16 *)
            BP := FrameTemp[15:1]; (* Bits 16 and above of applicable RBP/EBP are unmodified *)
            SP := SP − AllocSize;
FI;
END;

受影响的标志位¶

无。

保护模式异常¶

#SS(0) 如果 SP 或 ESP 寄存器的新值超出栈段限制。
#PF(错误码) 如果发生缺页异常，或者如果使用栈指针的最终值（在当前栈段内）进行写入会导致缺页异常。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#SS 如果 SP 或 ESP 寄存器的新值超出栈段限制。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#SS(0) 如果 SP 或 ESP 寄存器的新值超出栈段限制。
#PF(错误码) 如果发生缺页异常，或者如果使用栈指针的最终值（在当前栈段内）进行写入会导致缺页异常。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果栈地址为非规范形式。
#PF(错误码) 如果发生缺页异常，或者如果使用栈指针的最终值（在当前栈段内）进行写入会导致缺页异常。
#UD 如果使用了 LOCK 前缀。
