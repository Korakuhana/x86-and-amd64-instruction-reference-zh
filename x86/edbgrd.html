EDBGRD — 从调试 Enclave 中读取数据

操作码/指令 操作/编码 64/32位模式支持 CPUID功能标志 描述
EAX = 04H ENCLS[EDBGRD] IR V/V SGX1 此叶函数从调试 enclave 中读取一个双字/四字。

指令操作数编码¶

操作/编码 EAX RBX RCX
IR EDBGRD (输入) 返回错误代码 (输出) 从调试 enclave 中读取的数据 (输出) EPC 中源内存的地址 (输入)

描述¶

此叶函数将属于调试 enclave 的 EPC 页面中的一个四字/双字复制到 RBX 寄存器中。在 64 位模式下读取 8 个字节，在非 64 位模式下读取 4 个字节。读取数据的大小不能被覆盖。

EPC 内部源位置的有效地址在寄存器 RCX 中提供。

EDBGRD 内存参数语义¶

EDBGRD 异常条件¶

RCX 指向的页面是 SECS。 RCX 未解析为自然对齐的线性地址。
RCX 指向的页面不属于处于调试模式的 enclave。 RCX 指向 TCS 内部超出 TCS 架构大小限制 (SGX_TCS_LIMIT) 的位置。
导致任何段违规的操作数。 可能发生页面错误。
CPL > 0。

此指令忽略 enclave 页面上的 EPCM RWX 属性。因此，通过 EDBGRD 违反 EPCM RWX 属性不会导致 #GP 异常。

EDBGRD 操作流程中的临时变量¶

名称 类型 大小 (位) 描述
TMP_MODE64 二进制 1 ((IA32_EFER.LMA = 1) && (CS.L = 1))
TMP_SECS 有效地址 64 源操作数所属 enclave 的 SECS 的物理地址。

TMP_MODE64 := ((IA32_EFER.LMA = 1) && (CS.L = 1));

IF ( (TMP_MODE64 = 1) and (DS:RCX is not 8Byte Aligned) )

THEN #GP(0); FI;

IF ( (TMP_MODE64 = 0) and (DS:RCX is not 4Byte Aligned) )

THEN #GP(0); FI;

IF (DS:RCX does not resolve within an EPC)

THEN #PF(DS:RCX); FI;

(* make sure no other Intel SGX instruction is accessing the same EPCM entry *)

IF (Another instruction modifying the same EPCM entry is executing)

THEN #GP(0); FI;

IF (EPCM(DS:RCX).VALID = 0)

THEN #PF(DS:RCX); FI;

(* make sure that DS:RCX (SOURCE) is pointing to a PT_REG or PT_TCS or PT_VA or PT_SS_FIRST or PT_SS_REST *)

IF ( (EPCM(DS:RCX).PT ≠ PT_REG) and (EPCM(DS:RCX).PT ≠ PT_TCS) and (EPCM(DS:RCX).PT ≠ PT_VA)

and (EPCM(DS:RCX).PT ≠ PT_SS_FIRST) and (EPCM(DS:RCX).PT ≠ PT_SS_REST))

THEN #PF(DS:RCX); FI;

(* make sure that DS:RCX points to an accessible EPC page *)

IF (EPCM(DS:RCX).PENDING is not 0 or (EPCM(DS:RCX).MODIFIED is not 0) )

THEN

RFLAGS.ZF := 1;

RAX := SGX_PAGE_NOT_DEBUGGABLE;

GOTO DONE;

FI;

(* If source is a TCS, then make sure that the offset into the page is not beyond the TCS size*) IF ( ( EPCM(DS:RCX). PT = PT_TCS) and ((DS:RCX) & FFFH ≥ SGX_TCS_LIMIT) ) THEN #GP(0); FI;

(* make sure the enclave owning the PT_REG or PT_TCS page allow debug *)

IF ( (EPCM(DS:RCX).PT = PT_REG) or (EPCM(DS:RCX).PT = PT_TCS) )

THEN

TMP_SECS := GET_SECS_ADDRESS;

IF (TMP_SECS.ATTRIBUTES.DEBUG = 0)

THEN #GP(0); FI;

IF ( (TMP_MODE64 = 1) )

THEN RBX[63:0] := (DS:RCX)[63:0];

ELSE EBX[31:0] := (DS:RCX)[31:0];

FI;

ELSE

TMP_64BIT_VAL[63:0] := (DS:RCX)[63:0] & (~07H); // Read contents from VA slot

IF (TMP_MODE64 = 1)

THEN

IF (TMP_64BIT_VAL ≠ 0H)

THEN RBX[63:0] := 0FFFFFFFFFFFFFFFFH;

ELSE RBX[63:0] := 0H;

FI;

ELSE

IF (TMP_64BIT_VAL ≠ 0H)

THEN EBX[31:0] := 0FFFFFFFFH;

ELSE EBX[31:0] := 0H;

FI;

FI;

(* clear EAX and ZF to indicate successful completion *)

RAX := 0;

RFLAGS.ZF := 0;

DONE:

(* clear flags *)

RFLAGS.CF,PF,AF,OF,SF := 0;

影响的标志寄存器¶

如果页面是 MODIFIED 或 PENDING，则设置 ZF，RAX 包含错误代码。否则清除 ZF 且 RAX 设置为 0。清除 CF、PF、AF、OF、SF。

保护模式异常¶

#GP(0) 如果 RCX 中的地址违反 DS 段限制或访问权限。
如果 DS 段不可用。
如果 RCX 指向未按 4 字节对齐的内存位置。
如果 RCX 中的地址指向属于非调试 enclave 的页面。
如果 RCX 中的地址指向不是 PT_TCS、PT_REG 或 PT_VA 的页面。
如果 RCX 中的地址指向 TCS 内部超出 SGX_TCS_LIMIT 的位置。
#PF(错误代码) 如果访问内存操作数时发生页面错误。
如果 RCX 中的地址指向非 EPC 页面。
如果 RCX 中的地址指向无效的 EPC 页面。

64位模式异常¶

#GP(0) 如果 RCX 是规范形式之外的。
如果 RCX 指向未按 8 字节对齐的内存位置。
如果 RCX 中的地址指向属于非调试 enclave 的页面。
如果 RCX 中的地址指向不是 PT_TCS、PT_REG 或 PT_VA 的页面。
如果 RCX 中的地址指向 TCS 内部超出 SGX_TCS_LIMIT 的位置。
#PF(错误代码) 如果访问内存操作数时发生页面错误。
如果 RCX 中的地址指向非 EPC 页面。
如果 RCX 中的地址指向无效的 EPC 页面。
