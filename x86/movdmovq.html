描述¶

将双字从源操作数（第二个操作数）复制到目的操作数（第一个操作数）。源操作数和目的操作数可以是通用寄存器、MMX 技术寄存器、XMM 寄存器或 32 位内存位置。此指令可用于在 MMX 技术寄存器的低双字与通用寄存器或 32 位内存位置之间，或者在 XMM 寄存器的低双字与通用寄存器或 32 位内存位置之间来回移动双字。此指令不能用于在 MMX 技术寄存器之间、XMM 寄存器之间、通用寄存器之间或内存位置之间传输数据。

当目的操作数是 MMX 技术寄存器时，源操作数会被写入寄存器的低双字，且寄存器被零扩展到 64 位。当目的操作数是 XMM 寄存器时，源操作数会被写入寄存器的低双字，且寄存器被零扩展到 128 位。

在 64 位模式下，此指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问其他寄存器 (R8-R15)。使用 REX.W 前缀可将操作提升为 64 位。请参阅本节开头的汇总图表以获取编码数据和限制。

带 XMM 目的寄存器的 MOVD/Q：

从源操作数移动双字/四字整数并将其存储在目的 XMM 寄存器的低 32/64 位中。目的寄存器的高位被清零。源操作数可以是 32/64 位寄存器或 32/64 位内存位置。

128 位传统 SSE 版本：对应 YMM 目的寄存器的位 (MAXVL-1:128) 保持不变。四字操作需要使用 REX.W=1。

VEX.128 编码版本：目的寄存器的位 (MAXVL-1:128) 被清零。四字操作需要使用 VEX.W=1。

EVEX.128 编码版本：目的寄存器的位 (MAXVL-1:128) 被清零。四字操作需要使用 EVEX.W=1。

带 32/64 位寄存器/内存目的操作数的 MOVD/Q：

将源 XMM 寄存器的低双字/四字存储到 32/64 位内存位置或通用寄存器。四字操作需要使用 REX.W=1、VEX.W=1 或 EVEX.W=1。

注意：VEX.vvvv 和 EVEX.vvvv 是保留的，必须为 1111b，否则指令将产生 #UD 异常。

如果 VMOVD 或 VMOVQ 是用 VEX.L= 1 编码的，尝试执行用 VEX.L= 1 编码的指令将导致 #UD 异常。

操作¶

MOVD (When Destination Operand is an MMX Technology Register)¶

DEST[31:0] := SRC;
DEST[63:32] := 00000000H;

MOVD (When Destination Operand is an XMM Register)¶

DEST[31:0] := SRC;
DEST[127:32] := 000000000000000000000000H;
DEST[MAXVL-1:128] (Unmodified)

MOVD (When Source Operand is an MMX Technology or XMM Register)¶

DEST := SRC[31:0];

VMOVD (VEX-Encoded Version when Destination is an XMM Register)¶

DEST[31:0] := SRC[31:0]
DEST[MAXVL-1:32] := 0

MOVQ (When Destination Operand is an XMM Register)¶

DEST[63:0] := SRC[63:0];
DEST[127:64] := 0000000000000000H;
DEST[MAXVL-1:128] (Unmodified)

MOVQ (When Destination Operand is r/m64)¶

DEST[63:0] := SRC[63:0];

MOVQ (When Source Operand is an XMM Register or r/m64)¶

DEST := SRC[63:0];

VMOVQ (VEX-Encoded Version When Destination is an XMM Register)¶

DEST[63:0] := SRC[63:0]
DEST[MAXVL-1:64] := 0

VMOVD (EVEX-Encoded Version When Destination is an XMM Register)¶

DEST[31:0] := SRC[31:0]
DEST[MAXVL-1:32] := 0

VMOVQ (EVEX-Encoded Version When Destination is an XMM Register)¶

DEST[63:0] := SRC[63:0]
DEST[MAXVL-1:64] := 0
