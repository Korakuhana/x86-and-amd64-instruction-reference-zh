FTST — 测试

操作码 模式 传统模式 描述
D9 E4 将 ST(0) 与 0.0 进行比较。

描述¶

将 ST(0) 寄存器中的值与 0.0 进行比较，并根据结果设置 FPU 状态字中的条件码标志 C0、C2 和 C3（见下表）。

此指令执行“无序比较”。无序比较还会检查要比较的数字的类别（参阅本章中的“FXAM — 检查浮点”）。如果寄存器 ST(0) 中的值是 NaN 或处于未定义格式，则条件标志会被设置为“无序”，并产生无效操作异常。

零的符号被忽略，因此 (– 0.0 := +0.0)。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

CASE (relation of operands) OF
    Not comparable:
        C3, C2, C0 := 111;
    ST(0) > 0.0:
        C3, C2, C0 := 000;
    ST(0) < 0.0:
        C3, C2, C0 := 001;
    ST(0) = 0.0:
        C3, C2, C0 := 100;
ESAC;

受影响的 FPU 标志位¶

C1 设置为 0。
C0, C2, C3 参见表 3-40。

浮点异常¶

#IS 发生栈下溢。
#IA 源操作数是 NaN 值或处于不受支持的格式。
#D 源操作数是非正规化值。

保护模式异常¶

#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在待处理的 x87 FPU 异常。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

与保护模式下的异常相同。

与保护模式下的异常相同。

与保护模式下的异常相同。
