描述¶

VBROADCASTSD/VBROADCASTSS/VBROADCASTF128 从内存中的源操作数（第二个操作数）中将浮点值作为一个元组进行加载，并广播到目的操作数（第一个操作数）的所有元素。

VEX256 编码版本：目的操作数是 YMM 寄存器。源操作数可以是 32 位、64 位或 128 位内存位置。寄存器源编码保留，若使用会导致 #UD。目的寄存器的高位 (MAXVL-1:256) 清零。

EVEX 编码版本：目的操作数是 ZMM/YMM/XMM 寄存器，并根据写掩码 k1 进行更新。源操作数可以是 32 位、64 位内存位置，或是 XMM 寄存器的低位双字/四字元素。

VBROADCASTF32X2/VBROADCASTF32X4/VBROADCASTF64X2/VBROADCASTF32X8/VBROADCASTF64X4 从内存或寄存器中的源操作数（第二个操作数）中将浮点值作为元组加载，并广播到目的操作数（第一个操作数）的所有元素。目的操作数是根据写掩码 k1 更新的 YMM/ZMM 寄存器。源操作数可以是寄存器或 64 位/128 位/256 位内存位置。

VBROADCASTSD 以及 VBROADCASTF128,F32x4 和 F64x2 仅支持 256 位和 512 位及以上宽度版本。VBROADCASTSS 支持 128 位、256 位和 512 位宽度版本。F32x8 和 F64x4 仅支持 512 位宽度版本。

VBROADCASTF32X2/VBROADCASTF32X4/VBROADCASTF32X8 具有 32 位粒度。VBROADCASTF64X2 和 VBROADCASTF64X4 具有 64 位粒度。

注意：VEX.vvvv 和 EVEX.vvvv 保留，必须为 1111b，否则指令会导致 #UD。

如果 VBROADCASTSD 或 VBROADCASTF128 使用 VEX.L = 0 进行编码，则尝试执行使用 VEX.L = 0 编码的指令将导致 #UD 异常。

VBROADCASTSS（EVEX 编码版本）¶

(KL, VL) (4, 128), (8, 256),= (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN DEST[i+31:i] := SRC[31:0]
        ELSE
            IF *merging-masking*
                        ; merging-masking
                THEN *DEST[i+31:i] remains unchanged*
                ELSE
                        ; zeroing-masking
                    DEST[i+31:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
