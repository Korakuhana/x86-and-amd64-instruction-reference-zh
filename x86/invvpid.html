INVVPID — 基于 VPID 使转换失效

操作码/指令 Op/En 描述
66 0F 38 81 INVVPID r64, m128 RM 在 64 位模式下，基于 VPID 使 TLB 和分页结构缓存中的条目失效。
66 0F 38 81 INVVPID r32, m128 RM 在 64 位模式之外，基于 VPID 使 TLB 和分页结构缓存中的条目失效。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) NA NA

描述¶

基于虚拟处理器标识符 (VPID) 使翻译后备缓冲区 (TLB) 和分页结构缓存中的映射失效。（参见第 29 章，“地址转换的 VMX 支持”。）失效操作基于寄存器操作数中指定的 INVVPID 类型和内存操作数中指定的 INVVPID 描述符。

在 IA-32e 模式之外，寄存器操作数始终为 32 位，无论 CS.D 的值如何；在 64 位模式下，寄存器操作数为 64 位（该指令不能在兼容模式下执行）。

逻辑处理器支持的 INVVPID 类型在 IA32_VMX_EPT_VPID_CAP MSR 中报告（参见附录 A，“VMX 能力报告机制”）。当前定义了四种 INVVPID 类型：

  • 单条地址失效：如果 INVVPID 类型为 0，则逻辑处理器使 INVVPID 描述符中指定的线性地址和 VPID 的映射失效。在某些情况下，它也可能使其他线性地址（或其他 VPID）的映射失效。
  • 单上下文失效：如果 INVVPID 类型为 1，则逻辑处理器使标记有 INVVPID 描述符中指定的 VPID 的所有映射失效。在某些情况下，它也可能使其他 VPID 的映射失效。
  • 全上下文失效：如果 INVVPID 类型为 2，则逻辑处理器使标记有除 VPID 0000H 之外的所有 VPID 的所有映射失效。在某些情况下，它也可能使带有 VPID 0000H 的翻译失效。
  • 单上下文失效（保留全局翻译）：如果 INVVPID 类型为 3，则逻辑处理器使标记有 INVVPID 描述符中指定的 VPID 的所有映射失效（全局翻译除外）。在某些情况下，它也可能使全局翻译（以及带有其他 VPID 的映射）失效。有关全局翻译的信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》卷 3A 第 4 章中的“缓存翻译信息”部分。

如果指定了不支持的 INVVPID 类型，则指令失败。

INVVPID 使指示的 VPID 的所有指定映射失效，无论这些映射可能与之关联的 EPTP 和 PCID 值如何。

INVVPID 描述符由 128 位组成，并由 VPID 和线性地址组成，如图 31-2 所示。

操作¶

IF (not in VMX operation) or (CR0.PE = 0) or (RFLAGS.VM = 1) or (IA32_EFER.LMA = 1 and CS.L = 0)
    THEN #UD;
ELSIF in VMX non-root operation
    THEN VM exit;
ELSIF CPL > 0
    THEN #GP(0);
    ELSE
        INVVPID_TYPE := value of register operand;
        IF IA32_VMX_EPT_VPID_CAP MSR indicates that processor does not support
        INVVPID_TYPE
            THEN VMfail(Invalid operand to INVEPT/INVVPID);
            ELSE // INVVPID_TYPE must be in the range 0–3
                INVVPID_DESC := value of memory operand;
                IF INVVPID_DESC[63:16] ≠ 0
                    THEN VMfail(Invalid operand to INVEPT/INVVPID);
                    ELSE
                        CASE INVVPID_TYPE OF
                            0:
                                            // individual-address invalidation
                                VPID := INVVPID_DESC[15:0];
                                IF VPID = 0
                                    THEN VMfail(Invalid operand to INVEPT/INVVPID);
                                    ELSE
                                        GL_ADDR := INVVPID_DESC[127:64];
                                        IF (GL_ADDR is not in a canonical form)
                                            THEN
                                                VMfail(Invalid operand to INVEPT/INVVPID);
                                            ELSE
                                                Invalidate mappings for GL_ADDR tagged with VPID;
                                                VMsucceed;
                                        FI;
                                FI;
                                BREAK;
                            1:
                                            // single-context invalidation
                                VPID := INVVPID_DESC[15:0];
                                IF VPID = 0
                                    THEN VMfail(Invalid operand to INVEPT/INVVPID);
                                    ELSE
                                        Invalidate all mappings tagged with VPID;
                                        VMsucceed;
                                FI;
                                BREAK;
                            2:
                                            // all-context invalidation
                                Invalidate all mappings tagged with all non-zero VPIDs;
                                VMsucceed;
                                BREAK;
                            3:
                                            // single-context invalidation retaining globals
                                VPID := INVVPID_DESC[15:0];
                                IF VPID = 0
                                    THEN VMfail(Invalid operand to INVEPT/INVVPID);
                                    ELSE
                                        Invalidate all mappings tagged with VPID except global translations;
                                        VMsucceed;
                                FI;
                                BREAK;
                        ESAC;
                FI;
        FI;
FI;

受影响的标志位¶

参见操作部分和第 31.2 节。

保护模式异常¶

#GP(0) 如果当前特权级不为 0。
如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含不可用的段。
如果源操作数位于仅执行的代码段中。
#PF(错误码) 如果访问内存操作数时发生缺页异常。
#SS(0) 如果内存操作数有效地址超出了 SS 段限制。
如果 SS 寄存器包含不可用的段。
#UD 如果不在 VMX 操作中。
如果逻辑处理器不支持 VPID (IA32_VMX_PROCBASED_CTLS2[37]=0)。
如果逻辑处理器支持 VPID (IA32_VMX_PROCBASED_CTLS2[37]=1) 但不支持 INVVPID 指令 (IA32_VMX_EPT_VPID_CAP[32]=0)。

实地址模式异常¶

#UD INVVPID 指令在实地址模式下无法识别。

虚拟-8086 模式异常¶

#UD INVVPID 指令在虚拟-8086 模式下无法识别。

兼容模式异常¶

#UD INVVPID 指令在兼容模式下无法识别。

64位模式异常¶

#GP(0) 如果当前特权级不为 0。
如果内存操作数位于 CS、DS、ES、FS 或 GS 段中，且内存地址为非规范形式。
#PF(错误码) 如果访问内存操作数时发生缺页异常。
#SS(0) 如果内存目的操作数位于 SS 段中，且内存地址为非规范形式。
#UD 如果不在 VMX 操作中。
如果逻辑处理器不支持 VPID (IA32_VMX_PROCBASED_CTLS2[37]=0)。
如果逻辑处理器支持 VPID (IA32_VMX_PROCBASED_CTLS2[37]=1) 但不支持 INVVPID 指令 (IA32_VMX_EPT_VPID_CAP[32]=0)。
