VDIVPH — 除法打包 FP16 值

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
EVEX.128.NP.MAP5.W0 5E /r VDIVPH xmm1{k1}{z}, xmm2, xmm3/m128/m16bcst A V/V AVX512-FP16 AVX512VL 将 xmm2 中的打包 FP16 值除以 xmm3/m128/m16bcst 中的打包 FP16 值，并将结果存储在 xmm1 中，受写掩码 k1 控制。
EVEX.256.NP.MAP5.W0 5E /r VDIVPH ymm1{k1}{z}, ymm2, ymm3/m256/m16bcst A V/V AVX512-FP16 AVX512VL 将 ymm2 中的打包 FP16 值除以 ymm3/m256/m16bcst 中的打包 FP16 值，并将结果存储在 ymm1 中，受写掩码 k1 控制。
EVEX.512.NP.MAP5.W0 5E /r VDIVPH zmm1{k1}{z}, zmm2, zmm3/m512/m16bcst {er} A V/V AVX512-FP16 将 zmm2 中的打包 FP16 值除以 zmm3/m512/m16bcst 中的打包 FP16 值，并将结果存储在 zmm1 中，受写掩码 k1 控制。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Full ModRM:reg (w) EVEX.vvvv (r) ModRM:r/m (r) N/A

描述¶

此指令将第一个源操作数中的打包 FP16 值除以第二个源操作数中的对应元素，并将打包的 FP16 结果存储在目的操作数中。根据写掩码更新目的元素。

操作¶

VDIVPH (EVEX 编码版本) 当 SRC2 操作数为寄存器时¶

VL = 128, 256 or 512
KL := VL/16
IF (VL = 512) AND (EVEX.b = 1):
    SET_RM(EVEX.RC)
ELSE
    SET_RM(MXCSR.RC)
FOR j := 0 TO KL-1:
    IF k1[j] OR *no writemask*:
        DEST.fp16[j] := SRC1.fp16[j] / SRC2.fp16[j]
    ELSE IF *zeroing*:
        DEST.fp16[j] := 0
    // else dest.fp16[j] remains unchanged
DEST[MAXVL-1:VL] := 0

VDIVPH (EVEX 编码版本) 当 SRC2 操作数为内存源时¶

VL = 128, 256 or 512
KL := VL/16
FOR j := 0 TO KL-1:
    IF k1[j] OR *no writemask*:
        IF EVEX.b = 1:
            DEST.fp16[j] := SRC1.fp16[j] / SRC2.fp16[0]
        ELSE:
            DEST.fp16[j] := SRC1.fp16[j] / SRC2.fp16[j]
    ELSE IF *zeroing*:
        DEST.fp16[j] := 0
    // else dest.fp16[j] remains unchanged
DEST[MAXVL-1:VL] := 0
