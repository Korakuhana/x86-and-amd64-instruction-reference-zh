描述¶

从源操作数（第二个操作数）中复制双字，并根据顺序操作数（第三个操作数）选定的位置将其插入目的操作数（第一个操作数）中。图 4-16 展示了 256 位 VPSHUFD 指令的操作以及顺序操作数的编码。顺序操作数中的每个 2 位字段都在 128 位通道内选择一个双字位置的内容，并将其复制到目的操作数的目标元素中。例如，对于 256 位 VPSHUFD，顺序操作数的位 0 和位 1 靶向目的操作数低 128 位和高 128 位通道中的第一个双字元素。顺序操作数位 1:0 的编码值（见图 4-16 中的字段编码）决定了源操作数（来自各自 128 位通道）的哪个双字元素将被复制到目的操作数的双字 0。

对于 128 位操作，仅低位 128 位通道起作用。源操作数可以是 XMM 寄存器或 128 位内存位置。目的操作数是 XMM 寄存器。顺序操作数是一个 8 位立即数。注意，此指令允许将源操作数中的一个双字复制到目的操作数中的多个双字位置。

源操作数可以是 XMM 寄存器或 128 位内存位置。目的操作数是 XMM 寄存器。顺序操作数是一个 8 位立即数。注意，此指令允许将源操作数中的一个双字复制到目的操作数中的多个双字位置。

在 64 位模式下且未在 VEX/EVEX 中编码时，使用 REX.R 允许此指令访问 XMM8-XMM15。

128位传统 SSE 版本：对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：源操作数可以是 XMM 寄存器或 128 位内存位置。目的操作数是 XMM 寄存器。对应 ZMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：源操作数可以是 YMM 寄存器或 256 位内存位置。目的操作数是 YMM 寄存器。对应 ZMM 寄存器的位（MAXVL-1:256）清零。使用立即数字节作为顺序操作数，目的操作数的位 (255-1:128) 存储源操作数高 16 字节的置换结果。

EVEX 编码版本：源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 32 位内存位置广播的 512/256/128 位向量。目的操作数是根据写掩码更新的 ZMM/YMM/XMM 寄存器。

使用立即数字节作为顺序操作数，目的操作数的每个 128 位通道存储源操作数对应通道的置换结果。

注意：EVEX.vvvv 和 VEX.vvvv 是保留的，必须为 1111b，否则指令将产生 #UD。

VPSHUFD（EVEX 编码版本）¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF (EVEX.b = 1) AND (SRC *is memory*)
        THEN TMP_SRC[i+31:i] := SRC[31:0]
        ELSE TMP_SRC[i+31:i] := SRC[i+31:i]
    FI;
ENDFOR;
IF VL >= 128
    TMP_DEST[31:0] := (TMP_SRC[127:0] >> (ORDER[1:0] * 32))[31:0];
    TMP_DEST[63:32] := (TMP_SRC[127:0] >> (ORDER[3:2] * 32))[31:0];
    TMP_DEST[95:64] := (TMP_SRC[127:0] >> (ORDER[5:4] * 32))[31:0];
    TMP_DEST[127:96] := (TMP_SRC[127:0] >> (ORDER[7:6] * 32))[31:0];
FI;
IF VL >= 256
    TMP_DEST[159:128] := (TMP_SRC[255:128]
                        >> (ORDER[1:0] * 32))[31:0];
    TMP_DEST[191:160] := (TMP_SRC[255:128]
                        >> (ORDER[3:2] * 32))[31:0];
    TMP_DEST[223:192] := (TMP_SRC[255:128]
                        >> (ORDER[5:4] * 32))[31:0];
    TMP_DEST[255:224] := (TMP_SRC[255:128]
                        >> (ORDER[7:6] * 32))[31:0];
FI;
IF VL >= 512
    TMP_DEST[287:256] := (TMP_SRC[383:256]
                        >> (ORDER[1:0] * 32))[31:0];
    TMP_DEST[319:288] := (TMP_SRC[383:256]
                        >> (ORDER[3:2] * 32))[31:0];
    TMP_DEST[351:320] := (TMP_SRC[383:256]
                        >> (ORDER[5:4] * 32))[31:0];
    TMP_DEST[383:352] := (TMP_SRC[383:256]
                        >> (ORDER[7:6] * 32))[31:0];
    TMP_DEST[415:384] := (TMP_SRC[511:384]
                        >> (ORDER[1:0] * 32))[31:0];
    TMP_DEST[447:416] := (TMP_SRC[511:384]
                        >> (ORDER[3:2] * 32))[31:0];
    TMP_DEST[479:448] := (TMP_SRC[511:384]
                        >> (ORDER[5:4] * 32))[31:0];
    TMP_DEST[511:480] := (TMP_SRC[511:384]
                        >> (ORDER[7:6] * 32))[31:0];
FI;
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN DEST[i+31:i] := TMP_DEST[i+31:i]
        ELSE
            IF *merging-masking*
                            ; merging-masking
                THEN *DEST[i+31:i] remains unchanged*
                ELSE *zeroing-masking*
                                ; zeroing-masking
                    DEST[i+31:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
