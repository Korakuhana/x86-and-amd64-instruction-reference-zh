STI — 设置中断标志

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
FB STI ZO Valid Valid 设置中断标志位；在下一条指令执行完毕后，允许外部可屏蔽硬件中断。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

在大多数情况下，STI 设置 EFLAGS 寄存器中的中断标志（IF）。这允许处理器响应可屏蔽硬件中断。

如果 IF = 0，在执行 STI 之后的指令边界上，可屏蔽硬件中断仍被禁止。（此指令的延迟效应是为了允许在从过程或子程序返回之前启用中断。例如，如果 STI 指令后面紧跟着 RET 指令，则在识别外部中断之前允许执行 RET 指令。如果在 STI 执行后紧接着执行 CLI，则无法识别任何中断。）在传送另一个事件（例如异常）或执行下一条指令后，该禁止结束。

IF 标志以及 STI 和 CLI 指令并不禁止产生异常和不可屏蔽中断（NMI）。然而，在以 IF = 0 开始执行 STI 之后的指令边界上，NMI（和系统管理中断）可能会被禁止。

在以下两种定义模式下，操作有所不同：

  • PVI 模式 （保护模式虚拟中断）：CR0.PE = 1，EFLAGS.VM = 0，CPL = 3，且 CR4.PVI = 1；
  • VME 模式 （虚拟 8086 模式扩展）：CR0.PE = 1，EFLAGS.VM = 1，且 CR4.VME = 1。

如果 IOPL < 3，EFLAGS.VIP = 0，并且 VME 模式或 PVI 模式处于活动状态，STI 会设置 EFLAGS 寄存器中的 VIF 标志，而保持 IF 不受影响。

表 4-19 指示了 STI 指令的行为，具体取决于处理器的运行模式、IOPL、CPL 和 EFLAGS.VIP。

1. X = 此设置对指令操作无影响。

2. 在此表中，只要 CR0.PE = 1 且 EFLAGS.VM = 0，“保护模式”即适用；它包括兼容模式和 64 位模式。

3. PVI 模式和虚拟 8086 模式均意味着 CPL = 3。

操作¶

IF CR0.PE = 0 (* Executing in real-address mode *)
    THEN IF := 1; (* Set Interrupt Flag *)
    ELSE
        IF IOPL ≥ CPL (* CPL = 3 if EFLAGS.VM = 1 *)
            THEN IF := 1; (* Set Interrupt Flag *)
            ELSE
                IF VME mode OR PVI mode
                    THEN
                        IF EFLAGS.VIP = 0
                            THEN VIF := 1; (* Set Virtual Interrupt Flag *)
                            ELSE #GP(0);
                        FI;
                    ELSE #GP(0);
                FI;
        FI;
FI;

影响的标志位¶

IF 标志或 VIF 标志被设置为 1。其他标志不受影响。

保护模式异常¶

#GP(0) 如果 CPL 大于 IOPL 且 PVI 模式未激活。
如果 CPL 大于 IOPL 且 EFLAGS.VIP = 1。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#GP(0) 如果 IOPL 小于 3 且 VME 模式未激活。
如果 IOPL 小于 3 且 EFLAGS.VIP = 1。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

与保护模式下的异常相同。
