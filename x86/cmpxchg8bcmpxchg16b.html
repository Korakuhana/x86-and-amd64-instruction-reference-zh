CMPXCHG8B/CMPXCHG16B — 比较并交换字节

操作码/指令 Op/En 64位模式 兼容/传统模式 描述
0F C7 /1 CMPXCHG8B m64 M Valid Valid* 将 EDX:EAX 与 m64 进行比较。如果相等，则设置 ZF 并将 ECX:EBX 加载到 m64 中。否则，清除 ZF 并将 m64 加载到 EDX:EAX。
REX.W + 0F C7 /1 CMPXCHG16B m128 M Valid N.E. 将 RDX:RAX 与 m128 进行比较。如果相等，则设置 ZF 并将 RCX:RBX 加载到 m128 中。否则，清除 ZF 并将 m128 加载到 RDX:RAX。

* 请参阅下文的 IA-32 架构兼容性部分。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r, w) N/A N/A N/A

描述¶

将 EDX:EAX 中的 64 位值（或如果操作数大小为 128 位，则将 RDX:RAX 中的 128 位值）与操作数（目的操作数）进行比较。如果这些值相等，则将 ECX:EBX 中的 64 位值（或 RCX:RBX 中的 128 位值）存储在目的操作数中。否则，将目的操作数中的值加载到 EDX:EAX（或 RDX:RAX）中。目的操作数是一个 8 字节的内存位置（或如果操作数大小为 128 位，则为 16 字节的内存位置）。对于 EDX:EAX 和 ECX:EBX 寄存器对，EDX 和 ECX 包含 64 位值的高 32 位，EAX 和 EBX 包含低 32 位。对于 RDX:RAX 和 RCX:RBX 寄存器对，RDX 和 RCX 包含 128 位值的高 64 位，RAX 和 RBX 包含低 64 位。

此指令可以配合 LOCK 前缀使用，以允许指令原子地执行。为了简化与处理器总线的接口，不论比较结果如何，目的操作数都会接收一个写周期。如果比较失败，则将目的操作数写回；否则，将源操作数写入目的操作数。（处理器从不产生只有锁定读而没有锁定写的情况。）

在 64 位模式下，默认操作大小为 64 位。使用 REX.W 前缀可将操作提升至 128 位。注意，CMPXCHG16B 要求目的（内存）操作数必须 16 字节对齐。请参阅本节开头的汇总图表以获取编码数据和限制。有关指示 CMPXCHG16B 的 CPUID 标志的信息，请参阅第 3-243 页。

IA-32 架构兼容性¶

早于 Pentium 处理器的 Intel 处理器不支持此指令编码。

操作¶

IF (64-Bit Mode and OperandSize = 64)
    THEN
        TEMP128 := DEST
        IF (RDX:RAX = TEMP128)
            THEN
                ZF := 1;
                DEST := RCX:RBX;
            ELSE
                ZF := 0;
                RDX:RAX := TEMP128;
                DEST := TEMP128;
                FI;
        FI
    ELSE
        TEMP64 := DEST;
        IF (EDX:EAX = TEMP64)
            THEN
                ZF := 1;
                DEST := ECX:EBX;
            ELSE
                ZF := 0;
                EDX:EAX := TEMP64;
                DEST := TEMP64;
                FI;
        FI;
FI;

受影响的标志位¶

如果目的操作数与 EDX:EAX 相等，则设置 ZF 标志；否则清除该标志。CF, PF, AF, SF 和 OF 标志不受影响。

保护模式异常¶

#UD 如果目的地不是内存操作数。
#GP(0) 如果目的地位于不可写的段中。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。

实地址模式异常¶

#UD 如果目的地不是内存位置。
#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数的有效地址超出 SS 段限制。

虚拟-8086 模式异常¶

#UD 如果目的地不是内存位置。
#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查并且进行了未对齐的内存引用。

兼容模式异常¶

与保护模式下的异常相同。

如果 CMPXCHG16B 的内存操作数没有在 16 字节边界上对齐。
如果 CPUID.01H:ECX.CMPXCHG16B[bit 13] = 0。
