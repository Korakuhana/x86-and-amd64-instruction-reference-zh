描述¶

计算第一操作数（源 1 操作数）与第二操作数（源 2 操作数）的按位逻辑与（AND），并根据结果设置 SF、ZF 和 PF 状态标志。计算结果随后被丢弃。

在 64 位模式下，以 REX.R 的形式使用 REX 前缀可以访问额外的寄存器（R8-R15）。以 REX.W 的形式使用 REX 前缀可以将操作提升到 64 位。请参阅本节开头的汇总图表以获取编码数据和限制。

操作¶

TEMP := SRC1 AND SRC2;
SF := MSB(TEMP);
IF TEMP = 0
    THEN ZF := 1;
    ELSE ZF := 0;
FI:
PF := BitwiseXNOR(TEMP[0:7]);
CF := 0;
OF := 0;
(* AF 未定义 *)

受影响的标志位¶

OF 和 CF 标志被设置为 0。SF、ZF 和 PF 标志根据结果设置（请参阅上面的“操作”部分）。AF 标志的状态未定义。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段的界限。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出 SS 段的界限。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段的界限。
#SS 如果内存操作数有效地址超出 SS 段的界限。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段的界限。
#SS(0) 如果内存操作数有效地址超出 SS 段的界限。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果内存地址为非规范形式。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
