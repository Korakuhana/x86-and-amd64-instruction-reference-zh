UCOMISS — 无序比较标量单精度浮点值并设置 EFLAGS

操作码/指令 Op / En 64/32位模式支持 CPUID 特征标志 描述
NP 0F 2E /r UCOMISS xmm1, xmm2/m32 A V/V SSE 比较 xmm1 和 xmm2/mem32 中的低位单精度浮点值，并相应地设置 EFLAGS 标志。
VEX.LIG.0F.WIG 2E /r VUCOMISS xmm1, xmm2/m32 A V/V AVX 比较 xmm1 和 xmm2/mem32 中的低位单精度浮点值，并相应地设置 EFLAGS 标志。
EVEX.LLIG.0F.W0 2E /r VUCOMISS xmm1, xmm2/m32{sae} B V/V AVX512F 比较 xmm1 和 xmm2/mem32 中的低位单精度浮点值，并相应地设置 EFLAGS 标志。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4

描述¶

对操作数 1（第一个操作数）和操作数 2（第二个操作数）的低双字中的单精度浮点值进行无序比较，并根据结果（无序、大于、小于或等于）设置 EFLAGS 寄存器中的 ZF、PF 和 CF 标志。EFLAGS 寄存器中的 OF、SF 和 AF 标志被设置为 0。如果任一源操作数是 NaN（QNaN 或 SNaN），则返回无序（unordered）结果。

操作数 1 是 XMM 寄存器；操作数 2 可以是 XMM 寄存器或 32 位内存位置。

UCOMISS 指令与 COMISS 指令的不同之处在于，它仅在源操作数为 SNaN 时才触发 SIMD 浮点无效操作异常 (#I)。而 COMISS 指令在源操作数是 QNaN 或 SNaN 时均会触发无效操作异常。

如果生成了未屏蔽的 SIMD 浮点异常，则不会更新 EFLAGS 寄存器。

注意：VEX.vvvv 和 EVEX.vvvv 是保留的，必须为 1111b，否则指令将产生 #UD 异常。

软件应确保 VUCOMISS (或 VCOMISS) 使用 VEX.L=0 进行编码。对 VUCOMISS 使用 VEX.L=1 进行编码可能会在不同的处理器世代中遇到不可预测的行为。

操作¶

(V)UCOMISS（所有版本）¶

RESULT := UnorderedCompare(DEST[31:0] <> SRC[31:0]) {
(* 设置 EFLAGS *) CASE (RESULT) OF
    UNORDERED: ZF,PF,CF := 111;
    GREATER_THAN: ZF,PF,CF := 000;
    LESS_THAN: ZF,PF,CF := 001;
    EQUAL: ZF,PF,CF := 100;
ESAC;
OF, AF, SF := 0; }

SIMD 浮点异常¶

无效（如果是 SNaN 操作数）、非规范化（Denormal）。
