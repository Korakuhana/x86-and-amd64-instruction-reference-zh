描述¶

该指令有条件地预取最多 16 个 32 位或 8 个 64 位的字节数据元素。这些元素通过 VSIB 指定（即索引寄存器是保存打包索引的 zmm 寄存器）。仅当元素对应的掩码位为 1 时，才会执行预取。

缓存行将被带入由局部性提示 (T0) 指定的独占状态 (RFO)：

  • T0（时间数据）——将数据预取到一级缓存中。

[PS 数据] 对于双字索引，该指令将预取 16 个内存位置。对于四字索引，该指令将预取 8 个值。

[PD 数据] 对于双字和四字索引，该指令将预取 8 个内存位置。

请注意：

(1) 预取可能以任何顺序发生（或根本不发生）。该指令只是一个提示。

(2) 掩码保持不变。

(3) 在 16 位有效地址下无效，将触发 #UD 故障。

(4) 此指令不会产生任何浮点或内存故障。

(5) 预取不处理缓存行拆分。

(6) 如果内存操作数在编码时没有 SIB 字节，将发出 #UD 信号。

操作¶

BASE_ADDR 代表内存操作数基地址（通用寄存器）；可能不存在。
VINDEX 代表内存操作数索引向量（向量寄存器）。
SCALE 代表内存操作数比例因子（1, 2, 4 或 8）。
DISP 是可选的 1 字节、2 字节或 4 字节偏移量。
PREFETCH(mem, Level, State) 将 “mem” 指向的字节内存位置预取到由 “Level” 指定的缓存级别中；如果 “State” 为 1，则执行独占/所有权请求。请注意，该内存位置忽略缓存行拆分。此操作被认为是处理器的提示，并且可能会根据具体实现而被跳过。

VSCATTERPF0DPS (EVEX 编码版本)¶

(KL, VL) = (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j]
        Prefetch( [BASE_ADDR + SignExtend(VINDEX[i+31:i]) * SCALE + DISP], Level=0, RFO = 1)
    FI;
ENDFOR

VSCATTERPF0DPD (EVEX 编码版本)¶

(KL, VL) = (8, 512)
FOR j := 0 TO KL-1
    i := j * 64
    k := j * 32
    IF k1[j]
        Prefetch( [BASE_ADDR + SignExtend(VINDEX[k+31:k]) * SCALE + DISP], Level=0, RFO = 1)
    FI;
ENDFOR
