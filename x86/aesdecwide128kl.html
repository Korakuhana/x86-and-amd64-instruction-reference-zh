AESDECWIDE128KL — 使用 128 位密钥配合 Key Locker 对 8 个数据块执行十轮 AES 解密流程

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
F3 0F 38 D8 !(11):001:bbb AESDECWIDE128KL m384, <XMM0-7> A V/V AESKLEWIDE_KL 使用由 m384 处的句柄指示的 128 位 AES 密钥对 XMM0-7 进行解密，并将每个生成的数据块存储回其对应的寄存器。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2—9
A N/A ModRM:r/m (r) 隐式 XMM0-7 (r, w)

描述¶

AESDECWIDE128KL1 指令执行十轮 AES，以使用由第二操作数的句柄指示的 128 位密钥来解密 XMM0-7 中的八个数据块。如果操作成功（例如，未遇到句柄违规故障），它会将 XMM0-7 中的每个输入数据块替换为其对应的解密块。

操作¶

AESDECWIDE128KL

Handle := UnalignedLoad of 384 bit (SRC); // Load is not guaranteed to be atomic.
Illegal Handle = (HandleReservedBitSet (Handle) ||
                (Handle[0] AND (CPL > 0)) ||
                Handle [2] ||
                HandleKeyType (Handle) != HANDLE_KEY_TYPE_AES128);
IF (Illegal Handle)
    THEN RFLAGS.ZF := 1;
    ELSE
        (UnwrappedKey, Authentic) := UnwrapKeyAndAuthenticate384 (Handle[383:0], IWKey);
        IF Authentic == 0 {
            THEN RFLAGS.ZF := 1;
            ELSE
                    XMM0 := AES128Decrypt (XMM0, UnwrappedKey) ;
                    XMM1 := AES128Decrypt (XMM1, UnwrappedKey) ;
                    XMM2 := AES128Decrypt (XMM2, UnwrappedKey) ;
                    XMM3 := AES128Decrypt (XMM3, UnwrappedKey) ;
                    XMM4 := AES128Decrypt (XMM4, UnwrappedKey) ;
                    XMM5 := AES128Decrypt (XMM5, UnwrappedKey) ;
                    XMM6 := AES128Decrypt (XMM6, UnwrappedKey) ;
                    XMM7 := AES128Decrypt (XMM7, UnwrappedKey) ;
                    RFLAGS.ZF := 0;
        FI;
FI;
RFLAGS.OF, SF, AF, PF, CF := 0;

影响的标志¶

如果操作成功，ZF 设置为 0；如果因为句柄违规导致操作失败，ZF 设置为 1。其他算术标志（OF、SF、AF、PF、CF）清零。
