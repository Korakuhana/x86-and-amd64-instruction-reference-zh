描述¶

VPTERNLOGD/Q 接受三个长度为 512 位的位向量（分别位于第一、第二和第三操作数中）作为输入数据以形成一组 512 个索引，每个索引由来自每个输入向量的一位组成。imm8 字节指定一个布尔逻辑表，为每个 3 位索引值产生一个二进制值。最终的 512 位布尔结果在写掩码 k1 的控制下，以双字元素或四字元素的粒度写入目的操作数（第一操作数）。

目的操作数是 ZMM (EVEX.512)/YMM (EVEX.256)/XMM (EVEX.128) 寄存器。第一源操作数是 ZMM/YMM/XMM 寄存器。第二源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 32/64 位内存位置广播的 512/256/128 位向量。目的操作数是根据写掩码 k1 有条件更新的 ZMM 寄存器。

表 5-22 显示了由立即数值 0xE2 和 0xE4 指定的布尔函数的两个示例，查找结果列在第四列中，前三列包含 3 位索引的所有可能值。

在 imm8 中指定不同的值将允许在软件中使用 VPTERNLOGD/Q 实现任何任意的三输入布尔函数。表 5-11 和表 5-12 提供了所有 256 个可能 imm8 值与各种布尔表达式的映射。

操作¶

VPTERNLOGD (EVEX 编码版本)¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN
            FOR k := 0 TO 31
                IF (EVEX.b = 1) AND (SRC2 *is memory*)
                    THEN DEST[j][k] := imm[(DEST[i+k] << 2) + (SRC1[ i+k ] << 1) + SRC2[ k ]]
                    ELSE DEST[j][k] := imm[(DEST[i+k] << 2) + (SRC1[ i+k ] << 1) + SRC2[ i+k ]]
                FI;
                        ; table lookup of immediate bellow;
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[31+i:i] remains unchanged*
            ELSE ; zeroing-masking
                DEST[31+i:i] := 0
        FI;
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0

VPTERNLOGQ (EVEX 编码版本)¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no writemask*
        THEN
            FOR k := 0 TO 63
                IF (EVEX.b = 1) AND (SRC2 *is memory*)
                    THEN DEST[j][k] := imm[(DEST[i+k] << 2) + (SRC1[ i+k ] << 1) + SRC2[ k ]]
                    ELSE DEST[j][k] := imm[(DEST[i+k] << 2) + (SRC1[ i+k ] << 1) + SRC2[ i+k ]]
                FI; ; table lookup of immediate bellow;
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[63+i:i] remains unchanged*
                ELSE ; zeroing-masking
                    DEST[63+i:i] := 0
            FI;
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0
