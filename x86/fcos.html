FCOS — 余弦

操作码 模式 传统模式 描述
D9 FF 用其近似余弦值替换 ST(0)。

描述¶

计算寄存器 ST(0) 中源操作数的近似余弦值，并将结果存储在 ST(0) 中。源操作数必须以弧度给出，且必须在 -263 到 +263 的范围内。下表显示了计算各类数字的余弦时获得的结果。

F 表示有限浮点值。

* 表示浮点无效算术操作数 (#IA) 异常。

如果源操作数超出可接受范围，则设置 FPU 状态字中的 C2 标志，并且寄存器 ST(0) 中的值保持不变。当源操作数超出范围时，该指令不会引发异常。程序需要检查 C2 标志以获取超出范围的情况。超出 -263 到 +263 范围的源值可以通过减去 2π 的适当整数倍来减小到该指令的范围内。然而，即使在 -263 到 +263 的范围内，也可能会发生不准确的结果，因为内部用于参数减少的 π 的有限近似并非在所有情况下都足够。因此，为了获得准确的结果，将 FCOS 仅应用于在软件中精确减少的参数（绝对值小于 3π/8）是安全的。关于执行此类减少时使用的 π 的合适值，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 8 章中标题为“Approximation of Pi”和“Transcendental Instruction Accuracy”的部分。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

IF |ST(0)| < 263
THEN
	C2 := 0;
	ST(0) := FCOS(ST(0)); // approximation of cosine
ELSE (* Source operand is out-of-range *)
	C2 := 1;
FI;

影响的 FPU 标志寄存器¶

C1 如果发生栈下溢，则设置为 0。
如果结果被向上舍入，则设置；否则清除。
如果 C2 是 1，则未定义。
如果超出范围（-263 < 源操作数 < +263），则设置为 1；否则设置为 0。
C2
C0, C3 未定义。

浮点异常¶

#IS 发生栈下溢。
#IA 源操作数是 SNaN 值、∞，或不受支持的格式。
#D 源操作数是反常数。
#P 值无法在目标格式中精确表示。

保护模式异常¶

#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果有挂起的 x87 FPU 异常。
#UD 如果使用了 LOCK 前缀。
