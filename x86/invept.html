INVEPT — 使源自 EPT 的转换失效

操作码/指令 Op/En 描述
66 0F 38 80 INVEPT r64, m128 RM 使 TLB 和分页结构缓存中源自 EPT 的条目失效（在 64 位模式下）。
66 0F 38 80 INVEPT r32, m128 RM 使 TLB 和分页结构缓存中源自 EPT 的条目失效（在 64 位模式之外）。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) NA NA

描述¶

使翻译后备缓冲区 (TLB) 和分页结构缓存中源自扩展页表 (EPT) 的映射失效。（参见第 29 章，“地址转换的 VMX 支持”。）失效操作基于寄存器操作数中指定的 **INVEPT 类型**和内存操作数中指定的 **INVEPT 描述符**。

在 IA-32e 模式之外，寄存器操作数始终为 32 位，无论 CS.D 的值如何；在 64 位模式下，寄存器操作数为 64 位（该指令不能在兼容模式下执行）。

逻辑处理器支持的 INVEPT 类型在 IA32_VMX_EPT_VPID_CAP MSR 中报告（参见附录 A，“VMX 能力报告机制”）。当前定义了两种 INVEPT 类型：

  • 单上下文失效：如果 INVEPT 类型为 1，则逻辑处理器使与 INVEPT 描述符中指定的 EPT 指针 (EPTP) 的位 51:12 相关联的所有映射失效。它也可能使其他映射失效。
  • 全局失效：如果 INVEPT 类型为 2，则逻辑处理器使与所有 EPTP 相关联的映射失效。

如果指定了不支持的 INVEPT 类型，则指令失败。

INVEPT 使指示 of EPTP 的所有指定映射失效，无论这些映射可能与之关联的 VPID 和 PCID 值如何。

INVEPT 描述符由 128 位组成，并在位 63:0 中包含一个 64 位的 EPTP 值（见图 31-1）。

操作¶

IF (not in VMX operation) or (CR0.PE = 0) or (RFLAGS.VM = 1) or (IA32_EFER.LMA = 1 and CS.L = 0)
    THEN #UD;
ELSIF in VMX non-root operation
    THEN VM exit;
ELSIF CPL > 0
    THEN #GP(0);
    ELSE
        INVEPT_TYPE := value of register operand;
        IF IA32_VMX_EPT_VPID_CAP MSR indicates that processor does not support INVEPT_TYPE
            THEN VMfail(Invalid operand to INVEPT/INVVPID);
            ELSE // INVEPT_TYPE must be 1 or 2
                INVEPT_DESC := value of memory operand;
                EPTP := INVEPT_DESC[63:0];
                CASE INVEPT_TYPE OF
                    1:
                                    // single-context invalidation
                        IF VM entry with the “enable EPT“ VM execution control set to 1
                        would fail due to the EPTP value
                            THEN VMfail(Invalid operand to INVEPT/INVVPID);
                            ELSE
                                Invalidate mappings associated with EPTP[51:12];
                                VMsucceed;
                        FI;
                        BREAK;
                    2:
                                    // global invalidation
                        Invalidate mappings associated with all EPTPs;
                        VMsucceed;
                        BREAK;
                ESAC;
        FI;
FI;

受影响的标志位¶

参见操作部分和第 31.2 节。

保护模式异常¶

#GP(0) 如果当前特权级不为 0。
如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含不可用的段。
如果源操作数位于仅执行的代码段中。
#PF(错误码) 如果访问内存操作数时发生缺页异常。
#SS(0) 如果内存操作数有效地址超出了 SS 段限制。
如果 SS 寄存器包含不可用的段。
#UD 如果不在 VMX 操作中。
如果逻辑处理器不支持 EPT (IA32_VMX_PROCBASED_CTLS2[33]=0)。
如果逻辑处理器支持 EPT (IA32_VMX_PROCBASED_CTLS2[33]=1) 但不支持 INVEPT 指令 (IA32_VMX_EPT_VPID_CAP[20]=0)。

实地址模式异常¶

#UD INVEPT 指令在实地址模式下无法识别。

虚拟-8086 模式异常¶

#UD INVEPT 指令在虚拟-8086 模式下无法识别。

兼容模式异常¶

#UD INVEPT 指令在兼容模式下无法识别。

64位模式异常¶

#GP(0) 如果当前特权级不为 0。
如果内存操作数位于 CS、DS、ES、FS 或 GS 段中，且内存地址为非规范形式。
#PF(错误码) 如果访问内存操作数时发生缺页异常。
#SS(0) 如果内存操作数位于 SS 段中，且内存地址为非规范形式。
#UD 如果不在 VMX 操作中。
如果逻辑处理器不支持 EPT (IA32_VMX_PROCBASED_CTLS2[33]=0)。
如果逻辑处理器支持 EPT (IA32_VMX_PROCBASED_CTLS2[33]=1) 但不支持 INVEPT 指令 (IA32_VMX_EPT_VPID_CAP[20]=0)。
