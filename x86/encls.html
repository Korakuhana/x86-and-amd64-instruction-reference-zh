ENCLS — 执行指定叶函数编号的 Enclave 系统功能

操作码/指令 操作/编码 64/32位模式支持 CPUID功能标志 描述
NP 0F 01 CF ENCLS ZO V/V 无 此指令用于执行用于管理和调试 enclave 的特权级 Intel SGX 叶函数。

指令操作数编码¶

操作/编码 操作数 1 操作数 2 操作数 3 隐式寄存器操作数
ZO 无 无 无 参见第 38.3 节

描述¶

ENCLS 指令调用指定的特权级 Intel SGX 叶函数，用于管理 and 调试 enclave。软件通过在寄存器 EAX 中设置合适的值作为输入来指定叶函数。寄存器 RBX, RCX 和 RDX 具有特定于叶函数的用途，可以作为输入、作为输出，或者不被使用。在 64 位模式下，该指令忽略 RAX 寄存器的高 32 位。

如果 CR0.PE = 0 或 RFLAGS.VM = 1，或者指令在系统管理模式 (SMM) 下执行，ENCLS 指令会产生无效操作码异常 (#UD)。此外，当 CPL > 0 时尝试执行该指令会导致 #UD。如果 CR0.PG = 0 或尝试调用未定义的叶函数，该指令会产生通用保护异常 (#GP)。

在 VMX 非 root 操作中，如果“允许 ENCLS 退出 (enable ENCLS exiting)”的 VM 执行控制为 1，执行 ENCLS 可能会导致 VM 退出。在这种情况下，ENCLS 的各个叶函数的执行受 VMCS 中的“ENCLS 退出位图 (ENCLS-exiting bitmap)”字段管辖。该字段中的每一位对应一个 ENCLS 叶函数的索引（如 EAX 中提供）。

因此，VMX root 操作中的软件可以通过设置“允许 ENCLS 退出”VM 执行控制并设置 ENCLS 退出位图中的相应位，来拦截 VMX 非 root 操作中各种 ENCLS 叶函数的调用。

在 64 位模式之外 (IA32_EFER.LMA = 0 || CS.L = 0)，地址和操作数是 32 位的；在 64 位模式下 (IA32_EFER.LMA = 1 || CS.L = 1)，地址和操作数是 64 位的。CS.D 的值对地址计算没有影响。DS 段被用于创建线性地址。

段覆盖前缀和地址大小覆盖前缀被忽略，在 64 位模式下 REX 前缀也被忽略。

操作¶

IF TSX_ACTIVE
    THEN GOTO TSX_ABORT_PROCESSING; FI;
IF CR0.PE = 0 or RFLAGS.VM = 1 or in SMM or CPUID.SGX_LEAF.0:EAX.SE1 = 0
    THEN #UD; FI;
IF (CPL > 0)
    THEN #UD; FI;
IF in VMX non-root operation and the “enable ENCLS exiting“ VM-execution control is 1
    THEN
        IF EAX < 63 and ENCLS_exiting_bitmap[EAX] = 1 or EAX> 62 and ENCLS_exiting_bitmap[63] = 1
            THEN VM exit;
        FI;
FI;
IF IA32_FEATURE_CONTROL.LOCK = 0 or IA32_FEATURE_CONTROL.SGX_ENABLE = 0
    THEN #GP(0); FI;
IF (EAX is an invalid leaf number)
    THEN #GP(0); FI;
IF CR0.PG = 0
    THEN #GP(0); FI;
(* DS must not be an expanded down segment *)
IF not in 64-bit mode and DS.Type is expand-down data
    THEN #GP(0); FI;
Jump to leaf specific flow

影响的标志寄存器¶

参见各个具体的叶函数

保护模式异常¶

#UD 如果使用了 LOCK/66H/REP/VEX 前缀中的任何一个。
如果当前特权级别不为 0。
如果 CPUID.(EAX=12H,ECX=0):EAX.SGX1 [位 0] = 0。
如果逻辑处理器处于 SMM 模式。
#GP(0) 如果 IA32_FEATURE_CONTROL.LOCK = 0。
如果 IA32_FEATURE_CONTROL.SGX_ENABLE = 0。
如果 EAX 中的输入值编码了不受支持的叶函数。
如果数据段为向下扩展。
如果 CR0.PG = 0。

实地址模式异常¶

#UD ENCLS 在实模式下不被识别。

虚拟-8086 模式异常¶

#UD ENCLS 在虚拟-8086 模式下不被识别。

兼容模式异常¶

与保护模式下的异常相同。
