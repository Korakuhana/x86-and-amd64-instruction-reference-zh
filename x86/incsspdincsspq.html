INCSSPD/INCSSPQ — 增加影子栈指针

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
F3 0F AE /05 INCSSPD r32 R V/V CET_SS 将 SSP 增加 4 * r32[7:0]。
F3 REX.W 0F AE /05 INCSSPQ r64 R V/N.E. CET_SS 将 SSP 增加 8 * r64[7:0]。

指令操作数编码¶

Op/En Tuple 类型 操作数 1 操作数 2 操作数 3 操作数 4
R N/A ModRM:r/m (r) N/A N/A N/A

描述¶

此指令可用于将当前影子栈指针 (SSP) 增加该指令的操作数大小乘以源操作数中由位 7:0 指定的无符号 8 位值。该指令在源操作数位 7:0 中的无符号 8 位值指定的范围内，执行对影子栈中第一个和最后一个元素的弹出并丢弃操作。

操作¶

IF CPL = 3
    IF (CR4.CET & IA32_U_CET.SH_STK_EN) = 0
        THEN #UD; FI;
ELSE
    IF (CR4.CET & IA32_S_CET.SH_STK_EN) = 0
        THEN #UD; FI;
FI;
IF (operand size is 64-bit)
    THEN
        Range := R64[7:0];
        shadow_stack_load 8 bytes from SSP;
        IF Range > 0
            THEN shadow_stack_load 8 bytes from SSP + 8 * (Range - 1);
        FI;
        SSP := SSP + Range * 8;
    ELSE
        Range := R32[7:0];
        shadow_stack_load 4 bytes from SSP;
        IF Range > 0
            THEN shadow_stack_load 4 bytes from SSP + 4 * (Range - 1);
        FI;
        SSP := SSP + Range * 4;
FI;

受影响的标志位¶

无。

Intel C/C++ 编译器等价内部函数¶

INCSSPD void _incsspd(int);

INCSSPQ void _incsspq(int);

保护模式异常¶

#UD 如果使用了 LOCK 前缀。
如果 CR4.CET = 0。
如果 CPL = 3 且 IA32_U_CET.SH_STK_EN = 0。
如果 CPL < 3 且 IA32_S_CET.SH_STK_EN = 0。
#PF(错误码) 如果发生缺页异常。

实地址模式异常¶

#UD INCSSP 指令在实地址模式下无法识别。

虚拟-8086 模式异常¶

#UD INCSSP 指令在虚拟-8086 模式下无法识别。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

与保护模式下的异常相同。
