VRNDSCALESH — 对标量 FP16 值进行舍入以保留指定尾数位数

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.NP.0F3A.W0 0A /r /ib VRNDSCALESH xmm1{k1}{z}, xmm2, xmm3/m16 {sae}, imm8 A V/V AVX512-FP16 在写掩码 k1 控制下，将 xmm3/m16 中的低 FP16 值舍入到由 imm8 字段指定的尾数位数。将结果存储在 xmm1 中。将 xmm2 的 bits 127:16 复制到 xmm1[127:16]。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2 操作数 3 操作数 4
A Scalar ModRM:reg (w) VEX.vvvv (r) ModRM:r/m (r) imm8 (r)

描述¶

此指令根据立即数操作数中指定的舍入模式对第二源操作数中的低 FP16 值进行舍入（参见 表 5-32），并将结果存放在目的操作数中。

目的操作数的 bits 127:16 从第一源操作数的对应位复制。目的操作数的 bits MAXVL-1:128 清零。目的操作数的低 FP16 元素根据写掩码进行更新。

舍入过程将输入舍入为整数值，加上 imm8[7:4] 指定的保留在结果中的尾数位数，并将结果作为 FP16 值返回。

请注意，执行此指令时不会引起溢出（尽管源值会被 imm8[7:4] 的值缩放）。

立即数操作数还指定了舍入操作的控制字段。定义了三个位字段，如 表 5-32 “VRNDSCALEPH/VRNDSCALESH 的 Imm8 控制”所示。立即数字节的第 3 位控制处理器对精度异常的行为，第 2 位选择舍入模式控制的来源，而第 1:0 位指定一个非粘性舍入模式值。

根据立即数操作数发出精度浮点异常信号。如果任何源操作数是 SNaN，则它将被转换为 QNaN。

此指令结果的符号会被保留，包括零的符号。特殊情况见 表 5-33。

如果立即数操作数中该指令编码的 SPE 位（第 3 位）为 1，则 VRNDSCALESH 可以在未置位 MXCSR.PE 的情况下置位 MXCSR.UE。

VRNDSCALESH 对每个数据元素操作的公式为：

ROUND(x) = 2−M *Round_to_INT(x * 2M, round_ctrl),

round_ctrl = imm[3:0];

M=imm[7:4];

x * 2M 的运算是在假设指数范围无限的情况下计算的（即永远不会发生溢出）。

操作¶

VRNDSCALESH dest{k1}, src1, src2, imm8¶

IF k1[0] or *no writemask*:
    DEST.fp16[0] := round_fp16_to_integer(src2.fp16[0], imm8) // see VRNDSCALEPH
ELSE IF *zeroing*:
    DEST.fp16[0] := 0
//else DEST.fp16[0] remains unchanged
DEST[127:16] = src1[127:16]
DEST[MAXVL-1:128] := 0
