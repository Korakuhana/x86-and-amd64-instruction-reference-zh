FLD — 加载浮点值

操作码 指令 64位模式 兼容/传统模式 描述
D9 /0 FLD m32fp 有效 有效 将 m32fp 压入 FPU 寄存器栈。
DD /0 FLD m64fp 有效 有效 将 m64fp 压入 FPU 寄存器栈。
DB /5 FLD m80fp 有效 有效 将 m80fp 压入 FPU 寄存器栈。
D9 C0+i FLD ST(i) 有效 有效 将 ST(i) 压入 FPU 寄存器栈。

描述¶

将源操作数压入 FPU 寄存器栈。源操作数可以是单精度、双精度或双精度扩展浮点格式。如果源操作数是单精度或双精度浮点格式，则在压入栈之前会自动转换为双精度扩展浮点格式。

FLD 指令还可以将选定 FPU 寄存器 [ST(i)] 中的值压入栈。在这里，压入寄存器 ST(0) 会复制栈顶。

当 FLD 指令加载反常（denormal）值且 CW 中的 DM 位未屏蔽时，会触发异常，但该值仍会被压入 x87 栈。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

IF SRC is ST(i)
    THEN
        temp := ST(i);
FI;
TOP := TOP − 1;
IF SRC is memory-operand
    THEN
        ST(0) := ConvertToDoubleExtendedPrecisionFP(SRC);
    ELSE (* SRC is ST(i) *)
        ST(0) := temp;
FI;

影响的 FPU 标志寄存器¶

C1 如果发生栈溢出，则设置为 1；否则设置为 0。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈下溢或溢出。
#IA 源操作数是 SNaN。如果源操作数采用双精度扩展浮点格式（FLD m80fp 或 FLD ST(i)），则不会发生此异常。
#D 源操作数是反常（denormal）值。如果源操作数采用双精度扩展浮点格式，则不会发生此异常。

保护模式异常¶

#GP(0) 如果目的位于不可写段中。
如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存，并且它包含空段选择器。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果内存地址为非规范形式。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在未决的 x87 FPU 异常。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
