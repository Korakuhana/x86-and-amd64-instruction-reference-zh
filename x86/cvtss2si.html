CVTSS2SI — 将标量单精度浮点值转换为双字整数

操作码/指令 Op / En 64/32位模式支持 CPUID 特征标志 描述
F3 0F 2D /r CVTSS2SI r32, xmm1/m32 A V/V SSE 将 xmm1/m32 中的一个单精度浮点值转换为 r32 中的一个有符号双字整数。
F3 REX.W 0F 2D /r CVTSS2SI r64, xmm1/m32 A V/N.E. SSE 将 xmm1/m32 中的一个单精度浮点值转换为 r64 中的一个有符号四字整数。
VEX.LIG.F3.0F.W0 2D /r 1 VCVTSS2SI r32, xmm1/m32 A V/V AVX 将 xmm1/m32 中的一个单精度浮点值转换为 r32 中的一个有符号双字整数。
VEX.LIG.F3.0F.W1 2D /r 1 VCVTSS2SI r64, xmm1/m32 A V/N.E.2 AVX 将 xmm1/m32 中的一个单精度浮点值转换为 r64 中的一个有符号四字整数。
EVEX.LLIG.F3.0F.W0 2D /r VCVTSS2SI r32, xmm1/m32{er} B V/V AVX512F 将 xmm1/m32 中的一个单精度浮点值转换为 r32 中的一个有符号双字整数。
EVEX.LLIG.F3.0F.W1 2D /r VCVTSS2SI r64, xmm1/m32{er} B V/N.E.2 AVX512F 将 xmm1/m32 中的一个单精度浮点值转换为 r64 中的一个有符号四字整数。

1. 软件应确保将 VCVTSS2SI 编码为 VEX.L=0。将 VCVTSS2SI 编码为 VEX.L=1 可能会在不同的处理器世代中遇到不可预测的行为。

2. 非 64 位下的 VEX.W1/EVEX.W1 被忽略；指令的行为就如同使用 W0 版本一样。

描述¶

将源操作数（第二操作数）中的单精度浮点值转换为目标操作数（第一操作数）中的有符号双字整数（如果操作数大小为 64 位，则为有符号四字整数）。源操作数可以是 XMM 寄存器或内存位置。目标操作数是通用寄存器。当源操作数是 XMM 寄存器时，单精度浮点值包含在该寄存器的低双字中。

当转换不精确时，返回的值将根据 MXCSR 寄存器中的舍入控制位或嵌入式舍入控制位进行舍入。如果转换后的结果无法以目标格式表示，则会引发浮点无效异常，如果该异常被屏蔽，则返回不确定整数值（2w-1，其中 w 代表目标格式的位数）。

传统 SSE 指令：在 64 位模式下，使用 REX.W 前缀可将指令提升为生成 64 位数据。请参见本节开头的汇总图表以获取编码数据和限制。

VEX.W1 and EVEX.W1 版本：在 64 位模式下提升指令以生成 64 位数据。

注意：VEX.vvvv 和 EVEX.vvvv 是保留的，必须为 1111b，否则指令将引发 #UD。

软件应确保将 VCVTSS2SI 编码为 VEX.L=0。将 VCVTSS2SI 编码为 VEX.L=1 可能会在不同的处理器世代中遇到不可预测的行为。

操作¶

VCVTSS2SI (EVEX Encoded Version)¶

IF (SRC *is register*) AND (EVEX.b = 1)
	THEN
		SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(EVEX.RC);
	ELSE
		SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(MXCSR.RC);
FI;
IF 64-bit Mode and OperandSize = 64
THEN
	DEST[63:0] := Convert_Single_Precision_Floating_Point_To_Integer(SRC[31:0]);
ELSE
	DEST[31:0] := Convert_Single_Precision_Floating_Point_To_Integer(SRC[31:0]);
FI;

(V)CVTSS2SI (Legacy and VEX.128 Encoded Version)¶

IF 64-bit Mode and OperandSize = 64
THEN
	DEST[63:0] := Convert_Single_Precision_Floating_Point_To_Integer(SRC[31:0]);
ELSE
	DEST[31:0] := Convert_Single_Precision_Floating_Point_To_Integer(SRC[31:0]);
FI;
