VEXPANDPD — 从紧凑内存加载稀疏打包双精度浮点值

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
EVEX.128.66.0F38.W1 88 /r VEXPANDPD xmm1 {k1}{z}, xmm2/m128 A V/V AVX512VL AVX512F 使用写掩码 k1 将打包的双精度浮点值从 xmm2/m128 展开到 xmm1 中。
EVEX.256.66.0F38.W1 88 /r VEXPANDPD ymm1 {k1}{z}, ymm2/m256 A V/V AVX512VL AVX512F 使用写掩码 k1 将打包的双精度浮点值从 ymm2/m256 展开到 ymm1 中。
EVEX.512.66.0F38.W1 88 /r VEXPANDPD zmm1 {k1}{z}, zmm2/m512 A V/V AVX512F 使用写掩码 k1 将打包的双精度浮点值从 zmm2/m512 展开到 zmm1 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Tuple1 Scalar ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

将源操作数（第二个操作数）中输入向量的最多 8/4/2 个连续双精度浮点值展开（加载）到由写掩码 k1 选择的目的操作数（第一个操作数）中的稀疏元素。

目的操作数是一个 ZMM/YMM/XMM 寄存器，源操作数可以是一个 ZMM/YMM/XMM 寄存器或一个 512/256/128 位内存位置。

输入向量从源操作数的最低元素开始。写掩码寄存器 k1 选择目的元素（如果少于 8 个元素，则是部分向量或稀疏元素），以被输入向量中升序的元素替换。未被写掩码 k1 选择的目的元素将被保持不变或清零，这取决于 EVEX.z。

EVEX.vvvv 保留且必须为 1111b，否则指令将产生 #UD。

请注意，压缩位移假设了对应于单个元素大小的预缩放（N），而不是整个向量的大小。

操作¶

VEXPANDPD (EVEX 编码版本)¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
k := 0
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no writemask*
        THEN
            DEST[i+63:i] := SRC[k+63:k];
            k := k + 64
        ELSE
            IF *merging-masking*
                        ; merging-masking
                THEN *DEST[i+63:i] remains unchanged*
                ELSE
                        ; zeroing-masking
                    THEN DEST[i+63:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
