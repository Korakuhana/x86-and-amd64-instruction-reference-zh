描述¶

此指令根据立即数操作数中指定的舍入模式对源操作数中的 FP16 值进行舍入（参见 表 5-32），并将结果存放在目的操作数中。目的操作数根据写掩码有条件地更新。

舍入过程将输入舍入为整数值，加上 imm8[7:4] 指定的保留在结果中的尾数位数，并将结果作为 FP16 值返回。

请注意，执行此指令时不会引起溢出（尽管源值会被 imm8[7:4] 的值缩放）。

立即数操作数还指定了舍入操作的控制字段。定义了三个位字段，如表 5-32“VRNDSCALEPH/VRNDSCALESH 的 Imm8 控制”所示。立即数字节的第 3 位控制处理器对精度异常的行为，第 2 位选择舍入模式控制的来源，而第 1:0 位指定一个非粘性舍入模式值。

根据立即数操作数发出精度浮点异常信号。如果任何源操作数是 SNaN，则它将被转换为 QNaN。

此指令结果的符号会被保留，包括零的符号。特殊情况见表 5-33。

VRNDSCALEPH 对每个数据元素操作的公式为：

ROUND(x) = 2−M *Round_to_INT(x * 2M, round_ctrl),

round_ctrl = imm[3:0];

M=imm[7:4];

x * 2M 的运算是在假设指数范围无限的情况下计算的（即永远不会发生溢出）。

如果立即数操作数中该指令编码的 SPE 位（第 3 位）为 1，则 VRNDSCALEPH 可以在未置位 MXCSR.PE 的情况下置位 MXCSR.UE。

EVEX.vvvv 是保留的，必须为 1111b，否则指令将引发 #UD。

操作¶

def round_fp16_to_integer(src, imm8):
    if imm8[2] = 1:
        rounding_direction := MXCSR.RC
    else:
        rounding_direction := imm8[1:0]
    m := imm8[7:4] // scaling factor
    tsrc1 := 2^m * src
    if rounding_direction = 0b00:
        tmp := round_to_nearest_even_integer(trc1)
    else if rounding_direction = 0b01:
        tmp := round_to_equal_or_smaller_integer(trc1)
    else if rounding_direction = 0b10:
        tmp := round_to_equal_or_larger_integer(trc1)
    else if rounding_direction = 0b11:
        tmp := round_to_smallest_magnitude_integer(trc1)
    dst := 2^(-m) * tmp
    if imm8[3]==0: // check SPE
        if src != dst:
            MXCSR.PE := 1
    return dst
