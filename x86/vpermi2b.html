VPERMI2B — 从两个表中全置换字节并覆盖索引

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.128.66.0F38.W0 75 /r VPERMI2B xmm1 {k1}{z}, xmm2, xmm3/m128 A V/V AVX512VL AVX512_VBMI 使用 xmm1 中的字节索引置换 xmm3/m128 和 xmm2 中的字节，在写掩码 k1 下将字节结果存储在 xmm1 中。
EVEX.256.66.0F38.W0 75 /r VPERMI2B ymm1 {k1}{z}, ymm2, ymm3/m256 A V/V AVX512VL AVX512_VBMI 使用 ymm1 中的字节索引置换 ymm3/m256 和 ymm2 中的字节，在写掩码 k1 下将字节结果存储在 ymm1 中。
EVEX.512.66.0F38.W0 75 /r VPERMI2B zmm1 {k1}{z}, zmm2, zmm3/m512 A V/V AVX512_VBMI 使用 zmm1 中的字节索引置换 zmm3/m512 和 zmm2 中的字节，在写掩码 k1 下将字节结果存储在 zmm1 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Full Mem ModRM:reg (r, w) EVEX.vvvv (r) ModRM:r/m (r) N/A

描述¶

使用第一操作数（目的操作数）中的字节索引，置换第二操作数（第一源操作数）和第三操作数（第二源操作数）中的字节值，以从第二或第三源操作数中选择字节元素。选定的字节元素在写掩码 k1 下以字节粒度写入目的地。

第一和第二操作数是 ZMM/YMM/XMM 寄存器。第一操作数包含输入索引，用以从第二和第三操作数中的两个输入表中选择元素。第一操作数也是结果的目的地。第三操作数可以是 ZMM/YMM/XMM 寄存器或 512/256/128 位内存位置。在每个索引字节中，用于选择表格的 id 位是第 6/5/4 位，而 bits [5:0]/[4:0]/[3:0] 选择每个输入表中的元素。

注意，这些指令允许将源操作数中的某个字节值复制到目的操作数中的多个位置。此外，相同的表可以在随后的迭代中重用，但索引元素会被覆盖。

对于 VL=256,128，目的寄存器的高 bits (MAX_VL-1:256/128) 清零。

操作¶

VPERMI2B (EVEX 编码版本)¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
IF VL = 128:
    id := 3;
ELSE IF VL = 256:
    id := 4;
ELSE IF VL = 512:
    id := 5;
FI;
TMP_DEST[VL-1:0] := DEST[VL-1:0];
FOR j := 0 TO KL-1
    off := 8*SRC1[j*8 + id: j*8] ;
    IF k1[j] OR *no writemask*:
        DEST[j*8 + 7: j*8] := TMP_DEST[j*8+id+1]? SRC2[off+7:off] : SRC1[off+7:off];
    ELSE IF *zeroing-masking*
        DEST[j*8 + 7: j*8] := 0;
    *ELSE
        DEST[j*8 + 7: j*8] remains unchanged*
    FI;
ENDFOR
DEST[MAX_VL-1:VL] := 0;
