FCOM/FCOMP/FCOMPP — 比较浮点值

操作码 指令 64位模式 兼容/传统模式 描述
D8 /2 FCOM m32fp 有效 有效 将 ST(0) 与 m32fp 比较。
DC /2 FCOM m64fp 有效 有效 将 ST(0) 与 m64fp 比较。
D8 D0+i FCOM ST(i) 有效 有效 将 ST(0) 与 ST(i) 比较。
D8 D1 FCOM 有效 有效 将 ST(0) 与 ST(1) 比较。
D8 /3 FCOMP m32fp 有效 有效 将 ST(0) 与 m32fp 比较并弹出寄存器栈。
DC /3 FCOMP m64fp 有效 有效 将 ST(0) 与 m64fp 比较并弹出寄存器栈。
D8 D8+i FCOMP ST(i) 有效 有效 将 ST(0) 与 ST(i) 比较并弹出寄存器栈。
D8 D9 FCOMP 有效 有效 将 ST(0) 与 ST(1) 比较并弹出寄存器栈。
DE D9 FCOMPP 有效 有效 将 ST(0) 与 ST(1) 比较并双重弹出寄存器栈。

描述¶

比较寄存器 ST(0) 的内容与源值，并根据结果在 FPU 状态字中设置条件码标志 C0、C2 和 C3（见下表）。源操作数可以是数据寄存器或内存位置。如果未给出源操作数，则将 ST(0) 中的值与 ST(1) 中的值进行比较。忽略零的符号，即 -0.0 等于 +0.0。

* 如果产生了未屏蔽的无效算术操作数 (#IA) 异常，则不设置标志。

此指令检查正在比较的数字类别（参见本章中的“FXAM — 检查浮点数”）。如果任一操作数是 NaN 或处于不受支持的格式，则会引发无效算术操作数异常 (#IA)；如果该异常被屏蔽，则条件标志设置为“无序”。如果未屏蔽无效算术操作数异常，则不设置条件码标志。

FCOMP 指令在比较操作后弹出寄存器栈，而 FCOMPP 指令在比较操作后弹出寄存器栈两次。要弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。

FCOM 指令执行与 FUCOM 指令相同的操作。唯一的区别是它们如何处理 QNaN 操作数。当操作数中的一个或两个都是 NaN 值或处于不受支持的格式时，FCOM 指令会引发无效算术操作数异常 (#IA)。FUCOM 指令执行与 FCOM 指令相同的操作，区别在于它们不会针对 QNaN 产生无效算术操作数异常。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

CASE (relation of operands) OF
	ST > SRC:
					C3, C2, C0 := 000;
	ST < SRC:
					C3, C2, C0 := 001;
	ST = SRC:
					C3, C2, C0 := 100;
ESAC;
IF ST(0) or SRC = NaN or unsupported format
	THEN
		#IA
		IF FPUControlWord.IM = 1
			THEN
				C3, C2, C0 := 111;
		FI;
FI;
IF Instruction = FCOMP
	THEN
		PopRegisterStack;
FI;
IF Instruction = FCOMPP
	THEN
		PopRegisterStack;
		PopRegisterStack;
FI;

影响的 FPU 标志寄存器¶

C1 设置为 0。
C0, C2, C3 见前一页的表格。

浮点异常¶

#IS 发生栈下溢。
#IA 一个或两个操作数是 NaN 值或具有不受支持的格式。
寄存器被标记为空。
#D 一个或两个操作数是反常数。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(错误代码) 如果发生页面错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#UD 如果使用了 LOCK 前缀。
