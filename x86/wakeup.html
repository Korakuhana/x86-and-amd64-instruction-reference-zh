GETSEC[WAKEUP] — 在度量环境中唤醒睡眠中的处理器

操作码 指令 描述
NP 0F 37 (EAX=8) GETSEC[WAKEUP] 从 SENTER 睡眠状态中唤醒响应的逻辑处理器。

描述¶

GETSEC[WAKEUP] 叶函数向当前处于 SENTER 睡眠状态的所有逻辑处理器广播唤醒消息。此 GETSEC 叶函数必须仅由 ILP（初始化逻辑处理器）执行，以唤醒 RLP（响应逻辑处理器）。响应逻辑处理器在完成 SENTER 会合序列后进入 SENTER 睡眠状态。

GETSEC[WAKEUP] 指令只能在以下情况下执行：

  • 在由执行 GETSEC[SENTER] 启动的度量环境中。
  • 在已认证代码执行模式之外。
  • 除非处理器处于 CPL = 0 且 EFLAGS.VM = 0 的保护模式，否则不允许执行。
  • 此外，逻辑处理器必须被指定为引导处理器（通过设置 IA32_APIC_BASE.BSP = 1 配置）。

如果不满足这些条件，尝试执行 GETSEC[WAKEUP] 将导致通用保护异常。

RLP 退出 SENTER 睡眠状态，并开始执行以响应由 ILP 执行 GETSEC[WAKEUP] 启动的 WAKEUP 信号。RLP 检索指向包含用于从定义的入口点启用执行的信息的数据结构的指针。此数据结构使用 Intel® TXT 芯片组配置寄存器 LT.MLE.JOIN 中保存的物理地址进行定位。该寄存器在芯片组中可由所有处理器公开写入，不受 Intel® TXT 芯片组配置寄存器锁定状态的限制。此数据结构的格式在“表 7-12”中定义。

MLE JOIN 数据结构包含初始化 RLP 处理器状态并允许处理器加入度量环境所需的信息。GDTR、LIP 以及 CS、DS、SS 和 ES 选择器值使用此数据结构进行初始化。CS 选择器索引直接源自段选择器初始化值字段；DS、SS 和 ES 选择器初始化为 CS+8。段描述符字段隐式初始化为 BASE = 0, LIMIT = FFFFFH, G = 1, D = 1, P = 1, S = 1；DS、SS 和 ES 为读/写/访问；CS 为执行/读/访问。外部软件有责任建立由 MLE JOIN 数据结构指向的 GDT，该 GDT 包含与处理器初始化的隐式设置一致的描述符条目（请参阅表 7-6）。处理器在执行之前会检查表 7-12 内容的某些状态是否一致。任何一致性检查的失败都会导致 RLP 终止进入保护环境并发出 Intel® TXT 关机条件信号。执行的具体检查在本节稍后记录。在成功完成处理器一致性检查和随后的初始化后，度量环境中的 RLP 执行从偏移量 12 处的入口点开始（如表 7-12 中所示）。

操作¶

(* 内部标志 ACMODEFLAG 和 SENTERFLAG 的状态在指令边界之间持久化 *)
IF (CR4.SMXE=0)
    THEN #UD;
ELSE IF (in VMX non-root operation)
    THEN VM Exit (reason=”GETSEC instruction”);
ELSE IF (GETSEC leaf unsupported)
    THEN #UD;
ELSE IF ((CR0.PE=0) or (CPL>0) or (EFLAGS.VM=1) or (SENTERFLAG=0) or (ACMODEFLAG=1) or (IN_SMM=0) or (in VMX operation) or
(IA32_APIC_BASE.BSP=0) or (TXT chipset not present))
    THEN #GP(0);
ELSE
    SignalTXTMsg(WAKEUP);
END;

受影响的标志位¶

无。

前缀的使用¶

LOCK 前缀导致 #UD。

REP* 前缀导致 #UD（包括 REPNE/REPNZ 和 REP/REPE/REPZ）。

操作数大小前缀导致 #UD。

不允许使用 NP 66/F2/F3 前缀。

段超越前缀被忽略。

地址大小前缀被忽略。

REX 前缀被忽略。

保护模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[CAPABILITIES] 未报告支持 GETSEC[WAKEUP]。
#GP(0) 如果 CR0.PE=0 或 CPL>0 或 EFLAGS.VM=1。
如果处于 VMX 操作中。
如果保护分区尚未处于活动状态，或者处理器当前处于已认证代码模式。
如果处理器处于 SMM 中。

实地址模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[CAPABILITIES] 未报告支持 GETSEC[WAKEUP]。
#GP(0) GETSEC[WAKEUP] 在实地址模式下不被识别。

虚拟-8086 模式异常¶

#UD 如果 CR4.SMXE = 0。
如果 GETSEC[CAPABILITIES] 未报告支持 GETSEC[WAKEUP]。
#GP(0) GETSEC[WAKEUP] 在虚拟-8086 模式下不被识别。

兼容模式异常¶

所有保护模式异常均适用。

64位模式异常¶

所有保护模式异常均适用。

VM-exit 条件¶

退出原因 (GETSEC) 如果处于 VMX 非根操作中。
