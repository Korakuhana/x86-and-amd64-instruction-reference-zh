VMINSH — 返回标量 FP16 值的最小值

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.F3.MAP5.W0 5D /r VMINSH xmm1{k1}{z}, xmm2, xmm3/m16 {sae} A V/V AVX512-FP16 返回 xmm3/m16 与 xmm2 之间低位 FP16 值的最小值。在写掩码 k1 下将结果存储在 xmm1 中。xmm2 的第 127:16 位复制到 xmm1[127:16] 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Scalar ModRM:reg (w) VEX.vvvv (r) ModRM:r/m (r) N/A

描述¶

该指令对第一源操作数和第二源操作数中的低位打包 FP16 值进行比较，并将每对值中的最小值返回给目的操作数。

如果被比较的值都是 0.0（无论正负号），则返回第二操作数（源操作数）中的值。如果第二操作数中的值是 SNaN，则将 SNaN 原样转发到目的地（即不返回 SNaN 的 QNaN 版本）。

如果在此指令中只有一个值是 NaN（SNaN 或 QNaN），则将第二操作数（源操作数，可以是 NaN 或有效的浮点值）写入结果。如果需要返回 NaN 源操作数（无论是来自第一操作数还是第二操作数）来代替这种行为，可以使用一系列指令（例如比较，然后执行 AND、ANDN 和 OR）来模拟 VMINSH 的操作。

EVEX 编码版本：第一源操作数（第二操作数）是一个 ZMM/YMM/XMM 寄存器。第二源操作数可以是一个 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 16 位内存位置广播的 512/256/128 位向量。目的操作数是根据写掩码 k1 有条件更新的 ZMM/YMM/XMM 寄存器。

目的操作数的第 127:16 位复制自第一源操作数中的对应位。目的操作数的第 MAXVL-1:128 位清零。目的操作数的低位 FP16 元素根据写掩码更新。

操作¶

def MIN(SRC1, SRC2):
    IF (SRC1 = 0.0) and (SRC2 = 0.0):
        DEST := SRC2
    ELSE IF (SRC1 = NaN):
        DEST := SRC2
    ELSE IF (SRC2 = NaN):
        DEST := SRC2
    ELSE IF (SRC1 < SRC2):
        DEST := SRC1
    ELSE:
        DEST := SRC2

VMINSH dest, src1, src2¶

IF k1[0] OR *no writemask*:
    DEST.fp16[0] := MIN(SRC1.fp16[0], SRC2.fp16[0])
ELSE IF *zeroing*:
    DEST.fp16[0] := 0
// else dest.fp16[j] remains unchanged
DEST[127:16] := SRC1[127:16]
DEST[MAXVL-1:128] := 0
