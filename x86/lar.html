LAR — 加载访问权限字节

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 02 /r LAR r16, r16/m16 RM 有效 有效 r16 := r16/m16 引用的访问权限
0F 02 /r LAR reg, r32/m161 RM 有效 有效 reg := r32/m16 引用的访问权限

1. 对于所有加载（无论源操作数或目的操作数的大小如何），仅使用位 16-0。其他位被忽略。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

将第二个操作数（源操作数）指定的段描述符中的访问权限加载到第一个操作数（目的操作数）中，并设置标志寄存器中的 ZF 标志。源操作数（可以是寄存器或内存位置）包含要访问 of 段描述符的段选择子。如果源操作数是内存地址，则仅访问 16 位数据。目的操作数是通用寄存器。

作为加载过程的一部分，处理器执行访问检查。一旦加载到目的寄存器中，软件就可以对访问权限信息进行额外的检查。

段描述符的访问权限包括位于段描述符的第二个双字（字节 4-7）中的字段。LAR 指令加载以下字段：

  • 位 7:0 返回为 0
  • 位 11:8 返回段类型。
  • 位 12 返回 S 标志。
  • 位 14:13 返回 DPL。
  • 位 15 返回 P 标志。
  • 仅当操作数大小大于 16 位时，才会返回以下字段：
    • 位 19:16 未定义。
    • 位 19:16 未定义。
    • 位 20 返回描述符中软件可用的位。
    • 位 20 返回描述符中软件可用的位。
    • 位 21 返回 L 标志。
    • 位 21 返回 L 标志。
    • 位 22 返回 D/B 标志。
    • 位 22 返回 D/B 标志。
    • 位 23 返回 G 标志。
    • 位 23 返回 G 标志。
    • 位 31:24 返回为 0。
    • 位 31:24 返回为 0。

此指令在将访问权限加载到目的寄存器之前执行以下检查：

  • 检查段选择子是否不为 NULL。
  • 检查段选择子是否指向在要访问的 GDT 或 LDT 限制内的描述符。
  • 检查该描述符类型对于此指令是否有效。所有代码和数据段描述符对于 LAR 指令均有效（可使用 LAR 指令访问）。合法的系统段和门描述符类型在表 3-53 中给出。
  • 如果段不是一致代码段，它会检查指定的段描述符在 CPL 处是否可见（即，如果段选择子的 CPL 和 RPL 小于或等于段选择子的 DPL）。

如果段描述符无法访问或对于该指令是无效类型，则清除 ZF 标志，且目的操作数中不会加载访问权限。

LAR 指令只能在保护模式和 IA-32e 模式下执行。

操作¶

IF Offset(SRC) > descriptor table limit
    THEN
        ZF := 0;
    ELSE
        SegmentDescriptor := descriptor referenced by SRC;
        IF SegmentDescriptor(Type) ≠ conforming code segment
        and (CPL > DPL) or (RPL > DPL)
        or SegmentDescriptor(Type) is not valid for instruction
            THEN
                ZF := 0;
            ELSE
                DEST := access rights from SegmentDescriptor as given in Description section;
                ZF := 1;
        FI;
FI;

受影响的标志位¶

如果成功加载访问权限，则将 ZF 标志设置为 1；否则，将其清除为 0。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存且它包含一个 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出了 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时内存操作数有效地址未对齐。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD LAR 指令在实地址模式下无法识别。

虚拟-8086 模式异常¶

#UD LAR 指令在虚拟-8086 模式下无法执行。

兼容模式异常¶

与保护模式下的异常相同。

#SS(0) 如果指向 SS 段的内存操作数有效地址为非规范形式。
#GP(0) 如果内存操作数有效地址为非规范形式。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时内存操作数有效地址未对齐。
