BEXTR — 位域提取

操作码/指令 Op/En 64/32位模式 CPUID 功能标志 描述
VEX.LZ.0F38.W0 F7 /r BEXTR r32a, r/m32, r32b RMV V/V BMI1 以 r32b 为控制，从 r/m32 提取连续位；将结果存入 r32a。
VEX.LZ.0F38.W1 F7 /r BEXTR r64a, r/m64, r64b RMV V/N.E. BMI1 以 r64b 为控制，从 r/m64 提取连续位；将结果存入 r64a。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RMV ModRM:reg (w) ModRM:r/m (r) VEX.vvvv (r) N/A

描述¶

使用在第二源操作数（第三个操作数）中指定的索引值和长度值，从第一源操作数（第二个操作数）中提取连续的位。第二源操作数的第 7:0 位指定提取位的起始位位置。超过操作数大小的 START 值将不会从第一源操作数中提取任何位。第二源操作数的第 15:8 位指定从 START 位置开始提取的最大位数（LENGTH）。只提取第一源操作数中直到 (OperandSize - 1) 的位位置。提取的位被写入目的寄存器，从最低有效位开始。目的操作数中的所有高阶位（从位位置 LENGTH 开始）都清零。如果未提取任何位，则清除目的寄存器。

在实模式和虚拟 8086 模式下不支持此指令。如果不在 64 位模式下，操作数大小始终为 32 位。在 64 位模式下，操作数大小为 64 位时需要 VEX.W1。在非 64 位模式下，VEX.W1 被忽略。尝试在 VEX.L 不等于 0 的情况下执行此指令将导致 #UD。

操作¶

START := SRC2[7:0];
LEN := SRC2[15:8];
TEMP := ZERO_EXTEND_TO_512 (SRC1 );
DEST := ZERO_EXTEND(TEMP[START+LEN -1: START]);
ZF := (DEST = 0);

受影响的标志位¶

ZF 根据结果更新。AF、SF 和 PF 未定义。所有其他标志位被清除。

BEXTR unsigned __int32 _bextr_u32(unsigned __int32 src, unsigned __int32 start. unsigned __int32 len);

BEXTR unsigned __int64 _bextr_u64(unsigned __int64 src, unsigned __int32 start. unsigned __int32 len);

SIMD 浮点异常¶

无。
