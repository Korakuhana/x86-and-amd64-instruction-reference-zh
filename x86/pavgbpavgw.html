描述¶

计算源操作数（第二个操作数）和目的操作数（第一个操作数）的打包无符号整数的 SIMD 平均值，并将结果存储在目的操作数中。对于第一和第二操作数中的每个对应数据元素对，将这两个元素相加，在临时和上加 1，然后将该结果向右移一位。

(V)PAVGB 指令对打包无符号字节进行操作，(V)PAVGW 指令对打包无符号字进行操作。

在 64 位模式下且未使用 VEX/EVEX 编码时，使用 REX.R 形式的 REX 前缀允许此指令访问额外的寄存器（XMM8-XMM15）。

传统 SSE 指令：源操作数可以是 MMX 技术寄存器或 64 位内存位置。目的操作数可以是 MMX 技术寄存器。

128位传统 SSE 版本：第一个源操作数是 XMM 寄存器。第二个操作数可以是 XMM 寄存器或 128 位内存位置。目的操作数与第一个源 XMM 寄存器相同，且对应寄存器目的操作数的高位（MAXVL-1:128）保持不变。

EVEX.512 编码版本：第一个源操作数是 ZMM 寄存器。第二个源操作数是 ZMM 寄存器或 512 位内存位置。目的操作数是 ZMM 寄存器。

VEX.256 和 EVEX.256 编码版本：第一个源操作数是 YMM 寄存器。第二个源操作数是 YMM 寄存器或 256 位内存位置。目的操作数是 YMM 寄存器。

VEX.128 和 EVEX.128 编码版本：第一个源操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。目的操作数是 XMM 寄存器。对应寄存器目的操作数的高位（MAXVL-1:128）清零。

VPAVGB (EVEX encoded versions)
(KL, VL) = (16, 128), (32, 256), (64, 512)
FOR j := 0 TO KL-1
    i := j * 8
    IF k1[j] OR *no writemask*
        THEN DEST[i+7:i] := (SRC1[i+7:i] + SRC2[i+7:i] + 1) >> 1; (* Temp sum before shifting is 9 bits *)
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+7:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+7:i] = 0
            FI
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0
