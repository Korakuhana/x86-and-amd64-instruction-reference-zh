PMULLW — 打包有符号字乘法并存储低位结果

描述¶

对目的操作数（第一个操作数）和源操作数（第二个操作数）中的打包有符号字整数执行 SIMD 有符号乘法，并将每个中间 32 位结果的低 16 位存储在目的操作数中。（图 4-12 展示了使用 64 位操作数时的此操作。）

在 64 位模式下且未采用 VEX/EVEX 编码时，使用 REX.R 形式 of REX 前缀允许该指令访问其他寄存器（XMM8-XMM15）。

传统 SSE 版本 64 位操作数：源操作数可以是 MMX 技术寄存器或 64 位内存位置。目的操作数是 MMX 技术寄存器。

128位传统 SSE 版本：第一个源操作数和目的操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：第一个源操作数和目的操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。目的 YMM 寄存器的位（MAXVL-1:128）清零。VEX.L 必须为 0，否则指令将产生 #UD。

VEX.256 编码版本：第二个源操作数可以是 YMM 寄存器或 256 位内存位置。第一个源操作数和目的操作数是 YMM 寄存器。

EVEX 编码版本：第一个源操作数是 ZMM/YMM/XMM 寄存器。第二个源操作数是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置。目的操作数根据写掩码 k1 条件性地更新。

PMULLW（EVEX 编码版本）¶

(KL, VL) = (8, 128), (16, 256), (32, 512)
FOR j := 0 TO KL-1
    i := j * 16
    IF k1[j] OR *no writemask*
        THEN
            temp[31:0] := SRC1[i+15:i] * SRC2[i+15:i]
            DEST[i+15:i] := temp[15:0]
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+15:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+15:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
