CLRSSBSY — 清除监督者影子栈令牌中的 Busy 标志

操作码/指令 Op / En 64/32位模式支持 CPUID 特征标志 描述
F3 0F AE /6 CLRSSBSY m64 M V/V CET_SS 清除 m64 引用的监督者影子栈令牌中的忙碌 (Busy) 标志。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
M N/A ModRM:r/m (r, w) N/A N/A N/A

描述¶

清除 m64 引用的监督者影子栈令牌中的忙碌 (Busy) 标志。在将影子栈标记为非忙碌之后，SSP 将被加载为值 0。

操作¶

IF (CR4.CET = 0)
    THEN #UD; FI;
IF (IA32_S_CET.SH_STK_EN = 0)
    THEN #UD; FI;
IF CPL > 0
    THEN GP(0); FI;
SSP_LA = Linear_Address(mem operand)
IF SSP_LA not aligned to 8 bytes
    THEN #GP(0); FI;
expected_token_value=SSP_LA|BUSY_BIT (*busybit-bitposition0-mustbeset*)
new_token_value = SSP_LA (* Clear the busy bit *)
IF shadow_stack_lock_cmpxchg8b(SSP_LA, new_token_value, expected_token_value) != expected_token_value
    invalid_token := 1; FI
(* Set the CF if invalid token was detected *)
RFLAGS.CF = (invalid_token == 1) ? 1 : 0;
RFLAGS.ZF,PF,AF,OF,SF := 0;
SSP := 0

受影响的标志位¶

如果检测到无效令牌，则设置 CF，否则将其清除。ZF、PF、AF、OF 和 SF 被清除。

保护模式异常¶

#UD 如果使用了 LOCK 前缀。
如果 CR4.CET = 0。
如果 IA32_S_CET.SH_STK_EN = 0。
#GP(0) 如果内存操作数的线性地址未对齐到 8 字节。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果目的地位于不可写段中。
如果 DS、ES、FS 或 GS 寄存器用于访问内存且包含空段选择子。
如果 CPL 不为 0。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。

实地址模式异常¶

#UD 在实地址模式下不识别 CLRSSBSY 指令。

虚拟-8086 模式异常¶

#UD 在虚拟-8086 模式下不识别 CLRSSBSY 指令。

兼容模式异常¶

#UD 与保护模式下的异常相同。
#GP(0) 与保护模式下的异常相同。
#PF(错误码) 如果发生缺页异常。

64位模式异常¶

#UD 如果使用了 LOCK 前缀。
如果 CR4.CET = 0。
如果 IA32_S_CET.SH_STK_EN = 0。
#GP(0) 如果内存操作数的线性地址未对齐到 8 字节。
如果 CPL 不为 0。
如果内存地址处于非规范形式。
如果令牌无效。
#SS(0) 如果引用 SS 段的内存地址处于非规范形式。
#PF(错误码) 如果发生缺页异常。
