COMISS — 比较标量有序单精度浮点值并设置 EFLAGS

操作码/指令 Op / En 64/32位模式支持 CPUID 特征标志 描述
NP 0F 2F /r COMISS xmm1, xmm2/m32 A V/V SSE 比较 xmm1 和 xmm2/mem32 中的低位单精度浮点值，并相应地设置 EFLAGS 标志。
VEX.LIG.0F.WIG 2F /r VCOMISS xmm1, xmm2/m32 A V/V AVX 比较 xmm1 和 xmm2/mem32 中的低位单精度浮点值，并相应地设置 EFLAGS 标志。
EVEX.LLIG.0F.W0 2F /r VCOMISS xmm1, xmm2/m32{sae} B V/V AVX512F 比较 xmm1 和 xmm2/mem32 中的低位单精度浮点值，并相应地设置 EFLAGS 标志。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A N/A ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

比较操作数 1（第一操作数）和操作数 2（第二操作数）低位双字中的单精度浮点值，并根据结果（无序、大于、小于或等于）设置 EFLAGS 寄存器中的 ZF、PF 和 CF 标志。EFLAGS 寄存器中的 OF、SF 和 AF 标志被设置为 0。如果任一源操作数是 NaN（QNaN 或 SNaN），则返回无序（unordered）结果。

操作数 1 是 XMM 寄存器；操作数 2 可以是 XMM 寄存器或 32 位内存位置。

COMISS 指令与 UCOMISS 指令的不同之处在于，当源操作数是 QNaN 或 SNaN 时，它会触发 SIMD 浮点无效操作异常（#I）。UCOMISS 指令仅在源操作数是 SNaN 时触发无效操作异常。

如果生成了未屏蔽的 SIMD 浮点异常，则不更新 EFLAGS 寄存器。

VEX.vvvv 和 EVEX.vvvv 保留且必须为 1111b，否则指令将触发 #UD 异常。

软件应确保 VCOMISS 编码为 VEX.L=0。用 VEX.L=1 编码 VCOMISS 可能会在不同处理器代次之间遇到不可预测的行为。

操作¶

COMISS (All Versions)¶

RESULT := OrderedCompare(DEST[31:0] <> SRC[31:0]) {
(* Set EFLAGS *) CASE (RESULT) OF
    UNORDERED: ZF,PF,CF := 111;
    GREATER_THAN: ZF,PF,CF := 000;
    LESS_THAN: ZF,PF,CF := 001;
    EQUAL: ZF,PF,CF := 100;
ESAC;
OF, AF, SF := 0; }

SIMD 浮点异常¶

无效异常（如果为 SNaN 或 QNaN 操作数），非规范化值异常。
