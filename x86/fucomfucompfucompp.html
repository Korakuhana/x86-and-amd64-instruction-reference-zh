FUCOM/FUCOMP/FUCOMPP — 浮点数无序比较

操作码 指令 64位模式 兼容/传统模式 描述
DD E0+i FUCOM ST(i) Valid Valid 比较 ST(0) 与 ST(i)。
DD E1 FUCOM Valid Valid 比较 ST(0) 与 ST(1)。
DD E8+i FUCOMP ST(i) Valid Valid 比较 ST(0) 与 ST(i) 并弹出寄存器栈。
DD E9 FUCOMP Valid Valid 比较 ST(0) 与 ST(1) 并弹出寄存器栈。
DA E9 FUCOMPP Valid Valid 比较 ST(0) 与 ST(1) 并双重弹出寄存器栈。

描述¶

对寄存器 ST(0) 和 ST(i) 的内容进行无序比较，并根据结果设置 FPU 状态字中的条件码标志 C0、C2 和 C3（见下表）。如果未指定操作数，则比较寄存器 ST(0) 和 ST(1) 的内容。零的符号被忽略，因此 –0.0 等于 +0.0。

* 如果产生未屏蔽的无效算术操作数 (#IA) 异常，则不设置标志位。

无序比较会检查要比较的数字的类别（参阅本章中的“FXAM — 检查浮点”）。FUCOM/FUCOMP/FUCOMPP 指令执行与 FCOM/FCOMP/FCOMPP 指令相同的操作。唯一的区别是，FUCOM/FUCOMP/FUCOMPP 指令仅在其中一个或两个操作数为 SNaN 或处于不受支持的格式时才引发无效算术操作数异常 (#IA)；QNaN 会使条件码标志设置为无序，但不会导致产生异常。当其中一个或两个操作数是任何类型的 NaN 值或处于不受支持的格式时，FCOM/FCOMP/FCOMPP 指令会引发无效操作异常。

与 FCOM/FCOMP/FCOMPP 指令一样，如果操作导致引发无效算术操作数异常，则仅在屏蔽了该异常时才设置条件码标志。

FUCOMP 指令在比较操作之后弹出寄存器栈，而 FUCOMPP 指令在比较操作之后弹出寄存器栈两次。为了弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

CASE (relation of operands) OF
    ST > SRC:
                        C3, C2, C0 := 000;
    ST < SRC:
                        C3, C2, C0 := 001;
    ST = SRC:
                        C3, C2, C0 := 100;
ESAC;
IF ST(0) or SRC = QNaN, but not SNaN or unsupported format
    THEN
        C3, C2, C0 := 111;
    ELSE (* ST(0) or SRC is SNaN or unsupported format *)
            #IA;
        IF FPUControlWord.IM = 1
                THEN
                    C3, C2, C0 := 111;
        FI;
FI;
IF Instruction = FUCOMP
    THEN
        PopRegisterStack;
FI;
IF Instruction = FUCOMPP
    THEN
        PopRegisterStack;
FI;

受影响的 FPU 标志位¶

C1 如果发生栈下溢，则设置为 0。
C0, C2, C3 参见表 3-41。

浮点异常¶

#IS 发生栈下溢。
#IA 一个或两个操作数是 SNaN 值或具有不受支持的格式。检测 QNaN 值本身不会引发无效操作数异常。
#D 一个或两个操作数是非正规化值。

保护模式异常¶

#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在待处理的 x87 FPU 异常。
#UD 如果使用了 LOCK 前缀。
