LEA — 加载有效地址

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
8D /r LEA r16,m RM Valid Valid 将 m 的有效地址存储在寄存器 r16 中。
8D /r LEA r32,m RM Valid Valid 将 m 的有效地址存储在寄存器 r32 中。
REX.W + 8D /r LEA r64,m RM Valid N.E. 将 m 的有效地址存储在寄存器 r64 中。

指令操作数编码 ¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述 ¶

计算第二操作数（源操作数）的有效地址，并将其存储在第一操作数（目的操作数）中。源操作数是一个内存地址（偏移量部分），由处理器的某种寻址模式指定；目的操作数是一个通用寄存器。地址大小和操作数大小属性会影响该指令执行的操作，如下表所示。该指令的操作数大小属性由所选寄存器决定；地址大小属性由代码段的属性决定。

不同的汇编器可能会根据源操作数的大小属性和符号引用来使用不同的算法。

在 64 位模式下，该指令的目的操作数受操作数大小属性支配，默认操作数大小为 32 位。地址计算受地址大小属性支配，默认地址大小为 64 位。在 64 位模式下，无法编码 16 位的地址大小。参见 表 3-55。

操作 ¶ 

IF OperandSize = 16 and AddressSize = 16
    THEN
        DEST := EffectiveAddress(SRC); (* 16-bit address *)
    ELSE IF OperandSize = 16 and AddressSize = 32
        THEN
            temp := EffectiveAddress(SRC); (* 32-bit address *)
            DEST := temp[0:15]; (* 16-bit address *)
        FI;
    ELSE IF OperandSize = 32 and AddressSize = 16
        THEN
            temp := EffectiveAddress(SRC); (* 16-bit address *)
            DEST := ZeroExtend(temp); (* 32-bit address *)
        FI;
    ELSE IF OperandSize = 32 and AddressSize = 32
        THEN
            DEST := EffectiveAddress(SRC); (* 32-bit address *)
        FI;
    ELSE IF OperandSize = 16 and AddressSize = 64
        THEN
            temp := EffectiveAddress(SRC); (* 64-bit address *)
            DEST := temp[0:15]; (* 16-bit address *)
        FI;
    ELSE IF OperandSize = 32 and AddressSize = 64
        THEN
            temp := EffectiveAddress(SRC); (* 64-bit address *)
            DEST := temp[0:31]; (* 16-bit address *)
        FI;
    ELSE IF OperandSize = 64 and AddressSize = 64
        THEN
            DEST := EffectiveAddress(SRC); (* 64-bit address *)
        FI;
FI;
