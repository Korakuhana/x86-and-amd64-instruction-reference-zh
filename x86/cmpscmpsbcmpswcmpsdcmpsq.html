指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将第一个源操作数指定的字节、字、双字或四字与第二个源操作数指定的字节、字、双字或四字进行比较，并根据结果在 EFLAGS 寄存器中设置状态标志位。

两个源操作数都位于内存中。第一个源操作数的地址从 DS:SI、DS:ESI 或 RSI 读取（分别取决于指令的地址大小属性是 16、32 还是 64 位）。第二个源操作数的地址从 ES:DI、ES:EDI 或 RDI 读取（同样取决于指令的地址大小属性是 16、32 还是 64 位）。可以使用段覆盖前缀来覆盖 DS 段，但不能覆盖 ES 段。

在汇编代码级别，此指令允许两种形式：“显式操作数”形式和“无操作数”形式。显式操作数形式（使用 CMPS 助记符指定）允许显式指定两个源操作数。在这里，源操作数应为指示源值的大小和位置的符号。提供此显式操作数形式是为了便于编写文档。但是，请注意，此形式提供的文档可能会误导。也就是说，源操作数符号必须指定正确的类型（大小）（字节、字、双字、四字），但它们不必指定正确的位置。源操作数的位置始终由 DS:(E)SI（或 RSI）和 ES:(E)DI（或 RDI）寄存器指定，在执行比较字符串指令之前必须正确加载这些寄存器。

无操作数形式提供了 CMPS 指令的字节、字和双字版本的“简短形式”。在这里，处理器也假定由 DS:(E)SI（或 RSI）和 ES:(E)DI（或 RDI）寄存器指定源操作数的位置。源操作数的大小由助记符选择：CMPSB（字节比较）、CMPSW（字比较）、CMPSD（双字比较）或 CMPSQ（使用 REX.W 的四字比较）。

比较后，(E/R)SI 和 (E/R)DI 寄存器将根据 EFLAGS 寄存器中 DF 标志的设置自动递增或递减。（如果 DF 标志为 0，则 (E/R)SI 和 (E/R)DI 寄存器递增；如果 DF 标志为 1，则寄存器递减。）对于字节操作，寄存器递增或递减 1；对于字操作，递增或递减 2；对于双字操作，递增或递减 4。如果操作数大小为 64，对于四字操作，RSI 和 RDI 寄存器递增 8。

在进行数据块比较时，CMPS、CMPSB、CMPSW、CMPSD 和 CMPSQ 指令之前可以带有 REP 前缀。然而更常见的是，这些指令将在 LOOP 结构中使用，该结构在进行下一次比较之前根据状态标志位的设置采取某些行动。有关 REP 前缀的描述，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 2B 卷第 4 章中的“REP/REPE/REPZ /REPNE/REPNZ — 重复字符串操作前缀”。

在 64 位模式下，指令的默认地址大小为 64 位，使用前缀 67H 支持 32 位地址大小。使用 REX.W 前缀可将 doubleword 操作提升为 64 位（参见 CMPSQ）。有关编码数据和限制，请参阅本节开头的汇总表。

操作¶

temp := SRC1 - SRC2;
SetStatusFlags(temp);
IF (64-Bit Mode)
    THEN
        IF (Byte comparison)
        THEN IF DF = 0
            THEN
                (R|E)SI := (R|E)SI + 1;
                (R|E)DI := (R|E)DI + 1;
            ELSE
                (R|E)SI := (R|E)SI – 1;
                (R|E)DI := (R|E)DI – 1;
            FI;
        ELSE IF (Word comparison)
            THEN IF DF = 0
                THEN
                    (R|E)SI
                        := (R|E)SI + 2;
                    (R|E)DI
                        := (R|E)DI + 2;
                ELSE
                    (R|E)SI
                        := (R|E)SI – 2;
                    (R|E)DI
                        := (R|E)DI – 2;
                FI;
        ELSE IF (Doubleword
                        comparison)
            THEN IF DF = 0
                THEN
                    (R|E)SI
                        := (R|E)SI + 4;
                    (R|E)DI
                        := (R|E)DI + 4;
                ELSE
                    (R|E)SI
                        := (R|E)SI – 4;
                    (R|E)DI
                        := (R|E)DI – 4;
                FI;
        ELSE (* Quadword comparison *)
            THEN IF DF = 0
                (R|E)SI := (R|E)SI + 8;
                (R|E)DI := (R|E)DI + 8;
            ELSE
                (R|E)SI := (R|E)SI – 8;
                (R|E)DI := (R|E)DI – 8;
            FI;
        FI;
    ELSE (* Non-64-bit Mode *)
        IF (byte comparison)
        THEN IF DF = 0
            THEN
                (E)SI := (E)SI + 1;
                (E)DI := (E)DI + 1;
            ELSE
                (E)SI := (E)SI – 1;
                (E)DI := (E)DI – 1;
            FI;
        ELSE IF (Word comparison)
            THEN IF DF = 0
                (E)SI := (E)SI + 2;
                (E)DI := (E)DI + 2;
            ELSE
                (E)SI := (E)SI – 2;
                (E)DI := (E)DI – 2;
            FI;
        ELSE (* Doubleword comparison *)
            THEN IF DF = 0
                (E)SI := (E)SI + 4;
                (E)DI := (E)DI + 4;
            ELSE
                (E)SI := (E)SI – 4;
                (E)DI := (E)DI – 4;
            FI;
        FI;
FI;

影响的标志位¶

根据比较的临时结果设置 CF、OF、SF、ZF、AF 和 PF 标志。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择器。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀。
