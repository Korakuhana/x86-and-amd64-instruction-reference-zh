FISTTP — 截断存储整数

操作码 指令 64位模式 兼容/传统模式 描述
DF /1 FISTTP m16int 有效 有效 截断将 ST(0) 存储到 m16int 中。
DB /1 FISTTP m32int 有效 有效 截断将 ST(0) 存储到 m32int 中。
DD /1 FISTTP m64int 有效 有效 截断将 ST(0) 存储到 m64int 中。

描述¶

FISTTP 使用截断（chop）作为舍入模式将 ST 中的值转换为有符号整数，将结果传输到目的地，并弹出 ST。FISTTP 接受字、短整数和长整数目的操作数。

下表显示了在以整数格式存储各类数字时获得的结果。

F 表示有限浮点值。

I 表示整数。

* 表示浮点无效操作 (#IA) 异常。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

DEST := ST;
pop ST;

影响的标志寄存器¶

C1 被清除；C0、C2、C3 未定义。

数值异常¶

无效操作、栈无效（栈下溢）、精确度。

保护模式异常¶

#GP(0) 如果目的位于不可写段中。
如果 CS、DS、ES、FS 或 GS 段中的内存操作数有效地址非法。
#SS(0) 如果 SS 段中的地址非法。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#NM 如果 CR0.EM[bit 2] = 1。
如果 CR0.TS[bit 3] = 1。
#UD 如果 CPUID.01H:ECX.SSE3[bit 0] = 0。
如果使用了 LOCK 前缀。

GP(0) 如果操作数的任何部分超出从 0 到 0FFFFH 的有效地址空间。
