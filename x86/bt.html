描述 ¶

选择一个位串（由第一个操作数，即位基址指定）中由位偏移量（由第二个操作数指定）指示的位位置，并将该位的值存入 CF 标志中。位基址操作数可以是寄存器或内存地址；位偏移量操作数可以是寄存器或立即数：

  • 如果位基址操作数指定了寄存器，该指令将对位偏移量操作数进行模 16、32 或 64 的运算（取模大小取决于运行模式和寄存器大小；64 位操作数仅在 64 位模式下可用）。
  • 如果位基址操作数指定了内存地址，该操作数表示包含该位串位基址（所指定字节的第 0 位）的内存中字节的地址。偏移量操作数可以引用的位位置范围取决于操作数的大小。

另请参阅原书第 3-11 页的 Bit(BitBase, BitOffset)。

某些汇编器支持大于 31 的立即数位偏移量，方法是将立即数位偏移量字段与内存操作数的位移（displacement）字段结合使用。在这种情况下，立即数位偏移量的低 3 位或低 5 位（16 位操作数为 3 位，32 位操作数为 5 位）被保存在立即数位偏移量字段中，而高位部分由汇编器进行移位并与寻址模式中的字节位移相结合。如果高位不为零，处理器将忽略这些高位。

当访问内存中的位时，对于 32 位操作数大小，处理器可能会访问从该内存地址开始的 4 个字节，使用以下对应关系：

有效地址 + (4 ∗ (BitOffset DIV 32))

或者，对于 16 位操作数，它可能会访问从该内存地址开始的 2 个字节，使用以下对应关系：

有效地址 + (2 ∗ (BitOffset DIV 16))

即使只需访问单个字节即可到达给定的位，它也可能会这样做。在使用这种位寻址机制时，软件应避免引用靠近地址空间空洞的内存区域。特别地，它应避免引用内存映射的 I/O 寄存器。相反，软件应使用 MOV 指令从这些地址进行加载或向其进行存储，并使用这些指令的寄存器形式来操作数据。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 形式的 REX 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 形式的 REX 前缀可将操作提升至 64 位操作数。请参阅本节开头的汇总图表以获取编码数据和限制。

操作 ¶ 

CF := Bit(BitBase, BitOffset);

受影响的标志位 ¶

CF 标志包含选定位的值。ZF 标志不受影响。OF、SF、AF 和 PF 标志位未定义。

保护模式异常 ¶

#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数的有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀。
