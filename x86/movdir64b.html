MOVDIR64B — 以直接存储方式移动 64 字节

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
66 0F 38 F8 /r MOVDIR64B r16/r32/r64, m512 A 有效/有效 MOVDIR64B 将 64 字节作为直接存储移动，保证从源内存操作数地址到寄存器操作数中指定的 ES 段偏移量的目的内存地址的 64 字节写原子性。

描述¶

将 64 字节作为直接存储从源内存地址移动到目的内存地址，具有 64 字节写原子性。源操作数是普通的内存操作数。目的操作数是通用寄存器中指定的内存位置。寄存器内容被解释为进入 ES 段的偏移量，没有任何段超越。在 64 位模式下，寄存器操作数宽度为 64 位（使用 67H 前缀时为 32 位）。在 64 位模式之外，当 CS.D=1 时寄存器宽度为 32 位（使用 67H 前缀时为 16 位），当 CS.D=0 时寄存器宽度为 16 位（使用 67H 前缀时为 32 位）。MOVDIR64B 要求目的地址 64 字节对齐。源操作数没有对齐限制。

MOVDIR64B 首先从源内存地址读取 64 字节。然后对目的地址执行 64 字节的直接存储操作。加载操作遵循基于源地址内存类型的普通读取顺序。直接存储是通过使用写合并 (WC) 内存类型协议来写入数据来实现的。使用该协议，处理器不会将数据写入缓存层次结构，也不会从内存中将相应的缓存行提取到缓存层次结构中。如果目的地址被缓存，则在直接存储之前，该行会被写回（如果被修改）并从缓存中失效。

与允许目的地的 UC/WP 内存类型覆盖非临时提示的带非临时提示的存储不同，直接存储总是遵循 WC 内存类型协议，而不考虑目的地址内存类型（包括 UC/WP 类型）。与 WC 存储和带非临时提示的存储不同，直接存储有资格从写合并缓冲区中立即驱逐，因此不会与对相同地址的更晚存储（包括直接存储）进行合并。写合并缓冲区中保存的更早的 WC 和非临时存储可能会与相同地址的更晚直接存储进行合并。相对于其他存储，直接存储是弱有序的。需要更强顺序的软件应在直接存储之前或之后使用屏障指令（MFENCE 或 SFENCE）来强制执行所需的顺序。

源地址的 64 字节加载操作没有提供原子性保证，处理器实现可能会使用多次加载操作来读取这 64 字节。MOVDIR64B 发出的 64 字节直接存储保证了 64 字节的写完成原子性。这意味着数据在单个未被分割的 64 字节写入事务中到达目的地。

MOVDIR64B 指令的可用性由 CPUID 特征标志 MOVDIR64B 指示（叶 07H 中 ECX 寄存器的第 28 位，参见《Intel® 64 和 IA-32 架构软件开发人员手册》第 2A 卷中的“CPUID — CPU 识别”）。
