EDBGWR — 写入调试 Enclave

操作码/指令 操作/编码 64/32位模式支持 CPUID功能标志 描述
EAX = 05H ENCLS[EDBGWR] IR V/V SGX1 此叶函数向调试 enclave 写入双字/四字。

指令操作数编码¶

操作/编码 EAX RBX RCX
IR EDBGWR (输入) 返回错误代码 (输出) 要写入调试 enclave 的数据 (输入) EPC 中目标内存的地址 (输入)

描述¶

此叶函数将 EBX/RBX 中的内容复制到属于调试 enclave 的 EPC 页面。在 64 位模式下写入 8 字节，在非 64 位模式下写入 4 字节。数据大小无法覆盖。

EPC 内部目标位置的有效地址由寄存器 RCX 提供。

EDBGWR 内存参数语义¶

EPCQW
Enclave 允许的写访问

如果发生以下任何情况，指令将产生异常：

EDBGWR 异常条件¶

RCX 指向一个属于 SECS 的页面。 RCX 未解析为自然对齐的线性地址。
RCX 指向的页面不属于处于调试模式下的 enclave。 RCX 指向 TCS 内部且不是 FLAGS 字的位置。
导致任何段违规的操作数。 可能发生页面错误。
CPL > 0。

此指令忽略 enclave 页面上的 EPCM RWX 属性。因此，通过 EDBGRD 违反 EPCM RWX 属性不会导致 #GP。

EDBGWR 操作流程中的临时变量¶

名称 类型 大小 (位) 描述
TMP_MODE64 二进制 1 ((IA32_EFER.LMA = 1) && (CS.L = 1))。
TMP_SECS 物理地址 64 源操作数所属 enclave 的 SECS 物理地址。

TMP_MODE64 := ((IA32_EFER.LMA = 1) && (CS.L = 1));

IF ( (TMP_MODE64 = 1) and (DS:RCX is not 8Byte Aligned) )

THEN #GP(0); FI;

IF ( (TMP_MODE64 = 0) and (DS:RCX is not 4Byte Aligned) )

THEN #GP(0); FI;

IF (DS:RCX does not resolve within an EPC)

THEN #PF(DS:RCX); FI;

(* make sure no other Intel SGX instruction is accessing the same EPCM entry *)

IF (Another instruction modifying the same EPCM entry is executing)

THEN #GP(0); FI;

IF (EPCM(DS:RCX).VALID = 0)

THEN #PF(DS:RCX); FI;

(* make sure that DS:RCX (DST) is pointing to a PT_REG or PT_TCS or PT_SS_FIRST or PT_SS_REST *)

IF ( (EPCM(DS:RCX).PT ≠ PT_REG) and (EPCM(DS:RCX).PT ≠ PT_TCS)

and (EPCM(DS:RCX).PT ≠ PT_SS_FIRST) and (EPCM(DS:RCX).PT ≠ PT_SS_REST))

THEN #PF(DS:RCX); FI;

(* make sure that DS:RCX points to an accessible EPC page *)

IF ( (EPCM(DS:RCX).PENDING is not 0) or (EPCM(DS:RCS).MODIFIED is not 0) )

THEN

RFLAGS.ZF := 1;

RAX := SGX_PAGE_NOT_DEBUGGABLE;

GOTO DONE;

FI;

(* If destination is a TCS, then make sure that the offset into the page can only point to the FLAGS field*)

IF ( ( EPCM(DS:RCX). PT = PT_TCS) and ((DS:RCX) & FF8H ≠ offset_of_FLAGS & 0FF8H) )

THEN #GP(0); FI;

(* Locate the SECS for the enclave to which the DS:RCX page belongs *)

TMP_SECS := GET_SECS_PHYS_ADDRESS(EPCM(DS:RCX).ENCLAVESECS);

(* make sure the enclave owning the PT_REG or PT_TCS page allow debug *)

IF (TMP_SECS.ATTRIBUTES.DEBUG = 0)

THEN #GP(0); FI;

IF ( (TMP_MODE64 = 1) )

THEN (DS:RCX)[63:0] := RBX[63:0];

ELSE (DS:RCX)[31:0] := EBX[31:0];

FI;

(* clear EAX and ZF to indicate successful completion *)

RAX := 0;

RFLAGS.ZF := 0;

DONE:

(* clear flags *)

RFLAGS.CF,PF,AF,OF,SF := 0

影响的标志寄存器¶

如果页面为 MODIFIED 或 PENDING，则设置 ZF；RAX 包含错误代码。否则清除 ZF 且 RAX 设置为 0。清除 CF、PF、AF、OF、SF。

保护模式异常¶

#GP(0) 如果 RCX 中的地址违反了 DS 限制或访问权限。
如果 DS 段不可用。
如果 RCX 指向的内存位置未 4 字节对齐。
如果 RCX 中的地址指向属于非调试 enclave 的页面。
如果 RCX 中的地址指向不是 PT_TCS 或 PT_REG 的页面。
如果 RCX 中的地址指向 TCS 内部且不是 FLAGS 字的位置。
#PF(错误代码) 如果访问内存操作数时发生页面错误。
如果 RCX 中的地址指向非 EPC 页面。
如果 RCX 中的地址指向无效的 EPC 页面。

64位模式异常¶

#GP(0) 如果 RCX 为非规范形式。
如果 RCX 指向的内存位置未 8 字节对齐。
如果 RCX 中的地址指向属于非调试 enclave 的页面。
如果 RCX 中的地址指向不是 PT_TCS 或 PT_REG 的页面。
如果 RCX 中的地址指向 TCS 内部且不是 FLAGS 字的位置。
#PF(错误代码) 如果访问内存操作数时发生页面错误。
如果 RCX 中的地址指向非 EPC 页面。
如果 RCX 中的地址指向无效的 EPC 页面。
