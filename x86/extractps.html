EXTRACTPS — 提取打包的单精度浮点值

描述¶

从源操作数（第二个操作数）中由 imm8 指定的 32 位偏移量处提取一个单精度浮点值。忽略高于矢量长度最大有效偏移量的立即数位。

提取出的单精度浮点值存储在目标操作数的低 32 位中。

在 64 位模式下，目标寄存器操作数的默认操作数大小为 64 位。寄存器的高 32 位填充为零。忽略 REX.W。

VEX.128 和 EVEX 编码版本：在 64 位模式下使用 VEX.W1 或 EVEX.W1 形式且以通用寄存器（GPR）作为目标操作数时，打包的单精度量会被零扩展至 64 位。

VEX.vvvv/EVEX.vvvv 保留且必须为 1111b，否则指令将产生 #UD。

128 位传统 SSE 版本：在 64 位模式下使用 REX.W 前缀且以通用寄存器（GPR）作为目标操作数时，打包的单精度量会被零扩展至 64 位。

源寄存器是 XMM寄存器。Imm8[1:0] 决定开始提取 32 位浮点值的双字（DWORD）偏移量。

如果 VEXTRACTPS 编码有 VEX.L= 1，则尝试执行编码有 VEX.L= 1 的指令会导致 #UD 异常。

操作¶

VEXTRACTPS (EVEX 与 VEX.128 编码版本)¶

SRC_OFFSET := IMM8[1:0]
IF (64-Bit Mode and DEST is register)
	DEST[31:0] := (SRC[127:0] >> (SRC_OFFSET*32)) AND 0FFFFFFFFh
	DEST[63:32] := 0
ELSE
	DEST[31:0] := (SRC[127:0] >> (SRC_OFFSET*32)) AND 0FFFFFFFFh
FI

EXTRACTPS (128位传统 SSE 版本)¶

SRC_OFFSET := IMM8[1:0]
IF (64-Bit Mode and DEST is register)
	DEST[31:0] := (SRC[127:0] >> (SRC_OFFSET*32)) AND 0FFFFFFFFh
	DEST[63:32] := 0
ELSE
	DEST[31:0] := (SRC[127:0] >> (SRC_OFFSET*32)) AND 0FFFFFFFFh
FI
