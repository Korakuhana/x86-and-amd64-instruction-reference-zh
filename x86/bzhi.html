BZHI — 从指定位开始清零高位

操作码/指令 Op/En 64/32位模式 CPUID 功能标志 描述
VEX.LZ.0F38.W0 F5 /r BZHI r32a, r/m32, r32b RMV V/V BMI2 以 r32b 为指定的起始位置，将 r/m32 中的高位清零，结果存入 r32a。
VEX.LZ.0F38.W1 F5 /r BZHI r64a, r/m64, r64b RMV V/N.E. BMI2 以 r64b 为指定的起始位置，将 r/m64 中的高位清零，结果存入 r64a。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RMV ModRM:reg (w) ModRM:r/m (r) VEX.vvvv (r) N/A

描述¶

BZHI 将第一源操作数（第二个操作数）的位复制到目的操作数（第一个操作数）中，并根据第二源操作数（第三个操作数）指定的 INDEX 值清除目的操作数中的高位。INDEX 由第二源操作数的 7:0 位指定。INDEX 值饱和于 OperandSize - 1 的值。如果第三个操作数的低 8 位中包含的数值大于 OperandSize - 1，则设置 CF。

在实模式和虚拟 8086 模式下不支持此指令。如果不在 64 位模式下，操作数大小始终为 32 位。在 64 位模式下，操作数大小为 64 位时需要 VEX.W1。在非 64 位模式下，VEX.W1 被忽略。尝试在 VEX.L 不等于 0 的情况下执行此指令将导致 #UD。

操作¶

N := SRC2[7:0]
DEST := SRC1
IF (N < OperandSize)
    DEST[OperandSize-1:N] := 0
FI
IF (N > OperandSize - 1)
    CF := 1
ELSE
    CF := 0
FI

受影响的标志位¶

ZF 和 SF 标志根据结果更新。CF 标志根据“操作”部分中的规定进行设置。OF 标志位被清除。AF 和 PF 标志位是未定义的。
