OUT — 输出到端口

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
E6 ib OUT imm8, AL I Valid Valid 将 AL 中的字节输出到立即数 imm8 指定的 I/O 端口地址。
E7 ib OUT imm8, AX I Valid Valid 将 AX 中的字输出到立即数 imm8 指定 of I/O 端口地址。
E7 ib OUT imm8, EAX I Valid Valid 将 EAX 中的双字输出到立即数 imm8 指定的 I/O 端口地址。
EE OUT DX, AL ZO Valid Valid 将 AL 中的字节输出到 DX 中指定的 I/O 端口地址。
EF OUT DX, AX ZO Valid Valid 将 AX 中的字输出到 DX 中指定的 I/O 端口地址。
EF OUT DX, EAX ZO Valid Valid 将 EAX 中的双字输出到 DX 中指定的 I/O 端口地址。

1. 参见下文的 IA-32 架构兼容性部分。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
I imm8 N/A N/A N/A
ZO N/A N/A N/A N/A

描述¶

将第二个操作数（源操作数）的值复制到目标操作数（第一个操作数）指定的 I/O 端口中。源操作数可以是 AL、AX 或 EAX 寄存器，具体取决于正在访问的端口的大小（分别为 8 位、16 位或 32 位）；目标操作数可以是 8 位立即数或 DX 寄存器。使用 8 位立即数允许访问 0 到 255 的 I/O 端口地址；使用 DX 寄存器作为目标操作数允许访问 0 到 65,535 的 I/O 端口。

访问的 I/O 端口大小由 8 位 I/O 端口的操作码确定，或由 16 位或 32 位 I/O 端口的指令的操作数大小属性确定。

在机器码级别，访问 8 位 I/O 端口时的 I/O 指令更短。此时，端口地址的高 8 位将为 0。

此指令仅用于访问位于处理器 I/O 地址空间中的 I/O 端口。有关在 I/O 地址空间中访问 I/O 端口的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷中的第 19 章“输入/输出”。

此指令在非 64 位模式和 64 位模式下的操作相同。

IA-32 架构兼容性¶

执行 OUT 指令后，Pentium® 处理器在开始执行下一条指令之前，会确保已采样到 EWBE# 引脚为有效状态。（请注意，如果 EWBE# 无效，则可以预取指令，但只有在采样到 EWBE# 引脚为有效状态后才会执行。）仅 Pentium 处理器系列具有 EWBE# 引脚。

操作¶

IF ((PE = 1) and ((CPL > IOPL) or (VM = 1)))
    THEN (* Protected mode with CPL > IOPL or virtual-8086 mode *)
        IF (Any I/O Permission Bit for I/O port being accessed = 1)
            THEN (* I/O operation is not allowed *)
                #GP(0);
            ELSE ( * I/O operation is allowed *)
                DEST := SRC; (* Writes to selected I/O port *)
        FI;
    ELSE (Real Mode or Protected Mode with CPL ≤ IOPL *)
        DEST := SRC; (* Writes to selected I/O port *)
FI;

影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果 CPL 大于（特权级较低）I/O 特权级别（IOPL），并且正在访问的 I/O 端口在 TSS 中对应的任意 I/O 许可位为 1。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#GP(0) 如果所访问的 I/O 端口在 TSS 中的任意 I/O 许可位为 1。
#PF(fault-code) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

与保护模式下的异常相同。
