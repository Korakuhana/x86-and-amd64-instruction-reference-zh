MOV — 传送数据到/自调试寄存器

操作码/指令 Op/En 64位模式 兼容/传统模式 描述
0F 21/r MOV r32, DR0–DR7 MR N.E. Valid 将调试寄存器移动到 r32。
0F 21/r MOV r64, DR0–DR7 MR Valid N.E. 将扩展调试寄存器移动到 r64。
0F 23 /r MOV DR0–DR7, r32 RM N.E. Valid 将 r32 移动到调试寄存器。
0F 23 /r MOV DR0–DR7, r64 RM Valid N.E. 将 r64 移动到扩展调试寄存器。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
MR ModRM:r/m (w) ModRM:reg (r) N/A N/A
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

将调试寄存器（DR0、DR1、DR2、DR3、DR4、DR5、DR6 或 DR7）的内容移动到通用寄存器，反之亦然。在非 64 位模式下，不论操作数大小属性如何，这些指令的操作数大小始终为 32 位。（有关调试寄存器中标志和字段的详细描述，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3A 卷第 18.2 节“调试寄存器”。）

这些指令必须在特权级别 0 或实地址模式下执行。

当寄存器 CR4 中的调试扩展（DE）标志清除时，这些指令操作调试寄存器的方式与 Intel386 和 Intel486 处理器兼容。在这种模式下，引用 DR4 和 DR5 分别代表 DR6 和 DR7。当 CR4 中的 DE 标志置位时，尝试引用 DR4 和 DR5 会导致未定义操作码（#UD）异常。（CR4 寄存器是从 Pentium 处理器开始添加到 IA-32 架构中的。）

在操作码级别，ModR/M 字节内的 reg 字段指定加载或读取哪个调试寄存器。mod 字段中的两个位被忽略。r/m 字段指定加载或读取的通用寄存器。

在 64 位模式下，指令的默认操作大小为 64 位。使用 REX.B 前缀允许访问额外的寄存器（R8–R15）。忽略使用 REX.W 或 66H 前缀。使用 REX.R 前缀会导致无效操作码异常。请参阅本节开头的汇总图表以获取编码数据和限制。

操作¶

IF ((DE = 1) and (SRC or DEST = DR4 or DR5))
    THEN
        #UD;
    ELSE
        DEST := SRC;
FI;

受影响的标志位¶

OF, SF, ZF, AF, PF 和 CF 标志未定义。

保护模式异常¶

#GP(0) 如果当前特权级别不为 0。
#UD 如果 CR4.DE[bit 3] = 1（调试扩展）且执行了涉及 DR4 或 DR5 的 MOV 指令。
如果使用了 LOCK 前缀。
#DB 如果在 DR7.GD[bit 13] = 1 时访问任何调试寄存器。

实地址模式异常¶

#UD 如果 CR4.DE[bit 3] = 1（调试扩展）且执行了涉及 DR4 或 DR5 的 MOV 指令。
如果使用了 LOCK 前缀。
#DB 如果在 DR7.GD[bit 13] = 1 时访问任何调试寄存器。

虚拟-8086 模式异常¶

#GP(0) 在虚拟-8086 模式下无法加载或读取调试寄存器。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#GP(0) 如果当前特权级别不为 0。
如果试图在 DR6 的 bits 63:32 中写入 1。
如果试图在 DR7 的 bits 63:32 中写入 1。
#UD 如果 CR4.DE[bit 3] = 1（调试扩展）且执行了涉及 DR4 或 DR5 的 MOV 指令。
如果使用了 LOCK 前缀。
如果使用了 REX.R 前缀。
#DB 如果在 DR7.GD[bit 13] = 1 时访问任何调试寄存器。
