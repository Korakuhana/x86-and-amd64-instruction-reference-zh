LOOP/LOOPcc — 根据 ECX 计数器循环

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
E2 cb LOOP rel8 D Valid Valid 计数器减 1；如果计数器不为 0，则进行短跳转。
E1 cb LOOPE rel8 D Valid Valid 计数器减 1；如果计数器不为 0 且 ZF = 1，则进行短跳转。
E0 cb LOOPNE rel8 D Valid Valid 计数器减 1；如果计数器不为 0 且 ZF = 0，则进行短跳转。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
D Offset (偏移量) N/A N/A N/A

描述¶

使用 RCX、ECX 或 CX 寄存器作为计数器（取决于地址大小是 64 位、32 位还是 16 位）执行循环操作。请注意，LOOP 指令会忽略 REX.W；但可以使用 67H 前缀覆盖 64 位地址大小。

每次执行 LOOP 指令时，计数寄存器都会递减，然后检查是否为 0。如果计数为 0，则循环终止，程序继续执行 LOOP 指令后面的指令。如果计数不为零，则执行到目标（target）操作数的近跳转，该操作数通常是循环开始处的指令。

目标指令使用相对偏移量（相对于 IP/EIP/RIP 寄存器中指令指针当前值的有符号偏移量）指定。此偏移量在汇编代码中通常指定为标签，但在机器码级别，它被编码为有符号的 8 位立即数，该值会被加到指令指针中。此指令允许的偏移量为 -128 到 +127。

某些形式的循环指令（LOOPcc）还接受 ZF 标志作为在计数达到零之前终止循环的条件。对于这些形式的指令，每个指令都关联一个条件码（cc），以指示被测试的条件。在这里，LOOPcc 指令本身不会影响 ZF 标志的状态；ZF 标志是由循环中的其他指令改变的。

操作¶

IF (AddressSize = 32)
    THEN Count is ECX;
ELSE IF (AddressSize = 64)
    Count is RCX;
ELSE Count is CX;
FI;
Count := Count – 1;
IF Instruction is not LOOP
    THEN
        IF (Instruction := LOOPE) or (Instruction := LOOPZ)
            THEN IF (ZF = 1) and (Count ≠ 0)
                    THEN BranchCond := 1;
                    ELSE BranchCond := 0;
                FI;
            ELSE (Instruction = LOOPNE) or (Instruction = LOOPNZ)
                IF (ZF = 0 ) and (Count ≠ 0)
                    THEN BranchCond := 1;
                    ELSE BranchCond := 0;
        FI;
    ELSE (* Instruction = LOOP *)
        IF (Count ≠ 0)
            THEN BranchCond := 1;
            ELSE BranchCond := 0;
        FI;
FI;
IF BranchCond = 1
    THEN
        IF in 64-bit mode (* OperandSize = 64 *)
            THEN
                tempRIP := RIP + SignExtend(DEST);
                IF tempRIP is not canonical
                    THEN #GP(0);
                ELSE RIP := tempRIP;
                FI;
            ELSE
                tempEIP := EIP SignExtend(DEST);
                IF OperandSize 16
                    THEN tempEIP := tempEIP AND 0000FFFFH;
                FI;
                IF tempEIP is not within code segment limit
                    THEN #GP(0);
                    ELSE EIP := tempEIP;
                FI;
        FI;
    ELSE
        Terminate loop and continue program execution at (R/E)IP;
FI;

影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果跳转的偏移量超出了 CS 段的限制。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果跳转的偏移量超出了 CS 段的限制，或者超出了 0 到 FFFFH 的有效地址空间。如果使用了 32 位地址大小覆盖前缀，可能会发生这种情况。
#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

与实地址模式下的异常相同。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#GP(0) 如果跳转的偏移量是非规范形式。
#UD 如果使用了 LOCK 前缀。
