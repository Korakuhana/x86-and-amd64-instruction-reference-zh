LOADIWKEY — 使用 Key Locker 加载内部包装密钥

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
F3 0F 38 DC 11:rrr:bbb LOADIWKEY xmm1, xmm2, <EAX>, <XMM0> A 有效/有效 KL 从 xmm1、xmm2 和 XMM0 加载内部包装密钥。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2 操作数 3 操作数 4
A N/A ModRM:reg (r) ModRM:r/m (r) 隐式 EAX (r) 隐式 XMM0 (r)

描述¶

LOADIWKEY1 指令写入 Key Locker 内部包装密钥，该密钥称为 IWKey。该 IWKey 被 ENCODEKEY* 指令用于将密钥包装到句柄中。相反，AESENC/DEC*KL 指令使用 IWKey 从句柄中解包这些密钥并帮助验证句柄的完整性。出于安全原因，没有设计任何指令允许软件直接读取 IWKey 值。

IWKey 包括两个加密密钥以及元数据。这两个加密密钥是从寄存器源加载的，因此可以在密钥从未存在于内存中的情况下执行 LOADIWKEY。

密钥输入操作数为：

  • 256 位加密密钥从两个显式操作数加载。
  • 128 位完整性密钥从隐式操作数 XMM0 加载。

隐式操作数 EAX 指定 KeySource 以及是否允许备份密钥：

  • EAX[0] – 设置时，不允许将正在初始化的包装密钥备份到平台范围的存储中。
  • EAX[4:1] – 这指定了 KeySource，即密钥的类型。目前仅支持两种编码。KeySource 为 0 表示上述密钥输入操作数应直接存储为内部包装密钥。KeySource 为 1 的 LOADIWKEY 将使来自片上随机数发生器的随机数与源寄存器（包括 XMM0）进行异或，以便执行 LOADIWKEY 的软件不知道实际的 IWKey 加密和完整性密钥。软件可以选择将额外的随机数据放入源寄存器中，以便将其他随机数据源与硬件随机数发生器提供的值相结合。在使用 KeySource 为 1 执行 LOADIWKEY 后，软件应始终检查 ZF，因为该操作可能会因为无法从片上随机数发生器获取充足的满熵数据而失败。KeySource 为 0 和 1 都指定 IWKey 与 AES-GCM-SIV 算法一起使用。CPUID.19H.ECX[1] 列举了对 KeySource 1 的支持。所有其他 KeySource 编码均保留。
  • EAX[31:5] – 保留。

1. 关于 Key Locker 和该指令使用的进一步细节可以在这里找到：

https://software.intel.com/content/www/us/en/develop/download/intel-key-locker-specification.html.¶

操作¶

IF CPL > 0
                    // LOADKWKEY only allowed at ring 0 (supervisor mode)
    THEN #GP (0); FI;
IF EAX[4:1] > 1
                    // Reserved KeySource encoding used
    THEN #GP (0); FI;
IF EAX[31:5] != 0
                    // Reserved bit in EAX is set
    THEN #GP (0); FI;
IF EAX[0] AND (CPUID.19H.ECX[0] == 0)
                        // NoBackup is not supported on this part
    THEN #GP (0); FI;
IF (EAX[4:1] == 1) AND (CPUID.19H.ECX[1] == 0)
                        // KeySource of 1 is not supported on this part
    THEN #GP (0); FI;
IF (EAX[4:1] == 0) // KeySource of 0
    THEN
        IWKey.Encryption Key[127:0] := SRC2[127:0]:
        IWKey.Encryption Key[255:128] := SRC1[127:0];
        IWKey.IntegrityKey[127:0] := XMM0[127:0];
        IWKey.NoBackup = EAX [0];
        IWKey.KeySource = EAX [4:1];
        RFLAGS.ZF := 0;
    ELSE // KeySource of 1. See RDSEED definition for details of randomness
        IF HW_NRND_GEN.ready == 1 // Full-entropy random data from RDSEED hardware block was received
            THEN
                IWKey.Encryption Key[127:0] := SRC2[127:0] XOR HW_NRND_GEN.data[127:0];
                IWKey.Encryption Key[255:128] := SRC1[127:0] XOR HW_NRND_GEN.data[255:128];
                IWKey.IntegrityKey[127:0] := XMM0[127:0] XOR HW_NRND_GEN.data[383:256];
                IWKey.NoBackup = EAX [0];
                IWKey.KeySource = EAX [4:1];
                RFLAGS.ZF := 0;
            ELSE // Random data was not returned from RDSEED hardware block. IWKey was not loaded
                RFLAGS.ZF := 1;
        FI;
FI;
RFLAGS.OF, SF, AF, PF, CF := 0;

受影响的标志位¶

如果操作成功，ZF 设置为 0；如果由于未从 RDSEED 接收到满熵随机数据导致操作失败，则 ZF 设置为 1。其他算术标志位（OF、SF、AF、PF、CF）清除为 0。

异常（所有工作模式）¶

#GP 如果 CPL > 0。（不适用于实地址模式。）

如果 EAX[4:1] > 1。

如果 EAX[31:5] != 0。

如果 (EAX[0] == 1) 且 (CPUID.19H.ECX[0] == 0)。

如果 (EAX[4:1] == 1) 且 (CPUID.19H.ECX[1] == 0)。

#UD 如果使用了 LOCK 前缀。

如果 CPUID.07H:ECX.KL[bit 23] = 0。

如果 CR4.KL = 0。

如果 CR0.EM = 1。

如果 CR4.OSFXSR = 0。

#NM 如果 CR0.TS = 1。
