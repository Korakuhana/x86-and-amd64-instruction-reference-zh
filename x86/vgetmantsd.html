VGETMANTSD — 从 Float64 标量提取 Float64 规范化尾数

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.66.0F3A.W1 27 /r ib VGETMANTSD xmm1 {k1}{z}, xmm2, xmm3/m64{sae}, imm8 A V/V AVX512F 使用 imm8 进行符号控制和尾数区间规范化，提取 xmm3/m64 中低位 float64 元素的规范化尾数。在写掩码 k1 下将尾数存储到 xmm1 中，并与 xmm2 的其他元素合并。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Tuple1 Scalar ModRM:reg (w) EVEX.vvvv (r) ModRM:r/m (r) imm8

描述¶

将第二源操作数（第三操作数）低位四字元素中的双精度浮点值转换为具有由 imm8 字节指定的尾数规范化和符号控制的双精度浮点值，参见图 5-15。转换后的结果使用写掩码 k1 写入目的操作数（第一操作数）的低位四字元素。目的 XMM 寄存器的第 (127:64) 位复制自第一源操作数中的对应位。规范化尾数由 interv（imm8[1:0]）指定，符号控制（sc）由立即数字节的第 3:2 位指定。

转换操作为：

GetMant(x) = ±2k|x.significand|

其中：

1 <= |x.significand| < 2

无偏指数 k 可以是 0 或 -1，具体取决于由 interv 定义的区间范围、尾数（有效数字）的范围以及源的指数是偶数还是奇数。最终结果的符号由 sc 和源符号决定。imm8[1:0] 的编码值和符号控制如图 5-15 所示。

转换后的双精度浮点结果根据符号控制、无偏指数 k（加上偏置）和规范化到由 interv 指定范围的尾数进行编码。

在处理浮点特殊数值时，GetMant() 函数遵循表 5-18。

如果使用写掩码，目的操作数低位的四字元素将根据写掩码寄存器 k1 的值有条件地更新。如果不使用写掩码，目的操作数的低位四字元素将被无条件更新。

操作¶

// getmant_fp64(src, sign_control, normalization_interval) is defined in the operation section of VGETMANTPD

VGETMANTSD (EVEX 编码版本)¶

SignCtrl[1:0] := IMM8[3:2];
Interv[1:0] := IMM8[1:0];
IF k1[0] OR *no writemask*
    THEN DEST[63:0] :=
            getmant_fp64(src, sign_control, normalization_interval)
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[63:0] remains unchanged*
            ELSE ; zeroing-masking
                DEST[63:0] := 0
        FI
FI;
DEST[127:64] := SRC1[127:64]
DEST[MAXVL-1:128] := 0
