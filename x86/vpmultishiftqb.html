VPMULTISHIFTQB — 从四字源选择打包的未对齐字节

描述¶

该指令从第二源操作数（第三操作数）的每个输入四字元素中选择 8 个未对齐的字节，并为目的操作数（第一操作数）中的每个四字元素写入 8 个组合字节。每个字节结果是在第一源操作数（第二操作数）对应四字元素内使用字节粒度移位控制选择的。在写掩码 k1 的控制下更新目的操作数中的每个字节结果。

每个控制字节中仅低 6 位用于选择一个 8 位槽，以从第二源操作数的四字数据中提取输出字节。8 位槽的起始位可以相对于任何字节边界不对齐，并从输入四字源的控制字节低 6 位指定的位置提取。如果 8 位槽超出四字边界，则 8 位槽的超出边界部分会回绕，从输入四字元素的 bit 0 开始。

第一源操作数是 ZMM/YMM/XMM 寄存器。第二源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 64 位内存位置广播的 512/256/128 位向量。目的操作数是 ZMM/YMM/XMM 寄存器。

操作¶

VPMULTISHIFTQB DEST, SRC1, SRC2 (EVEX 编码版本)¶

(KL, VL) = (2, 128),(4, 256), (8, 512)
FOR i := 0 TO KL-1
    IF EVEX.b=1 AND src2 is memory THEN
            tcur := src2.qword[0]; //broadcasting
    ELSE
            tcur := src2.qword[i];
    FI;
    FOR j := 0 to 7
        ctrl := src1.qword[i].byte[j] & 63;
        FOR k := 0 to 7
            res.bit[k] := tcur.bit[ (ctrl+k) mod 64 ];
        ENDFOR
        IF k1[i*8+j] or no writemask THEN
            DEST.qword[i].byte[j] := res;
        ELSE IF zeroing-masking THEN
            DEST.qword[i].byte[j] := 0;
    ENDFOR
ENDFOR
DEST.qword[MAX_VL-1:VL] := 0;
