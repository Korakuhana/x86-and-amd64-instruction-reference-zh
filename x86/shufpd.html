SHUFPD — 打包双精度浮点值对的交错移位

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
66 0F C6 /r ib SHUFPD xmm1, xmm2/m128, imm8 A V/V SSE2 使用 imm8 从各对中进行选择，对来自 xmm1 和 xmm2/m128 的两对双精度浮点值进行交错移位（shuffle），交错后的结果存储在 xmm1 中。
VEX.128.66.0F.WIG C6 /r ib VSHUFPD xmm1, xmm2, xmm3/m128, imm8 B V/V AVX 使用 imm8 从各对中进行选择，对来自 xmm2 和 xmm3/m128 的两对双精度浮点值进行交错移位，交错后的结果存储在 xmm1 中。
VEX.256.66.0F.WIG C6 /r ib VSHUFPD ymm1, ymm2, ymm3/m256, imm8 B V/V AVX 使用 imm8 从各对中进行选择，对来自 ymm2 和 ymm3/m256 的四对双精度浮点值进行交错移位，交错后的结果存储在 xmm1 中。
EVEX.128.66.0F.W1 C6 /r ib VSHUFPD xmm1{k1}{z}, xmm2, xmm3/m128/m64bcst, imm8 C V/V AVX512VL AVX512F 使用 imm8 从各对中进行选择，对来自 xmm2 和 xmm3/m128/m64bcst 的两对双精度浮点值进行交错移位。在写掩码 k1 的控制下将交错后的结果存储在 xmm1 中。
EVEX.256.66.0F.W1 C6 /r ib VSHUFPD ymm1{k1}{z}, ymm2, ymm3/m256/m64bcst, imm8 C V/V AVX512VL AVX512F 使用 imm8 从各对中进行选择，对来自 ymm2 和 ymm3/m256/m64bcst 的四对双精度浮点值进行交错移位。在写掩码 k1 的控制下将交错后的结果存储在 ymm1 中。
EVEX.512.66.0F.W1 C6 /r ib VSHUFPD zmm1{k1}{z}, zmm2, zmm3/m512/m64bcst, imm8 C V/V AVX512F 使用 imm8 从各对中进行选择，对来自 zmm2 和 zmm3/m512/m64bcst 的八对双精度浮点值进行交错移位。在写掩码 k1 的控制下将交错后的结果存储在 zmm1 中。

描述¶

使用位控制从输入对中选择一个双精度浮点值，并将其移动到目的操作数的指定元素中。目的操作数的双精度元素按照从低到高的顺序，在第一源操作数和第二源操作数之间以 128 位输入对的粒度进行交错。imm8 字节中的每一位（从位 0 开始）都是目的操作数对应元素的物理选择控制，用以接收输入对交错移位后的结果。

EVEX 编码版本：第一源操作数是 ZMM/YMM/XMM 寄存器。第二源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 64 位内存位置广播的 512/256/128 位向量。目的操作数是根据写掩码更新的 ZMM/YMM/XMM 寄存器。选择控制是 imm8 字节的低 8/4/2 位。

VEX.256 编码版本：第一源操作数是 YMM 寄存器。第二源操作数可以是 YMM 寄存器或 256 位内存位置。目的操作数是 YMM 寄存器。选择控制是 imm8 字节的第 3:0 位，imm8[7:4] 被忽略。

VEX.128 编码版本：第一源操作数是 XMM 寄存器。第二源操作数可以是 XMM 寄存器或 128 位内存位置。目的操作数是 XMM 寄存器。对应 ZMM 寄存器目的操作数的高位 (MAXVL-1:128) 被清零。选择控制是 imm8 字节的第 1:0 位，imm8[7:2] 被忽略。

128 位传统 SSE 版本：第二源操作数可以是 XMM 寄存器或 128 位内存位置。目的操作数与第一源操作数相同，且为 XMM 寄存器。对应 ZMM 寄存器目的操作数的高位 (MAXVL-1:128) 保持不变。选择控制是 imm8 字节的第 1:0 位，imm8[7:2] 被忽略。

操作¶

VSHUFPD（当 SRC2 为向量寄存器时的 EVEX 编码版本）¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
IF IMM0[0] = 0
    THEN TMP_DEST[63:0] := SRC1[63:0]
    ELSE TMP_DEST[63:0] := SRC1[127:64] FI;
IF IMM0[1] = 0
    THEN TMP_DEST[127:64] := SRC2[63:0]
    ELSE TMP_DEST[127:64] := SRC2[127:64] FI;
IF VL >= 256
    IF IMM0[2] = 0
        THEN TMP_DEST[191:128] := SRC1[191:128]
        ELSE TMP_DEST[191:128] := SRC1[255:192] FI;
    IF IMM0[3] = 0
        THEN TMP_DEST[255:192] := SRC2[191:128]
        ELSE TMP_DEST[255:192] := SRC2[255:192] FI;
FI;
IF VL >= 512
    IF IMM0[4] = 0
        THEN TMP_DEST[319:256] := SRC1[319:256]
        ELSE TMP_DEST[319:256] := SRC1[383:320] FI;
    IF IMM0[5] = 0
        THEN TMP_DEST[383:320] := SRC2[319:256]
        ELSE TMP_DEST[383:320] := SRC2[383:320] FI;
    IF IMM0[6] = 0
        THEN TMP_DEST[447:384] := SRC1[447:384]
        ELSE TMP_DEST[447:384] := SRC1[511:448] FI;
    IF IMM0[7] = 0
        THEN TMP_DEST[511:448] := SRC2[447:384]
        ELSE TMP_DEST[511:448] := SRC2[511:448] FI;
FI;
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no writemask*
        THEN DEST[i+63:i] := TMP_DEST[i+63:i]
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+63:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+63:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
