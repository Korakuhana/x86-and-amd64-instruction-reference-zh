SIDT — 存储中断描述符表寄存器

操作码1

指令 Op/En 64位模式 兼容/传统模式 描述
0F 01 /1 SIDT m M Valid Valid 存储 IDTR 到 m。

1. 请参阅下文的 IA-32 架构兼容性部分。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (w) N/A N/A N/A

描述¶

将中断描述符表寄存器（IDTR）的内容存储在目的操作数中。目的操作数指定一个 6 字节的内存位置。

在非 64 位模式下，寄存器的 16 位限长字段存储在内存位置的低 2 字节中，32 位基地址存储在高 4 字节中。

在 64 位模式下，操作数大小固定为 8+2 字节。该指令存储 8 字节的基地址和 2 字节的限长值。

SIDT 仅在操作系统软件中有用。然而，如果 CR4.UMIP = 0，它可以在应用程序中使用而不会引发异常。有关加载 GDTR 和 IDTR 的信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 2A 卷第 3 章中的“LGDT/LIDT—加载全局/中断描述符表寄存器”。

IA-32 架构兼容性¶

如果不引用高 8 位，则 16 位格式 of SIDT 与 Intel 286 处理器兼容。Intel 286 处理器将这些位填充为 1；Intel 286 处理器之后的处理器代将这些位填充为 0。

操作¶

IF instruction is SIDT
    THEN
        IF OperandSize =16 or OperandSize = 32 (* Legacy or Compatibility Mode *)
            THEN
                DEST[0:15] := IDTR(Limit);
                DEST[16:47] := IDTR(Base); FI; (* Full 32-bit base address stored *)
            ELSE (* 64-bit Mode *)
                DEST[0:15] := IDTR(Limit);
                DEST[16:79] := IDTR(Base); (* Full 64-bit base address stored *)
        FI;
FI;

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果目的地位于不可写的段中。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器用于访问内存且包含 NULL 段选择子。
如果 CR4.UMIP = 1 且 CPL > 0。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在 CPL = 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数的有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 CR4.UMIP = 1。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查并且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#UD 如果使用了 LOCK 前缀。
#GP(0) 如果内存地址为非规范形式。
如果 CR4.UMIP = 1 且 CPL > 0。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在 CPL = 3 时进行了未对齐的内存引用。
