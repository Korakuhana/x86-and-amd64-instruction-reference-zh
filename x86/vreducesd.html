VREDUCESD — 对标量双精度浮点值执行归约转换

操作码/指令 Op / En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.66.0F3A.W1 57 VREDUCESD xmm1 {k1}{z}, xmm2, xmm3/m64{sae}, imm8/r A V/V AVX512DQ 通过减去 imm8 字段指定的若干尾数位，对 xmm3/m64 中的标量双精度浮点值执行归约转换。此外，将 xmm2 的高双精度浮点值 (bits[127:64]) 复制到 xmm1[127:64]。将结果存储在 xmm1 寄存器中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4

描述¶

对第二源操作数（第三操作数）低四字元素中的二进制编码双精度浮点值执行归约转换，并在写掩码 k1 控制下，将归约后的二进制浮点格式结果存储在目的操作数（第一操作数）的低四字元素中。目的操作数的 bits 127:64 从第一源操作数（第二操作数）的对应四字元素复制。

归约转换从二进制浮点源值中减去整数部分和前导 M 个尾数位，其中 M 是由 imm8[7:4] 指定的无符号整数，参见 图 5-28。具体而言，归约转换可以表示为：

dest = src – (ROUND(2M*src))*2-M;

其中 “Round()” 将 “src”、“2M” 以及它们的乘积视为具有规范化有效位数 and 偏置指数的二进制浮点数。

归约结果的幅度可以通过令 src = 2p*man2 来表示，

其中 'man2' 是规范化的有效位数，'p' 是无偏指数。

如果 RC = RNE：0 <=|归约结果| <= 2p-M-1

如果 RC ≠ RNE：0 <=|归约结果| < 2p-M

此指令可能会导致精度异常置位。然而，在设置了 SPE（即精度异常抑制，即 imm8[3]=1）的情况下，将不报告精度异常。

该操作受写掩码控制。

输入值的特殊情况处理列在 表 5-29 中。

操作¶

ReduceArgumentDP(SRC[63:0], imm8[7:0])
{
    // Check for NaN
    IF (SRC [63:0] = NAN) THEN
        RETURN (Convert SRC[63:0] to QNaN); FI;
    M := imm8[7:4]; // Number of fraction bits of the normalized significand to be subtracted
    RC := imm8[1:0];// Round Control for ROUND() operation
    RC source := imm[2];
    SPE := imm[3];// Suppress Precision Exception
    TMP[63:0] := 2-M *{ROUND(2M*SRC[63:0], SPE, RC_source, RC)}; // ROUND() treats SRC and 2M as standard binary FP values
    TMP[63:0] := SRC[63:0] – TMP[63:0]; // subtraction under the same RC,SPE controls
    RETURN TMP[63:0]; // binary encoded FP with biased exponent and normalized significand
}
