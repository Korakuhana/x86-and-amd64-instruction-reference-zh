ETRACK — 激活 EBLOCK 检查

操作码/指令 操作/编码 64/32位模式支持 CPUID功能标志 描述
EAX = 0CH ENCLS[ETRACK] IR V/V SGX1 此叶函数激活 EBLOCK 检查。

指令操作数编码¶

操作/编码 EAX RCX
IR ETRACK (输入) 返回错误代码 (输出) 指向 EPC 页面的 SECS 指针 (输入)

描述¶

此叶函数提供了硬件机制，用于追踪软件是否已成功完成所需的 TLB 地址清除。该指令只能在当前特权级别为 0 时执行。

RCX 的内容是一个 EPC 页面的有效地址。

下表提供了有关 ETRACK 叶函数的内存参数的附加信息。

操作¶

IF (DS:RCX is not 4KByte Aligned)
	THEN #GP(0); FI;
IF (DS:RCX does not resolve within an EPC)
	THEN #PF(DS:RCX); FI;
(* Check concurrency with other Intel SGX instructions *)
IF (Other Intel SGX instructions using tracking facility on this SECS)
	THEN
		IF (<<VMX non-root operation>> AND <<ENABLE_EPC_VIRTUALIZATION_EXTENSIONS>>)
			THEN
				VMCS.Exit_reason := SGX_CONFLICT;
				VMCS.Exit_qualification.code := TRACKING_RESOURCE_CONFLICT;
				VMCS.Exit_qualification.error := 0;
				VMCS.Guest-physical_address := SECS(TMP_SECS).ENCLAVECONTEXT;
				VMCS.Guest-linear_address := 0;
			Deliver VMEXIT;
			ELSE
				#GP(0);
		FI;
FI;
IF (EPCM(DS:RCX). VALID = 0)
	THEN #PF(DS:RCX); FI;
IF (EPCM(DS:RCX).PT ≠ PT_SECS)
	THEN #PF(DS:RCX); FI;
(* All processors must have completed the previous tracking cycle*)
IF ( (DS:RCX).TRACKING ≠ 0) )
	THEN
		IF (<<VMX non-root operation>> AND <<ENABLE_EPC_VIRTUALIZATION_EXTENSIONS>>)
			THEN
				VMCS.Exit_reason := SGX_CONFLICT;
				VMCS.Exit_qualification.code := TRACKING_REFERENCE_CONFLICT;
				VMCS.Exit_qualification.error := 0;
				VMCS.Guest-physical_address := SECS(TMP_SECS).ENCLAVECONTEXT;
				VMCS.Guest-linear_address := 0;
			Deliver VMEXIT;
		FI;
		RFLAGS.ZF := 1;
		RAX := SGX_PREV_TRK_INCMPL;
		GOTO DONE;
	ELSE
		RAX := 0;
		RFLAGS.ZF := 0;
FI;
DONE:
RFLAGS.CF,PF,AF,OF,SF := 0;

影响的标志寄存器¶

如果 SECS 正在使用或无效，则设置 ZF；否则清除。清除 CF、PF、AF、OF、SF。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 DS 段限制。
如果内存操作数未正确对齐。
如果另一个线程正在并发地在此 SECS 上使用追踪设施。
#PF(错误代码) 如果访问内存操作数时发生页面错误。
如果内存操作数不是 EPC 页面。

64位模式异常¶

#GP(0) 如果内存操作数为非规范形式。
如果内存操作数未正确对齐。
如果指定的 EPC 资源正在使用中。
#PF(错误代码) 如果访问内存操作数时发生页面错误。
如果内存操作数不是 EPC 页面。
