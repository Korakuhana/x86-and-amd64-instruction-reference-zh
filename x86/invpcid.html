INVPCID — 使进程上下文标识符失效

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
66 0F 38 82 /r INVPCID r32, m128 RM N.E./V INVPCID 基于 r32 中的失效类型和 m128 中的描述符，使 TLB 和分页结构缓存中的条目失效。
66 0F 38 82 /r INVPCID r64, m128 RM V/N.E. INVPCID 基于 r64 中的失效类型和 m128 中的描述符，使 TLB 和分页结构缓存中的条目失效。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) N/A N/A

描述¶

基于进程上下文标识符 (PCID) 使翻译后备缓冲区 (TLB) 和分页结构缓存中的映射失效。（参见《Intel® 64 和 IA-32 架构软件开发人员手册》卷 3A 第 4.10 节“缓存转换信息”。）失效操作基于寄存器操作数中指定的 INVPCID 类型 and 内存操作数中指定的 INVPCID 描述符。

在 64 位模式之外，寄存器操作数始终为 32 位，无论 CS.D 的值如何。在 64 位模式下，寄存器操作数为 64 位。

当前定义了四种 INVPCID 类型：

  • 单条地址失效：如果 INVPCID 类型为 0，则逻辑处理器使 INVPCID 描述符中指定的线性地址和 PCID 的映射失效（全局翻译除外）。1 在某些情况下，该指令也可能使全局翻译或其他线性地址（或其他 PCID）的映射失效。
  • 单上下文失效：如果 INVPCID 类型为 1，则逻辑处理器使与 INVPCID 描述符中指定的 PCID 相关联的所有映射失效（全局翻译除外）。在某些情况下，该指令也可能使全局翻译或其他 PCID 的映射失效。
  • 全上下文失效（包括全局翻译）：如果 INVPCID 类型为 2，则逻辑处理器使与任何 PCID 相关联的所有映射失效（包括全局翻译）。
  • 全上下文失效：如果 INVPCID 类型为 3，则逻辑处理器使与任何 PCID 相关联的所有映射失效（全局翻译除外）。在某些情况下，该指令也可能使全局翻译失效。

INVPCID 描述符由 128 位组成，并由 PCID 和线性地址组成，如图 3-25 所示。对于 INVPCID 类型为 0，即使在 64 位模式之外，处理器也使用完整的 64 位线性地址；对于其他 INVPCID 类型，不使用线性地址。

1. 如果分页结构使用大于 4 KB 的页映射线性地址，并且该页存在多个 TLB 条目（参见《Intel® 64 和 IA-32 架构软件开发人员手册》卷 3A 第 4.10.2.3 节“TLB 使用的细节”），则该指令将使所有这些条目失效。

如果 CR4.PCIDE = 0，逻辑处理器将不缓存除 000H 之外的任何 PCID 信息。在这种情况下，只有当 INVPCID 描述符中指定的 PCID 为 000H时，才允许执行 INVPCID 类型为 0 和 1 的操作；执行 INVPCID 类型为 2 和 3 的操作仅使 PCID 000H 的映射失效。请注意，在 IA-32e 模式之外，CR4.PCIDE 必须为 0（参见《Intel® 64 和 IA-32 架构软件开发人员手册》卷 3A 第 4.10.1 节“进程上下文标识符 (PCID)”）。

操作¶

INVPCID_TYPE := value of register operand; // must be in the range of 0–3
INVPCID_DESC := value of memory operand;
CASE INVPCID_TYPE OF
    0:
            // individual-address invalidation
        PCID := INVPCID_DESC[11:0];
        L_ADDR := INVPCID_DESC[127:64];
        Invalidate mappings for L_ADDR associated with PCID except global translations;
        BREAK;
    1:
            // single PCID invalidation
        PCID := INVPCID_DESC[11:0];
        Invalidate all mappings associated with PCID except global translations;
        BREAK;
    2:
            // all PCID invalidation including global translations
        Invalidate all mappings for all PCIDs, including global translations;
        BREAK;
    3:
            // all PCID invalidation retaining global translations
        Invalidate all mappings for all PCIDs except global translations;
        BREAK;
ESAC;

SIMD 浮点异常¶

无。

保护模式异常¶

#GP(0) 如果当前特权级不为 0。
如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含不可用的段。
如果源操作数位于仅执行的代码段中。
如果寄存器操作数中指定了无效的类型，即 INVPCID_TYPE > 3。
如果 INVPCID_DESC 的位 63:12 不全为零。
如果 INVPCID_TYPE 为 0 或 1，且 INVPCID_DESC[11:0] 不为零。
如果 INVPCID_TYPE 为 0，且 INVPCID_DESC[127:64] 中的线性地址为非规范形式。
#PF(错误码) 如果访问内存操作数时发生缺页异常。
#SS(0) 如果内存操作数有效地址超出了 SS 段限制。
如果 SS 寄存器包含不可用的段。
#UD 如果 CPUID.(EAX=07H, ECX=0H):EBX.INVPCID[位 10] = 0。
如果使用了 LOCK 前缀。
