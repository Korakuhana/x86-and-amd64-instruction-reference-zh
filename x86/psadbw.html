PSADBW — 计算绝对差值之和

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
NP 0F F6 /r1 PSADBW mm1, mm2/m64 A V/V SSE 计算 mm2/m64 和 mm1 中打包无符号字节整数的绝对差值；然后对这些差值求和以产生无符号字整数结果。
66 0F F6 /r PSADBW xmm1, xmm2/m128 A V/V SSE2 计算 xmm2/m128 和 xmm1 中打包无符号字节整数的绝对差值；然后分别对 8 个低位差值和 8 个高位差值求和，以产生两个无符号字整数结果。
VEX.128.66.0F.WIG F6 /r VPSADBW xmm1, xmm2, xmm3/m128 B V/V AVX 计算 xmm3/m128 和 xmm2 中打包无符号字节整数的绝对差值；然后分别对 8 个低位差值和 8 个高位差值求和，以产生两个无符号字整数结果。
VEX.256.66.0F.WIG F6 /r VPSADBW ymm1, ymm2, ymm3/m256 B V/V AVX2 计算 ymm3/m256 和 ymm2 中打包无符号字节整数的绝对差值；然后对每组连续的 8 个差值分别求和，以产生四个无符号字整数结果。
EVEX.128.66.0F.WIG F6 /r VPSADBW xmm1, xmm2, xmm3/m128 C V/V AVX512VL AVX512BW 计算 xmm3/m128 和 xmm2 中打包无符号字节整数的绝对差值；然后对每组连续 of 8 个差值分别求和，以产生两个无符号字整数结果。
EVEX.256.66.0F.WIG F6 /r VPSADBW ymm1, ymm2, ymm3/m256 C V/V AVX512VL AVX512BW 计算 ymm3/m256 和 ymm2 中打包无符号字节整数的绝对差值；然后对每组连续的 8 个差值分别求和，以产生四个无符号字整数结果。
EVEX.512.66.0F.WIG F6 /r VPSADBW zmm1, zmm2, zmm3/m512 C V/V AVX512BW 计算 zmm3/m512 和 zmm2 中打包无符号字节整数的绝对差值；然后对每组连续的 8 个差值分别求和，以产生八个无符号字整数结果。

1. 请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 2A 卷第 2.5 节“Intel® AVX 和 Intel® SSE 指令异常分类”中的注释，以及《Intel® 64 和 IA-32 架构软件开发人员手册》第 3B 卷第 23.25.3 节“操作 MMX 寄存器的传统 SIMD 指令的异常条件”中的注释。

描述¶

计算源操作数（第二个操作数）与目的操作数（第一个操作数）中 8 个无符号字节整数之差的绝对值。然后对这 8 个差值求和，产生一个无符号字整数结果，并存储在目的操作数中。图 4-14 展示了使用 64 位操作数时 PSADBW 指令的操作。

当配合 64 位操作数工作时，字整数结果存储在目的操作数的低位字中，目的操作数中的其余字节全部清零。

当配合 128 位操作数工作时，会计算两个打包的结果。此时，对源操作数 and 目的操作数的 8 个低位字节进行操作，产生一个字结果并存储在目的操作数的低位字中，对 8 个高位字节进行操作，产生一个字结果并存储在目的操作数的位 64 到 79 中。目的操作数的其余字节均被清零。

对于 256 位版本，第三组的 8 个差值求和产生一个无符号字，存储在目的寄存器的位 [143:128] 中，第四组的 8 个差值求和产生一个无符号字，存储在目的寄存器的位 [207:192] 中。目的寄存器的其余字被设置为 0。

对于 512 位版本，第五组的结果存储在目的寄存器的位 [271:256] 中。第六组的结果存储在位 [335:320] 中。第七组和第八组的结果分别存储在位 [399:384] 和位 [463:447] 中。目的寄存器中的其余位被设置为 0。

在 64 位模式下且未采用 VEX/EVEX 前缀编码时，使用 REX.R 形式的 REX 前缀允许此指令访问其他寄存器（XMM8-XMM15）。

传统 SSE 版本：源操作数可以是 MMX 技术寄存器或 64 位内存位置。目的操作数是 MMX 技术寄存器。

128位传统 SSE 版本：第一个源操作数和目的寄存器是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。对应 ZMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 和 EVEX.128 编码版本：第一个源操作数和目的寄存器是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。对应 ZMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 和 EVEX.256 编码版本：第一个源操作数和目的寄存器是 YMM 寄存器。第二个源操作数是 YMM 寄存器或 256 位内存位置。对应 ZMM 寄存器的位（MAXVL-1:256）清零。

EVEX.512 编码版本：第一个源操作数和目的寄存器是 ZMM 寄存器。第二个源操作数是 ZMM 寄存器或 512 位内存位置。

操作¶

VPSADBW（EVEX 编码版本）¶

VL = 128, 256, 512
TEMP0 := ABS(SRC1[7:0] - SRC2[7:0])
(* Repeat operation for bytes 1 through 15 *)
TEMP15 := ABS(SRC1[127:120] - SRC2[127:120])
DEST[15:0] := SUM(TEMP0:TEMP7)
DEST[63:16] := 000000000000H
DEST[79:64] := SUM(TEMP8:TEMP15)
DEST[127:80] := 00000000000H
IF VL >= 256
    (* Repeat operation for bytes 16 through 31*)
    TEMP31 := ABS(SRC1[255:248] - SRC2[255:248])
    DEST[143:128] := SUM(TEMP16:TEMP23)
    DEST[191:144] := 000000000000H
    DEST[207:192] := SUM(TEMP24:TEMP31)
    DEST[223:208] := 00000000000H
FI;
IF VL >= 512
(* Repeat operation for bytes 32 through 63*)
    TEMP63 := ABS(SRC1[511:504] - SRC2[511:504])
    DEST[271:256] := SUM(TEMP0:TEMP7)
    DEST[319:272] := 000000000000H
    DEST[335:320] := SUM(TEMP8:TEMP15)
    DEST[383:336] := 00000000000H
    DEST[399:384] := SUM(TEMP16:TEMP23)
    DEST[447:400] := 000000000000H
    DEST[463:448] := SUM(TEMP24:TEMP31)
    DEST[511:464] := 00000000000H
FI;
DEST[MAXVL-1:VL] := 0
