UIRET — 用户中断返回

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
F3 0F 01 EC UIRET ZO V/I UINTR 从处理用户中断中返回。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A N/A

描述¶

UIRET 从处理用户中断中返回。无论处于哪个特权级 (CPL)，均可执行此指令。

在事务区域内部执行 UIRET 会导致事务中止；该中止会像执行 IRET 引起的中止一样加载 EAX。

UIRET 可以通过架构分支记录 (LBR)、Intel 处理器跟踪 (Intel PT) 以及性能监视进行跟踪。对于 Intel PT 和 LBR，UIRET 记录的方式与 IRET 完全相同。因此，对于 LBR，UIRET 属于 OTHER_BRANCH 类别，这意味着必须设置 IA32_LBR_CTL.OTHER_BRANCH[bit 22] 来记录用户中断的递交，并且 IA32_LBR_x_INFO.BR_TYPE 字段对于任何记录的用户中断都将指示为 OTHER_BRANCH。对于 Intel PT，必须通过设置 IA32_RTIT_CTL.BranchEn[bit 13] 来启用控制流跟踪。

UIRET 还会递增启用了 BR_INST_RETIRED.FAR_BRANCH 计数的性能计数器。

操作¶

Pop tempRIP;
Pop tempRFLAGS; // 见下文了解如何使用它加载 RFLAGS
Pop tempRSP;
IF tempRIP is not canonical in current paging mode
    THEN #GP(0);
FI;
IF ShadowStackEnabled(CPL)
    THEN
        PopShadowStack SSRIP;
        IF SSRIP ≠ tempRIP
            THEN #CP (FAR-RET/IRET);
        FI;
FI;
RIP := tempRIP;
// 在 RFLAGS 中仅更新 CF、PF、AF、ZF、SF、TF、DF、OF、NT、RF、AC 和 ID
RFLAGS := (RFLAGS & ~254DD5H) | (tempRFLAGS & 254DD5H);
RSP := tempRSP;
UIF := 1;
Clear any cache-line monitoring established by MONITOR or UMONITOR;

受影响的标志位¶

请参阅“操作”部分。

保护模式异常¶

#UD UIRET 指令在保护模式下无法识别。

实地址模式异常¶

#UD UIRET 指令在实地址模式下无法识别。

虚拟-8086 模式异常¶

#UD UIRET 指令在虚拟-8086 模式下无法识别。

兼容模式异常¶

#UD UIRET 指令在兼容模式下无法识别。

64位模式异常¶

#GP(0) 如果返回指令指针是非规范的。
#SS(0) 如果尝试从栈中弹出值导致引用非规范地址。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#CP 如果来自栈与影子栈的返回指令指针不匹配。
#UD 如果使用了 LOCK 前缀。
如果是在安全包围圈 (Enclave) 中执行。
如果 CR4.UINTR = 0。
如果 CPUID.07H.0H:EDX.UINTR[bit 5] = 0。
