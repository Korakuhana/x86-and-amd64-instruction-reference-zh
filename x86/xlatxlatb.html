XLAT/XLATB — 查表转换

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
D7 XLAT m8 ZO Valid Valid 将 AL 设置为内存字节 DS:[(E)BX + 无符号 AL]。
D7 XLATB ZO Valid Valid 将 AL 设置为内存字节 DS:[(E)BX + 无符号 AL]。
REX.W + D7 XLATB ZO Valid N.E. 将 AL 设置为内存字节 [RBX + 无符号 AL]。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

使用 AL 寄存器的内容作为表索引定位内存表中的字节条目，然后将表条目的内容复制回 AL 寄存器。AL 寄存器中的索引被视为无符号整数。XLAT 和 XLATB 指令从 DS:EBX 或 DS:BX 寄存器获取内存中表的基地址（分别取决于指令的地址大小属性，即 32 或 16）。（DS 段可以使用段超越前缀进行覆盖。）

在汇编代码级别，允许使用该指令的两种形式：“显式操作数”形式和“无操作数”形式。显式操作数形式（通过助记符 XLAT 指定）允许使用符号显式指定表的基地址。提供这种显式操作数形式是为了便于编写文档；然而，请注意，此形式提供的文档可能会引起误解。也就是说，该符号不必指定正确的基地址。基地址始终由 DS:(E)BX 寄存器指定，在执行 XLAT 指令之前必须正确加载这些寄存器。

无操作数形式 (XLATB) 提供了 XLAT 指令的“短形式”。在这里，处理器同样假设 DS:(E)BX 寄存器包含表的基地址。

在 64 位模式下，操作类似于传统或兼容模式。AL 用于指定表索引（操作数大小固定为 8 位）。然而，RBX 用于指定表的基地址。有关编码数据和限制，请参阅本节开头的汇总图表。

操作¶

IF AddressSize = 16
    THEN
        AL := (DS:BX + ZeroExtend(AL));
    ELSE IF (AddressSize = 32)
        AL := (DS:EBX + ZeroExtend(AL)); FI;
    ELSE (AddressSize = 64)
        AL := (RBX + ZeroExtend(AL));
FI;

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含一个空段选择子。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果内存地址为非规范形式。
#PF(错误码) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。
