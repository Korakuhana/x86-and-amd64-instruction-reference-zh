RDTSCP — 读取时间戳计数器和处理器 ID

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 01 F9 RDTSCP ZO Valid Valid 将 64 位时间戳计数器和 IA32_TSC_AUX 的值读取到 EDX:EAX 和 ECX 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将处理器时间戳计数器（一个 64 位 MSR）的当前值读取到 EDX:EAX 寄存器中，同时将 IA32_TSC_AUX MSR（地址 C0000103H）的值读取到 ECX 寄存器中。EDX 寄存器加载 IA32_TSC MSR 的高 32 位；EAX 寄存器加载 IA32_TSC MSR 的低 32 位；ECX 寄存器加载 IA32_TSC_AUX MSR 的低 32 位。在支持 Intel 64 架构的处理器上，RAX、RDX 和 RCX 各自的高 32 位均被清零。

处理器在每个时钟周期单调递增时间戳计数器 MSR，并在处理器复位时将其重置为 0。有关时间戳计数器行为的特定细节，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3B 卷第 18 章中的“时间戳计数器”。

寄存器 CR4 中的时间戳禁用 (TSD) 标志限制 RDTSCP 指令的使用，如下所述。当该标志清除时，RDTSCP 指令可以在任何特权级执行；当设置该标志时，该指令只能在特权级 0 执行。

RDTSCP 指令不是序列化指令，但它确实会等待，直到所有先前的指令都已执行且所有先前的加载在全局可见。1 但它不等待先前的存储在全局可见，并且随后的指令可能会在执行读取操作之前开始执行。以下项可以指导寻求控制 RDTSCP 执行顺序的软件：

  • 如果软件要求仅在所有先前的存储在全局可见之后才执行 RDTSCP，它可以在 RDTSCP 之前立即执行 MFENCE。
  • 如果软件要求在执行任何随后的指令（包括任何 memory 访问）之前执行 RDTSCP，它可以在 RDTSCP 之后立即执行 LFENCE。

有关此指令在 VMX 非根操作中行为的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3C 卷第 26 章中的“VMX 非根操作中指令行为的变化”。

1. 当确定要加载的值时，加载被认为在全局可见。

操作¶

IF (CR4.TSD = 0) or (CPL = 0) or (CR0.PE = 0)
    THEN
        EDX:EAX := TimeStampCounter;
        ECX := IA32_TSC_AUX[31:0];
    ELSE (* CR4.TSD = 1 且 (CPL = 1、2 或 3) 且 CR0.PE = 1 *)
        #GP(0);
FI;

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果寄存器 CR4 中的 TSD 标志被设置且 CPL 大于 0。
#UD 如果使用了 LOCK 前缀。
如果 CPUID.80000001H:EDX.RDTSCP[位 27] = 0。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。
如果 CPUID.80000001H:EDX.RDTSCP[位 27] = 0。

虚拟-8086 模式异常¶

#GP(0) 如果寄存器 CR4 中的 TSD 标志被设置。
#UD 如果使用了 LOCK 前缀。
如果 CPUID.80000001H:EDX.RDTSCP[位 27] = 0。
