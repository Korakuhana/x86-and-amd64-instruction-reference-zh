MASKMOVDQU — 存储双四字的选定字节

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
66 0F F7 /r MASKMOVDQU xmm1, xmm2 RM 有效/有效 SSE2 使用 xmm2 中的字节掩码，选择性地将字节从 xmm1 写入内存位置。默认内存位置由 DS:DI/EDI/RDI 指定。
VEX.128.66.0F.WIG F7 /r VMASKMOVDQU xmm1, xmm2 RM 有效/有效 AVX 使用 xmm2 中的字节掩码，选择性地将字节从 xmm1 写入内存位置。默认内存位置由 DS:DI/EDI/RDI 指定。

1. 需要 ModRM.MOD = 011B。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) N/A N/A

描述¶

将源操作数（第一个操作数）中的选定字节存储到 128 位内存位置中。掩码操作数（第二个操作数）选择将源操作数中的哪些字节写入内存。源和掩码操作数是 XMM 寄存器。内存位置由 DI/EDI/RDI 寄存器中的有效地址指定（默认段寄存器是 DS，但可以使用段超越前缀进行覆盖）。内存位置不需要对齐在自然边界上。（存储地址的大小取决于地址大小属性。）

掩码操作数中每个字节的最高有效位决定是否将源操作数中的对应字节写入内存中的对应字节位置：0 表示不写入，1 表示写入。

MASKMOVDQU 指令向处理器生成非临时提示，以最大限度地减少缓存污染。非临时提示是通过使用写合并 (WC) 内存类型协议来实现的（参见《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 10 章中的“临时与非临时数据的缓存”）。由于 WC 协议使用弱有序的内存一致性模型，如果多个处理器可能会使用不同的内存类型来读写目的内存位置，则应该将使用 SFENCE 或 MFENCE 指令实现的屏障操作与 MASKMOVDQU 指令结合使用。

掩码全为 0 时的行为如下：

  • 没有数据会被写入内存。
  • 不保证断点（代码或数据）的触发；不同的处理器实现可能会触发或不触发这些断点。
  • 与寻址内存相关的异常和缺页异常可能仍会触发（取决于具体实现）。
  • 如果目的内存区域映射为 UC 或 WP，则不保证强制执行这些内存类型的相关语义（即保留），且与具体实现相关。

MASKMOVDQU 指令可用于提高需要按字节合并数据的算法的性能。MASKMOVDQU 不应导致所有权读取；这样做会产生不必要的带宽，因为数据是使用字节掩码直接写入的，而在存储前无需分配旧数据。

在 64 位模式下，使用 REX.R 前缀允许此指令访问额外的寄存器 (XMM8-XMM15)。

注意：在 VEX 编码版本中，VEX.vvvv 是保留的，必须为 1111b，否则指令将产生 #UD 异常。

如果 VMASKMOVDQU 用 VEX.L= 1 编码，尝试执行用 VEX.L= 1 编码的指令将导致 #UD 异常。

操作¶

IF (MASK[7] = 1)
    THEN DEST[DI/EDI] := SRC[7:0] ELSE (* Memory location unchanged *); FI;
IF (MASK[15] = 1)
    THEN DEST[DI/EDI +1] := SRC[15:8] ELSE (* Memory location unchanged *); FI;
    (* Repeat operation for 3rd through 14th bytes in source operand *)
IF (MASK[127] = 1)
    THEN DEST[DI/EDI +15] := SRC[127:120] ELSE (* Memory location unchanged *); FI;
