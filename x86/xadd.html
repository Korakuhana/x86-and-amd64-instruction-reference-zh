XADD — 交换并相加

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F C0 /r XADD r/m8, r8 MR Valid Valid 交换 r8 和 r/m8；将和加载到 r/m8 中。
REX + 0F C0 /r XADD r/m8*, r8* MR Valid N.E. 交换 r8 和 r/m8；将和加载到 r/m8 中。
0F C1 /r XADD r/m16, r16 MR Valid Valid 交换 r16 和 r/m16；将和加载到 r/m16 中。
0F C1 /r XADD r/m32, r32 MR Valid Valid 交换 r32 和 r/m32；将和加载到 r/m32 中。
REX.W + 0F C1 /r XADD r/m64, r64 MR Valid N.E. 交换 r64 和 r/m64；将和加载到 r/m64 中。

* 在 64 位模式下，如果使用 REX 前缀，则无法将 r/m8 编码为访问以下字节寄存器：AH、BH、CH、DH。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
MR ModRM:r/m (r, w) ModRM:reg (r, w) N/A N/A

描述¶

将第一个操作数（目的操作数）与第二个操作数（源操作数）进行交换，然后将这两个值的和加载到目的操作数中。目的操作数可以是寄存器或内存位置；源操作数是寄存器。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 形式的 REX 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 形式的 REX 前缀可将操作提升至 64 位。请参阅本节开头的汇总表以获取编码数据和限制。

该指令可以与 LOCK 前缀配合使用，以允许指令原子地执行。

IA-32 架构兼容性¶

早于 Intel486 处理器的 IA-32 处理器无法识别此指令。如果使用此指令，您应该提供在早期处理器上运行的等效代码序列。

操作¶

TEMP := SRC + DEST;
SRC := DEST;
DEST := TEMP;

受影响的标志位¶

CF, PF, AF, SF, ZF 和 OF 标志根据加法的结果进行设置，该结果存储在目的操作数中。

保护模式异常¶

#GP(0) 如果目的地位于不可写的段中。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀，但目的地不是内存操作数。
