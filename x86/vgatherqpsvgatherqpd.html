描述¶

汇集由基址 BASE_ADDR 和带有比例因子 SCALE 的索引向量 V_INDEX 指向的一组最多 8 个单精度/双精度浮点内存位置。结果写入向量寄存器。元素通过 VSIB 指定（即索引寄存器是包含打包索引的向量寄存器）。只有当元素的相应掩码位为 1 时，才会加载该元素。如果元素的掩码位未设置，则目的寄存器的相应元素保持不变。除非触发异常，否则此指令会将整个掩码寄存器清零。

如果至少有一个元素已经被汇集（即异常是由最右边已设置掩码位之外的元素触发的），则该指令可以因异常而挂起。发生这种情况时，目的寄存器和掩码寄存器 (k1) 会被部分更新；那些已被汇集的元素被放入目的寄存器中，并将其掩码位清零。如果已汇集的元素有任何挂起的陷阱或中断，它们将代替异常被传递；在这种情况下，EFLAG.RF 被设置为 1，以便在指令继续执行时不会重新触发指令断点。

如果数据元素大小小于索引元素大小，则目的寄存器的高位部分和掩码寄存器不对应于正在汇集的任何元素。该指令将这些高位部分清零。即使指令触发异常，甚至在汇集任何元素之前指令就已经触发了异常，它也可能会更新这些未使用的元素到这两个寄存器中的一个或两个。

请注意：

  • 可以按任何顺序从内存中读取 these 值。与其他指令的内存排序遵循 Intel-64 内存排序模型。
  • 故障按从右到左的方式传递。也就是说，如果某个元素触发了故障并传递，则所有靠近目的 zmm 寄存器 LSB 的元素都将完成（且无故障）。靠近 MSB 的单个元素可能完成，也可能未完成。如果给定元素触发了多个故障，它们将按常规顺序传递。
  • 可以按任何顺序汇集元素，但故障必须按从右到左的顺序传递；因此，在传递故障之前，可能会汇集故障元素左侧的元素。此指令的特定实现是可重复的——在给定相同输入值和架构状态的情况下，将汇集故障元素左侧的同一组元素。
  • 此指令不执行 AC 检查，因此永远不会传递 AC 故障。
  • 对于 16 位有效地址无效。将导致 #UD 故障。

请注意，在此指令中强制使用 VSIB 字节。因此，如果 ModRM.rm 不等于 100b，指令将产生 #UD 故障。

此指令有特殊的 disp8*N 和对齐规则。N 被视为单个向量元素的大小。

表示比例索引所需的位数可能多于处理器使用的地址位数（例如，在 32 位模式下，如果比例因子大于 1）。在这种情况下，超出地址位数的最高有效位将被忽略。

如果目的向量 zmm1 与索引向量 VINDEX 相同，指令将产生 #UD 故障。如果指定了 k0 掩码寄存器，指令也将产生 #UD 故障。

操作¶

BASE_ADDR stands for the memory operand base address (a GPR); may not exist
VINDEX stands for the memory operand vector of indices (a ZMM register)
SCALE stands for the memory operand scalar (1, 2, 4 or 8)
DISP is the optional 1 or 4 byte displacement

VGATHERQPS (EVEX 编码版本)¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 32
    k := j * 64
    IF k1[j] OR *no writemask*
        THEN DEST[i+31:i] :=
            MEM[BASE_ADDR + (VINDEX[k+63:k]) * SCALE + DISP]
            k1[j] := 0
        ELSE *DEST[i+31:i] := remains unchanged*
    FI;
ENDFOR
k1[MAX_KL-1:KL] := 0
DEST[MAXVL-1:VL/2] := 0

VGATHERQPD (EVEX 编码版本)¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no writemask*
        THEN DEST[i+63:i] := MEM[BASE_ADDR + (VINDEX[i+63:i]) * SCALE + DISP]
            k1[j] := 0
        ELSE *DEST[i+63:i] := remains unchanged*
    FI;
ENDFOR
k1[MAX_KL-1:KL] := 0
DEST[MAXVL-1:VL] := 0
