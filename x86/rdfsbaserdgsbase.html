RDFSBASE/RDGSBASE — 读取 FS/GS 段基地址

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
F3 0F AE /0 RDFSBASE r32 M V/I FSGSBASE 用 FS 基地址加载 32 位目的寄存器。
F3 REX.W 0F AE /0 RDFSBASE r64 M V/I FSGSBASE 用 FS 基地址加载 64 位目的寄存器。
F3 0F AE /1 RDGSBASE r32 M V/I FSGSBASE 用 GS 基地址加载 32 位目的寄存器。
F3 REX.W 0F AE /1 RDGSBASE r64 M V/I FSGSBASE 用 GS 基地址加载 64 位目的寄存器。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (w) N/A N/A N/A

描述¶

用 FS 或 GS 段基地址加载 ModR/M:r/m 字段指示的通用寄存器。

目的操作数可以是 32 位或 64 位通用寄存器。REX.W 前缀指示操作数大小为 64 位。如果不使用 REX.W 前缀，则操作数大小为 32 位；源基地址（FS 或 GS）的高 32 位被忽略，目的寄存器的高 32 位被清零。

此指令仅在 64 位模式下受支持。

操作¶

DEST := FS/GS segment base address;

受影响的标志位¶

无。

C/C++ 编译器等价内部函数¶

RDFSBASE unsigned int _readfsbase_u32(void );
RDFSBASE unsigned __int64 _readfsbase_u64(void );
RDGSBASE unsigned int _readgsbase_u32(void );
RDGSBASE unsigned __int64 _readgsbase_u64(void );

保护模式异常¶

#UD RDFSBASE 和 RDGSBASE 指令在保护模式下不被识别。

实地址模式异常¶

#UD RDFSBASE 和 RDGSBASE 指令在实地址模式下不被识别。

虚拟-8086 模式异常¶

#UD RDFSBASE 和 RDGSBASE 指令在虚拟-8086 模式下不被识别。

兼容模式异常¶

#UD RDFSBASE 和 RDGSBASE 指令在兼容模式下不被识别。

64位模式异常¶

#UD 如果使用了 LOCK 前缀。
如果 CR4.FSGSBASE[bit 16] = 0。
如果 CPUID.07H.0H:EBX.FSGSBASE[bit 0] = 0。
