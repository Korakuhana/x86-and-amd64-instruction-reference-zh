ENCLU — 执行指定叶函数编号的 Enclave 用户功能

操作码/指令 操作/编码 64/32位模式支持 CPUID功能标志 描述
NP 0F 01 D7 ENCLU ZO V/V 无 此指令用于执行非特权级 Intel SGX 叶函数。

指令操作数编码¶

操作/编码 操作数 1 操作数 2 操作数 3 隐式寄存器操作数
ZO 无 无 无 参见第 38.4 节

描述¶

ENCLU 指令调用指定的非特权级 Intel SGX 叶函数。软件通过在寄存器 EAX 中设置合适的值作为输入来指定叶函数。寄存器 RBX, RCX 和 RDX 具有特定于叶函数的用途，可以作为输入、作为输出，或者不被使用。在 64 位模式下，该指令忽略 RAX 寄存器的高 32 位。

ENCLU 指令产生一个无效操作码异常 (#UD) 如果 CR0.PE = 0 或 RFLAGS.VM = 1，或者指令在系统管理模式 (SMM) 下执行。此外，当 CPL < 3 时尝试执行该指令会导致 #UD。如果 CR0.PG 或 CR0.NE 为 0，或者尝试调用未定义的叶函数，该指令会产生通用保护异常 (#GP)。如果 CR0.TS = 1，ENCLU 指令会产生设备不可用异常 (#NM)。

在 64 位模式之外 (IA32_EFER.LMA = 0 || CS.L = 0)，地址和操作数是 32 位的；在 64 位模式下 (IA32_EFER.LMA = 1 || CS.L = 1)，地址和操作数是 64 位的。CS.D 的值对地址计算没有影响。DS 段被用于创建线性地址。

段覆盖前缀和地址大小覆盖前缀被忽略，在 64 位模式下 REX 前缀也被忽略。

操作¶

IN_64BIT_MODE := 0;
IF TSX_ACTIVE
        THEN GOTO TSX_ABORT_PROCESSING; FI;
(* If enclosing app has CET indirect branch tracking enabled then if it is not ERESUME leaf cause a #CP fault *)
(* If the ERESUME is not successful it will leave tracker in WAIT_FOR_ENDBRANCH *)
TRACKER = (CPL == 3) ? IA32_U_CET.TRACKER : IA32_S_CET.TRACKER
IF EndbranchEnabledAndNotSuppressed(CPL) and TRACKER = WAIT_FOR_ENDBRANCH and
    (EAX != ERESUME or CR0.TS or (in SMM) or (CPUID.SGX_LEAF.0:EAX.SE1 = 0) or (CPL < 3))
        THEN
            Handle CET State machine violation (* see Section 17.3.6, “Legacy Compatibility Treatment,” in the
                Intel® 64 and IA-32 Architectures Software Developer’s Manual, Volume 1. *)
FI;
IF CR0.PE= 0 or RFLAGS.VM = 1 or in SMM or CPUID.SGX_LEAF.0:EAX.SE1 = 0
        THEN #UD; FI;
IF CR0.TS = 1
        THEN #NM; FI;
IF CPL < 3
        THEN #UD; FI;
IF IA32_FEATURE_CONTROL.LOCK = 0 or IA32_FEATURE_CONTROL.SGX_ENABLE = 0
        THEN #GP(0); FI;
IF EAX is invalid leaf number
        THEN #GP(0); FI;
IF CR0.PG = 0 or CR0.NE = 0
        THEN #GP(0); FI;
IN_64BIT_MODE := IA32_EFER.LMA AND CS.L ? 1 : 0;
(* Check not in 16-bit mode and DS is not a 16-bit segment *)
IF not in 64-bit mode and CS.D = 0
        THEN #GP(0); FI;
IF CR_ENCLAVE_MODE = 1 and (EAX = 2 or EAX = 3) (* EENTER or ERESUME *)
        THEN #GP(0); FI;
IF CR_ENCLAVE_MODE = 0 and (EAX = 0 or EAX = 1 or EAX = 4 or EAX = 5 or EAX = 6 or EAX = 7 or EAX = 9)
(* EREPORT, EGETKEY, EEXIT, EACCEPT, EMODPE, EACCEPTCOPY, or EDECCSSA *)
        THEN #GP(0); FI;
Jump to leaf specific flow

影响的标志寄存器¶

参见各个具体的叶函数

保护模式异常¶

#UD 如果使用了 LOCK/66H/REP/VEX 前缀中的任何一个。
如果当前特权级别不为 3。
如果 CPUID.(EAX=12H,ECX=0):EAX.SGX1 [位 0] = 0。
如果逻辑处理器处于 SMM 模式。
#GP(0) 如果 IA32_FEATURE_CONTROL.LOCK = 0。
如果 IA32_FEATURE_CONTROL.SGX_ENABLE = 0。
如果 EAX 中的输入值编码了不受支持的叶函数。
如果 EAX 中的输入值编码了 EENTER/ERESUME 且 ENCLAVE_MODE = 1。
如果 EAX 中的输入值编码了 EGETKEY/EREPORT/EEXIT/EACCEPT/EACCEPTCOPY/EMODPE 且 ENCLAVE_MODE = 0。
如果运行在 16 位模式下。
如果数据段处于 16 位模式下。
如果 CR0.PG = 0 或 CR0.NE = 0。
#NM 如果 CR0.TS = 1。

实地址模式异常¶

#UD ENCLS 在实模式下不被识别（应为 ENCLU 在实模式下不被识别，但为完全符合原版说明，在此与原版一致）。

虚拟-8086 模式异常¶

#UD ENCLS 在虚拟-8086 模式下不被识别（应为 ENCLU，与原版保持一致）。

兼容模式异常¶

与保护模式下的异常相同。
