描述¶

对源操作数（第二个操作数）和目的操作数（第一个操作数）的打包整数进行 SIMD 加法，并将打包整数结果存储在目的操作数中。有关 SIMD 操作的图示，请参阅 Intel® 64 和 IA-32 架构软件开发人员手册第 1 卷中的图 9-4。溢出通过回绕处理，如下文所述。

PADDB 和 VPADDB 指令将来自第一个源操作数和第二个源操作数的打包字节整数相加，并将打包整数结果存储在目的操作数中。当单个结果太大而无法用 8 位表示（溢出）时，结果将回绕，并且低 8 位写入目的操作数（即忽略进位）。

PADDW 和 VPADDW 指令将来自第一个源操作数和第二个源操作数的打包字整数相加，并将打包整数结果存储在目的操作数中。当单个结果太大而无法用 16 位表示（溢出）时，结果将回绕，并且低 16 位写入目的操作数（即忽略进位）。

PADDD 和 VPADDD 指令将来自第一个源操作数和第二个源操作数的打包双字整数相加，并将打包整数结果存储在目的操作数中。当单个结果太大而无法用 32 位表示（溢出）时，结果将回绕，并且低 32 位写入目的操作数（即忽略进位）。

PADDQ 和 VPADDQ 指令将来自第一个源操作数和第二个源操作数的打包四字整数相加，并将打包整数结果存储在目的操作数中。当四字结果太大而无法用 64 位表示（溢出）时，结果将回绕，并且低 64 位写入目的操作数（即忽略进位）。

注意，(V)PADDB、(V)PADDW、(V)PADDD 和 (V)PADDQ 指令可以对无符号或有符号（补码表示法）打包整数进行操作；然而，它不会在 EFLAGS 寄存器中设置位来指示溢出 and/or 进位。为了防止未检测到的溢出情况，软件必须控制所操作值的范围。

EVEX 编码的 VPADDD/Q：第一个源操作数是 ZMM/YMM/XMM 寄存器。第二个源操作数是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置或从 32/64 位内存位置广播的 512/256/128 位向量。目的操作数是根据写掩码更新的 ZMM/YMM/XMM 寄存器。

EVEX 编码的 VPADDB/W：第一个源操作数是 ZMM/YMM/XMM 寄存器。第二个源操作数是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置。目的操作数是根据写掩码更新的 ZMM/YMM/XMM 寄存器。

VEX.256 编码版本：第一个源操作数是 YMM 寄存器。第二个源操作数是 YMM 寄存器或 256 位内存位置。目的操作数是 YMM 寄存器。目的操作数的高位（MAXVL-1:256）清零。

VEX.128 编码版本：第一个源操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。目的操作数是 XMM 寄存器。对应 ZMM 寄存器目的操作数的高位（MAXVL-1:128）清零。

128位传统 SSE 版本：第一个源操作数是 XMM 寄存器。第二个操作数可以是 XMM 寄存器或 128 位内存位置。目的操作数与第一个源 XMM 寄存器相同，且对应 ZMM 寄存器目的操作数的高位（MAXVL-1:128）保持不变。

VPADDB（EVEX 编码版本）¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
FOR j := 0 TO KL-1
    i := j * 8
    IF k1[j] OR *no writemask*
        THEN DEST[i+7:i] := SRC1[i+7:i] + SRC2[i+7:i]
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+7:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+7:i] = 0
            FI
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0
