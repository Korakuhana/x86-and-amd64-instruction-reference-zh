GETSEC[CAPABILITIES] — 报告 SMX 功能

操作码 指令 描述
NP 0F 37 (EAX = 0) GETSEC[CAPABILITIES] 报告 SMX 功能。功能索引在 EBX 中输入，结果在 EAX 中返回。

描述¶

GETSEC[CAPABILITIES] 函数返回受支持的 GETSEC 叶函数的位向量。在入口处将 EAX 设置为 0，以选择 GETSEC 的 CAPABILITIES 叶。EBX 用作在 EAX 中返回位向量字段的选择器。GETSEC[CAPABILITIES] 可以在所有特权级别下执行，但必须设置 CR4.SMXE 位，否则将返回未定义操作码异常 (#UD)。

在执行 GETSEC[CAPABILITIES] 时，如果 EBX = 0，则 EAX 返回一个位向量，表示支持 Intel® TXT 的芯片组的存在状态以及前 30 个可用的 GETSEC 叶函数。返回的位向量格式在表 7-3 中提供。

如果第 0 位被设置为 1，则表示处理器采样到了支持 Intel® TXT 的芯片组存在。如果设置了 1-30 范围内的位，则对应的 GETSEC 叶函数可用。如果给定位索引处的位值为 0，则该索引对应的 GETSEC 叶函数不受支持，尝试执行会导致 #UD。

EAX 的第 31 位指示是否支持更多的叶索引。如果 Extended Leafs（扩展叶）第 31 位被设置，则可以通过重复执行 GETSEC[CAPABILITIES] 并将 EBX 递增 1 来访问其他叶函数。当 EAX 的最高有效位未设置时，则不支持其他 GETSEC 叶函数；将 EBX 索引为更高的值会导致 EAX 返回零。

操作¶

IF (CR4.SMXE=0)
    THEN #UD;
ELSIF (in VMX non-root operation)
    THEN VM Exit (reason=”GETSEC instruction”);
IF (EBX=0) THEN
        BitVector := 0;
        IF (TXT chipset present)
            BitVector[Chipset present] := 1;
        IF (ENTERACCS Available)
            THEN BitVector[ENTERACCS] := 1;
        IF (EXITAC Available)
            THEN BitVector[EXITAC] := 1;
        IF (SENTER Available)
            THEN BitVector[SENTER] := 1;
        IF (SEXIT Available)
            THEN BitVector[SEXIT] := 1;
        IF (PARAMETERS Available)
            THEN BitVector[PARAMETERS] := 1;
        IF (SMCTRL Available)
            THEN BitVector[SMCTRL] := 1;
        IF (WAKEUP Available)
            THEN BitVector[WAKEUP] := 1;
        EAX := BitVector;
ELSE
    EAX := 0;
END;;

受影响的标志位¶

无。

前缀的使用¶

LOCK 前缀导致 #UD。

REP* 前缀导致 #UD（包括 REPNE/REPNZ 和 REP/REPE/REPZ）。

操作数大小前缀导致 #UD。

不允许 66/F2/F3 前缀。

段超越前缀被忽略。

地址大小前缀被忽略。

REX 前缀被忽略。

保护模式异常¶

#UD 如果 CR4.SMXE = 0。

实地址模式异常¶

#UD 如果 CR4.SMXE = 0。

虚拟-8086 模式异常¶

#UD 如果 CR4.SMXE = 0。

兼容模式异常¶

#UD 如果 CR4.SMXE = 0。

64位模式异常¶

#UD 如果 CR4.SMXE = 0。

VM-exit 条件¶

原因 (GETSEC) 如果处于 VMX 非根操作中。
