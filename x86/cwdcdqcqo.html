CWD/CDQ/CQO — 字转换为双字/双字转换为四字

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
99 CWD ZO Valid Valid DX:AX := AX 的符号扩展。
99 CDQ ZO Valid Valid EDX:EAX := EAX 的符号扩展。
REX.W + 99 CQO ZO Valid N.E. RDX:RAX := RAX 的符号扩展。

指令操作数编码 ¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述 ¶

通过符号扩展将寄存器 AX、EAX 或 RAX 中的操作数大小翻倍（取决于操作数大小），并分别将结果存储在寄存器对 DX:AX、EDX:EAX 或 RDX:RAX 中。CWD 指令将 AX 寄存器中值的符号位（第 15 位）复制到 DX 寄存器的每一个位中。CDQ 指令将 EAX 寄存器中值的符号位（第 31 位）复制到 EDX 寄存器的每一个位中。CQO 指令（仅在 64 位模式下可用）将 RAX 寄存器中值的符号位（第 63 位）复制到 RDX 寄存器的每一个位中。

CWD 指令可用于在字除法之前从字产生双字被除数。CDQ 指令可用于在双字除法之前从双字产生四字被除数。CQO 指令可用于在四字除法之前从四字产生双四字被除数。

CWD 和 CDQ 助记符指向相同的操作码。CWD 指令旨在在操作数大小属性为 16 时使用，CDQ 指令旨在在操作数大小属性为 32 时使用。某些汇编器在使用 CWD 时可能会强制操作数大小为 16，而在使用 CDQ 时可能会强制操作数大小为 32。其他汇编器可能将这些助记符视为同义词（CWD/CDQ），并使用操作数大小属性的当前设置来确定要转换的值的大小，而不管使用的是哪个助记符。

在 64 位模式下，使用 REX.W 前缀可将操作提升至 64 位。CQO 助记符与 CWD/CDQ 指向相同的操作码。请参阅本节开头的汇总图表以获取编码数据和限制。

操作 ¶ 

IF OperandSize = 16 (* CWD instruction *)
    THEN
        DX := SignExtend(AX);
    ELSE IF OperandSize = 32 (* CDQ instruction *)
        EDX := SignExtend(EAX); FI;
    ELSE IF 64-Bit Mode and OperandSize = 64 (* CQO instruction*)
        RDX := SignExtend(RAX); FI;
FI;

受影响的标志位 ¶

无。

异常（所有运行模式） ¶

#UD 如果使用了 LOCK 前缀。
