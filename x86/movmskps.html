MOVMSKPS — 提取打包单精度浮点符号掩码

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
NP 0F 50 /r MOVMSKPS reg, xmm RM 有效/有效 SSE 从 xmm 中提取 4 位符号掩码并存储在 reg 中。r32 或 r64 的高位填充为零。
VEX.128.0F.WIG 50 /r VMOVMSKPS reg, xmm2 RM 有效/有效 AVX 从 xmm2 中提取 4 位符号掩码并存储在 reg 中。r32 或 r64 的高位被清零。
VEX.256.0F.WIG 50 /r VMOVMSKPS reg, ymm2 RM 有效/有效 AVX 从 ymm2 中提取 8 位符号掩码并存储在 reg 中。r32 或 r64 的高位被清零。

1. 需要 ModRM.MOD = 011B。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

从源操作数（第二个操作数）中的打包单精度浮点值中提取符号位，将其格式化为 4 位或 8 位掩码，并将该掩码存储在目的操作数（第一个操作数）中。源操作数是 XMM 或 YMM 寄存器，目的操作数是通用寄存器。该掩码存储在目的操作数的低 4 位或 8 位中。目的操作数掩码之外的高位填充为零。

在 64 位模式下，当与 REX.R 前缀一起使用时，该指令可以访问其他寄存器 (XMM8-XMM15, R8-R15)。在 64 位模式下，默认操作数大小为 64 位。

128 位版本：源操作数是 YMM（注：此处原文应为 XMM 寄存器，但遵循原文） 寄存器。目的操作数是通用寄存器。

VEX.256 编码版本：源操作数是 YMM 寄存器。目的操作数是通用寄存器。

注意：在 VEX 编码版本中，VEX.vvvv 是保留的，必须为 1111b，否则指令将产生 #UD 异常。

操作¶

DEST[0] := SRC[31];
DEST[1] := SRC[63];
DEST[2] := SRC[95];
DEST[3] := SRC[127];
IF DEST = r32
    THEN DEST[31:4] := ZeroExtend;
    ELSE DEST[63:4] := ZeroExtend;
FI;

(V)MOVMSKPS (128-bit version)¶

DEST[0] := SRC[31]
DEST[1] := SRC[63]
DEST[2] := SRC[95]
DEST[3] := SRC[127]
IF DEST = r32
    THEN DEST[31:4] := 0;
    ELSE DEST[63:4] := 0;
FI

VMOVMSKPS (VEX.256 encoded version)¶

DEST[0] := SRC[31]
DEST[1] := SRC[63]
DEST[2] := SRC[95]
DEST[3] := SRC[127]
DEST[4] := SRC[159]
DEST[5] := SRC[191]
DEST[6] := SRC[223]
DEST[7] := SRC[255]
IF DEST = r32
    THEN DEST[31:8] := 0;
    ELSE DEST[63:8] := 0;
FI

SIMD 浮点异常¶

无。
