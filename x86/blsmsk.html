BLSMSK — 获取掩码至最低设置位

操作码/指令 Op/En 64/32位模式 CPUID 功能标志 描述
VEX.LZ.0F38.W0 F3 /2 BLSMSK r32, r/m32 VM V/V BMI1 从第 0 位到 r/m32 中的最低设置位，将 r32 中的所有低位设置为“1”。
VEX.LZ.0F38.W1 F3 /2 BLSMSK r64, r/m64 VM V/N.E. BMI1 从第 0 位到 r/m64 中的最低设置位，将 r64 中的所有低位设置为“1”。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
VM VEX.vvvv (w) ModRM:r/m (r) N/A N/A

描述¶

将目的操作数的所有低位设置为“1”，直到并包括源操作数中的最低设置位（=1）。如果源操作数为零，则 BLSMSK 将目的操作数的所有位设置为 1，并将 CF 标志设置为 1。

在实模式和虚拟 8086 模式下不支持此指令。如果不在 64 位模式下，操作数大小始终为 32 位。在 64 位模式下，操作数大小为 64 位时需要 VEX.W1。在非 64 位模式下，VEX.W1 被忽略。尝试在 VEX.L 不等于 0 的情况下执行此指令将导致 #UD。

操作¶

temp := (SRC-1) XOR (SRC) ;
SF := temp[OperandSize -1];
ZF := 0;
IF SRC = 0
    CF := 1;
ELSE
    CF := 0;
FI
DEST := temp;

受影响的标志位¶

SF 根据结果更新。如果源操作数为零，则设置 CF。ZF 和 OF 标志位被清除。AF 和 PF 标志位是未定义的。

BLSMSK unsigned __int32 _blsmsk_u32(unsigned __int32 src);

BLSMSK unsigned __int64 _blsmsk_u64(unsigned __int64 src);
