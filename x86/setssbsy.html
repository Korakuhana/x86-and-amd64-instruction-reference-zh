SETSSBSY — 将影子栈标记为忙

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
F3 0F 01 E8 SETSSBSY ZO V/V CET_SS 在由 IA32_PL0_SSP 引用的主管影子栈令牌中设置 busy（忙）标志。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

SETSSBSY 指令验证在 IA32_PL0_SSP MSR 地址处存在非忙的主管影子栈令牌，并将其标记为忙。成功执行该指令后，SSP 被设置为 IA32_PL0_SSP MSR 的值。

操作¶

IF (CR4.CET = 0)
    THEN #UD; FI;
IF (IA32_S_CET.SH_STK_EN = 0)
    THEN #UD; FI;
IF CPL > 0
    THEN GP(0); FI;
SSP_LA = IA32_PL0_SSP
If SSP_LA not aligned to 8 bytes
    THEN #GP(0); FI;
expected_token_value = SSP_LA
new_token_value = SSP_LA | BUSY_BIT
IF shadow_stack_lock_cmpxchg8B(SSP_LA, new_token_value, expected_token_value) != expected_token_value
    THEN #CP(SETSSBSY); FI;
SSP = SSP_LA

受影响的标志位¶

无。

C/C++ 编译器等价内部函数¶

SETSSBSYvoid _setssbsy(void);

保护模式异常¶

#UD 如果使用了 LOCK 前缀。
如果 CR4.CET = 0。
如果 IA32_S_CET.SH_STK_EN = 0。
#GP(0) 如果 IA32_PL0_SSP 未对齐到 8 字节。
如果 CPL 不为 0。
#CP(setssbsy) 如果令牌中的 busy 位被设置。
如果处于 32 位或兼容模式，且令牌中的地址不低于 4G。
#PF(错误码) 如果发生缺页异常。

实地址模式异常¶

#UD 在实地址模式下无法识别 SETSSBSY 指令。

虚拟-8086 模式异常¶

#UD 在虚拟-8086 模式下无法识别 SETSSBSY 指令。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

与保护模式下的异常相同。
