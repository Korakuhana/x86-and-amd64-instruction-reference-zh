FADD/FADDP/FIADD — 加法

操作码 指令 64位模式 兼容/传统模式 描述
D8 /0 FADD m32fp 有效 有效 将 m32fp 加到 ST(0) 并将结果存储在 ST(0) 中。
DC /0 FADD m64fp 有效 有效 将 m64fp 加到 ST(0) 并将结果存储在 ST(0) 中。
D8 C0+i FADD ST(0), ST(i) 有效 有效 将 ST(0) 加到 ST(i) 并将结果存储在 ST(0)。
DC C0+i FADD ST(i), ST(0) 有效 有效 将 ST(i) 加到 ST(0) 并将结果存储在 ST(i)。
DE C0+i FADDP ST(i), ST(0) 有效 有效 将 ST(0) 加到 ST(i)，将结果存储在 ST(i)，并弹出寄存器栈。
DE C1 FADDP 有效 有效 将 ST(0) 加到 ST(1)，将结果存储在 ST(1)，并弹出寄存器栈。
DA /0 FIADD m32int 有效 有效 将 m32int 加到 ST(0) 并将结果存储在 ST(0) 中。
DE /0 FIADD m16int 有效 有效 将 m16int 加到 ST(0) 并将结果存储在 ST(0) 中。

描述¶

将目的操作数和源操作数相加，并将和存储在目的位置。目的操作数始终是 FPU 寄存器；源操作数可以是寄存器或内存位置。内存中的源操作数可以是单精度或双精度浮点格式，也可以是字或双字整数格式。

无操作数版本的指令将 ST(0) 寄存器的内容加到 ST(1) 寄存器中。单操作数版本将内存位置的内容（浮点值或整数值）加到 ST(0) 寄存器的内容中。双操作数版本将 ST(0) 寄存器的内容加到 ST(i) 寄存器中，反之亦然。ST(0) 中的值可以通过以下代码加倍：

FADD ST(0), ST(0);

FADDP 指令在存储结果后执行弹出 FPU 寄存器栈的附加操作。要弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。（无操作数版本的浮点加法指令总是会导致寄存器栈被弹出。在某些汇编器中，此指令的助记符是 FADD 而不是 FADDP。）

FIADD 指令在执行加法之前将整数源操作数转换为双精度扩展浮点格式。

下页的表格显示了在相加各类数字时获得的结果，假设既没有发生溢出也没有发生下溢。

当两个符号相反的操作数之和为 0 时，结果为 +0，但向 -∞ 舍入的模式除外，在这种情况下结果为 -0。当源操作数是整数 0 时，它被视为 +0。

当两个操作数都是相同符号的无穷大时，结果是预期符号的 ∞。如果两个操作数是相反符号的无穷大，则会产生无效操作异常。参见表 3-18。

F 表示有限浮点值。

I 表示整数。

* 表示浮点无效算术操作数 (#IA) 异常。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

IF Instruction = FIADD
	THEN
		DEST := DEST + ConvertToDoubleExtendedPrecisionFP(SRC);
	ELSE (* Source operand is floating-point value *)
		DEST := DEST + SRC;
FI;
IF Instruction = FADDP
	THEN
		PopRegisterStack;
FI;

影响的 FPU 标志寄存器¶

C1 如果发生栈下溢，则设置为 0。
如果结果被向上舍入，则设置；否则清除。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈下溢。
#IA 操作数是 SNaN 值或不受支持的格式。
操作数是异号的无穷大。
#D 源操作数是反常数。
#U 结果对于目标格式来说太小。
#O 结果对于目标格式来说太大。
#P 值无法在目标格式中精确表示。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(错误代码) 如果发生页面错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
