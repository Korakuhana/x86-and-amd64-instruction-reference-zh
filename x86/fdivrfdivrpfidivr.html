FDIVR/FDIVRP/FIDIVR — 反向除法

操作码 指令 64位模式 兼容/传统模式 描述
D8 /7 FDIVR m32fp 有效 有效 将 m32fp 除以 ST(0)，并将结果存储在 ST(0) 中。
DC /7 FDIVR m64fp 有效 有效 将 m64fp 除以 ST(0)，并将结果存储在 ST(0) 中。
D8 F8+i FDIVR ST(0), ST(i) 有效 有效 将 ST(i) 除以 ST(0)，并将结果存储在 ST(0) 中。
DC F0+i FDIVR ST(i), ST(0) 有效 有效 将 ST(0) 除以 ST(i)，并将结果存储在 ST(i) 中。
DE F0+i FDIVRP ST(i), ST(0) 有效 有效 将 ST(0) 除以 ST(i)，将结果存储在 ST(i)，并弹出寄存器栈。
DE F1 FDIVRP 有效 有效 将 ST(0) 除以 ST(1)，将结果存储在 ST(1)，并弹出寄存器栈。
DA /7 FIDIVR m32int 有效 有效 将 m32int 除以 ST(0)，并将结果存储在 ST(0) 中。
DE /7 FIDIVR m16int 有效 有效 将 m16int 除以 ST(0)，并将结果存储在 ST(0) 中。

描述¶

将源操作数除以目的操作数，并将结果存储在目的位置。目的操作数（除数）始终在 FPU 寄存器中；源操作数（被除数）可以是寄存器或内存位置。内存中的源操作数可以是单精度或双精度浮点格式，也可以是字或双字整数格式。

这些指令执行与 FDIV、FDIVP 和 FIDIV指令相反的操作。提供它们是为了支持更高效的编码。

无操作数版本的指令将 ST(0) 寄存器的内容除以 ST(1) 寄存器的内容。单操作数版本将内存位置的内容（浮点值或整数值）除以 ST(0) 寄存器的内容。双操作数版本将 ST(i) 寄存器的内容除以 ST(0) 寄存器的内容，反之亦然。

FDIVRP 指令在存储结果后执行弹出 FPU 寄存器栈的附加操作。要弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。无操作数版本的浮点除法指令总是会导致寄存器栈被弹出。在某些汇编器中，此指令的助记符是 FDIVR 或者是 FDIVRP。

FIDIVR 指令在执行除法之前将整数源操作数转换为双精度扩展浮点格式。

如果产生未屏蔽的除以零异常 (#Z)，则不存储结果；如果屏蔽了该异常，则在目的操作数中存储适当符号的 ∞。

下表显示了在除以各类数字时获得的结果，假设既没有发生溢出也没有发生下溢。

F 表示有限浮点值。

I 表示整数。

* 表示浮点无效算术操作数 (#IA) 异常。

** 表示浮点除以零 (#Z) 异常。

当源操作数是整数 0 时，它被视为 +0。此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

IF DEST = 0
	THEN
		#Z;
	ELSE
		IF Instruction = FIDIVR
			THEN
				DEST := ConvertToDoubleExtendedPrecisionFP(SRC) / DEST;
			ELSE (* Source operand is floating-point value *)
				DEST := SRC / DEST;
		FI;
FI;
IF Instruction = FDIVRP
	THEN
		PopRegisterStack;
FI;

影响的 FPU 标志寄存器¶

C1 如果发生栈下溢，则设置为 0。
如果结果被向上舍入，则设置；否则清除。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈下溢。
#IA 操作数是 SNaN 值或不受支持的格式。
±∞ / ±∞; ±0 / ±0
#D 源操作数是反常数。
#Z SRC / ±0，其中 SRC 不等于 ±0。
#U 结果对于目标格式来说太小。
#O 结果对于目标格式来说太大。
#P 值无法在目标格式中精确表示。
