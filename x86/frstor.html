FRSTOR — 恢复 x87 FPU 状态

操作码 模式 传统模式 描述
DD /4 有效 有效 从 m94byte 或 m108byte 加载 FPU 状态。

描述¶

从源操作数指定的内存区域加载 FPU 状态（工作环境和寄存器栈）。该状态数据通常是由之前的 FSAVE/FNSAVE 指令写入到指定的内存位置的。

FPU 工作环境包括 FPU 控制字、状态字、标记字（tag word）、指令指针、数据指针和上一次的操作码。《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷中的图 8-9 至图 8-12 显示了在内存中存储的工作环境布局，具体取决于处理器的运行模式（保护模式或实模式）和当前的操作数大小属性（16 位或 32 位）。在虚拟-8086 模式下，使用实模式的布局。FPU 寄存器栈的内容存储在紧随工作环境映像之后的 80 字节中。

FRSTOR 指令应当在与对应的 FSAVE/FNSAVE 指令相同的运行模式下执行。

如果在新的 FPU 状态字中设置了一个或多个未被掩码屏蔽的异常位，则在执行下一条浮点指令时将产生浮点异常（不等待的浮点指令除外，参见《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 8 章中标题为“Software Exception Handling”的部分）。为避免在加载新工作环境时引发异常，请清除正在加载 of FPU 状态字中的所有异常标志。

此指令的操作在非 64 位模式和 64 位模式下相同。

FPUTagWord := SRC[FPUTagWord];

影响的 FPU 标志寄存器¶

加载 C0, C1, C2, C3 标志。

浮点异常¶

无；但是，如果在状态字中加载了未屏蔽的异常，它将在执行下一条“等待”浮点指令时产生。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存且其包含 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误.
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果内存地址为非规范形式。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
