FST/FSTP — 存储浮点值

操作码 指令 64位模式 兼容/传统模式 描述
D9 /2 FST m32fp 有效 有效 将 ST(0) 复制到 m32fp。
DD /2 FST m64fp 有效 有效 将 ST(0) 复制到 m64fp。
DD D0+i FST ST(i) 有效 有效 将 ST(0) 复制到 ST(i)。
D9 /3 FSTP m32fp 有效 有效 将 ST(0) 复制到 m32fp 并弹出寄存器栈。
DD /3 FSTP m64fp 有效 有效 将 ST(0) 复制到 m64fp 并弹出寄存器栈。
DB /7 FSTP m80fp 有效 有效 将 ST(0) 复制到 m80fp 并弹出寄存器栈。
DD D8+i FSTP ST(i) 有效 有效 将 ST(0) 复制到 ST(i) 并弹出寄存器栈。

描述¶

FST 指令将 ST(0) 寄存器中的值复制到目的操作数，目的操作数可以是内存位置或 FPU 寄存器栈中的另一个寄存器。将值存储在内存中时，该值将被转换为单精度或双精度浮点格式。

FSTP 指令执行与 FST 指令相同的操作，然后弹出寄存器栈。为了弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。FSTP 指令还可以将值以双精度扩展浮点格式存储在内存中。

如果目的操作数是内存位置，则该操作数指定要存储目的值第一个字节的地址。如果目的操作数是寄存器，则该操作数指定寄存器栈中相对于栈顶的寄存器。

如果目的尺寸是单精度或双精度，则将要存储的值的尾数舍入到目的的宽度（根据 FPU 控制字中 RC 字段指定的舍入模式），并且将指数转换为目的格式的宽度和偏差（bias）。如果要存储的值对于目的格式来说太大，则会产生数值溢出异常 (#O)，并且如果该异常未被掩码屏蔽，则不会在目的操作数中存储任何值。如果要存储的值是反常值，则不会产生反常异常 (#D)。这种情况仅被视为数值下溢异常 (#U) 状态。

如果要存储的值是 ±0、±∞ 或 NaN，则会截断尾数和指数的最低有效位以适应目的格式。此操作保留该值作为 0、∞ 或 NaN 的身份。

如果目的操作数是非空寄存器，则不会产生无效操作异常。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

DEST := ST(0);
IF Instruction = FSTP
    THEN
        PopRegisterStack;
FI;

影响的 FPU 标志寄存器¶

C1 如果发生栈下溢，则设置为 0。
如果产生浮点不精确异常 (#P)，则指示舍入方向：0 := 未向上舍入；1 := 向上舍入。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈下溢。
#IA 如果目的结果是 SNaN 值或不受支持的格式，除非目的格式是双精度扩展浮点格式。
#U 结果对于目的格式来说太小。
#O 结果对于目的格式来说太大。
#P 值无法在目的格式中精确表示。

保护模式异常¶

#GP(0) 如果目的位于不可写段中。
如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存且其包含 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。
