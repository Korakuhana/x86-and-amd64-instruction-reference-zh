POP — 从栈中弹出一个值

描述 ¶

将栈顶的值加载到目的操作数（或显式操作码）指定的位置，然后递增栈指针。目的操作数可以是一个通用寄存器、内存位置或段寄存器。

地址和操作数大小的确定和使用如下：

  • 地址大小。当前代码段描述符中的 D 标志决定默认的地址大小；它可以通过指令前缀（67H）进行覆盖。地址大小仅在写入内存中的目的操作数时使用。
  • 操作数大小。当前代码段描述符中的 D 标志决定默认的操作数大小；它可以通过指令前缀（66H 或 REX.W）进行覆盖。操作数大小（16、32 或 64 位）决定栈指针递增的量（2、4 或 8）。
  • 栈地址大小。在 64 位模式之外，当前栈段描述符中的 B 标志决定栈指针的大小（16 或 32 位）；在 64 位模式下，栈指针的大小始终为 64 位。

栈地址大小决定从内存中的栈读取数据以及递增栈指针时栈指针的宽度。（如上所述，栈指针递增的量由操作数大小决定。）

如果目的操作数是段寄存器 DS、ES、FS、GS 或 SS 之一，则加载到寄存器中的值必须是有效的段选择子。在保护模式下，将段选择子弹入段寄存器会自动将与该段选择子关联的描述符信息加载到段寄存器的隐藏（阴影）部分，并导致对选择子和描述符信息进行验证。

可以将 NULL 值（0000-0003）弹入 DS、ES、FS 或 GS 寄存器，而不会导致通用保护故障。然而，任何随后尝试引用其相应段寄存器加载了 NULL 值的段的操作都会引发通用保护异常（#GP）。在这种情况下，不会发生内存引用，并且段寄存器的保存值为 NULL。

POP 指令不能将值弹入 CS 寄存器。要从栈中加载 CS 寄存器，请使用 RET 指令。

如果 ESP 寄存器被用作寻址内存中目的操作数的基址寄存器，则 POP 指令在递增 ESP 寄存器之后计算操作数的有效地址。对于因为 POP 指令而导致 ESP 缠绕回 0H 的 16 位栈情况，产生内存写入的位置是特定于处理器系列的。

POP ESP 指令在将旧栈顶的数据写入目的地之前递增栈指针（ESP）。

使用 POP 指令加载 SS 寄存器会抑制或禁止某些调试异常，并禁止在下一条指令边界上的中断。（该禁止在引发异常或执行下一条指令后结束。）这种行为允许在发生事件之前通过下一条指令（POP ESP）将栈指针加载到 ESP 寄存器中。参见 Intel® 64 和 IA-32 架构软件开发人员手册第 3A 卷第 6.8.3 节“切换栈时屏蔽异常和中断”。Intel 建议软件使用 LSS 指令同时加载 SS 寄存器和 ESP。

在 64 位模式下，使用 REX.R 形式的 REX 前缀允许访问额外的寄存器（R8-R15）。处于 64 位模式时，使用 32 位操作数的 POP 无法编码，且向 DS、ES、SS 的 POP 无效。参见本节开头的汇总图表以获取编码数据和限制。

操作 ¶ 

IF StackAddrSize = 32
    THEN
        IF OperandSize = 32
                THEN
                    DEST := SS:ESP; (* Copy a doubleword *)
                    ESP := ESP + 4;
                ELSE (* OperandSize = 16*)
                    DEST := SS:ESP; (* Copy a word *)
                    ESP := ESP + 2;
        FI;
    ELSE IF StackAddrSize = 64
        THEN
                IF OperandSize = 64
                    THEN
                        DEST := SS:RSP; (* Copy quadword *)
                        RSP := RSP + 8;
                    ELSE (* OperandSize = 16*)
                        DEST := SS:RSP; (* Copy a word *)
                        RSP := RSP + 2;
                FI;
        FI;
    ELSE StackAddrSize = 16
        THEN
                IF OperandSize = 16
                    THEN
                        DEST := SS:SP; (* Copy a word *)
                        SP := SP + 2;
                    ELSE (* OperandSize = 32 *)
                        DEST := SS:SP; (* Copy a doubleword *)
                        SP := SP + 4;
                FI;
FI;

                OR segment is not a data or readable code segment

PREOTECTED MODE OR COMPATIBILITY MODE;
IF SS is loaded;
    THEN
        IF segment selector is NULL
                THEN #GP(0);
        FI;
        IF segment selector index is outside descriptor table limits
                or segment selector's RPL ≠ CPL
                or segment is not a writable data segment
                or DPL ≠ CPL
                    THEN #GP(selector);

保护模式异常 ¶

#GP(0) 如果试图使用 NULL 段选择子加载 SS 寄存器。
如果目的操作数处于不可写的段中。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段界限。
如果 DS、ES、FS 或 GS 寄存器被用于访问内存且它包含 NULL 段选择子。
#GP(选择子) 如果段选择子索引超出描述符表限制。
如果正在加载 SS 寄存器，并且段选择子的 RPL 和段描述符的 DPL 不等于 CPL。
如果正在加载 SS 寄存器，并且指向的段是不可写的数据段。
如果正在加载 DS、ES、FS 或 GS 寄存器，并且指向的段不是数据或可读代码段。
如果正在加载 DS、ES、FS 或 GS 寄存器，且指向的段是数据或非一致性代码段，但 RPL 和 CPL 都大于 DPL。
#SS(0) 如果当前的栈顶不在栈段内。
如果内存操作数的有效地址超出 SS 段界限。
#SS(选择子) 如果正在加载 SS 寄存器，且指向的段被标记为不存在。
#NP 如果正在加载 DS、ES、FS 或 GS 寄存器，且指向的段被标记为不存在。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果当当前特权级别为 3 且启用了对齐检查时，进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
