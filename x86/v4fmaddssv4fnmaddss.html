V4FMADDSS/V4FNMADDSS — 标量单精度浮点融合乘加（4次迭代）

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
EVEX.LLIG.F2.0F38.W0 9B /r V4FMADDSS xmm1{k1}{z}, xmm2+3, m128 A V/V AVX512_4FMAPS 将 xmm2 指示的源寄存器块中的标量单精度浮点值与来自 m128 的值相乘，并将结果累加到 xmm1 中。
EVEX.LLIG.F2.0F38.W0 AB /r V4FNMADDSS xmm1{k1}{z}, xmm2+3, m128 A V/V AVX512_4FMAPS 将 xmm2 指示的源寄存器块中的标量单精度浮点值与来自 m128 的值相乘并取反，并将结果累加到 xmm1 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Tuple1_4X ModRM:reg (r, w) EVEX.vvvv (r) ModRM:r/m (r) N/A

描述¶

此指令计算 4 个连续的标量融合单精度浮点乘加指令，在四个步骤的每个步骤中均有顺序选择的内存操作数。

在上面的框中，记号“+3”用于表示该指令根据该操作数访问 4 个源寄存器；源寄存器是连续的，从 4 的倍数边界开始，并包含编码 of 寄存器操作数。

此指令支持内存故障抑制。如果最低有效掩码位设置为 1，或者如果使用“无掩码”编码，则加载整个内存操作数。

元组类型 Tuple1_4X 意味着此指令的内存操作部分引用了四个 32 位元素（16 字节）。

在每个 FMA 边界处执行舍入。异常也按顺序获取。第一 FMA 的计算前和计算后异常优先于第二 FMA 的计算前和计算后异常，依此类推。

操作¶

src_reg_id is the 5 bit index of the vector register specified in the instruction as the src1 register.
define NFMA_SS(vl, dest, k1, msrc, regs_loaded, src_base, posneg):
    tmpdest := dest
    // reg[] is an array representing the SIMD register file.
    IF k1[0] or *no writemask*:
        FOR j := 0 to regs_loaded - 1:
            IF posneg = 0:
                tmpdest.single[0] := RoundFPControl_MXCSR(tmpdest.single[0] - reg[src_base + j ].single[0] * msrc.single[j])
            ELSE:
                tmpdest.single[0] := RoundFPControl_MXCSR(tmpdest.single[0] + reg[src_base + j ].single[0] * msrc.single[j])
    ELSE IF *zeroing*:
        tmpdest.single[0] := 0
    dest := tmpdst
    dest[MAX_VL-1:VL] := 0
