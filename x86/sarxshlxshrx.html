SARX/SHLX/SHRX — 移位且不影响标志位

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
VEX.LZ.F3.0F38.W0 F7 /r SARX r32a, r/m32, r32b RMV V/V BMI2 将 r/m32 算术右移，移位次数由 r32b 指定。
VEX.LZ.66.0F38.W0 F7 /r SHLX r32a, r/m32, r32b RMV V/V BMI2 将 r/m32 逻辑左移，移位次数由 r32b 指定。
VEX.LZ.F2.0F38.W0 F7 /r SHRX r32a, r/m32, r32b RMV V/V BMI2 将 r/m32 逻辑右移，移位次数由 r32b 指定。
VEX.LZ.F3.0F38.W1 F7 /r SARX r64a, r/m64, r64b RMV V/N.E. BMI2 将 r/m64 算术右移，移位次数由 r64b 指定。
VEX.LZ.66.0F38.W1 F7 /r SHLX r64a, r/m64, r64b RMV V/N.E. BMI2 将 r/m64 逻辑左移，移位次数由 r64b 指定。
VEX.LZ.F2.0F38.W1 F7 /r SHRX r64a, r/m64, r64b RMV V/N.E. BMI2 将 r/m64 逻辑右移，移位次数由 r64b 指定。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RMV ModRM:reg (w) ModRM:r/m (r) VEX.vvvv (r) N/A

描述¶

将第一个源操作数（第二个操作数）的位根据第二个源操作数（第三个操作数）中指定的 COUNT 值向左或向右移位。结果被写入目的操作数（第一个操作数）。

算术右移 (SARX) 和 逻辑右移 (SHRX) 指令将目的操作数的位向右移（移向低有效位位置），SARX 在移位时保留并传播最高有效位（符号位）。

逻辑左移 (SHLX) 将目的操作数的位向左移（移向高有效位位置）。

在实模式和虚拟-8086 模式下不支持此指令。如果不在 64 位模式下，操作数大小始终为 32 位。在 64 位模式下，操作数大小为 64 需要 VEX.W1。在非 64 位模式下，VEX.W1 被忽略。尝试在 VEX.L 不等于 0 的情况下执行此指令将导致 #UD。

如果第一个源操作数中指定的值超出 OperandSize - 1，则对 COUNT 值进行掩码处理。

SARX、SHRX 和 SHLX 指令不更新标志位。

操作¶

TEMP := SRC1;
IF VEX.W1 and CS.L = 1
THEN
    countMASK := 3FH;
ELSE
    countMASK := 1FH;
FI
COUNT := (SRC2 AND countMASK)
DEST[OperandSize -1] = TEMP[OperandSize -1];
DO WHILE (COUNT ≠ 0)
    IF instruction is SHLX
        THEN
            DEST[] := DEST *2;
        ELSE IF instruction is SHRX
            THEN
                DEST[] := DEST /2; // 无符号除
        ELSE // SARX
                DEST[] := DEST /2; // 有符号除，向负无穷舍入
    FI;
    COUNT := COUNT - 1;
OD
