MAXSS — 返回标量单精度浮点值最大值

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
F3 0F 5F /r MAXSS xmm1, xmm2/m32 A 有效/有效 SSE 返回 xmm2/m32 和 xmm1 之间最大标量单精度浮点值。
VEX.LIG.F3.0F.WIG 5F /r VMAXSS xmm1, xmm2, xmm3/m32 B 有效/有效 AVX 返回 xmm3/m32 和 xmm2 之间最大标量单精度浮点值。

描述¶

比较第一个源操作数和第二个源操作数中的低单精度浮点值，并将最大值返回到目的操作数的低双字中。

如果被比较的值都是 0.0（无论符号如何），则返回第二个源操作数中的值。如果第二个源操作数中的值是 SNaN，则将该 SNaN 原样返回到目的地（即不返回 SNaN 的 QNaN 版本）。

如果对于此指令只有一个值是 NaN（SNaN 或 QNaN），则将第二个源操作数，无论是 NaN 还是有效的浮点值，写入结果中。如果需要返回两个源操作数中任意一个的 NaN，可以通过指令序列来模拟 MAXSS 的操作，例如先进行比较，然后进行 AND、ANDN 和 OR。

第二个源操作数可以是 XMM 寄存器或 32 位内存位置。第一个源和目的操作数是 XMM 寄存器。

128 位传统 SSE 版本：目的操作数与第一个源操作数相同。对应目的寄存器的高位 (MAXVL:32) 保持不变。

VEX.128 和 EVEX 编码版本：第一个源操作数是 VEX.vvvv 编码的 XMM 寄存器。XMM 目的寄存器的位 (127:32) 从第一个源操作数的相应位复制。目的寄存器的高位 (MAXVL:128) 被清零。

EVEX 编码版本：目的操作数的低双字元素根据写掩码进行更新。

软件应确保 VMAXSS 编码为 VEX.L=0。将 VMAXSS 编码为 VEX.L=1 可能会在不同的处理器世代中遇到不可预测的行为。

操作¶

MAX(SRC1, SRC2)
{
    IF ((SRC1 = 0.0) and (SRC2 = 0.0)) THEN DEST := SRC2;
        ELSE IF (SRC1 = NaN) THEN DEST := SRC2; FI;
        ELSE IF (SRC2 = NaN) THEN DEST := SRC2; FI;
        ELSE IF (SRC1 > SRC2) THEN DEST := SRC1;
        ELSE DEST := SRC2;
    FI;
}

VMAXSS (EVEX Encoded Version)¶

IF k1[0] or *no writemask*
    THEN DEST[31:0] := MAX(SRC1[31:0], SRC2[31:0])
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[31:0] remains unchanged*
            ELSE ; zeroing-masking
                THEN DEST[31:0] := 0
        FI;
FI;
DEST[127:32] := SRC1[127:32]
DEST[MAXVL-1:128] := 0

VMAXSS (VEX.128 Encoded Version)¶

DEST[31:0] := MAX(SRC1[31:0], SRC2[31:0])
DEST[127:32] := SRC1[127:32]
DEST[MAXVL-1:128] := 0

MAXSS (128-bit Legacy SSE Version)¶

DEST[31:0] := MAX(DEST[31:0], SRC[31:0])
DEST[MAXVL-1:32] (Unmodified)

受影响的标志位¶

SIMD 浮点异常¶
