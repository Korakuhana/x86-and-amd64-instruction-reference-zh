LLDT — 加载局部描述符表寄存器

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 00 /2 LLDT r/m16 M Valid Valid 将段选择子 r/m16 加载到 LDTR 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r) N/A N/A N/A

描述¶

将源操作数加载到局部描述符表寄存器（LDTR）的段选择子字段中。源操作数（通用寄存器或内存位置）包含一个指向局部描述符表（LDT）的段选择子。在 LDTR 中加载段选择子之后，处理器使用该段选择子在全局描述符表（GDT）中定位该 LDT 的段描述符。然后，它将 LDT 的段限长和基地址从段描述符加载到 LDTR 中。段寄存器 DS, ES, SS, FS, GS 和 CS 不受此指令影响，当前任务的任务状态段（TSS）中的 LDTR 字段也不受影响。

如果源操作数的第 2-15 位为 0，则 LDTR 被标记为无效，并且 LLDT 指令静默完成。然而，随后所有对 LDT 中描述符的引用（除了 LAR, VERR, VERW 或 LSL 指令之外）都会引起通用保护异常（#GP）。

操作数大小属性对此指令没有影响。

LLDT 指令供操作系统软件使用；不应在应用程序中使用。该指令只能在保护模式或 64 位模式下执行。

在 64 位模式下，操作数大小固定为 16 位。

操作¶

IF SRC(Offset) > descriptor table limit
    THEN #GP(segment selector); FI;
IF segment selector is valid
    Read segment descriptor;
    IF SegmentDescriptor(Type) ≠ LDT
        THEN #GP(segment selector); FI;
    IF segment descriptor is not present
        THEN #NP(segment selector); FI;
    LDTR(SegmentSelector) := SRC;
    LDTR(SegmentDescriptor) := GDTSegmentDescriptor;
ELSE LDTR := INVALID
FI;

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果当前特权级别不为 0。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#GP(选择子) 如果选择子操作数不指向全局描述符表，或者 GDT 中的项不是局部描述符表。
段选择子超出 GDT 限制。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#NP(选择子) 如果 LDT 描述符不存在。
#PF(错误码) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 在实地址模式下无法识别 LLDT 指令。

虚拟-8086 模式异常¶

#UD 在虚拟-8086 模式下无法识别 LLDT 指令。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果当前特权级别不为 0。
如果内存地址为非规范形式。
#GP(选择子) 如果选择子操作数不指向全局描述符表，或者 GDT 中的项不是局部描述符表。
段选择子超出 GDT 限制。
#NP(选择子) 如果 LDT 描述符不存在。
#PF(错误码) 如果发生缺页异常。
#UD 如果使用了 LOCK 前缀。
