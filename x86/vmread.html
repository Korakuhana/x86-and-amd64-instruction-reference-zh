VMREAD — 从虚拟机控制结构读取字段

操作码/指令 Op/En 描述
NP 0F 78 VMREAD r/m64, r64 MR 读取指定的 VMCS 字段（在 64 位模式下）。
NP 0F 78 VMREAD r/m32, r32 MR 读取指定的 VMCS 字段（在 64 位模式以外）。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
MR ModRM:r/m (w) ModRM:reg (r) NA NA

描述¶

从 VMCS 中读取指定的字段，并将其存储到指定的目的操作数（寄存器或内存）中。在 VMX 根操作中，该指令从当前 VMCS 中读取。如果在 VMX 非根操作中执行，该指令将从当前 VMCS 中“VMCS 链接指针”字段所引用的 VMCS 中读取。

VMCS 字段由寄存器源操作数中包含的 VMCS 字段编码指定。在 IA-32e 模式外，无论 CS.D 的值如何，源操作数都有 32 位。在 64 位模式下，源操作数有 64 位。

目的操作数（可以是寄存器或在内存中）的有效大小在 IA-32e 模式外始终为 32 位（就操作数大小而言，CS.D 的设置将被忽略），而在 64 位模式下为 64 位。如果由源操作数指定的 VMCS 字段短于此有效操作数大小，则目的操作数的高位将被清零为 0。如果 VMCS 字段较长，则不读取该字段的高位。

请注意，访问内存目的操作数所导致的任何故障，只能在下方的操作部分确定相关 VMCS 指针有效且支持指定的 VMCS 字段之后才会发生。

操作¶

IF (not in VMX operation) or (CR0.PE = 0) or (RFLAGS.VM = 1) or (IA32_EFER.LMA = 1 and CS.L = 0)
    THEN #UD;
ELSIF in VMX non-root operation AND (“VMCS shadowing” is 0 OR source operand sets bits in range 63:15 OR
VMREAD bit corresponding to bits 14:0 of source operand is 1)1
    THEN VMexit;
ELSIF CPL > 0
    THEN #GP(0);
ELSIF (in VMX root operation AND current-VMCS pointer is not valid) OR
(in VMX non-root operation AND VMCS link pointer is not valid)
    THEN VMfailInvalid;
ELSIF source operand does not correspond to any VMCS field
    THEN VMfailValid(VMREAD/VMWRITE from/to unsupported VMCS component);
    ELSE
        IF in VMX root operation
            THEN destination operand := contents of field indexed by source operand in current VMCS;
            ELSE destination operand := contents of field indexed by source operand in VMCS referenced by VMCS link pointer;
        FI;
        VMsucceed;
FI;

1. 源操作数的 VMREAD 位定义如下。设 x 为源操作数第 14:0 位的值，设 addr 为 VMREAD 位图地址。对应的 VMREAD 位处于物理地址 addr | (x » 3) 处的字节中的 bit 位置 x & 7。

受影响的标志位¶

请参阅操作部分和第 31.2 节。

保护模式异常¶

#GP(0) 如果当前特权级别不为 0。
如果内存目的操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含不可用的段。
如果目的操作数位于只读数据段或任何代码段中。
#PF(错误码) 如果访问内存目的操作数时发生缺页异常。
#SS(0) 如果内存目的操作数有效地址超出 SS 段限制。
如果 SS 寄存器包含不可用的段。
#UD 如果不在 VMX 操作中。

实地址模式异常¶

#UD 在实地址模式下无法识别 VMREAD 指令。

虚拟-8086 模式异常¶

#UD 在虚拟-8086 模式下无法识别 VMREAD 指令。

兼容模式异常¶

#UD 在兼容模式下无法识别 VMREAD 指令。

64位模式异常¶

#GP(0) 如果当前特权级别不为 0。
如果目的操作数位于 CS、DS、ES、FS 或 GS 段中，且内存地址为非规范形式。
#PF(错误码) 如果访问内存目的操作数时发生缺页异常。
#SS(0) 如果目的操作数位于 SS 段中，且内存地址为非规范形式。
#UD 如果不在 VMX 操作中。
