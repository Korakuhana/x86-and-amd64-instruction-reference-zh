DIVSD — 除以标量双精度浮点值

指令/指令格式 操作/编码 64/32位模式支持 CPUID功能标志 描述
F2 0F 5E /r DIVSD xmm1, xmm2/m64 A V/V SSE2 将 xmm1 中的低位双精度浮点值除以 xmm2/m64 中的低位双精度浮点值。
VEX.LIG.F2.0F.WIG 5E /r VDIVSD xmm1, xmm2, xmm3/m64 B V/V AVX 将 xmm2 中的低位双精度浮点值除以 xmm3/m64 中的低位双精度浮点值。
EVEX.LLIG.F2.0F.W1 5E /r VDIVSD xmm1 {k1}{z}, xmm2, xmm3/m64{er} C V/V AVX512F 将 xmm2 中的低位双精度浮点值除以 xmm3/m64 中的低位双精度浮点值。

指令操作数编码¶

操作/编码 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A N/A ModRM:reg (r, w) ModRM:r/m (r) N/A N/A

描述¶

将第一个源操作数中的低位双精度浮点值除以第二个源操作数中的低位双精度浮点值，并将双精度浮点结果存储在目标操作数中。第二个源操作数可以是 XMM 寄存器或 64 位内存位置。第一个源操作数和目标操作数是 XMM 寄存器。

128 位传统 SSE 版本：第一个源操作数和目标操作数相同。对应 ZMM 目标寄存器的位（MAXVL-1:64）保持不变。

VEX.128 编码版本：第一个源操作数是编码在 VEX.vvvv 的 XMM 寄存器。目标操作数在位 127:64 处的四字从第一个源操作数的对应四字复制。目标寄存器的位（MAXVL-1:128）清零。

EVEX.128 编码版本：第一个源操作数是编码在 EVEX.vvvv 的 XMM 寄存器。目标操作数在位 127:64 处的四字元素从第一个源操作数复制。目标寄存器的位（MAXVL-1:128）清零。

EVEX 版本：根据写掩码更新目标的低四字元素。

软件应确保 VDIVSD 编码为 VEX.L=0。将 VDIVSD 编码为 VEX.L=1 可能会在不同的处理器世代中遇到不可预测的行为。

操作¶

VDIVSD (EVEX 编码版本)¶

IF (EVEX.b = 1) AND SRC2 *is a register*
    THEN
        SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(EVEX.RC);
    ELSE
        SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(MXCSR.RC);
FI;
IF k1[0] or *no writemask*
    THEN DEST[63:0] := SRC1[63:0] / SRC2[63:0]
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[63:0] remains unchanged*
            ELSE ; zeroing-masking
                THEN DEST[63:0] := 0
        FI;
FI;
DEST[127:64] := SRC1[127:64]
DEST[MAXVL-1:128] := 0

VDIVSD (VEX.128 编码版本)¶

DEST[63:0] := SRC1[63:0] / SRC2[63:0]
DEST[127:64] := SRC1[127:64]
DEST[MAXVL-1:128] := 0

DIVSD (128位传统 SSE 版本)¶

DEST[63:0] := DEST[63:0] / SRC[63:0]
DEST[MAXVL-1:64] (Unmodified)
