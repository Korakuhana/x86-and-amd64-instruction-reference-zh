描述¶

将第一个源操作数偶数编号（基于零的索引）元素中的打包有符号双字整数与第二个源操作数对应元素中的打包有符号双字整数相乘，并将打包的有符号四字结果存储在目的操作数中。

128位传统 SSE 版本：输入的有符号双字整数取自源操作数的偶数编号元素，即第一个（低位）和第三个双字元素。对于 128 位内存操作数，从内存中读取 128 位，但在计算中仅使用第一个和第三个双字。第一个源操作数与目的 XMM 操作数相同。第二个源操作数可以是 XMM 寄存器或 128 位内存位置。对应目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：输入的有符号双字整数取自源操作数的偶数编号元素，即第一个（低位）和第三个双字元素。对于 128 位内存操作数，从内存中读取 128 位，但在计算中仅使用第一个和第三个双字。第一个源操作数与目的操作数是 XMM 寄存器。第二个源操作数可以是 XMM 寄存器或 128 位内存位置。对应目的寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：输入的有符号双字整数取自源操作数的偶数编号元素，即第一个、第三个、第五个、第七个双字元素。对于 256 位内存操作数，从内存中读取 256 位，但在计算中仅使用四个偶数编号的双字。第一个源操作数与目的操作数是 YMM 寄存器。第二个源操作数可以是 YMM 寄存器或 256 位内存位置。对应目的寄存器的位（MAXVL-1:256）清零。

EVEX 编码版本：输入的有符号双字整数取自源操作数的偶数编号元素。第一个源操作数是 ZMM/YMM/XMM 寄存器。第二个源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 64 位内存位置广播的 512/256/128 位向量。目的操作数是 ZMM/YMM/XMM 寄存器，并根据写掩码以 64 位粒度进行更新。

操作¶

VPMULDQ（EVEX 编码版本）¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no writemask*
        THEN
            IF (EVEX.b = 1) AND (SRC2 *is memory*)
                THEN DEST[i+63:i] := SignExtend64( SRC1[i+31:i]) * SignExtend64( SRC2[31:0])
                ELSE DEST[i+63:i] := SignExtend64( SRC1[i+31:i]) * SignExtend64( SRC2[i+31:i])
            FI;
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+63:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+63:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
