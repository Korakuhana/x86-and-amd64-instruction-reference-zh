TPAUSE — 定时暂停

操作码 / 指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
66 0F AE /6 TPAUSE r32, <edx>, <eax> A V/V WAITPKG 指示处理器进入与实现相关的优化状态，直到 TSC 达到 EDX:EAX 中的值。

描述¶

TPAUSE 指示处理器进入与实现相关的优化状态。有两种这样的优化状态可供选择：轻量级功耗/性能优化状态，以及改进的功耗/性能优化状态。两者之间的选择由显式输入寄存器源操作数的 bit[0] 控制。

当 CPUID.7.0:ECX.WAITPKG[bit 5] 枚举为 1 时，TPAUSE 可用。TPAUSE 可以在任何特权级执行。该指令在非 64 位模式和 64 位模式下的操作相同。

与 PAUSE 不同，当在事务区域内使用时，TPAUSE 指令不会导致事务中止，这在《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 16 章“使用 Intel® 事务同步扩展进行编程”中有所描述。

1. ModR/M 字节的 Mod 字段的值必须为 11B。

输入寄存器包含处理器应进入的首选优化状态等信息，如下表所述。除 bit 0 以外的其他位均保留，如果为非零值，将导致 #GP 异常。

当时间戳计数器达到或超过隐式 EDX:EAX 64 位输入值时，指令执行唤醒。

在执行 TPAUSE 指令之前，操作系统可以指定其允许处理器暂停运行的最大延迟。它可以通过向以下 32 位 MSR（MSR 索引为 E1H 的 IA32_UMWAIT_CONTROL）写入 TSC 份量值来实现：

  • IA32_UMWAIT_CONTROL[31:2] — 确定处理器可以驻留在 C0.1 或 C0.2 中的最大时间（以 TSC 份量为单位）。零值表示没有最大时间。最大时间值是一个 32 位值，其中高 30 位来自此字段，低两位为零。
  • IA32_UMWAIT_CONTROL[1] — 保留。
  • IA32_UMWAIT_CONTROL[0] — 操作系统不允许 C0.2。值“1”表示所有 C0.2 请求都恢复为 C0.1。

如果执行 TPAUSE 指令的处理器由于操作系统时间限制过期而唤醒，则指令设置 RFLAGS.CF；否则，清除该标志。

以下附加事件会导致处理器退出与实现相关的优化状态：向事务区域内的读取集范围进行存储写入、NMI 或 SMI、调试异常、机器检查异常、BINIT# 信号、INIT# 信号以及 RESET# 信号。

其他与实现相关的事件可能会导致处理器退出与实现相关的优化状态，从而继续执行紧跟在 TPAUSE 之后的指令。此外，外部中断会导致处理器退出与实现相关的优化状态，而不管可屏蔽中断是否被禁止 (EFLAGS.IF = 0)。应该注意的是，如果可屏蔽中断被禁止，执行将继续进行紧跟在 TPAUSE 之后的指令。

操作¶

os_deadline := TSC+(IA32_UMWAIT_CONTROL[31:2]<<2)
instr_deadline := UINT64(EDX:EAX)
IF os_deadline < instr_deadline:
    deadline := os_deadline
    using_os_deadline := 1
ELSE:
    deadline := instr_deadline
    using_os_deadline := 0
WHILE TSC < deadline:
    implementation_dependent_optimized_state(Source register, deadline, IA32_UMWAIT_CONTROL[0])
IF using_os_deadline AND TSC ≥ deadline:
    RFLAGS.CF := 1
ELSE:
    RFLAGS.CF := 0
RFLAGS.AF,PF,SF,ZF,OF := 0

数值异常¶

无。

异常（所有运行模式）¶

#GP(0) 如果 src[31:1] != 0。

如果 CR4.TSD = 1 且 CPL != 0。

#UD 如果 CPUID.7.0:ECX.WAITPKG[bit 5] = 0。
