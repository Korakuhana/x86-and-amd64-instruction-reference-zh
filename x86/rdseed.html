RDSEED — 读取随机种子

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
NFx 0F C7 /7 RDSEED r16 M V/V RDSEED 读取符合 NIST SP800-90B & C 标准的 16 位随机值并存储在目的寄存器中。
NFx 0F C7 /7 RDSEED r32 M V/V RDSEED 读取符合 NIST SP800-90B & C 标准的 32 位随机值并存储在目的寄存器中。
NFx REX.W + 0F C7 /7 RDSEED r64 M V/I RDSEED 读取符合 NIST SP800-90B & C 标准的 64 位随机值并存储在目的寄存器中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (w) N/A N/A N/A

描述¶

加载硬件生成的随机值并将其存储在目的寄存器中。该随机值由符合 NIST SP800-90B 和 NIST SP800-90C 规范（处于 XOR 构建模式）的增强型 NRBG（非确定性随机比特生成器）生成。随机值的大小由目的寄存器大小和工作模式确定。进位标志 (CF) 指示在执行指令时随机值是否可用。CF=1 表示目的操作数中的数据有效。否则 CF=0，目的操作数中的数据将在指定宽度内返回为零。在任何一种情况下，所有其他标志均被强制置为 0。软件必须检查 CF=1 的状态以确定是否返回了有效的随机种子值，否则需要循环并重试执行 RDSEED（请参阅第 1.2 节）。

RDSEED 指令在所有特权级下均可用。RDSEED 指令在事务区域内部或外部均可正常执行。

在 64 位模式下，该指令的默认操作数大小为 32 位。使用 REX.B 形式的 REX 前缀允许访问其他寄存器（R8-R15）。使用 REX.W 形式的 REX 前缀将操作提升为 64 位操作数。请参阅本节开头的汇总图表以获取编码数据和限制。

操作¶

IF HW_NRND_GEN.ready = 1
    THEN
        CASE of
            操作数大小为 64: DEST[63:0] := HW_NRND_GEN.data;
            操作数大小为 32: DEST[31:0] := HW_NRND_GEN.data;
            操作数大小为 16: DEST[15:0] := HW_NRND_GEN.data;
        ESAC;
        CF := 1;
    ELSE
        CASE of
            操作数大小为 64: DEST[63:0] := 0;
            操作数大小为 32: DEST[31:0] := 0;
            操作数大小为 16: DEST[15:0] := 0;
        ESAC;
        CF := 0;
FI;
OF, SF, ZF, AF, PF := 0;

受影响的标志位¶

根据结果设置 CF 标志（请参阅上文的“操作”部分）。OF、SF、ZF、AF 和 PF 标志设置为 0。
