SYSCALL — 快速系统调用

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 05 SYSCALL ZO Valid Invalid 快速调用特权级 0 系统过程。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

SYSCALL 在特权级 0 处调用操作系统系统调用处理程序。它通过从 IA32_LSTAR MSR 加载 RIP 来执行此操作（在将紧跟在 SYSCALL 之后的指令地址保存到 RCX 之后）。（WRMSR 指令确保 IA32_LSTAR MSR 始终包含规范形式的地址。）

SYSCALL 还将 RFLAGS 保存到 R11 中，然后使用 IA32_FMASK MSR（MSR 地址 C0000084H）屏蔽 RFLAGS；具体而言，处理器会清除 RFLAGS 中与 IA32_FMASK MSR 中设置的位对应的所有位。

SYSCALL 用派生自 IA32_STAR MSR 的第 47:32 位的值加载 CS 和 SS 选择子。然而，CS 和 SS 描述符缓存不会从那些选择子引用的描述符（在 GDT 或 LDT 中）加载。相反，描述符缓存被加载为固定值。详情请参阅操作（Operation）部分。操作系统软件有责任确保这些选择子值引用的描述符（在 GDT 或 LDT 中）与加载到描述符缓存中的固定值相对应；SYSCALL 指令并不保证这种对应关系。

SYSCALL 指令不保存栈指针（RSP）。如果操作系统系统调用处理程序要更改栈指针，则软件有责任保存栈指针的先前值。这可以在执行 SYSCALL 之前完成，软件通过紧跟在 SYSCALL 之后的指令（该指令将在 SYSRET 之后执行）恢复栈指针。或者，操作系统系统调用处理程序可以保存栈指针并在执行 SYSRET 之前将其恢复。

当在调用 SYSCALL 指令的特权级启用阴影栈时，SSP 将保存到 IA32_PL3_SSP MSR 中。如果阴影栈在特权级 0 启用，则 SSP 加载为 0。有关额外的 CET 详细信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 6 章“过程调用、中断和异常”和第 17 章“控制流强制技术（CET）”。

指令顺序。 在早先指令完成执行之前，可能会从内存中获取 SYSCALL 之后的指令，但是在 SYSCALL 之前的所有指令都完成执行之前，它们将不会执行（即使是猜测性执行）（后续指令可能会在早先指令存储的数据变得全局可见之前执行）。

操作¶

IF (CS.L ≠ 1 ) or (IA32_EFER.LMA ≠ 1) or (IA32_EFER.SCE ≠ 1)
(* Not in 64-Bit Mode or SYSCALL/SYSRET not enabled in IA32_EFER *)
    THEN #UD;
FI;
RCX := RIP; (* Will contain address of next instruction *)
RIP := IA32_LSTAR;
R11 := RFLAGS;
RFLAGS := RFLAGS AND NOT(IA32_FMASK);
CS.Selector := IA32_STAR[47:32] AND FFFCH (* Operating system provides CS; RPL forced to 0 *)
(* Set rest of CS to a fixed value *)
CS.Base := 0;
                (* Flat segment *)
CS.Limit := FFFFFH;
                (* With 4-KByte granularity, implies a 4-GByte limit *)
CS.Type := 11;
                (* Execute/read code, accessed *)
CS.S := 1;
CS.DPL := 0;
CS.P := 1;
CS.L := 1;
                (* Entry is to 64-bit mode *)
CS.D := 0;
                (* Required if CS.L = 1 *)
CS.G := 1;
                (* 4-KByte granularity *)
IF ShadowStackEnabled(CPL)
    THEN (* adjust so bits 63:N get the value of bit N–1, where N is the CPU’s maximum linear-address width *)
        IA32_PL3_SSP := LA_adjust(SSP);
            (* With shadow stacks enabled the system call is supported from Ring 3 to Ring 0 *)
            (* OS supporting Ring 0 to Ring 0 system calls or Ring 1/2 to ring 0 system call *)
            (* Must preserve the contents of IA32_PL3_SSP to avoid losing ring 3 state *)
FI;
CPL := 0;
IF ShadowStackEnabled(CPL)
    SSP := 0;
FI;
IF EndbranchEnabled(CPL)
    IA32_S_CET.TRACKER = WAIT_FOR_ENDBRANCH
    IA32_S_CET.SUPPRESS = 0
FI;
SS.Selector := IA32_STAR[47:32] + 8;
                (* SS just above CS *)
(* Set rest of SS to a fixed value *)
SS.Base := 0;
                (* Flat segment *)
SS.Limit := FFFFFH;
                (* With 4-KByte granularity, implies a 4-GByte limit *)
SS.Type := 3;
                (* Read/write data, accessed *)
SS.S := 1;
SS.DPL := 0;
SS.P := 1;
SS.B := 1;
                (* 32-bit stack segment *)
SS.G := 1;
                (* 4-KByte granularity *)
