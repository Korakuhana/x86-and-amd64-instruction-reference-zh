XGETBV — 获取扩展控制寄存器的值

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
NP 0F 01 D0 XGETBV ZO Valid Valid 将 ECX 指定的 XCR 读取到 EDX:EAX 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将 ECX 寄存器中指定的扩展控制寄存器 (XCR) 的内容读取到寄存器 EDX:EAX 中。（在支持 Intel 64 架构的处理器上，RCX 的高 32 位被忽略。）EDX 寄存器加载有 XCR 的高 32 位，EAX 寄存器加载有低 32 位。（在支持 Intel 64 架构的处理器上，RAX 和 RDX 各自的高 32 位被清零。）如果在所读取的 XCR 中实现的位数少于 64 位，则返回到 EDX:EAX 中未实现位位置的值是未定义的。

在支持 XGETBV 指令的任何处理器上均支持 XCR0。如果 CPUID.(EAX=0DH,ECX=1):EAX.XG1[bit 2] = 1，则执行 ECX = 1 的 XGETBV 会在 EDX:EAX 中返回 XCR0 与 XINUSE 状态组件位图当前值的逻辑与 (AND)。这允许软件发现 XSAVEOPT 和 XSAVES 使用的初始化优化的状态。请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》卷 1 第 13 章“使用 XSAVE 功能集管理状态”。

在 ECX 中使用任何其他值都会导致通用保护 (#GP) 异常。

操作¶

EDX:EAX := XCR[ECX];

受影响的标志位¶

无。

Intel C/C++ 编译器等价内部函数¶

XGETBV unsigned __int64 _xgetbv( unsigned int index);

保护模式异常¶

#GP(0) 如果 ECX 中指定了无效的 XCR（如果 CPUID.(EAX=0DH,ECX=1):EAX.XG1[bit 2] = 0，则包括 ECX = 1）。
#UD 如果 CPUID.01H:ECX.XSAVE[bit 26] = 0。
如果 CR4.OSXSAVE[bit 18] = 0。
如果使用了 LOCK 前缀。

实地址模式异常¶

#GP(0) 如果 ECX 中指定了无效的 XCR（如果 CPUID.(EAX=0DH,ECX=1):EAX.XG1[bit 2] = 0，则包括 ECX = 1）。
#UD 如果 CPUID.01H:ECX.XSAVE[bit 26] = 0。
如果 CR4.OSXSAVE[bit 18] = 0。
如果使用了 LOCK 前缀。
