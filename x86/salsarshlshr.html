描述 ¶

将第一个操作数（目的操作数）中的位向左或向右移位第二个操作数（计数操作数）所指定的位数。超出目的操作数边界的位首先移入 CF 标志位，然后被丢弃。在移位操作结束时，CF 标志位包含从目的操作数中移出的最后一位。

目的操作数可以是寄存器或内存地址。计数操作数可以是立即数或 CL 寄存器。计数被屏蔽（掩码）为 5 位（对于 64 位操作数为 6 位）。计数范围限制在 0 到 31 之间（对于 64 位操作数为 0 到 63）。针对计数为 1 提供了一种特殊的操作码编码方式。

算术左移 (SAL) 和逻辑左移 (SHL) 指令执行相同的操作；它们将目的操作数中的位向左移位（朝向更高有效位位置）。对于每一次移位，目的操作数的最高有效位移入 CF 标志，而最低有效位被清零（参见 Intel® 64 和 IA-32 架构软件开发人员手册第 1 卷中的图 7-7）。

算术右移 (SAR) 和逻辑右移 (SHR) 指令将目的操作数中的位向右移位（朝向更低有效位位置）。对于每一次移位，目的操作数的最低有效位被移入 CF 标志，而最高有效位根据指令类型被设置为 1 或清零。SHR 指令将最高有效位清零（参见 Intel® 64 和 IA-32 架构软件开发人员手册第 1 卷中的图 7-8）；SAR 指令将最高有效位设置或清零，以与目的操作数中原始值的符号（最高有效位）一致。实际上，SAR 指令在移位后空出的高位位置填充未移位值原本的符号（参见 Intel® 64 和 IA-32 架构软件开发人员手册第 1 卷中的图 7-9）。

SAR 和 SHR 指令可分别用于执行目的操作数除以 2 的幂的有符号或无符号除法。例如，使用 SAR 指令将一个有符号整数向右移位 1 位相当于将该值除以 2。

使用 SAR 指令执行除法操作所产生的结果与 IDIV 指令不同。IDIV 指令的商是向零取整的，而 SAR 指令的“商”是向负无穷方向舍入的。这种差异仅在负数时显现。例如，当使用 IDIV 指令将 -9除以 4 时，结果为 -2 且余数为 -1。如果使用 SAR 指令将 -9 向右移位两位，则结果为 -3，而“余数”为 +3；然而，SAR 指令仅在 CF 标志中保存余数的最高有效位。

仅在 1 位移位时才会影响 OF 标志。对于左移，如果结果的最高有效位与 CF 标志相同（即原始操作数的最高两位相同），则 OF 标志被设置为 0；否则被设置为 1。对于 SAR 指令，所有 1 位移位的 OF 标志都会被清零。对于 SHR 指令，OF 标志被设置为原始操作数的最高有效位。

在 64 位模式下，该指令的默认操作大小为 32 位，并且 CL 的掩码宽度为 5 位。使用 REX.R 形式的 REX 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 形式 of REX 前缀可将操作提升为 64 位，并将 CL 的掩码宽度设置为 6 位。请参阅本节开头的汇总图表以获取编码数据和限制。

IA-32 架构兼容性 ¶

8086 不会对移位计数进行掩码。然而，所有其他 IA-32 处理器（从 Intel 286 处理器开始）都会将移位计数掩码为 5 位，从而使最大计数为 31。这种掩码操作在所有运行模式下进行（包括虚拟-8086 模式），以减少指令的最大执行时间。

IF OperandSize = 64
    THEN
        countMASK := 3FH;
    ELSE
        countMASK := 1FH;
FI
tempCOUNT := (COUNT AND countMASK);
tempDEST := DEST;
WHILE (tempCOUNT ≠ 0)
DO
    IF instruction is SAL or SHL
        THEN
            CF := MSB(DEST);
        ELSE (* Instruction is SAR or SHR *)
            CF := LSB(DEST);
    FI;
    IF instruction is SAL or SHL
        THEN
            DEST := DEST ∗ 2;
        ELSE
            IF instruction is SAR
                THEN
                    DEST := DEST / 2; (* Signed divide, rounding toward negative infinity *)
                ELSE (* Instruction is SHR *)
                    DEST := DEST / 2 ; (* Unsigned divide *)
            FI;
    FI;
    tempCOUNT := tempCOUNT – 1;
OD;
(* Determine overflow for the various instructions *)
IF (COUNT and countMASK) = 1
    THEN
        IF instruction is SAL or SHL
            THEN
                OF := MSB(DEST) XOR CF;
            ELSE
                IF instruction is SAR
                    THEN
                        OF := 0;
                    ELSE (* Instruction is SHR *)
                        OF := MSB(tempDEST);
                FI;
        FI;
    ELSE IF (COUNT AND countMASK) = 0
        THEN
            All flags unchanged;
        ELSE (* COUNT not 1 or 0 *)
            OF := undefined;
    FI;
FI;

受影响的标志位 ¶

CF 标志包含从目的操作数中移出的最后一位的值；对于移位计数大于或等于目的操作数大小（以位为单位）的 SHL 和 SHR 指令，CF 标志是未定义的。OF 标志仅对 1 位移位有影响（参见上面的“描述”）；否则它是未定义的。SF、ZF 和 PF 标志根据结果设置。如果计数为 0，则标志不受影响。对于非零计数，AF 标志是未定义的。
