BSWAP — 字节交换

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F C8+rd BSWAP r32 O Valid* Valid 反转 32 位寄存器的字节顺序。
REX.W + 0F C8+rd BSWAP r64 O Valid N.E. 反转 64 位寄存器的字节顺序。

* 请参阅下文的 IA-32 架构传统兼容性部分。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
O opcode + rd (r, w) N/A N/A N/A

描述¶

反转 32 位或 64 位（目的）寄存器的字节顺序。提供此指令是为了将小端（little-endian）值转换为大端（big-endian）格式，反之亦然。要在字值（16 位寄存器）中交换字节，请使用 XCHG 指令。当 BSWAP 指令引用 16 位寄存器时，结果是未定义的。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 形式的 REX 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 形式 of REX 前缀可将操作提升至 64 位。请参阅本节开头的汇总图表以获取编码数据和限制。

IA-32 架构传统兼容性¶

早于 Intel486™ 处理器系列的 IA-32 处理器不支持 BSWAP 指令。为了与此指令兼容，软件应包含功能等效的代码，以便在早于 Intel486 处理器的 Intel 处理器上执行。

操作¶

TEMP := DEST
IF 64-bit mode AND OperandSize = 64
    THEN
        DEST[7:0] := TEMP[63:56];
        DEST[15:8] := TEMP[55:48];
        DEST[23:16] := TEMP[47:40];
        DEST[31:24] := TEMP[39:32];
        DEST[39:32] := TEMP[31:24];
        DEST[47:40] := TEMP[23:16];
        DEST[55:48] := TEMP[15:8];
        DEST[63:56] := TEMP[7:0];
    ELSE
        DEST[7:0] := TEMP[31:24];
        DEST[15:8] := TEMP[23:16];
        DEST[23:16] := TEMP[15:8];
        DEST[31:24] := TEMP[7:0];
FI;

受影响的标志位¶

无。

异常（所有操作模式）¶

#UD 如果使用了 LOCK 前缀。
