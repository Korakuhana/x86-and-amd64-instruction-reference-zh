描述¶

SHLD 指令用于 64 位或更多位的多精度移位。

该指令将第一个操作数（目的操作数）向左移动由第三个操作数（计数操作数）指定的位数。第二个操作数（源操作数）提供从右侧移入的位（从目的操作数的第 0 位开始）。

目的操作数可以是寄存器或内存位置；源操作数是寄存器。计数操作数是一个无符号整数，可以存储在立即数字节或 CL 寄存器中。如果计数操作数是 CL，则移位计数是 CL 与计数掩码的逻辑与结果。在非 64 位模式和默认的 64 位模式下，仅使用计数的高/低 0 到 4 位（即将计数掩码设置为 0 到 31 之间的值）。如果计数大于操作数大小，则结果是未定义的。

如果计数为 1 或更大，则 CF 标志位填充从目的操作数中移出的最后一位。对于 1 位移位，如果发生符号变化，则设置 OF 标志；否则清除该标志。如果计数操作数为 0，则标志位不受影响。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 形式的 REX 前缀允许访问额外的寄存器 (R8-R15)。使用 REX.W 形式 of REX 前缀可将操作提升为 64 位（将计数掩码升级为 6 位）。请参阅本节开头的汇总图表以获取编码数据和限制。

操作¶

IF (In 64-Bit Mode and REX.W = 1)
    THEN COUNT := COUNT MOD 64;
    ELSE COUNT := COUNT MOD 32;
FI
SIZE := OperandSize;
IF COUNT = 0
    THEN
        No operation;
    ELSE
        IF COUNT > SIZE
            THEN (* 错误参数 *)
                DEST is undefined;
                CF, OF, SF, ZF, AF, PF are undefined;
            ELSE (* 执行移位 *)
                CF := BIT[DEST, SIZE – COUNT];
                (* 退出时移出的最后一位 *)
                FOR i := SIZE – 1 DOWN TO COUNT
                    DO
                        Bit(DEST, i) := Bit(DEST, i – COUNT);
                    OD;
                FOR i := COUNT – 1 DOWN TO 0
                    DO
                        BIT[DEST, i] := BIT[SRC, i – COUNT + SIZE];
                    OD;
        FI;
FI;

受影响的标志位¶

如果计数为 1 或更大，则 CF 标志填充从目的操作数移出的最后一位，并且 SF、ZF 和 PF 标志根据结果的值设置。对于 1 位移位，如果发生符号变化，则设置 OF 标志；否则清除。对于大于 1 位的移位，OF 标志是未定义的。如果发生移位，AF 标志是未定义的。如果计数操作数为 0，则标志不受影响。如果计数大于操作数大小，则标志是未定义的。

保护模式异常¶

#GP(0) 如果目标操作数位于不可写的段中。
如果内存操作数的有效地址超出了 CS、DS、ES、FS 或 GS 段的界限。
如果 DS、ES、FS 或 GS 寄存器包含一个 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出了 SS 段的界限。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数的有效地址超出了 CS、DS、ES、FS 或 GS 段的界限。
#SS 如果内存操作数的有效地址超出了 SS 段的界限。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数的有效地址超出了 CS、DS、ES、FS 或 GS 段的界限。
#SS(0) 如果内存操作数的有效地址超出了 SS 段的界限。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。

与保护模式下的异常相同。
