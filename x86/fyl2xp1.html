FYL2XP1 — 计算 y ∗ log2(x + 1)

操作码 指令 64位模式 兼容/传统模式 描述
D9 F9 FYL2XP1 Valid Valid 用 ST(1) ∗ log2(ST(0) + 1.0) 替换 ST(1)，并弹出寄存器栈。

描述¶

计算 (ST(1) ∗ log2(ST(0) + 1.0))，将结果存储在寄存器 ST(1) 中，并弹出 FPU 寄存器栈。ST(0) 中的源操作数必须在以下范围内：

ST(1) 中的源操作数范围可以从 −∞ 到 +∞。如果 ST(0) 操作数超出其可接受范围，则结果未定义，并且软件不应依赖于产生异常。在某些情况下，当 ST(0) 超出范围时可能会产生异常，但此行为特定于具体实现且不作保证。

下表显示了在不发生下溢的情况下，计算各类数字的对数 epsilon 时获得的结果。

F 表示有限浮点值。

* 表示浮点数无效操作 (#IA) 异常。

此指令对于与 0 接近的 epsilon 值 [寄存器 ST(0) 中的值] 提供最佳精度。对于较小的 epsilon (ε) 值，使用 FYL2XP1 指令比使用 (ε+1) 作为 FYL2X 指令的参数，可以保留更多有效数字。表达式 (ε+1) 通常见于复利和年金计算。通过在 ST(1) 源操作数中包含比例因子，结果可以简单地转换为另一个对数底数的值。以下等式用于计算特定对数底数的比例因子，其中 n 是 FYL2XP1 指令结果所需的对数底数：

比例因子 := logn 2

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

ST(1) := ST(1) ∗ log2(ST(0) + 1.0);
PopRegisterStack;

受影响的 FPU 标志位¶

C1 如果发生栈下溢，则设置为 0。
如果结果向上舍入，则设置；否则清除。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈下溢。
#IA 任一操作数是 SNaN 值或处于不受支持的格式。
#D 源操作数是非正规化值。
#U 结果对于目的格式来说太小。
#O 结果对于目的格式来说太大。
#P 值无法在目的格式中精确表示。

保护模式异常¶

#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在待处理的 x87 FPU 异常。
#UD 如果使用了 LOCK 前缀。
