描述¶

VEXTRACTI128/VEXTRACTI32x4 和 VEXTRACTI64x2 从源操作数（第二个操作数）中提取 128 位双字整数值，并存储到目的操作数（第一个操作数）的低 128 位中。128 位数据提取发生在由作为乘数的 imm8[0]（256 位）或 imm8[1:0] 指定的 128 位粒度偏移处。目的操作数可以是向量寄存器或 128 位内存位置。

VEXTRACTI32x4：目的操作数的低 128 位根据写掩码以 32 位粒度更新。

VEXTRACTI64x2：目的操作数的低 128 位根据写掩码以 64 位粒度更新。

VEXTRACTI32x8 和 VEXTRACTI64x4 从源操作数（第二个操作数）中提取 256 位四字整数值，并存储到目的操作数（第一个操作数）的低 256 位中。256 位数据提取发生在由作为乘数的 imm8[0]（256 位）或 imm8[0] 指定的 256 位粒度偏移处。目的操作数可以是向量寄存器或 256 位内存位置。

VEXTRACTI32x8：目的操作数的低 256 位根据写掩码以 32 位粒度更新。

VEXTRACTI64x4：目的操作数的低 256 位根据写掩码以 64 位粒度更新。

VEX.vvvv 和 EVEX.vvvv 保留且必须为 1111b，否则指令将产生 #UD。

立即数的高 7 位（在 EVEX.512 中为高 6 位）被忽略。

如果 VEXTRACTI128 使用 VEX.L= 0 编码，尝试执行使用 VEX.L= 0 编码的指令将导致 #UD 异常。

操作¶

VEXTRACTI32x4 (EVEX 编码版本) 当目的操作数为寄存器时¶

VL = 256, 512
IF VL = 256
    CASE (imm8[0]) OF
        0: TMP_DEST[127:0] := SRC1[127:0]
        1: TMP_DEST[127:0] := SRC1[255:128]
    ESAC.
FI;
IF VL = 512
    CASE (imm8[1:0]) OF
        00: TMP_DEST[127:0] := SRC1[127:0]
        01: TMP_DEST[127:0] := SRC1[255:128]
        10: TMP_DEST[127:0] := SRC1[383:256]
        11: TMP_DEST[127:0] := SRC1[511:384]
    ESAC.
FI;
FOR j := 0 TO 3
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN DEST[i+31:i] := TMP_DEST[i+31:i]
        ELSE
            IF *merging-masking*
                        ; merging-masking
                THEN *DEST[i+31:i] remains unchanged*
                ELSE *zeroing-masking*
                            ; zeroing-masking
                    DEST[i+31:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:128] := 0
