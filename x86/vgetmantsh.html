VGETMANTSH — 从 FP16 标量提取 FP16 规范化尾数

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.LLIG.NP.0F3A.W0 27 /r /ib VGETMANTSH xmm1{k1}{z}, xmm2, xmm3/m16 {sae}, imm8 A V/V AVX512-FP16 使用 imm8 进行符号控制和尾数区间规范化，提取 xmm3/m16 中低位 FP16 元素的规范化尾数。在写掩码 k1 下将尾数存储到 xmm1 中，并与 xmm2 的其他元素合并。xmm2 的第 127:16 位复制到 xmm1[127:16] 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4

描述¶

该指令将第二源操作数低位元素中的 FP16 值转换为具有由 imm8 字节指定的尾数规范化和符号控制的 FP16 值，参见表 5-19。转换后的结果使用写掩码 k1 写入目的操作数的低位元素。规范化尾数由 interv（imm8[1:0]）指定，符号控制（SC）由立即数字节的第 3:2 位指定。

目的操作数的第 127:16 位复制自第一源操作数中的对应位。目的操作数的第 MAXVL-1:128 位清零。目的操作数的低位 FP16 元素根据写掩码更新。

对于每个输入的 FP16 值 x，转换操作为：

GetMant(x) = ±2k|x.significand|

其中：

1 ≤ |x.significand| < 2

无偏指数 k 取决于由 interv 定义的区间范围，以及源的指数是偶数还是奇数。最终结果的符号由符号控制、源符号和前导小数位决定。

imm8[1:0] 的编码值和符号控制如表 5-19 所示。

每个转换后的 FP16 结果都根据符号控制、无偏指数 k（加上偏置）和规范化到由 interv 指定范围的尾数进行编码。

在处理浮点特殊数值时，GetMant() 函数遵循表 5-20。

操作¶

VGETMANTSH dest{k1}, src1, src2, imm8¶

sign_control := imm8[3:2]
normalization_interval := imm8[1:0]
IF k1[0] or *no writemask*:
    dest.fp16[0] := getmant_fp16(src2.fp16[0],
            // see VGETMANTPH
        normalization_interval)
ELSE IF *zeroing*:
    dest.fp16[0] := 0
//else dest.fp16[0] remains unchanged
DEST[127:16] := src1[127:16]
DEST[MAXVL-1:128] := 0
