描述 ¶

检查 EFLAGS 寄存器中一个或多个状态标志位的状态（CF、OF、PF、SF 和 ZF），如果标志位处于指定的状态（条件满足），则执行跳转到目的操作数指定的目标指令。每个指令都关联一个条件码（cc）以指示所测试的条件。如果条件不满足，则不执行跳转，程序继续执行 Jcc 指令之后的下一条指令。

目标指令是通过相对偏移量（相对于 EIP 寄存器中当前指令指针值的有符号偏移量）指定的。相对偏移量（rel8、rel16 或 rel32）在汇编代码中通常指定为标签，但在机器码级别，它被编码为有符号的 8 位或 32 位立即数，并被加到指令指针上。对于 –128 到 +127 的偏移量，指令编码最有效。如果操作数大小属性为 16，则清除 EIP 寄存器的高两个字节，从而使最大指令指针大小为 16 位。

每个 Jcc 助记符的条件在上一页表格的“描述”列中给出。术语“大于（greater）”和“小于（less）”用于有符号整数的比较，而“高于（above）”和“低于（below）”用于无符号整数的比较。

由于状态标志位的特定状态有时可以有两种解释方式，因此为某些操作码定义了两个助记符。例如，JA（如果高于则跳转）指令和 JNBE（如果非低于或等于则跳转）指令是操作码 77H 的备用助记符。

Jcc 指令不支持远跳转（跳转到其他代码段）。当条件跳转的目标位于不同的段中时，请使用与 Jcc 指令测试的条件相反的条件，然后使用无条件远跳转（JMP 指令）访问其他段中的目标。例如，以下条件远跳转是非法的：

JZ FARLABEL;

要实现此远跳转，请使用以下两条指令：

JNZ BEYOND;

JMP FARLABEL;

BEYOND:

JRCXZ、JECXZ 和 JCXZ 指令与其他 Jcc 指令不同，因为它们不检查状态标志位。相反，它们检查 RCX、ECX 或 CX 是否为 0。具体检查哪个寄存器由地址大小属性决定。当在以条件循环指令（如 LOOPNE）结束的循环开头使用这些指令时，它们非常有用。它们可用于在 RCX、ECX 或 CX 为 0 时防止指令序列进入循环。否则，循环将执行 264、232 或 64K 次（而不是零次）。

无论跳转地址或是否可缓存，所有条件跳转都会转换为一个或两个缓存行的代码提取。

在 64 位模式下，操作数大小固定为 64 位。短跳转是 RIP = RIP + 符号扩展至 64 位的 8 位偏移量。近跳转是 RIP = RIP + 符号扩展至 64 位的 32 位偏移量。

操作 ¶ 

IF condition
    THEN
        tempEIP := EIP + SignExtend(DEST);
        IF OperandSize = 16
            THEN tempEIP := tempEIP AND 0000FFFFH;
        FI;
    IF tempEIP is not within code segment limit
        THEN #GP(0);
        ELSE EIP := tempEIP
    FI;
FI;

受影响的标志位 ¶

无。

保护模式异常 ¶

#GP(0) 如果跳转的目标偏移量超出了 CS 段的限制。
#UD 如果使用了 LOCK 前缀。

实地址模式异常 ¶

#GP 如果跳转的目标偏移量超出 CS 段的限制，或者在 0 到 FFFFH 的有效地址空间之外。如果使用了 32 位地址大小覆盖前缀，则可能会发生这种情况。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常 ¶

与实地址模式下的异常相同。

兼容模式异常 ¶

与保护模式下的异常相同。

64位模式异常 ¶

#GP(0) 如果内存地址为非规范形式。
#UD 如果使用了 LOCK 前缀。
