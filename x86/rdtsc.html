RDTSC — 读取时间戳计数器

操作码* 指令 Op/En 64位模式 兼容/传统模式 描述
0F 31 RDTSC ZO Valid Valid 将时间戳计数器读取到 EDX:EAX 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将处理器时间戳计数器（一个 64 位的 MSR）的当前值读取到 EDX:EAX 寄存器中。EDX 寄存器加载 MSR 的高 32 位，EAX 寄存器加载低 32 位。（在支持 Intel 64 架构的处理器上，RAX 和 RDX 各自的高 32 位被清除。）

处理器在每个时钟周期单调递增时间戳计数器 MSR，并在处理器复位时将其复位为 0。有关时间戳计数器行为的具体细节，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3B 卷第 18 章中的“时间戳计数器”。

CR4 寄存器中的时间戳禁用（TSD）标志限制 RDTSC 指令的使用如下：当该标志清零时，可以在任何特权级别执行 RDTSC 指令；当该标志置位时，该指令只能在特权级别 0 执行。

在特权级别 0 执行时，也可以使用 RDMSR 指令读取时间戳计数器。

RDTSC 指令不是序列化指令。在读取计数器之前，它不一定会等待所有先前的指令执行完毕。同样，后续指令可能会在执行读取操作之前开始执行。以下项目可以指导寻求对 RDTSC 执行进行排序的软件：

  • 如果软件要求仅在所有先前指令都已执行且所有先前加载都全局可见之后才执行 RDTSC1，则它可以在 RDTSC 之前立即执行 LFENCE。
  • 如果软件要求仅在所有先前指令都已执行且所有先前加载和存储都全局可见之后才执行 RDTSC，则它可以在 RDTSC 之前立即执行序列 MFENCE;LFENCE。
  • 如果软件要求在执行任何后续指令（包括任何 memory 访问）之前执行 RDTSC，则它可以在 RDTSC 之后立即执行序列 LFENCE。

此指令由 Pentium 处理器引入。

有关此指令在 VMX 非根操作中的行为的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3C 卷第 26 章中的“VMX 非根操作中的指令行为更改”。

1. 当要加载的值被确定时，加载被认为是全局可见的。

操作¶

IF (CR4.TSD = 0) or (CPL = 0) or (CR0.PE = 0)
    THEN EDX:EAX := TimeStampCounter;
    ELSE (* CR4.TSD = 1 and (CPL = 1, 2, or 3) and CR0.PE = 1 *)
        #GP(0);
FI;

影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果寄存器 CR4 中的 TSD 标志置位且 CPL 大于 0。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#GP(0) 如果寄存器 CR4 中的 TSD 标志置位。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

与保护模式下的异常相同。
