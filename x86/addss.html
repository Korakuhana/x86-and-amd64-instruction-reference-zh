ADDSS — 加法标量单精度浮点值

操作码/指令 Op / En 64/32位模式支持 CPUID 特征标志 描述
F3 0F 58 /r ADDSS xmm1, xmm2/m32 A V/V SSE 将打包在 xmm2/mem 中的低位单精度浮点值加到 xmm1，并将结果存入 xmm1。
VEX.LIG.F3.0F.WIG 58 /r VADDSS xmm1,xmm2, xmm3/m32 B V/V AVX 将打包在 xmm3/mem 中的低位单精度浮点值加到 xmm2，并将结果存入 xmm1。
EVEX.LLIG.F3.0F.W0 58 /r VADDSS xmm1{k1}{z}, xmm2, xmm3/m32{er} C V/V AVX512F 使用写掩码 k1，将打包在 xmm3/m32 中的低位单精度浮点值加到 xmm2，并将结果存入 xmm1。

描述¶

将第二源操作数中的低位单精度浮点值与第一源操作数中的低位单精度浮点值相加，并将双精度浮点结果（注：原文如此，实际应为单精度）存入目标操作数。

第二源操作数可以是 XMM 寄存器或 64 位（注：实际为 32 位）内存地址。第一源和目标操作数是 XMM 寄存器。

128 位传统 SSE 版本：第一源和目标操作数相同。相应目标寄存器的高位（MAXVL-1:32）保持不变。

EVEX 和 VEX.128 编码版本：第一源操作数由 EVEX.vvvv/VEX.vvvv 编码。目标 XMM 寄存器的位（127:32）从第一源操作数中的相应位复制。目标寄存器的位（MAXVL-1:128）清零。

EVEX 版本：目标的低双字（doubleword）元素根据写掩码进行更新。

软件应确保 VADDSS 以 VEX.L=0 编码。在不同的处理器世代中，以 VEX.L=1 编码 VADDSS 可能会遇到不可预测的行为。

操作¶

VADDSS (EVEX Encoded Versions)

IF (EVEX.b = 1) AND SRC2 *is a register*
    THEN
        SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(EVEX.RC);
    ELSE
        SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(MXCSR.RC);
FI;
IF k1[0] or *no writemask*
    THEN DEST[31:0] := SRC1[31:0] + SRC2[31:0]
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[31:0] remains unchanged*
            ELSE ; zeroing-masking
                THEN DEST[31:0] := 0
        FI;
FI;
DEST[127:32] := SRC1[127:32]
DEST[MAXVL-1:128] := 0

VADDSS DEST, SRC1, SRC2 (VEX.128 Encoded Version)

DEST[31:0] := SRC1[31:0] + SRC2[31:0]
DEST[127:32] := SRC1[127:32]
DEST[MAXVL-1:128] := 0

ADDSS DEST, SRC (128-bit Legacy SSE Version)

DEST[31:0] := DEST[31:0] + SRC[31:0]
DEST[MAXVL-1:32] (Unmodified)
