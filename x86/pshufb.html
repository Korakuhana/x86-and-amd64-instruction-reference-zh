描述¶

PSHUFB 根据源操作数（第二个操作数）中的置换控制掩码，对目的操作数（第一个操作数）中的字节进行就位置换。该指令对目的操作数中的数据进行排列，保持置换掩码不受影响。如果置换控制掩码每个字节的最高有效位（位 [7]）被设置，则在结果字节中写入常数零。置换控制掩码中的每个字节都形成一个索引，用于排列目的操作数中的对应字节。每个索引的值是置换控制字节的最低 4 位（128 位操作）或 3 位（64 位操作）。当源操作数是 128 位内存操作数时，操作数必须对齐到 16 字节边界，否则将产生通用保护异常 (#GP)。

在 64 位模式下且未采用 VEX/EVEX 编码时，使用 REX 前缀可以访问 XMM8-XMM15 寄存器。

传统 SSE 版本 64 位操作数：两个操作数都可以是 MMX 寄存器。

128位传统 SSE 版本：第一个源操作数与目的操作数相同。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：目的操作数是第一个操作数，第一个源操作数是第二个操作数，第二个源操作数是第三个操作数。目的 YMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：目的 YMM 寄存器的位 (255:128) 存储使用第二个源操作数的高 16 字节作为控制掩码对第一个源操作数的高 16 字节进行 16 字节置换的结果。

高 128 位通道的每个索引值是对应置换控制字节的最低 4 位。索引值在每个 128 位通道内选择一个源数据元素。

EVEX 编码版本：第二个源操作数是 ZMM/YMM/XMM 寄存器或 512/256/128 位内存位置。第一个源操作数和目的操作数是 ZMM/YMM/XMM 寄存器。目的操作数在写掩码 k1 下条件性地更新。

EVEX 和 VEX 编码版本：四/两个通道内 128 位置换。

操作¶

PSHUFB（配合 64 位操作数）¶

TEMP := DEST
for i = 0 to 7 {
    if (SRC[(i * 8)+7] = 1 ) then
            DEST[(i*8)+7...(i*8)+0] := 0;
    else
            index[2..0] := SRC[(i*8)+2 .. (i*8)+0];
            DEST[(i*8)+7...(i*8)+0] := TEMP[(index*8+7)..(index*8+0)];
    endif;
}
PSHUFB (with 128 bit operands)
TEMP := DEST
for i = 0 to 15 {
    if (SRC[(i * 8)+7] = 1 ) then

        else
            index[3..0] := SRC[(i*8)+3 .. (i*8)+0];
            DEST[(i*8)+7..(i*8)+0] := TEMP[(index*8+7)..(index*8+0)];

}

VPSHUFB（VEX.128 编码版本）¶

for i = 0 to 15 {
    if (SRC2[(i * 8)+7] = 1) then
        DEST[(i*8)+7..(i*8)+0] := 0;
        else
        index[3..0] := SRC2[(i*8)+3 .. (i*8)+0];
        DEST[(i*8)+7..(i*8)+0] := SRC1[(index*8+7)..(index*8+0)];
    endif
}
DEST[MAXVL-1:128] := 0

VPSHUFB（VEX.256 编码版本）¶

for i = 0 to 15 {
    if (SRC2[(i * 8)+7] == 1 ) then
        DEST[(i*8)+7..(i*8)+0] := 0;
        else
        index[3..0] := SRC2[(i*8)+3 .. (i*8)+0];
        DEST[(i*8)+7..(i*8)+0] := SRC1[(index*8+7)..(index*8+0)];
    endif
    if (SRC2[128 + (i * 8)+7] == 1 ) then
        DEST[128 + (i*8)+7..(i*8)+0] := 0;
        else
        index[3..0] := SRC2[128 + (i*8)+3 .. (i*8)+0];
        DEST[128 + (i*8)+7..(i*8)+0] := SRC1[128 + (index*8+7)..(index*8+0)];
    endif
}

VPSHUFB（EVEX 编码版本）¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
jmask := (KL-1) & ~0xF
                // 0x00, 0x10, 0x30 depending on the VL
FOR j = 0 TO KL-1
                // dest
    IF kl[ i ] or no_masking
        index := src.byte[ j ];
        IF index & 0x80
            Dest.byte[ j ] := 0;
        ELSE
            index := (index & 0xF) + (j & jmask);
                // 16-element in-lane lookup
            Dest.byte[ j ] := src.byte[ index ];
    ELSE if zeroing
        Dest.byte[ j ] := 0;
DEST[MAXVL-1:VL] := 0;
