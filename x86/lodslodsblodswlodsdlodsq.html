描述¶

从源操作数中将一个字节、字或双字分别加载到 AL、AX 或 EAX 寄存器中。源操作数是一个内存位置，其地址从 DS:ESI 或 DS:SI 寄存器读取（分别取决于指令的地址大小属性是 32 还是 16）。可以使用段覆盖前缀来覆盖 DS 段。

在汇编代码级别，此指令允许两种形式：“显式操作数”形式和“无操作数”形式。显式操作数形式（使用 LODS 助记符指定）允许显式指定源操作数。在此处，源操作数应为指示源值的大小的符号。目标操作数然后自动选择以匹配源操作数的大小（AL 寄存器用于字节操作，AX 用于字操作，EAX 用于双字操作）。提供此显式操作数形式是为了便于编写文档；然而，请注意，此形式提供的文档可能会误导。也就是说，源操作数符号必须指定正确的类型（大小）（字节、字或双字），但不必指定正确的位置。位置始终由 DS:(E)SI 寄存器指定，在执行加载字符串指令之前必须正确加载这些寄存器。

无操作数形式提供了 LODS 指令的字节、字和双字版本的“简短形式”。在这里，也假定 DS:(E)SI 是源操作数，并且 AL、AX 或 EAX 寄存器被假定为目标操作数。源和目标操作数的大小由助记符选择：LODSB（字节加载到 AL 寄存器）、LODSW（字加载到 AX）、或者 LODSD（双字加载到 EAX）。

在将字节、字或双字从内存位置传输到 AL、AX 或 EAX 寄存器之后，(E)SI 寄存器根据 EFLAGS 寄存器中 DF 标志的设置自动递增或递减。（如果 DF 标志为 0，则 (E)SI 寄存器递增；如果 DF 标志为 1，则 ESI 寄存器递减。）对于字节操作，(E)SI 寄存器递增或递减 1；对于字操作，递增或递减 2；对于双字操作，递增或递减 4。

在 64 位模式下，使用 REX.W 前缀可将操作提升为 64 位。LODS/LODSQ 将地址 (R)SI 处的四字加载到 RAX 中。然后根据 EFLAGS 寄存器中 DF 标志的设置，自动递增或递减 (R)SI 寄存器。

LODS、LODSB、LODSW 和 LODSD 指令之前可以带有 REP 前缀，用于 ECX 个字节、字或双字的块加载。然而更常见的是，这些指令在 LOOP 结构中使用，因为在下一次传输之前，通常需要对移动到寄存器中的数据进行进一步处理。有关 REP 前缀的描述，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 2B 卷第 4 章中的“REP/REPE/REPZ /REPNE/REPNZ — 重复字符串操作前缀”。

操作¶

IF AL := SRC; (* Byte load *)
    THEN AL := SRC; (* Byte load *)
        IF DF = 0
            THEN (E)SI := (E)SI + 1;
            ELSE (E)SI := (E)SI – 1;
        FI;
ELSE IF AX := SRC; (* Word load *)
    THEN IF DF = 0
            THEN (E)SI := (E)SI + 2;
            ELSE (E)SI := (E)SI – 2;
        IF;
    FI;
ELSE IF EAX := SRC; (* Doubleword load *)
    THEN IF DF = 0
            THEN (E)SI := (E)SI + 4;
            ELSE (E)SI := (E)SI – 4;
        FI;
    FI;
ELSE IF RAX := SRC; (* Quadword load *)
    THEN IF DF = 0
            THEN (R)SI := (R)SI + 8;
            ELSE (R)SI := (R)SI – 8;
        FI;
    FI;
FI;

影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择器。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
