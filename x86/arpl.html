ARPL — 调整段选择子的 RPL 字段

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
63 /r ARPL r/m16, r16 MR N. E. Valid 调整 r/m16 的 RPL，使其不小于 r16 的 RPL。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
MR ModRM:r/m (w) ModRM:reg (r) N/A N/A

描述¶

比较两个段选择子的 RPL 字段。第一操作数（目标操作数）包含一个段选择子，第二操作数（源操作数）包含另一个。（RPL 字段位于每个操作数的第 0 和 1 位。）如果目标操作数的 RPL 字段小于源操作数的 RPL 字段，则设置 ZF 标志，并将目标操作数的 RPL 字段增加到与源操作数的 RPL 字段相匹配。否则，清除 ZF 标志，且不对目标操作数进行任何更改。（目标操作数可以是一个字寄存器或内存地址；源操作数必须是一个字寄存器。）

ARPL 指令供操作系统程序使用（不过，它也可以由应用程序使用）。它通常用于调整应用程序传递给操作系统的段选择子的 RPL，使其与应用程序的特权级相匹配。在此，传递给操作系统的段选择子被放置在目标操作数中，而应用程序代码段的段选择子被放置在源操作数中。（源操作数中的 RPL 字段代表应用程序的特权级。）执行 ARPL 指令可以确保操作系统接收到的段选择子的 RPL 不低于（即不具有更高的特权）应用程序的特权级（在过程调用后，可以从栈中读取应用程序代码段的段选择子）。

该指令在兼容模式和传统模式下的执行情况如上所述。在 64 位模式下无法编码。

有关使用该指令的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3A 卷第 3 章“保护模式内存管理”中的“检查调用者访问权限”。

操作¶

IF 64-BIT MODE
    THEN
        See MOVSXD;
    ELSE
        IF DEST[RPL] < SRC[RPL]
            THEN
                ZF := 1;
                DEST[RPL] := SRC[RPL];
            ELSE
                ZF := 0;
        FI;
FI;

影响的标志¶

如果目标操作数的 RPL 字段小于源操作数的 RPL 字段，则将 ZF 标志设置为 1；否则，将其设置为 0。

保护模式异常¶

#GP(0) 如果目标位于不可写段中。
如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段界限。
如果使用 DS、ES、FS 或 GS 寄存器访问内存，且它包含空段选择子。
#SS(0) 如果内存操作数有效地址超出了 SS 段界限。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 在实地址模式下无法识别 ARPL 指令。
如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#UD 在虚拟-8086 模式下无法识别 ARPL 指令。
如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

不适用。
