MOVBE — 交换字节后移动数据

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
0F 38 F0 /r MOVBE r16, m16 RM 有效/有效 MOVBE 反转 m16 中的字节顺序并移动到 r16。
0F 38 F0 /r MOVBE r32, m32 RM 有效/有效 MOVBE 反转 m32 中的字节顺序并移动到 r32。
REX.W + 0F 38 F0 /r MOVBE r64, m64 RM 有效/非有效 MOVBE 反转 m64 中的字节顺序并移动 to r64。
0F 38 F1 /r MOVBE m16, r16 MR 有效/有效 MOVBE 反转 r16 中的字节顺序并移动到 m16。
0F 38 F1 /r MOVBE m32, r32 MR 有效/有效 MOVBE 反转 r32 中的字节顺序并移动到 m32。
REX.W + 0F 38 F1 /r MOVBE m64, r64 MR 有效/非有效 MOVBE 反转 r64 中的字节顺序并移动到 m64。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (w) ModRM:r/m (r) N/A N/A
MR ModRM:r/m (w) ModRM:reg (r) N/A N/A

描述¶

对从第二个操作数（源操作数）复制的数据执行字节交换操作，并将结果存储在第一个操作数（目的操作数）中。源操作数可以是通用寄存器或内存位置；目的操作数可以是通用寄存器或内存位置；但两个操作数不能同时为寄存器，且只有一个操作数可以是内存位置。两个操作数必须具有相同的大小，可以是字、双字或四字。

提供 MOVBE 指令用于在从内存读取或写入内存时交换字节；从而支持将小端（little-endian）值转换为大端（big-endian）格式，反之亦然。

在 64 位模式下，此指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问其他寄存器 (R8-R15)。使用 REX.W 前缀可将操作提升为 64 位。请参阅本节开头的汇总图表以获取编码数据和限制。

操作¶

TEMP := SRC
IF ( OperandSize = 16)
    THEN
        DEST[7:0] := TEMP[15:8];
        DEST[15:8] := TEMP[7:0];
    ELES IF ( OperandSize = 32)
        DEST[7:0] := TEMP[31:24];
        DEST[15:8] := TEMP[23:16];
        DEST[23:16] := TEMP[15:8];
        DEST[31:23] := TEMP[7:0];
    ELSE IF ( OperandSize = 64)
        DEST[7:0] := TEMP[63:56];
        DEST[15:8] := TEMP[55:48];
        DEST[23:16] := TEMP[47:40];
        DEST[31:24] := TEMP[39:32];
        DEST[39:32] := TEMP[31:24];
        DEST[47:40] := TEMP[23:16];
        DEST[55:48] := TEMP[15:8];
        DEST[63:56] := TEMP[7:0];
FI;

受影响的标志位¶

无。

保护模式异常¶

#GP(0) 如果目的操作数位于不可写的段中。
如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含一个 NULL 段选择子。
#SS(0) 如果内存操作数有效地址超出了 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
#UD 如果 CPUID.01H:ECX.MOVBE[位 22] = 0。
如果使用了 LOCK 前缀。
如果使用了 REP (F3H) 前缀。

#GP 如果内存操作数有效地址超出了 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出了 SS 段限制。
#UD 如果 CPUID.01H:ECX.MOVBE[位 22] = 0。
