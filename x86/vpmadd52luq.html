VPMADD52LUQ — 打包无符号 52 位整数乘法并将低 52 位乘积加到 64 位累加器

操作码/指令 Op/En 64/32位模式支持 CPUID 描述
EVEX.128.66.0F38.W1 B4 /r VPMADD52LUQ xmm1 {k1}{z}, xmm2,xmm3/m128/m64bcst A V/V AVX512_IFMA AVX512VL 将 xmm2 和 xmm3/m128 中的无符号 52 位整数相乘，并在写掩码 k1 的控制下将 104 位乘积的低 52 位加到 xmm1 中的四字无符号整数中。
EVEX.256.66.0F38.W1 B4 /r VPMADD52LUQ ymm1 {k1}{z}, ymm2, ymm3/m256/m64bcst A V/V AVX512_IFMA AVX512VL 将 ymm2 和 ymm3/m256 中的无符号 52 位整数相乘，并在写掩码 k1 的控制下将 104 位乘积的低 52 位加到 ymm1 中的四字无符号整数中。
EVEX.512.66.0F38.W1 B4 /r VPMADD52LUQ zmm1 {k1}{z}, zmm2,zmm3/m512/m64bcst A V/V AVX512_IFMA 将 zmm2 和 zmm3/m512 中的无符号 52 位整数相乘，并在写掩码 k1 的控制下将 104 位乘积的低 52 位加到 zmm1 中的四字无符号整数中。

指令操作数编码¶

描述¶

将第一源操作数（第二操作数）每个四字元素中的打包无符号 52 位整数，与第二源操作数（第三操作数）对应元素中的打包无符号 52 位整数相乘，以形成打包的 104 位中间结果。在写掩码 k1 控制下，将每个 104 位乘积的低 52 位无符号整数加到目的操作数（第一操作数）的对应四字无符号整数中。

第一源操作数是 ZMM/YMM/XMM 寄存器。第二源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置，或从 64 位内存位置广播的 512/256/128 位向量。目的操作数是一个 ZMM/YMM/XMM 寄存器，以 64 位粒度在写掩码 k1 控制下有条件地更新。

操作¶

VPMADD52LUQ (EVEX 编码)¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 64;
    IF k1[j] OR *no writemask* THEN
        IF src2 is Memory AND EVEX.b=1 THEN
            tsrc2[63:0] := ZeroExtend64(src2[51:0]);
        ELSE
            tsrc2[63:0] := ZeroExtend64(src2[i+51:i];
        FI;
        Temp128[127:0] := ZeroExtend64(src1[i+51:i]) * tsrc2[63:0];
        Temp2[63:0] := DEST[i+63:i] + ZeroExtend64(temp128[51:0]) ;
        DEST[i+63:i] := Temp2[63:0];
    ELSE
        IF *zeroing-masking* THEN
            DEST[i+63:i] := 0;
        ELSE *merge-masking*
            DEST[i+63:i] is unchanged;
        FI;
    FI;
ENDFOR
DEST[MAX_VL-1:VL] := 0;
