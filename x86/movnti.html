MOVNTI — 使用非临时提示存储双字

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
NP 0F C3 /r MOVNTI m32, r32 MR 有效/有效 SSE2 使用非临时提示将双字从 r32 移动到 m32。
NP REX.W + 0F C3 /r MOVNTI m64, r64 MR 有效/非有效 SSE2 使用非临时提示将四字从 r64 移动到 m64。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
MR ModRM:r/m (w) ModRM:reg (r) N/A N/A

描述¶

将源操作数（第二个操作数）中的双字整数使用非临时提示移动到目的操作数（第一个操作数）中，以在写入内存期间最大程度减少缓存污染。源操作数是通用寄存器。目的操作数是 32 位内存位置。

非临时提示是通过在将数据写入内存时使用写合并 (WC) 内存类型协议来实现的。使用该协议，处理器不会将数据写入缓存层次结构，也不会从内存中将相应的缓存行提取到缓存层次结构中。如果要写入的区域的内存类型在不可缓存 (UC) 或写保护 (WP) 内存区域中，则为非临时存储指定的内存地址可以覆盖非临时提示。有关非临时存储的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷第 10 章中的“时间数据与非时间数据的缓存”。

由于 WC 协议使用弱有序的内存一致性模型，如果多个处理器可能会使用不同的内存类型来读取/写入目的内存位置，则应将使用 SFENCE 或 MFENCE 指令实现的屏障操作与 MOVNTI 指令结合使用。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 前缀允许访问其他寄存器 (R8-R15)。使用 REX.W 前缀可将操作提升为 64 位。请参阅本节开头的汇总图表以获取编码数据和限制。

SIMD 浮点异常¶

无。

保护模式异常¶

#GP(0) 如果 CS、DS、ES、FS 或 GS 段中存在非法的内存操作数有效地址。
#SS(0) 如果 SS 段中的地址非法。
#PF(错误码) 如果发生缺页异常。
#UD 如果 CPUID.01H:EDX.SSE2[位 26] = 0。
如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果操作数的任何部分超出了从 0 到 FFFFH 的有效地址空间。
#UD 如果 CPUID.01H:EDX.SSE2[位 26] = 0。
如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

与实地址模式下的异常相同。

#PF(错误码) 如果发生缺页异常。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果指向 SS 段的内存地址为非规范形式。
#GP(0) 如果内存地址为非规范形式。
#PF(错误码) 如果发生缺页异常。
#UD 如果 CPUID.01H:EDX.SSE2[位 26] = 0。
如果使用了 LOCK 前缀。
#AC(0) 如果启用了对齐检查，并且在当前特权级为 3 时进行了未对齐的内存引用。
