描述 ¶

递减栈指针，然后将源操作数存储在栈顶。地址和操作数大小的确定和使用如下：

  • 地址大小。当前代码段描述符中的 D 标志决定默认的地址大小；它可以通过指令前缀（67H）进行覆盖。地址大小仅在引用内存中的源操作数时使用。
  • 操作数大小。当前代码段描述符中的 D 标志决定默认的操作数大小；它可以通过指令前缀（66H 或 REX.W）进行覆盖。操作数大小（16、32 或 64 位）决定栈指针递减的量（2、4 或 8）。

如果源操作数是大小小于操作数大小的立即数，则将符号扩展后的值压入栈中。如果源操作数是段寄存器（16 位）且操作数大小为 64 位，则将零扩展后的值压入栈中；如果操作数大小为 32 位，则要么将零扩展后的值压入栈中，要么使用 16 位传送将段选择子写入栈中。对于最后一种情况，所有最近的 Intel Core 和 Intel Atom 处理器都会执行 16 位传送，使栈位置的高位部分保持未修改状态。

  • 栈地址大小。在 64 位模式之外，当前栈段描述符中的 B 标志决定栈指针的大小（16 或 32 位）；在 64 位模式下，栈指针的大小始终为 64 位。

栈地址大小决定在向内存中的栈写入数据以及递减栈指针时栈指针的宽度。（如上所述，栈指针递减的量由操作数大小决定。）

如果操作数大小小于栈地址大小，则 PUSH 指令可能会导致栈指针未对齐（即栈指针未对齐在双字或四字边界上）。

PUSH ESP 指令压入在执行该指令之前已存在的 ESP 寄存器值。如果 PUSH 指令使用的内存操作数中使用了 ESP 寄存器来计算操作数地址，则在递减 ESP 寄存器之前计算操作数的地址。

如果在实地址模式下执行 PUSH 指令时 ESP 或 SP 寄存器为 1，则会生成栈故障异常（#SS）（因为违反了栈段的界限）。其交付会遇到第二个栈故障异常（出于同样的原因），从而导致生成双重故障异常（#DF）。双重故障异常的交付会遇到第三个栈故障异常，逻辑处理器进入关机模式。参见 Intel® 64 和 IA-32 架构软件开发人员手册第 3A 卷第 6 章中关于双重故障异常的讨论。

IA-32 架构兼容性 ¶

对于从 Intel 286 开始的 IA-32 处理器，PUSH ESP 指令压入在执行该指令之前已存在的 ESP 寄存器值。（对于 Intel 64 架构、IA-32 架构的实地址和虚拟-8086 模式也是如此。）对于 Intel® 8086 处理器，PUSH SP 指令压入 SP 寄存器的新值（即递减 2 后的值）。

操作 ¶ 

(* See Description section for possible sign-extension or zero-extension of source operand and for *)
(* a case in which the size of the memory store may be
                    smaller than the instruction’s operand size *)
IF StackAddrSize = 64
    THEN
        IF OperandSize = 64
            THEN
                RSP := RSP – 8;
                Memory[SS:RSP] := SRC;
                    (* push quadword *)
        ELSE IF OperandSize = 32
            THEN
                RSP := RSP – 4;
                Memory[SS:RSP] := SRC;
                    (* push dword *)
            ELSE (* OperandSize = 16 *)
                RSP := RSP – 2;
                Memory[SS:RSP] := SRC;
                    (* push word *)
        FI;
ELSE IF StackAddrSize = 32
    THEN
        IF OperandSize = 64
            THEN
                ESP := ESP – 8;
                Memory[SS:ESP] := SRC;
                    (* push quadword *)
        ELSE IF OperandSize = 32
            THEN
                ESP := ESP – 4;
                Memory[SS:ESP] := SRC;
                    (* push dword *)
            ELSE (* OperandSize = 16 *)
                ESP := ESP – 2;
                Memory[SS:ESP] := SRC;
                    (* push word *)
        FI;
    ELSE (* StackAddrSize = 16 *)
        IF OperandSize = 32
            THEN
                SP := SP – 4;
                Memory[SS:SP] := SRC;
                    (* push dword *)
            ELSE (* OperandSize = 16 *)
                SP := SP – 2;
                Memory[SS:SP] := SRC;
                    (* push word *)
        FI;
FI;

受影响的标志位 ¶

无。

保护模式异常 ¶

#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段界限。
如果使用 DS、ES、FS 或 GS 寄存器访问内存且它包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段界限。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常 ¶

#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段界限。
#SS 如果内存操作数的有效地址超出 SS 段界限。
如果 SP 或 ESP 寄存器的新值超出了栈段限制。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常 ¶

#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段界限。
#SS(0) 如果内存操作数的有效地址超出 SS 段界限。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常 ¶

与保护模式下的异常相同。

64位模式异常 ¶

#GP(0) 如果内存地址为非规范形式。
#SS(0) 如果栈地址为非规范形式。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
如果 PUSH 的是 CS、SS、DS 或 ES 段寄存器。
