描述¶

VFNMADD132SS：将来自第一个源操作数的低位打包单精度浮点值与第三个源操作数中的低位打包单精度浮点值相乘，将取反的无限精度中间结果与第二个源操作数中的低位打包单精度浮点值相加，执行舍入并将结果的打包单精度浮点值存储到目的操作数（第一个源操作数）中。

VFNMADD213SS：将来自第二个源操作数的低位打包单精度浮点值与第一个源操作数中的低位打包单精度浮点值相乘，将取反的无限精度中间结果与第三个源操作数中的低位打包单精度浮点值相加，执行舍入并将结果的打包单精度浮点值存储到目的操作数（第一个源操作数）中。

VFNMADD231SS：将来自第二个源操作数的低位打包单精度浮点值与第三个源操作数中的低位打包单精度浮点值相乘，将取反的无限精度中间结果与第一个源操作数中的低位打包单精度浮点值相加，执行舍入并将结果的打包单精度浮点值存储到目的操作数（第一个源操作数）中。

VEX.128 和 EVEX 编码版本：目的操作数（也是第一个源操作数）在 reg_field 中编码。第二个源操作数在 VEX.vvvv/EVEX.vvvv 中编码。第三个源操作数在 rm_field 中编码。目的寄存器的位 127:32 保持不变。目的寄存器的位 MAXVL-1:128 被清零。

EVEX 编码版本：目的寄存器的低双字元素根据写掩码进行更新。

编译器工具可以为汇总表的操作码/指令列中列出的每个指令助记符有选择地支持互补助记符。互补助记符在涉及 NaN 的情况下的行为由操作码/指令列中定义的指令助记符的定义所控制。

操作¶

在以下操作中，“*”和“+”符号表示使用无限精度输入和输出进行乘法 and 加法（无舍入）。

VFNMADD132SS DEST, SRC2, SRC3 (EVEX 编码版本)¶

IF (EVEX.b = 1) and SRC3 *is a register*
    THEN
        SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(EVEX.RC);
    ELSE
        SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(MXCSR.RC);
FI;
IF k1[0] or *no writemask*
    THEN DEST[31:0] := RoundFPControl(-(DEST[31:0]*SRC3[31:0]) + SRC2[31:0])
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[31:0] remains unchanged*
            ELSE ; zeroing-masking
                THEN DEST[31:0] := 0
        FI;
FI;
DEST[127:32] := DEST[127:32]
DEST[MAXVL-1:128] := 0
