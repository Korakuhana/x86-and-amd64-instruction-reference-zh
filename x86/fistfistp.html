FIST/FISTP — 存储整数

操作码 指令 64位模式 兼容/传统模式 描述
DF /2 FIST m16int 有效 有效 将 ST(0) 存储到 m16int 中。
DB /2 FIST m32int 有效 有效 将 ST(0) 存储到 m32int 中。
DF /3 FISTP m16int 有效 有效 将 ST(0) 存储到 m16int 中并弹出寄存器栈。
DB /3 FISTP m32int 有效 有效 将 ST(0) 存储到 m32int 中并弹出寄存器栈。
DF /7 FISTP m64int 有效 有效 将 ST(0) 存储到 m64int 中并弹出寄存器栈。

描述¶

FIST 指令将 ST(0) 寄存器中的值转换为有符号整数，并将结果存储在目的操作数中。值可以以字或双字整数格式存储。目的操作数指定要存储目的值第一个字节的地址。

FISTP 指令执行与 FIST 指令相同的操作，然后弹出寄存器栈。为了弹出寄存器栈，处理器将 ST(0) 寄存器标记为空，并将栈指针 (TOP) 递增 1。FISTP 指令还可以以四字整数格式存储值。

下表显示了在以整数格式存储各类数字时获得的结果。

如果源值是非整数值，则根据 FPU 控制字的 RC 字段指定的舍入模式将其舍入为整数值。

如果转换后的值对于目的格式过大，或者源操作数是 ∞、SNaN、QNaN 或处于不受支持的格式，则会触发无效算术操作数条件。如果未屏蔽无效操作异常，则会产生无效算术操作数异常 (#IA)，且不会在目的操作数中存储任何值。如果屏蔽了无效操作异常，则在内存中存储整数不确定值。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

DEST := Integer(ST(0));
IF Instruction = FISTP
    THEN
        PopRegisterStack;
FI;

影响的 FPU 标志寄存器¶

C1 如果发生栈下溢，则设置为 0。
如果产生不精确异常 (#P)，则指示舍入方向：0 := 未向上舍入；1 := 向上舍入。
否则设置为 0。
C0, C2, C3 未定义。

浮点异常¶

#IS 发生栈下溢。
#IA 转换后的值对于目的格式过大。
源操作数是 SNaN、QNaN、±∞ 或不受支持的格式。
#P 值无法在目的格式中精确表示。

保护模式异常¶

#GP(0) 如果目的位于不可写段中。
如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果使用 DS、ES、FS 或 GS 寄存器访问内存，并且它包含空段选择器。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#UD 如果使用了 LOCK 前缀。

虚拟-8086 模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果内存地址为非规范形式。
#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在未决的 x87 FPU 异常。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
