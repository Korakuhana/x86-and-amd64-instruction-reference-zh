NEG — 求补码（求相反数）

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
F6 /3 NEG r/m8 M Valid Valid 对 r/m8 取二进制补码（求相反数）。
REX + F6 /3 NEG r/m81 M Valid N.E. 对 r/m8 取二进制补码（求相反数）。
F7 /3 NEG r/m16 M Valid Valid 对 r/m16 取二进制补码（求相反数）。
F7 /3 NEG r/m32 M Valid Valid 对 r/m32 取二进制补码（求相反数）。
REX.W + F7 /3 NEG r/m64 M Valid N.E. 对 r/m64 取二进制补码（求相反数）。

1. 在 64 位模式下，如果使用 REX 前缀，则无法编码 r/m8 以访问以下字节寄存器：AH、BH、CH、DH。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r, w) N/A N/A N/A

描述¶

将操作数（目标操作数）的值替换为其二进制补码。（此操作等效于从 0 中减去该操作数。）目标操作数位于通用寄存器或内存位置中。

此指令可与 LOCK 前缀一起使用，以允许以原子方式执行该指令。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 形式的 REX 前缀允许访问其他寄存器（R8-R15）。使用 REX.W 形式的 REX 前缀可将操作提升为 64 位。有关编码数据和限制，请参阅本节开头的汇总表。

操作¶

IF DEST = 0
    THEN CF := 0;
    ELSE CF := 1;
FI;
DEST := [– (DEST)]

影响的标志位¶

如果源操作数为 0，则 CF 标志设为 0；否则设为 1。根据结果设置 OF、SF、ZF、AF 和 PF 标志。

保护模式异常¶

#GP(0) 如果目标操作数位于不可写的段中。
如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择器。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀，但目标不是内存操作数。

实地址模式异常¶

#GP 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀，但目标不是内存操作数。

#SS(0) 如果内存操作数有效地址超出 SS 段限制.

兼容模式异常¶

与保护模式下的异常相同。
