LGDT/LIDT — 加载全局/中断描述符表寄存器

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 01 /2 LGDT m16&32 M N.E. Valid 加载 m 到 GDTR。
0F 01 /3 LIDT m16&32 M N.E. Valid 加载 m 到 IDTR。
0F 01 /2 LGDT m16&64 M Valid N.E. 加载 m 到 GDTR。
0F 01 /3 LIDT m16&64 M Valid N.E. 加载 m 到 IDTR。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r) N/A N/A N/A

描述¶

将源操作数中的值加载到全局描述符表寄存器（GDTR）或中断描述符表寄存器（IDTR）中。源操作数指定一个 6 字节的内存位置，其中包含全局描述符表（GDT）或中断描述符表（IDT）的基地址（线性地址）和限长（以字节为单位的表大小）。如果操作数大小属性为 32 位，则将 16 位限长（6 字节数据操作数的低 2 字节）和 32 位基地址（数据操作数的高 4 字节）加载到寄存器中。如果操作数大小属性为 16 位，则加载 16 位限长（低 2 字节）和 24 位基地址（第三、第四和第五字节）。在这里，操作数的高位字节不使用，且 GDTR 或 IDTR 中基地址的高位字节用零填充。

LGDT 和 LIDT 指令仅在操作系统软件中使用，不用于应用程序。它们是保护模式下唯一能直接加载线性地址（即非段相对地址）和限长的指令。它们通常在实地址模式下执行，以便在切换到保护模式之前进行处理器初始化。

在 64 位模式下，指令的操作数大小固定为 8+2 字节（8 字节基地址和 2 字节限长）。请参阅本节开头的汇总图表以获取编码数据和限制。

有关存储 GDTR 和 IDTR 内容的信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 2B 卷第 4 章中的“SGDT—存储全局描述符表寄存器”。

操作¶

IF Instruction is LIDT
    THEN
        IF OperandSize = 16
            THEN
                IDTR(Limit) := SRC[0:15];
                IDTR(Base) := SRC[16:47] AND 00FFFFFFH;
            ELSE IF 32-bit Operand Size
                THEN
                    IDTR(Limit) := SRC[0:15];
                    IDTR(Base) := SRC[16:47];
                FI;
            ELSE IF 64-bit Operand Size (* In 64-Bit Mode *)
                THEN
                    IDTR(Limit) := SRC[0:15];
                    IDTR(Base) := SRC[16:79];
                FI;
        FI;
    ELSE (* Instruction is LGDT *)
        IF OperandSize = 16
            THEN
                GDTR(Limit) := SRC[0:15];
                GDTR(Base) := SRC[16:47] AND 00FFFFFFH;
            ELSE IF 32-bit Operand Size
                THEN
                    GDTR(Limit) := SRC[0:15];
                    GDTR(Base) := SRC[16:47];
                FI;
            ELSE IF 64-bit Operand Size (* In 64-Bit Mode *)
                THEN
                    GDTR(Limit) := SRC[0:15];
                    GDTR(Base) := SRC[16:79];
                FI;
        FI;
FI;

受影响的标志位¶

无。

保护模式异常¶

#UD 如果使用了 LOCK 前缀。
#GP(0) 如果当前特权级别不为 0。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器用于访问内存且包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。
#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数的有效地址超出 SS 段限制。

虚拟-8086 模式异常¶

#UD 如果使用了 LOCK 前缀。
#GP 如果当前特权级别不为 0。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果引用 SS 段的内存地址为非规范形式。
#GP(0) 如果当前特权级别不为 0。
如果内存地址为非规范形式。
#UD 如果使用了 LOCK 前缀。
#PF(错误码) 如果发生缺页异常。
