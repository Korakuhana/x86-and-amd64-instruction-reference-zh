SWAPGS — 交换 GS 基址寄存器

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F 01 F8 SWAPGS ZO Valid Invalid 将当前 GS 基址寄存器的值与 MSR 地址 C0000102H 中包含的值进行交换。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

SWAPGS 将当前 GS 基址寄存器的值与 MSR 地址 C0000102H (IA32_KERNEL_GS_BASE) 中包含的值进行交换。SWAPGS 指令是专为系统软件使用的特权指令。

当使用 SYSCALL 实现系统调用时，在 OS 入口点处没有内核栈。也没有一种简单的方法来获取指向内核结构的指针（可以从中读取内核栈指针）。因此，内核无法保存通用寄存器或引用内存。

根据设计，SWAPGS 不需要任何通用寄存器或内存操作数。在使用该指令之前不需要保存任何寄存器。SWAPGS 将来自 IA32_KERNEL_GS_BASE MSR 的 CPL 0 数据指针与 GS 基址寄存器进行交换。然后，内核可以在正常的内存引用上使用 GS 前缀来访问内核数据结构。类似地，当使用中断或异常进入 OS 内核时（此时内核栈已经设置好），可以使用 SWAPGS 快速获取指向内核数据结构的指针。

IA32_KERNEL_GS_BASE MSR 本身只能使用 RDMSR/WRMSR 指令进行访问。这些指令只能在特权级 0 处访问。WRMSR 指令可确保 IA32_KERNEL_GS_BASE MSR 包含规范地址。

操作¶

IF CS.L ≠ 1 (* Not in 64-Bit Mode *)
    THEN
        #UD; FI;
IF CPL ≠ 0
    THEN #GP(0); FI;
tmp := GS.base;
GS.base := IA32_KERNEL_GS_BASE;
IA32_KERNEL_GS_BASE := tmp;

受影响的标志位¶

无。

保护模式异常¶

#UD 如果模式不是 64 位。

实地址模式异常¶

#UD 如果模式不是 64 位。

虚拟-8086 模式异常¶

#UD 如果模式不是 64 位。

兼容模式异常¶

#UD 如果模式不是 64 位。

64位模式异常¶

#GP(0) 如果 CPL != 0。
#UD 如果使用了 LOCK 前缀。
