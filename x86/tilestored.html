TILESTORED — 存储 Tile

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
VEX.128.F3.0F38.W0 4B !(11):rrr:100 TILESTORED sibmem, tmm1 A V/N.E. AMX-TILE 根据 tmm1 中的指定将 tile 存储在 sibmem 中。

指令操作数编码¶

Op/En 元组 操作数 1 操作数 2 操作数 3 操作数 4
A N/A ModRM:r/m (w) ModRM:reg (r) N/A N/A

描述¶

此指令要求使用 SIB 寻址。变址寄存器用作步长（stride）指示器。如果 SIB 编码中省略了变址寄存器，则变址寄存器的内容假定为零。

此指令根据 tile 配置指定的行和列来存储 tile 源寄存器。

TILECFG 数据中的 TILECFG.start_row 应初始化为 '0' 以便存储整个 tile，并在 TILESTORED 指令成功完成时设置为零。TILESTORED 是一个可重启的指令，当在指令执行期间发生可重启事件时，TILECFG.start_row 将为非零值。

仅支持内存操作数，并且只能使用 SIB 寻址模式进行访问，类似于 V[P]GATHER*/V[P]SCATTER* 指令。

在 Intel TSX 事务中尝试执行 TILESTORED 指令都会导致事务中止。

操作¶

TILESTORED tsib, tsrc
start := tilecfg.start_row
membegin := tsib.base + displacement
// 如果 SIB 编码中没有变址寄存器，则使用值零。
stride := tsib.index << tsib.scale
while start < tdest.rows:
    memptr := membegin + start * stride
    write_memory(memptr, tsrc.colsb, tsrc.row[start])
    start := start + 1
zero_tilecfg_start()
// 在指令中间发生内存故障的情况下，tilecfg.start_row := start

Intel C/C++ 编译器等价内部函数¶

TILESTORED void _tile_stored(__tile src, void *base, int stride);

受影响的标志位¶

无。
