描述¶

该指令使用 qword 索引，从内存操作数（第二个操作数）指定的内存地址中有条件地加载最多 2 或 4 个双精度浮点值。内存操作数使用 SIB 字节的 VSIB 形式来指定一个通用寄存器操作数作为通用基址、一个向量寄存器作为相对于基址的索引数组，以及一个常数比例因子。

掩码操作数（第三个操作数）指定从每个内存地址进行的条件加载操作，以及目的操作数（第一个操作数）的每个数据元素的相应更新。条件性由掩码寄存器中每个数据元素的最高有效位指定。如果某个元素的掩码位未设置，则目的寄存器的相应元素保持不变。目的寄存器和掩码寄存器中的数据元素宽度是相同的。除非指令引发异常，否则该指令将把整个掩码寄存器清零。

使用掩码寄存器下半部中的 dword 索引，该指令从 VSIB 寻址内存操作数中有条件地加载最多 2 或 4 个双精度浮点值，并更新目的寄存器。

如果至少有一个元素已经被汇集（即异常是由最右边已设置掩码位之外的元素触发的），则该指令可以因异常而挂起。发生这种情况时，目的寄存器和掩码操作数会被部分更新；那些已被汇集的元素被放入目的寄存器中，并将其掩码位清零。如果已汇集的元素有任何挂起的陷阱或中断，它们将代替异常被传递；在这种情况下，EFLAG.RF 被设置为 1，以便在指令继续执行时不会重新触发指令断点。

如果数据大小和索引大小不同，则目的寄存器的一部分和掩码寄存器的一部分不对应于正在汇集的任何元素。该指令将这些部分清零。即使指令触发异常，甚至在汇集任何元素之前指令就已经触发了异常，它也可能会对这两个寄存器中的一个或两个执行此操作。

VEX.128 版本：该指令将汇集两个双精度浮点值。对于 dword 索引，仅使用向量索引寄存器中的低两个索引。

VEX.256 版本：该指令将汇集四个双精度浮点值。对于 dword 索引，仅使用向量索引寄存器中的低四个索引。

请注意：

  • 如果索引、掩码或目的寄存器中的任何一对相同，此指令会导致 #UD 故障。
  • 可以按任何顺序从内存中读取这些值。与其他指令的内存排序遵循 Intel-64 内存排序模型。
  • 故障按从右到左的方式传递。也就是说，如果某个元素触发了故障并传递，则所有靠近目的寄存器 LSB 的元素都将完成（且无故障）。靠近 MSB 的单个元素可能完成，也可能未完成。如果给定元素触发了多个故障，它们将按常规顺序传递。
  • 可以按任何顺序汇集元素，但故障必须按从右到左的顺序传递；因此，在传递故障之前，可能会汇集故障元素左侧 of 的元素。此指令的特定实现是可重复的——在给定相同输入值和架构状态的情况下，将汇集故障元素左侧的同一组元素。
  • 此指令不执行 AC 检查，因此永远不会传递 AC 故障。
  • 如果地址大小属性为 16 位，此指令将导致 #UD。
  • 如果内存操作数的编码中没有 SIB 字节，此指令将导致 #UD。
  • 此指令不应用于访问内存映射 I/O，因为它执行的各个加载的顺序是特定于实现的，并且某些实现可能使用大于数据元素大小的加载，或者加载元素的次数不确定。
  • 表示比例索引所需的位数可能多于处理器使用的地址位数（例如，在 32 位模式下，如果比例因子大于 1）。在这种情况下，超出地址位数的最高有效位将被忽略。

操作¶

DEST := SRC1;
BASE_ADDR: base register encoded in VSIB addressing;
VINDEX: the vector index register encoded by VSIB addressing;
SCALE: scale factor encoded by SIB:[7:6];
DISP: optional 1, 4 byte displacement;
MASK := SRC3;

VGATHERDPD (VEX.128 版本)¶

MASK[MAXVL-1:128] := 0;
FOR j := 0 to 1
    i := j * 64;
    IF MASK[63+i] THEN
        MASK[i +63:i] := FFFFFFFF_FFFFFFFFH; // extend from most significant bit
    ELSE
        MASK[i +63:i] := 0;
    FI;
ENDFOR
FOR j := 0 to 1
    k := j * 32;
    i := j * 64;
    DATA_ADDR := BASE_ADDR + (SignExtend(VINDEX[k+31:k])*SCALE + DISP;
    IF MASK[63+i] THEN
        DEST[i +63:i] := FETCH_64BITS(DATA_ADDR); // a fault exits the instruction
    FI;
    MASK[i +63: i] := 0;
ENDFOR
DEST[MAXVL-1:128] := 0;
