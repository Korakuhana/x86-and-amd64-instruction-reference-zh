VRCP14PD — 计算打包双精度浮点值的近似倒数

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.128.66.0F38.W1 4C /r VRCP14PD xmm1 {k1}{z}, xmm2/m128/m64bcst A V/V AVX512VL AVX512F 计算 xmm2/m128/m64bcst 中打包双精度浮点值的近似倒数，并在写掩码控制下将结果存储在 xmm1 中。
EVEX.256.66.0F38.W1 4C /r VRCP14PD ymm1 {k1}{z}, ymm2/m256/m64bcst A V/V AVX512VL AVX512F 计算 ymm2/m256/m64bcst 中打包双精度浮点值的近似倒数，并在写掩码控制下将结果存储在 ymm1 中。
EVEX.512.66.0F38.W1 4C /r VRCP14PD zmm1 {k1}{z}, zmm2/m512/m64bcst A V/V AVX512F 计算 zmm2/m512/m64bcst 中打包双精度浮点值的近似倒数，并在写掩码控制下将结果存储在 zmm1 中。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Full ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

此指令对源操作数（第二操作数）中的八个/四个/两个打包双精度浮点值执行 SIMD 近似倒数计算，并将打包双精度浮点结果存储在目的操作数中。该近似的最大相对误差小于 2-14。

源操作数可以是 ZMM 寄存器、512 位内存位置，或从 64 位内存位置广播的 512 位向量。目的操作数是根据写掩码有条件更新的 ZMM 寄存器。

VRCP14PD 指令不受 MXCSR 寄存器中舍入控制位的影响。当源值为 0.0 时，返回带有源值符号的 ∞。仅当 MXCSR 中设置了 DAZ 位时，渐近下溢 (denormal) 源值才会被视为零。否则它会被正确处理（即不视为 0.0）。仅当 MXCSR 中设置了 FTZ 位时，下溢结果才会刷新为零。否则，它将被正确处理（即写入正确的下溢结果）并保留操作数的符号。当源值是 SNaN 或 QNaN 时，SNaN 被转换为 QNaN，或者返回源 QNaN。

EVEX.vvvv 是保留的，必须为 1111b，否则指令将引发 #UD。

MXCSR 异常标志不受此指令影响，且不报告浮点异常。

* 在这种情况下，尾数向右移动一位或两位

操作¶

VRCP14PD (EVEX 编码版本)¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no writemask* THEN
            IF (EVEX.b = 1) AND (SRC *is memory*)
                THEN DEST[i+63:i] := APPROXIMATE(1.0/SRC[63:0]);
                ELSE DEST[i+63:i] := APPROXIMATE(1.0/SRC[i+63:i]);
            FI;
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[i+63:i] remains unchanged*
            ELSE ; zeroing-masking
                DEST[i+63:i] := 0
        FI;
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0
