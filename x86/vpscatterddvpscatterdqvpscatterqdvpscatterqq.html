指令操作数编码¶

描述¶

将双字向量中最多 16 个元素（四字索引为 8 个元素）或四字向量中最多 8 个元素存储到基地址 BASE_ADDR 和带有比例因子 SCALE 的索引向量 VINDEX 指向的内存位置。这些元素通过 VSIB 指定（即索引寄存器是保存打包索引的向量寄存器）。仅当元素对应的掩码位为 1 时，才会存储该元素。除非触发异常，否则此指令会将整个掩码寄存器设置为 0。

如果至少已经分发了一个元素（即，如果异常是由其掩码位置位的最右侧元素以外 of 元素触发的），则该指令可以被异常暂停。当发生这种情况时，目的地寄存器和掩码寄存器会被部分更新。如果来自已分发元素的任何陷阱或中断处于挂起状态，它们将代替异常被递送；在这种情况下，EFLAG.RF 被设置为 1，以便在指令继续执行时不会重新触发指令断点。

注意：

  • 只有对重叠向量索引的写入才被保证彼此有序（从源寄存器的 LSB 到 MSB）。请注意，这也包括部分重叠的向量索引。未重叠的写入可以以任何顺序发生。与其他指令的内存排序遵循 Intel-64 内存排序模型。请注意，这不考虑映射到相同物理地址位置的非重叠索引。
  • If two or more destination indices completely overlap, the “earlier” write(s) may be skipped.
  • 故障以从右到左的方式递送。也就是说，如果一个元素触发了故障并递送，所有更靠近目的地 ZMM 寄存器 LSB 的元素都将被完成（且无故障）。更靠近 MSB 的个别元素可能完成，也可能未完成。如果给定元素触发多个故障，它们将按常规顺序递送。
  • 元素可以以任何顺序分发，但故障必须按从右到左的顺序递送；因此，在递送故障之前，可能会在故障元素左侧分发元素。此指令的给定实现是可重复的——在相同的输入值和架构状态下，将分发故障元素左侧的同一组元素。
  • 此指令不执行 AC 检查，因此永远不会递送 AC 故障。
  • 对 16 位有效地址无效。将递送 #UD 故障。
  • 如果此指令自我重写然后发生故障，则在递送故障之前可能仅完成元素的一个子集（如上所述）。如果故障处理程序完成并尝试重新执行此指令，则将执行新指令，并且分发不会完成。

注意，此指令中强制要求存在 VSIB 字节。因此，如果 ModRM.rm 不等于 100b，指令将导致 #UD 故障。

此指令具有特殊的 disp8*N 和对齐规则。N 被认为是单个向量元素的大小。

与处理器使用的地址位相比，缩放后的索引可能需要更多位来表示（例如，在 32 位模式下，如果比例因子大于 1）。在这种情况下，超出地址位数的最显著位将被忽略。

如果指定了 k0 掩码寄存器，指令将导致 #UD 故障。

如果 EVEX.Z = 1，指令将导致 #UD 故障。

操作¶

BASE_ADDR 代表内存操作数基地址（通用寄存器）；可能不存在
VINDEX 代表内存操作数索引向量（ZMM 寄存器）
SCALE 代表内存操作数比例因子 (1, 2, 4 或 8)
DISP 是可选的 1 或 4 字节位移

VPSCATTERDD (EVEX 编码版本)¶

(KL, VL)= (4, 128), (8, 256), (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN MEM[BASE_ADDR +SignExtend(VINDEX[i+31:i]) * SCALE + DISP] := SRC[i+31:i]
            k1[j] := 0
    FI;
ENDFOR
k1[MAX_KL-1:KL] := 0

VPSCATTERDQ (EVEX 编码版本)¶

(KL, VL)= (2, 128), (4, 256), (8, 512)
FOR j := 0 TO KL-1
    i := j * 64
    k := j * 32
    IF k1[j] OR *no writemask*
        THEN MEM[BASE_ADDR +SignExtend(VINDEX[k+31:k]) * SCALE + DISP] := SRC[i+63:i]
            k1[j] := 0
    FI;
ENDFOR
