PTEST — 逻辑比较

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
66 0F 38 17 /r PTEST xmm1, xmm2/m128 RM V/V SSE4_1 如果 xmm2/m128 AND xmm1 的结果全为 0，则将 ZF 置 1。如果 xmm2/m128 AND NOT xmm1 的结果全为 0，则将 CF 置 1。
VEX.128.66.0F38.WIG 17 /r VPTEST xmm1, xmm2/m128 RM V/V AVX 根据源操作数的按位 AND 和 ANDN 结果设置 ZF 和 CF。
VEX.256.66.0F38.WIG 17 /r VPTEST ymm1, ymm2/m256 RM V/V AVX 根据源操作数的按位 AND 和 ANDN 结果设置 ZF 和 CF。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) N/A N/A

描述¶

如果第一个源操作数（第一个操作数）与第二个源操作数（第二个操作数）按位 AND 的结果中所有位都为 0，则 PTEST 和 VPTEST 设置 ZF 标志。如果第二个源操作数与目的操作数的逻辑非按位 AND 的结果中所有位都为 0，则 VPTEST 设置 CF 标志。

第一个源寄存器由 ModR/M reg 字段指定。

128 位版本：第一个源寄存器是一个 XMM 寄存器。第二个源寄存器可以是 XMM 寄存器或 128 位内存位置。目的寄存器不被修改。

VEX.256 编码版本：第一个源寄存器是一个 YMM 寄存器。第二个源寄存器可以是 YMM 寄存器或 256 位内存位置。目的寄存器不被修改。

注意：在 VEX 编码的版本中，VEX.vvvv 是保留的，必须为 1111b，否则指令将触发 #UD 异常。

操作¶

(V)PTEST（128位版本）¶

IF (SRC[127:0] BITWISE AND DEST[127:0] = 0)
    THEN ZF := 1;
    ELSE ZF := 0;
IF (SRC[127:0] BITWISE AND NOT DEST[127:0] = 0)
    THEN CF := 1;
    ELSE CF := 0;
DEST (unmodified)
AF := OF := PF := SF := 0;

VPTEST（VEX.256 编码版本）¶

IF (SRC[255:0] BITWISE AND DEST[255:0] = 0) THEN ZF := 1;
    ELSE ZF := 0;
IF (SRC[255:0] BITWISE AND NOT DEST[255:0] = 0) THEN CF := 1;
    ELSE CF := 0;
DEST (unmodified)
AF := OF := PF := SF := 0;

受影响的标志位¶

OF、AF、PF、SF 标志清零，ZF、CF 标志根据操作结果设置。

SIMD 浮点异常¶

无。

其他异常¶

请参阅表 2-21，“第 4 类异常条件”，此外：

#UD 如果 VEX.vvvv ̸= 1111B。
