ADDSUBPS — 打包单精度浮点加法/减法

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
F2 0F D0 /r ADDSUBPS xmm1, xmm2/m128 RM V/V SSE3 对来自 xmm2/m128 和 xmm1 的单精度浮点值进行加法/减法交替操作（偶数通道减，奇数通道加），结果存入 xmm1。
VEX.128.F2.0F.WIG D0 /r VADDSUBPS xmm1, xmm2, xmm3/m128 RVM V/V AVX 对来自 xmm3/mem 和 xmm2 的打包单精度浮点值进行加法/减法交替操作，并将结果存入 xmm1。
VEX.256.F2.0F.WIG D0 /r VADDSUBPS ymm1, ymm2, ymm3/m256 RVM V/V AVX 对来自 ymm3/mem 和 ymm2 的打包单精度浮点值进行加法/减法交替操作，并将结果存入 ymm1。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r, w) ModRM:r/m (r) N/A N/A
RVM ModRM:reg (w) VEX.vvvv (r) ModRM:r/m (r) N/A

描述¶

将第一源操作数（第二操作数）中奇数编号（如通道 1, 3, 5, 7）的单精度浮点值与第二源操作数（第三操作数）中对应的单精度浮点值相加；并将结果存入目标操作数（第一操作数）的奇数编号通道中。从第一源操作数中对应的偶数编号（如通道 0, 2, 4, 6）的单精度浮点值中减去第二源操作数中偶数编号的单精度浮点值；并将结果存入目标操作数的偶数编号通道中。

在 64 位模式下，使用 REX.R 形式的 REX 前缀允许此指令访问额外的寄存器（XMM8-XMM15）。

128 位传统 SSE 版本：第二源操作数可以是 XMM 寄存器或 128 位内存地址。目标操作数与第一源 XMM 寄存器相同，并且相应 YMM 寄存器目标的高位（MAXVL-1:128）保持不变。请参阅图 3-4。

VEX.128 编码版本：第一源操作数是 XMM 寄存器或 128 位内存地址。目标操作数是 XMM 寄存器。相应 YMM 寄存器目标的高位（MAXVL-1:128）清零。

VEX.256 编码版本：第一源操作数是 YMM 寄存器。第二源操作数可以是 YMM 寄存器或 256 位内存地址。目标操作数是 YMM 寄存器。

操作¶

ADDSUBPS (128-bit Legacy SSE Version)

DEST[31:0] := DEST[31:0] - SRC[31:0]
DEST[63:32] := DEST[63:32] + SRC[63:32]
DEST[95:64] := DEST[95:64] - SRC[95:64]
DEST[127:96] := DEST[127:96] + SRC[127:96]
DEST[MAXVL-1:128] (Unmodified)

VADDSUBPS (VEX.128 Encoded Version)

DEST[31:0] := SRC1[31:0] - SRC2[31:0]
DEST[63:32] := SRC1[63:32] + SRC2[63:32]
DEST[95:64] := SRC1[95:64] - SRC2[95:64]
DEST[127:96] := SRC1[127:96] + SRC2[127:96]
DEST[MAXVL-1:128] := 0

VADDSUBPS (VEX.256 Encoded Version)

DEST[31:0] := SRC1[31:0] - SRC2[31:0]
DEST[63:32] := SRC1[63:32] + SRC2[63:32]
DEST[95:64] := SRC1[95:64] - SRC2[95:64]
DEST[127:96] := SRC1[127:96] + SRC2[127:96]
DEST[159:128] := SRC1[159:128] - SRC2[159:128]
DEST[191:160] := SRC1[191:160] + SRC2[191:160]
DEST[223:192] := SRC1[223:192] - SRC2[223:192]
DEST[255:224] := SRC1[255:224] + SRC2[255:224]

异常¶

如果源操作数是内存操作数，则该操作数必须对齐到 16 字节边界，否则将生成通用保护异常（#GP）。

SIMD 浮点异常¶

溢出、欠载、无效、精度、非规范化数。
