XRSTOR — 恢复处理器扩展状态

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
NP 0F AE /5 XRSTOR mem M V/V XSAVE 从 mem 恢复由 EDX:EAX 指定的状态组件。
NP REX.W + 0F AE /5 XRSTOR64 mem M V/N.E. XSAVE 从 mem 恢复由 EDX:EAX 指定的状态组件。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r) N/A N/A N/A

描述¶

从由源操作数指定的内存地址处的 XSAVE 区域执行对处理器状态组件的完全或部分恢复。隐式的 EDX:EAX 寄存器对指定了 64 位指令掩码。恢复的具体状态组件对应于请求功能位图 (RFBM) 中设置的位，该位图是 EDX:EAX 和 XCR0 的逻辑与。

XSAVE 区域的格式在《Intel® 64 和 IA-32 架构软件开发人员手册》卷 1 第 13.4 节“XSAVE 区域”中详细介绍。像 FXRSTOR 和 FXSAVE 一样，用于 x87 状态的内存格式取决于 REX.W 前缀；参见《Intel® 64 和 IA-32 架构软件开发人员手册》卷 1 第 13.5.1 节“x87 状态”。

《Intel® 64 和 IA-32 架构软件开发人员手册》卷 1 第 13.8 节“XRSTOR 的操作”提供了关于 XRSTOR 指令操作的详细描述。以下条目提供了高级大纲：

  • XRSTOR 的执行可以采用两种形式之一：标准形式和紧凑形式。XSAVE 标头中 XCOMP_BV 字段的第 63 位决定了使用哪种形式：值 0 指定标准形式，而值 1 指定紧凑形式。
  • 如果 RFBM[i] = 0，XRSTOR 不会更新状态组件 i。1
  • 如果 RFBM[i] = 1 且 XSAVE 标头中 XSTATE_BV 字段的位 i 为 0，则 XRSTOR 会初始化状态组件 i。
  • 如果 RFBM[i] = 1 且 XSTATE_BV[i] = 1，则 XRSTOR 从 XSAVE 区域加载状态组件 i。
  • 标准形式的 XRSTOR 对 MXCSR（它是状态组件 1 — SSE 的一部分）的处理方式与 XMM 寄存器不同。如果任一形式尝试用非法值加载 MXCSR，就会发生通用保护异常 (#GP)。
  • XRSTOR 加载内部值 XRSTOR_INFO，这可用于优化随后执行的 XSAVEOPT 或 XSAVES。
  • 紧随 XRSTOR 执行之后，处理器会将 RFBM[i] = 1 且 XSTATE_BV[i] = 1 的任何状态组件 i 跟踪为使用中（不在初始配置中）；它将 RFBM[i] = 0 的任何状态组件 i 跟踪为已修改。

使用未对齐到 64 字节边界的源操作数（对于 64 位和 32 位模式）会导致通用保护 (#GP) 异常。在 64 位模式下，RDX 和 RAX 的高 32 位被忽略。

有关位图 XINUSE 和 XMODIFIED 以及数量 XRSTOR_INFO 的讨论，请参见《Intel® 64 和 IA-32 架构软件开发人员手册》卷 1 第 13.6 节“XSAVE 管理状态的处理器跟踪”。

1. 如果 RFBM[1] = 0 且 RFBM[2] = 1，则存在特例。在这种情况下，标准形式的 XRSTOR 将从内存中加载 MXCSR，即使 MXCSR 是状态组件 1 — SSE 的一部分。紧凑形式的 XRSTOR 不会做此特例处理。

操作¶

RFBM := XCR0 AND EDX:EAX; /* bitwise logical AND */
COMPMASK := XCOMP_BV field from XSAVE header;
RSTORMASK := XSTATE_BV field from XSAVE header;
IF COMPMASK[63] = 0
    THEN
        /* 标准形式的 XRSTOR */
        TO_BE_RESTORED := RFBM AND RSTORMASK;
        TO_BE_INITIALIZED := RFBM AND NOT RSTORMASK;
        IF TO_BE_RESTORED[0] = 1
            THEN
                XINUSE[0] := 1;
                load x87 state from legacy region of XSAVE area;
        ELSIF TO_BE_INITIALIZED[0] = 1
            THEN
                XINUSE[0] := 0;
                initialize x87 state;
        FI;
        IF RFBM[1] = 1 OR RFBM[2] = 1
            THEN load MXCSR from legacy region of XSAVE area;
        FI;
        IF TO_BE_RESTORED[1] = 1
            THEN
                XINUSE[1] := 1;
                load XMM registers from legacy region of XSAVE area; // this step does not load MXCSR
        ELSIF TO_BE_INITIALIZED[1] = 1
            THEN
                XINUSE[1] := 0;
                set all XMM registers to 0; // this step does not initialize MXCSR
        FI;
        FOR i := 2 TO 62
            IF TO_BE_RESTORED[i] = 1
                THEN
                    XINUSE[i] := 1;
                    load XSAVE state component i at offset n from base of XSAVE area;
                        // n enumerated by CPUID(EAX=0DH,ECX=i):EBX)
            ELSIF TO_BE_INITIALIZED[i] = 1
                THEN
                    XINUSE[i] := 0;
                    initialize XSAVE state component i;
            FI;
        ENDFOR;
    ELSE
        /* 紧凑形式的 XRSTOR */
        IF CPUID.(EAX=0DH,ECX=1):EAX.XSAVEC[bit 1] = 0
            THEN /* compacted form not supported */
                #GP(0);
        FI;
        FORMAT = COMPMASK AND 7FFFFFFF_FFFFFFFFH;
        RESTORE_FEATURES = FORMAT AND RFBM;
        TO_BE_RESTORED := RESTORE_FEATURES AND RSTORMASK;
        FORCE_INIT := RFBM AND NOT FORMAT;
        TO_BE_INITIALIZED = (RFBM AND NOT RSTORMASK) OR FORCE_INIT;
        IF TO_BE_RESTORED[0] = 1
            THEN
                XINUSE[0] := 1;
                load x87 state from legacy region of XSAVE area;
        ELSIF TO_BE_INITIALIZED[0] = 1
            THEN
                XINUSE[0] := 0;
                initialize x87 state;
        FI;
        IF TO_BE_RESTORED[1] = 1
            THEN
                XINUSE[1] := 1;
                load SSE state from legacy region of XSAVE area; // this step loads the XMM registers and MXCSR
        ELSIF TO_BE_INITIALIZED[1] = 1
            THEN
                set all XMM registers to 0;
                XINUSE[1] := 0;
                MXCSR := 1F80H;
        FI;
        NEXT_FEATURE_OFFSET = 576;
                                // Legacy area and XSAVE header consume 576 bytes
        FOR i := 2 TO 62
            IF FORMAT[i] = 1
                THEN
                    IF TO_BE_RESTORED[i] = 1
                        THEN
                            XINUSE[i] := 1;
                            load XSAVE state component i at offset NEXT_FEATURE_OFFSET from base of XSAVE area;
                    FI;
                    NEXT_FEATURE_OFFSET = NEXT_FEATURE_OFFSET + n (n enumerated by CPUID(EAX=0DH,ECX=i):EAX);
            FI;
            IF TO_BE_INITIALIZED[i] = 1
                THEN
                    XINUSE[i] := 0;
                    initialize XSAVE state component i;
            FI;
        ENDFOR;
FI;
XMODIFIED := NOT RFBM;
IF in VMX non-root operation
    THEN VMXNR := 1;
    ELSE VMXNR := 0;
FI;
LAXA := linear address of XSAVE area;

保护模式异常¶

#GP(0) 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。
无论段如何，如果内存操作数未在 64 字节边界上对齐。
如果 XSAVE 标头的 XCOMP_BV 字段的第 63 位为 1 且 CPUID.(EAX=0DH,ECX=1):EAX.XSAVEC[bit 1] = 0。
如果执行了标准形式，且 XCR0 中的一位为 0，且 XSAVE 标头的 XSTATE_BV 字段中的对应位为 1。
如果执行了标准形式，且 XSAVE 标头中 23:8 字节不全为零。
如果执行了紧凑形式，且 XCR0 中的一位为 0，且 XSAVE 标头的 XCOMP_BV 字段中的对应位为 1。
如果执行了紧凑形式，且 XSAVE 标头中 XCOMP_BV 字段的一位为 0，且 XSTATE_BV 字段中的对应位为 1。
如果执行了紧凑形式，且 XSAVE 标头中 63:16 字节不全为零。
如果尝试向 MXCSR 寄存器的任何保留位写入 1。
#SS(0) 如果内存操作数有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#NM 如果 CR0.TS[bit 3] = 1。
#UD 如果 CPUID.01H:ECX.XSAVE[bit 26] = 0。
如果 CR4.OSXSAVE[bit 18] = 0。
如果使用了 LOCK 前缀。
#AC 如上所述，如果此异常被禁用，且内存操作数未对齐在 64 字节边界上，则会发出通用保护异常 (#GP)。如果启用了对齐检查异常 (#AC)（且 CPL 为 3），则不保证会发出 #AC，并且具体情况可能因实现而异，如下所示。在所有未发出 #AC 的实现中，会发出通用保护异常来代替它。此外，对齐检查的宽度也可能因实现而异。例如，对于特定的实现，可能会针对 2 字节未对齐发出对齐检查异常，而针对所有其他未对齐（4、8 或 16 字节未对齐）可能会发出通用保护异常。
