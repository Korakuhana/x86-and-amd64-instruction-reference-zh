TZCNT — 计算末尾零位的数量

操作码/指令 Op/En 64/32位模式 CPUID 特征标志 描述
F3 0F BC /r TZCNT r16, r/m16 A V/V BMI1 计算 r/m16 中末尾的零位数量，并在 r16 中返回结果。
F3 0F BC /r TZCNT r32, r/m32 A V/V BMI1 计算 r/m32 中末尾的零位数量，并在 r32 中返回结果。
F3 REX.W 0F BC /r TZCNT r64, r/m64 A V/N.E. BMI1 计算 r/m64 中末尾的零位数量，并在 r64 中返回结果。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
A ModRM:reg (w) ModRM:r/m (r) N/A N/A

描述¶

TZCNT 计算源操作数（第二个操作数）中末尾的最低有效零位的数量，并在目的操作数（第一个操作数）中返回结果。TZCNT 是 BSF 指令的扩展。TZCNT 和 BSF 指令的关键区别在于，当源操作数为零时，TZCNT 提供操作数大小作为输出，而对于 BSF 指令，如果源操作数为零，目的操作数的内容是未定义的。在不支持 TZCNT 的处理器上，该指令的字节编码按 BSF 执行。

操作¶

temp := 0
DEST := 0
DO WHILE ( (temp < OperandSize) and (SRC[ temp] = 0) )
    temp := temp +1
    DEST := DEST+ 1
OD
IF DEST = OperandSize
    CF := 1
ELSE
    CF := 0
FI
IF DEST = 0
    ZF := 1
ELSE
    ZF := 0
FI

受影响的标志位¶

如果是零输出（源的最低有效位已设置），ZF 被设置为 1，否则为 0；如果输入为零，CF 被设置为 1，否则被清除。OF、SF、PF 和 AF 标志未定义。
