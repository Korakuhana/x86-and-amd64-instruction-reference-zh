SQRTSS — 计算标量单精度值的平方根

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
F3 0F 51 /r SQRTSS xmm1, xmm2/m32 A V/V SSE 计算 xmm2/m32 中低位单精度浮点值的平方根，并将结果存储在 xmm1 中。
VEX.LIG.F3.0F.WIG 51 /r VSQRTSS xmm1, xmm2, xmm3/m32 B V/V AVX 计算 xmm3/m32 中低位单精度浮点值的平方根，并将结果存储在 xmm1 中。此外，将 xmm2 的高位单精度浮点值（位[127:32]）复制到 xmm1[127:32]。
EVEX.LLIG.F3.0F.W0 51 /r VSQRTSS xmm1 {k1}{z}, xmm2, xmm3/m32{er} C V/V AVX512F 在写掩码 k1 的控制下计算 xmm3/m32 中低位单精度浮点值的平方根，并将结果存储在 xmm1 中。此外，将 xmm2 的高位单精度浮点值（位[127:32]）复制到 xmm1[127:32]。

描述¶

计算第二个源操作数中低位单精度浮点值的平方根，并将单精度浮点结果存储在目的操作数中。第二个源操作数可以是 XMM 寄存器或 32 位内存位置。第一源操作数和目的操作数是 XMM 寄存器。

128 位传统 SSE 版本：第一源操作数和目的操作数相同。对应 YMM 目的寄存器的高位 (MAXVL-1:32) 保持不变。

VEX.128 和 EVEX 编码版本：目的操作数的 127:32 位从第一源操作数的对应位复制。目的 ZMM 寄存器的高位 (MAXVL-1:128) 被清零。

EVEX 编码版本：目的操作数的低双字元素根据写掩码进行更新。

软件应确保 VSQRTSS 使用 VEX.L=0 进行编码。对 VSQRTSS 使用 VEX.L=1 进行编码可能会在不同的处理器世代中遇到不可预测的行为。

操作¶

VSQRTSS（EVEX 编码版本）¶

IF (EVEX.b = 1) AND (SRC2 *is register*)
    THEN
        SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(EVEX.RC);
    ELSE
        SET_ROUNDING_MODE_FOR_THIS_INSTRUCTION(MXCSR.RC);
FI;
IF k1[0] or *no writemask*
    THEN DEST[31:0] := SQRT(SRC2[31:0])
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[31:0] remains unchanged*
            ELSE ; zeroing-masking
                DEST[31:0] := 0
        FI;
FI;
DEST[127:32] := SRC1[127:32]
DEST[MAXVL-1:128] := 0

VSQRTSS（VEX.128 编码版本）¶

DEST[31:0] := SQRT(SRC2[31:0])
DEST[127:32] := SRC1[127:32]
DEST[MAXVL-1:128] := 0

SQRTSS（128 位传统 SSE 版本）¶

DEST[31:0] := SQRT(SRC2[31:0])
DEST[MAXVL-1:32] (Unmodified)
