FPREM — 部分余数

操作码 指令 64位模式 兼容/传统模式 描述
D9 F8 FPREM 有效 有效 用 ST(0) 除以 ST(1) 得到的余数替换 ST(0)。

描述¶

计算用 ST(0) 寄存器中的值（被除数）除以 ST(1) 寄存器中的值（除数或模数）得到的余数，并将结果存储在 ST(0) 中。余数代表以下值：

Remainder := ST(0) − (Q ∗ ST(1))

这里，Q 是通过将 [ST(0) / ST(1)] 的浮点数商向零截断而获得的整数值。余数的符号与被除数的符号相同。余数的大小（绝对值）小于模数的大小，除非计算了部分余数（如下所述）。

此指令产生精确的结果；不发生不精确结果异常，且舍入控制不起作用。下表显示了在计算各类数字的余数时获得的结果，假设没有发生下溢。

F 表示有限浮点值。

* 表示浮点无效算术操作数 (#IA) 异常。

当结果为 0 时，其符号与被除数的符号相同。当模数为 ∞ 时，结果等于 ST(0) 中的值。

FPREM 指令不计算 IEEE Std 754 中指定的余数。IEEE 指定的余数可以使用 FPREM1 指令计算。提供 FPREM 指令是为了与 Intel 8087 和 Intel 287 数学协处理器兼容。

FPREM 指令因其计算余数的方式而得名“部分余数”。此指令通过迭代相减得出余数。但是，在执行一次该指令时，它最多只能将 ST(0) 的指数减少 63。如果指令成功产生小于模数的余数，则操作完成，并清除 FPU 状态字中的 C2 标志。否则，设置 C2，且 ST(0) 中的结果被称为部分余数。部分余数的指数将比原始被除数的指数至少小 32。软件可以重新执行该指令（使用 ST(0) 中的部分余数作为被除数），直到清除 C2。（请注意，在执行此类余数计算循环时，需要使用 FPU 的更高优先级中断例程可以强制在循环中的指令之间进行上下文切换。）

FPREM 指令的一个重要用途是减少周期函数的自变量。当变量减少（reduction）完成时，该指令将商的三个最低有效位存储在 FPU 状态字的 C3、C1 和 C0 标志中。此信息在正切函数的自变量减少（使用 π/4 的模数）中非常重要，因为它将原始角度定位在单位圆的八个扇区中的正确位置。

此指令的操作在非 64 位模式和 64 位模式下相同。

操作¶

D := exponent(ST(0)) – exponent(ST(1));
IF D < 64
    THEN
        Q := Integer(TruncateTowardZero(ST(0) / ST(1)));
        ST(0) := ST(0) – (ST(1) ∗ Q);
        C2 := 0;
        C0, C3, C1 := LeastSignificantBits(Q); (* Q2, Q1, Q0 *)
    ELSE
        C2 := 1;
        N := An implementation-dependent number between 32 and 63;
        QQ := Integer(TruncateTowardZero((ST(0) / ST(1)) / 2(D − N)));
        ST(0) := ST(0) – (ST(1) ∗ QQ ∗ 2(D − N));
FI;

影响的 FPU 标志寄存器¶

C0 设置为商的第 2 位 (Q2)。
C1 如果发生栈下溢，则设置为 0；否则，设置为商的最低有效位 (Q0)。
C2 如果变量减少完成，则设置为 0；如果未完成，则设置为 1。
C3 设置为商的第 1 位 (Q1)。

浮点异常¶

#IS 发生栈下溢。
#IA 源操作数是 SNaN 值、模数为 0、被除数为 ∞，或是不受支持的格式。
#D 源操作数是反常（denormal）值。
#U 结果对于目的格式过小。

保护模式异常¶

#NM CR0.EM[bit 2] 或 CR0.TS[bit 3] = 1。
#MF 如果存在未决的 x87 FPU 异常。
#UD 如果使用了 LOCK 前缀。
