ROUNDSD — 对标量双精度浮点值进行舍入

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
66 0F 3A 0B /r ib ROUNDSD xmm1, xmm2/m64, imm8 RMI V/V SSE4_1 对 xmm2/m64 中的低位打包双精度浮点值进行舍入，并将结果保存在 xmm1 中。舍入模式由 imm8 决定。
VEX.LIG.66.0F3A.WIG 0B /r ib VROUNDSD xmm1, xmm2, xmm3/m64, imm8 RVMI V/V AVX 对 xmm3/m64 中的低位打包双精度浮点值进行舍入，并将结果保存在 xmm1 中。舍入模式由 imm8 决定。来自 xmm2 的高位打包双精度浮点值（位[127:64]）被复制到 xmm1[127:64] 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RMI ModRM:reg (w) ModRM:r/m (r) imm8 N/A
RVMI ModRM:reg (w) VEX.vvvv (r) ModRM:r/m (r) imm8

描述¶

对源操作数（第二个操作数）较低四字中的双精度浮点值使用立即数操作数（第三个操作数）中指定的舍入模式进行舍入，并将结果保存在目的操作数（第一个操作数）中。舍入过程将双精度浮点输入舍入为整数值，并将该整数结果作为双精度浮点值返回到最低位置。目的操作数中的高位双精度浮点值将保持不变。

立即数操作数指定舍入操作的控制字段，定义了三个位字段，如图 4-24 所示（参见 ROUNDPD 页面）。立即数字节的第 3 位控制处理器对精度异常的行为，第 2 位选择舍入模式控制的来源。位 1:0 指定非粘性舍入模式值（表 4-18 列出了舍入模式字段的编码值）。

根据立即数操作数发出精度浮点异常信号。如果任何源操作数是 SNaN，则它将被转换为 QNaN。如果 DAZ 设置为 '1'，则在舍入前反常数将被转换为零。

128位传统 SSE 版本：第一个源操作数与目的操作数相同。对应 YMM 目的寄存器的位（MAXVL-1:64）保持不变。

VEX.128 编码版本：目的 YMM 寄存器的位（MAXVL-1:128）清零。

操作¶

IF (imm[2] = ‘1)
    THEN // 舍入模式由 MXCSR.RC 决定
        DEST[63:0] := ConvertDPFPToInteger_M(SRC[63:0]);
    ELSE // 舍入模式由 IMM8.RC 决定
        DEST[63:0] := ConvertDPFPToInteger_Imm(SRC[63:0]);
FI;
DEST[127:63] remains unchanged ;

ROUNDSD（128位传统 SSE 版本）¶

DEST[63:0] := RoundToInteger(SRC[63:0], ROUND_CONTROL)
DEST[MAXVL-1:64] (Unmodified)

VROUNDSD（VEX.128 编码版本）¶

DEST[63:0] := RoundToInteger(SRC2[63:0], ROUND_CONTROL)
DEST[127:64] := SRC1[127:64]
DEST[MAXVL-1:128] := 0

SIMD 浮点异常¶

无效（仅当 SRC = SNaN 时发出信号）。

精度（仅当 imm[3] = '0 时发出信号；如果 imm[3] = '1，则忽略 MXCSR 中的精度掩码，且不发出精度异常信号。）

请注意，ROUNDSD 不会发出反常数（Denormal）信号。
