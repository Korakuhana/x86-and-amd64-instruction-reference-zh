BTS — 位测试并置位

0F AB /r BTS r/m16, r16 MR Valid Valid 将选定的位存入 CF 标志位，并将其置 1。

描述 ¶

选择一个位串（由第一个操作数，即位基址指定）中由位偏移量操作数（第二个操作数）指示的位位置，并将该位的值存入 CF 标志中，然后将位串中选定的位置为 1。位基址操作数可以是寄存器或内存地址；位偏移量操作数可以是寄存器或立即数：

  • 如果位基址操作数指定了寄存器，该指令将对位偏移量操作数进行模 16、32 或 64 的运算（取模大小取决于运行模式和寄存器大小；64 位操作数仅在 64 位模式下可用）。这允许选择任何位位置。
  • 如果位基址操作数指定了内存地址，该操作数表示包含该位串位基址（所指定字节的第 0 位）的内存中字节的地址。偏移量操作数可以引用的位位置范围取决于操作数的大小。

另请参阅原书第 3-11 页的 Bit(BitBase, BitOffset)。

某些汇编器支持大于 31 的立即数位偏移量，方法是将立即数位偏移量字段与内存操作数的位移字段结合使用。有关此寻址机制的更多信息，请参阅本章中的“BT—位测试”。

该指令可以与 LOCK 前缀一起使用，以允许该指令原子地执行。

在 64 位模式下，该指令的默认操作大小为 32 位。使用 REX.R 形式的 REX 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 形式的 REX 前缀可将操作提升至 64 位。请参阅本节开头的汇总图表以获取编码数据和限制。

操作 ¶ 

CF := Bit(BitBase, BitOffset);
Bit(BitBase, BitOffset) := 1;

受影响的标志位 ¶

CF 标志包含被置位前选定位的值。ZF 标志不受影响。OF、SF、AF 和 PF 标志位未定义。

保护模式异常 ¶

#GP(0) 如果目的操作数指向非可写段。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含 NULL 段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀，但目的地不是内存操作数。

实地址模式异常 ¶

#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数的有效地址超出 SS 段限制。
#UD 如果使用了 LOCK 前缀，但目的地不是内存操作数。

虚拟-8086 模式异常 ¶

#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数的有效地址超出 SS 段限制。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀，但目的地不是内存操作数。

兼容模式异常 ¶

与保护模式下的异常相同。
