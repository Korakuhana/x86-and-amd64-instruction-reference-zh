BOUND — 检查数组索引是否超出边界

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
62 /r BOUND r16, m16&16 RM Invalid Valid 检查 r16（数组索引）是否在 m16&16 指定的边界内。
62 /r BOUND r32, m32&32 RM Invalid Valid 检查 r32（数组索引）是否在 m32&32 指定的边界内。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) N/A N/A

描述¶

BOUND 指令确定第一个操作数（数组索引）是否在第二个操作数（边界操作数）指定的数组边界内。数组索引是位于寄存器中的有符号整数。边界操作数是一个内存位置，包含一对有符号的双字整数（当操作数大小属性为 32 时）或一对有符号的字整数（当操作数大小属性为 16 时）。第一个双字（或字）是数组的下界，第二个双字（或字）是数组的上界。数组索引必须大于或等于下界，且小于或等于上界。如果索引不在边界内，则会发出 BOUND 范围超出异常（#BR）。当产生此异常时，保存的返回指令指针指向该 BOUND 指令。

边界限制数据结构（包含数组下限和上限的两个字或双字）通常放置在数组本身之前，使得限制可以通过距数组开头的恒定偏移量来寻址。因为数组的地址已经存在于寄存器中，所以这种做法避免了获取数组边界有效地址的额外总线周期。

此指令在兼容模式和传统模式下如上所述执行。在 64 位模式下无效。

操作¶

IF 64bit Mode
    THEN
        #UD;
    ELSE
        IF (ArrayIndex < LowerBound OR ArrayIndex > UpperBound) THEN
        (* Below lower bound or above upper bound *)
            IF <equation for PL enabled> THEN BNDSTATUS := 0
            #BR;
        FI;
FI;

受影响的标志位¶

无。

保护模式异常¶

#BR 如果边界测试失败。
#UD 如果第二个操作数不是内存位置。
如果使用了 LOCK 前缀。
#GP(0) 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
如果 DS、ES、FS 或 GS 寄存器包含空段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段限制。
#PF(fault-code) 如果发生页错误。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。

#BR 如果边界测试失败。
#UD 如果第二个操作数不是内存位置。
如果使用了 LOCK 前缀。
#GP 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。
#SS 如果内存操作数的有效地址超出 SS 段限制。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#UD 如果处于 64 位模式。
