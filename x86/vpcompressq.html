VPCOMPRESSQ — 将稀疏打包的四字整数值存储到紧凑内存/寄存器中

操作码/指令 Op/En 64/32位模式支持 CPUID 功能标志 描述
EVEX.128.66.0F38.W1 8B /r VPCOMPRESSQ xmm1/m128 {k1}{z}, xmm2 A V/V AVX512VL AVX512F 在控制掩码 k1 下，将 xmm2 中的打包四字整数值压缩存储到 xmm1/m128。
EVEX.256.66.0F38.W1 8B /r VPCOMPRESSQ ymm1/m256 {k1}{z}, ymm2 A V/V AVX512VL AVX512F 在控制掩码 k1 下，将 ymm2 中的打包四字整数值压缩存储到 ymm1/m256。
EVEX.512.66.0F38.W1 8B /r VPCOMPRESSQ zmm1/m512 {k1}{z}, zmm2 A V/V AVX512F 在控制掩码 k1 下，将 zmm2 中的打包四字整数值压缩存储到 zmm1/m512。

指令操作数编码¶

Op/En 元组类型 操作数 1 操作数 2 操作数 3 操作数 4
A Tuple1 Scalar ModRM:r/m (w) ModRM:reg (r) N/A N/A

描述¶

将源操作数（第二操作数）中的最多 8/4/2 个四字整数值压缩（存储）到目的操作数（第一操作数）中。源操作数是 ZMM/YMM/XMM 寄存器，目的操作数可以是 ZMM/YMM/XMM 寄存器或 512/256/128 位内存位置。

opmask 寄存器 k1 从源操作数中选择活动元素（如果少于 8 个活动元素，则为部分向量，可能非连续），以压缩为连续向量。该连续向量从目的操作数的低位元素开始写入目的地。

内存目的版本：只有连续的向量被写入目的内存位置。EVEX.z 必须为零。

寄存器目的版本：如果连续向量的向量长度小于源操作数中输入向量的向量长度，且未设置 EVEX.z，则目的寄存器的高位不会被修改，否则高位清零。

注意：EVEX.vvvv 是保留的，必须为 1111b，否则指令会导致 #UD。

注意，压缩位移假设对应的预缩放比例（N）为单个元素的大小，而不是整个向量的大小。

操作¶

VPCOMPRESSQ (EVEX 编码版本) 存储形式¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
SIZE := 64
k := 0
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no controlmask*
        THEN
            DEST[k+SIZE-1:k] := SRC[i+63:i]
            k := k + SIZE
    FI;
ENFOR

VPCOMPRESSQ (EVEX 编码版本) 寄存器-寄存器形式¶

(KL, VL) = (2, 128), (4, 256), (8, 512)
SIZE := 64
k := 0
FOR j := 0 TO KL-1
    i := j * 64
    IF k1[j] OR *no controlmask*
        THEN
                DEST[k+SIZE-1:k] := SRC[i+63:i]
                k := k + SIZE
    FI;
ENDFOR
IF *merging-masking*
            THEN *DEST[VL-1:k] remains unchanged*
            ELSE DEST[VL-1:k] := 0
FI
DEST[MAXVL-1:VL] := 0
