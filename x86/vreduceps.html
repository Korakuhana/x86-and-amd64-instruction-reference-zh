描述¶

对源操作数（第二操作数）中的打包二进制编码单精度浮点值执行归约转换，并在写掩码 k1 控制下，将归约后的二进制浮点格式结果存储在目的操作数（第一操作数）中。

归约转换从二进制浮点源值中减去整数部分和前导 M 个尾数位，其中 M 是由 imm8[7:4] 指定的无符号整数，参见 图 5-28。具体而言，归约转换可以表示为：

dest = src – (ROUND(2M*src))*2-M;

其中 “Round()” 将 “src”、“2M” 以及它们的乘积视为具有规范化有效位数和偏置指数的二进制浮点数。

归约结果的幅度可以通过令 src = 2p*man2 来表示，

其中 'man2' 是规范化的有效位数，'p' 是无偏指数。

如果 RC = RNE：0 <=|归约结果| <= 2p-M-1

如果 RC ≠ RNE：0 <=|归约结果| < 2p-M

此指令可能会导致精度异常置位。然而，在设置了 SPE（即精度异常抑制，即 imm8[3]=1）的情况下，将不报告精度异常。

EVEX.vvvv 是保留的，必须为 1111b，否则指令将引发 #UD。

输入值的特殊情况处理列在 表 5-29 中。

操作¶

ReduceArgumentSP(SRC[31:0], imm8[7:0])
{
    // Check for NaN
    IF (SRC [31:0] = NAN) THEN
        RETURN (Convert SRC[31:0] to QNaN); FI
    M := imm8[7:4]; // Number of fraction bits of the normalized significand to be subtracted
    RC := imm8[1:0];// Round Control for ROUND() operation
    RC source := imm[2];
    SPE := imm[3];// Suppress Precision Exception
    TMP[31:0] := 2-M *{ROUND(2M*SRC[31:0], SPE, RC_source, RC)}; // ROUND() treats SRC and 2M as standard binary FP values
    TMP[31:0] := SRC[31:0] – TMP[31:0]; // subtraction under the same RC,SPE controls
    RETURN TMP[31:0]; // binary encoded FP with biased exponent and normalized significand
}

VREDUCEPS (EVEX 编码版本)¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask* THEN
            IF (EVEX.b == 1) AND (SRC *is memory*)
                THEN DEST[i+31:i] := ReduceArgumentSP(SRC[31:0], imm8[7:0]);
                ELSE DEST[i+31:i] := ReduceArgumentSP(SRC[i+31:i], imm8[7:0]);
            FI;
    ELSE
        IF *merging-masking* ; merging-masking
            THEN *DEST[i+31:i] remains unchanged*
            ELSE ; zeroing-masking
                DEST[i+31:i] = 0
        FI;
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0

SIMD 浮点异常¶

无效, 精度。

如果启用了 SPE，则不报告精度异常（无论 MXCSR 异常掩码如何）。
