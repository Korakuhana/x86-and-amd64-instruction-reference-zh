描述¶

对源操作数（第二个操作数）和目的操作数（第一个操作数）的打包无符号整数进行 SIMD 加法，并将打包整数结果存储在目的操作数中。有关 SIMD 操作的图示，请参阅 Intel® 64 和 IA-32 架构软件开发人员手册第 1 卷中的图 9-4。溢出通过无符号饱和处理，如下文所述。

(V)PADDUSB 执行第一个源操作数和第二个源操作数的打包无符号字节整数的 SIMD 饱和加法，并将打包整数结果存储在目的操作数中。当单个字节结果超出无符号字节整数的范围（即大于 FFH）时，将饱和值 FFH 写入目的操作数。

(V)PADDUSW 执行第一个源操作数和第二个源操作数的打包无符号字整数的 SIMD 饱和加法，并将打包整数结果存储在目的操作数中。当单个字结果超出无符号字整数的范围（即大于 FFFFH）时，将饱和值 FFFFH 写入目的操作数。

EVEX 编码版本：第一个源操作数是 ZMM/YMM/XMM 寄存器。第二个源操作数是 ZMM/YMM/XMM 寄存器或 512/256/128 位内存位置。目的操作数是 ZMM/YMM/XMM 寄存器。

VEX.256 编码版本：第一个源操作数是 YMM 寄存器。第二个源操作数是 YMM 寄存器或 256 位内存位置。目的操作数是 YMM 寄存器。

VEX.128 编码版本：第一个源操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。目的操作数是 XMM 寄存器。对应目的寄存器目的操作数的高位（MAXVL-1:128）清零。

128位传统 SSE 版本：第一个源操作数是 XMM 寄存器。第二个操作数可以是 XMM 寄存器或 128 位内存位置。目的操作数与第一个源 XMM 寄存器相同，且对应寄存器目的操作数的高位（MAXVL-1:128）保持不变。

VPADDUSB（EVEX 编码版本）¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
FOR j := 0 TO KL-1
    i := j * 8
    IF k1[j] OR *no writemask*
        THEN DEST[i+7:i] := SaturateToUnsignedByte (SRC1[i+7:i] + SRC2[i+7:i])
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+7:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+7:i] = 0
            FI
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0

(KL, VL) = (8, 128), (16, 256), (32, 512)
FOR j := 0 TO KL-1
    i := j * 16
    IF k1[j] OR *no writemask*
        THEN DEST[i+15:i] := SaturateToUnsignedWord (SRC1[i+15:i] + SRC2[i+15:i])
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+15:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+15:i] = 0
            FI
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0
