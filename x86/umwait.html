描述¶

UMWAIT 指示处理器在监视地址范围时进入与实现相关的优化状态。该优化状态可以是轻量级功耗/性能优化状态，也可以是改进的功耗/性能优化状态。两种状态之间的选择由显式输入寄存器的 bit[0] 控制。

当 CPUID.7.0:ECX.WAITPKG[bit 5] 枚举为 1 时，UMWAIT 可用。UMWAIT 可以在任何特权级执行。此指令在非 64 位模式和 64 位模式下的操作相同。

输入寄存器包含处理器应进入的首选优化状态等信息，如下表所述。除 bit 0 以外的其他位均保留，如果为非零值，将导致 #GP 异常。

当时间戳计数器达到或超过隐式 EDX:EAX 64 位输入值时，指令执行唤醒（如果监视硬件未提前触发）。

在执行 UMWAIT 指令之前，操作系统可以指定其允许处理器暂停运行的最大延迟。它可以通过向以下 32 位 MSR（MSR 索引为 E1H 的 IA32_UMWAIT_CONTROL）写入 TSC 份量值来实现：

  • IA32_UMWAIT_CONTROL[31:2] — 确定处理器可以驻留在 C0.1 或 C0.2 中的最大时间（以 TSC 份量为单位）。零值表示没有最大时间。最大时间值是一个 32 位值，其中高 30 位来自此字段，低两位为零。
  • IA32_UMWAIT_CONTROL[1] — 保留。
  • IA32_UMWAIT_CONTROL[0] — 操作系统不允许 C0.2。值“1”表示所有 C0.2 请求都恢复为 C0.1。

如果执行 UMWAIT 指令的处理器由于操作系统时间限制过期而唤醒，则指令设置 RFLAGS.CF；否则，清除该标志。

在事务区域内部使用 UMWAIT 指令会导致事务中止。

UMWAIT 指令与 UMONITOR 指令配合使用。这两条指令允许定义要等待的地址 (UMONITOR) 以及在等待时执行与实现相关的优化操作 (UMWAIT)。执行 UMWAIT 是对处理器的提示，即在等待由 UMONITOR 准备好的地址范围发生事件或存储写入操作时，它可以进入与实现相关的优化状态。如果在 UMWAIT 之前且在最后一次执行 UMONITOR 之后执行了以下任一指令，则 UMWAIT 指令不会等待（不会进入与实现相关的优化状态）：IRET、MONITOR、SYSEXIT、SYSRET 和远（far）RET（如果是后者更改了 CPL）。

以下附加事件会导致处理器退出与实现相关的优化状态：向由 UMONITOR 指令准备好的地址范围进行存储写入、NMI 或 SMI、调试异常、机器检查异常、BINIT# 信号、INIT# 信号以及 RESET# 信号。其他与实现相关的事件也可能导致处理器退出与实现相关的优化状态。

此外，外部中断会导致处理器退出与实现相关的优化状态，而不管可屏蔽中断是否被禁止 (EFLAGS.IF = 0)。

在退出与实现相关的优化状态之后，控制权转移到 UMWAIT 之后的指令。在执行该指令之前，可以交付尚未屏蔽的悬挂（pending）中断（包括 NMI 或 SMI）。

与 HLT 指令不同，UMWAIT 指令在处理完 SMI 之后不会重新在 UMWAIT 指令处重新启动。

If the preceding UMONITOR instruction did not successfully arm an address range or if UMONITOR was not executed prior to executing UMWAIT and following the most recent execution of the legacy MONITOR instruction (UMWAIT does not interoperate with MONITOR), then the processor will not enter an optimized state. Execution will continue to the instruction following UMWAIT.

如果向 UMONITOR 准备好的地址范围进行的存储写入是由其他处理器代理发起的，或者是由非处理器代理发起的，都将导致处理器退出 UMWAIT 状态。

操作¶

os_deadline := TSC+(IA32_UMWAIT_CONTROL[31:2]<<2)
instr_deadline := UINT64(EDX:EAX)
IF os_deadline < instr_deadline:
    deadline := os_deadline
    using_os_deadline := 1
ELSE:
    deadline := instr_deadline
    using_os_deadline := 0
WHILE monitor hardware armed AND TSC < deadline:
    implementation_dependent_optimized_state(Source register, deadline, IA32_UMWAIT_CONTROL[0] )
IF using_os_deadline AND TSC ≥ deadline:
    RFLAGS.CF := 1
ELSE:
    RFLAGS.CF := 0
RFLAGS.AF,PF,SF,ZF,OF := 0
