PCMPISTRI — 比较打包隐式长度字符串，返回索引

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
66 0F 3A 63 /r imm8 PCMPISTRI xmm1, xmm2/m128, imm8 RM V/V SSE4_2 对隐式长度的字符串数据进行打包比较，生成一个索引，并将结果存储在 ECX 中。
VEX.128.66.0F3A.WIG 63 /r ib VPCMPISTRI xmm1, xmm2/m128, imm8 RM V/V AVX 对隐式长度的字符串数据进行打包比较，生成一个索引，并将结果存储在 ECX 中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
RM ModRM:reg (r) ModRM:r/m (r) imm8 N/A

描述¶

此指令根据 imm8 控制字节中的编码值（请参阅第 4.1 节，“PCMPESTRI / PCMPESTRM / PCMPISTRI / PCMPISTRM 的 imm8 控制字节操作”）比较来自两个字符串的数据，并生成存储到 ECX 的索引。

每个字符串由单个值表示。该值是 xmm（对于第二个操作数，可能是 m128），包含字符串的数据元素（字节或字数据）。每个输入字节/字都被标记为有效/无效。只有当字节/字的索引低于最低有效空（null）字节/字时，它才被认为是有效的。（最低有效空字节/字也被认为是无效的。）

根据 imm8 位字段的编码值（见第 4.1 节）执行比较和聚合操作。IntRes2 的第一个（或根据 imm8[6] 为最后一个）置位位的索引返回到 ECX 中。如果 IntRes2 中没有置位位，则 ECX 设置为 16 (8)。

注意，写入算术标志位的方式是非标准的，以便提供最相关的信息：

CFlag – 如果 IntRes2 等于零则重置，否则置位

ZFlag – 如果 xmm2/mem128 的任何字节/字为空（null）则置位，否则重置

SFlag – 如果 xmm1 的任何字节/字为空（null）则置位，否则重置

OFlag – IntRes2[0]

AFlag – 重置

PFlag – 重置

注意：在 VEX.128 编码版本中，VEX.vvvv 是保留的，必须为 1111b，VEX.L 必须为 0，否则指令将导致 #UD。

有效操作数大小¶

操作模式/大小 操作数 1 操作数 2 结果
16 位 xmm xmm/m128 ECX
32 位 xmm xmm/m128 ECX
64 位 xmm xmm/m128 ECX

用于返回索引的 Intel C/C++ 编译器等价内部函数¶

int _mm_cmpistri (__m128i a, __m128i b, const int mode);

用于读取 EFlag 结果的 Intel C/C++ 编译器内部函数¶

int _mm_cmpistra (__m128i a, __m128i b, const int mode);

int _mm_cmpistrc (__m128i a, __m128i b, const int mode);

int _mm_cmpistro (__m128i a, __m128i b, const int mode);

int _mm_cmpistrs (__m128i a, __m128i b, const int mode);

int _mm_cmpistrz (__m128i a, __m128i b, const int mode);

SIMD 浮点异常¶

无。

其他异常¶

请参阅表 2-21，“第 4 类异常条件”，此外，如果内存操作数未对齐到 16 字节边界，此指令不会导致 #GP，并且：

#UD 如果 VEX.L = 1。
如果 VEX.vvvv ≠ 1111B。
