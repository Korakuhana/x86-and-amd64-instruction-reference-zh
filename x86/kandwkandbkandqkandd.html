KANDW/KANDB/KANDQ/KANDD — 按位逻辑与掩码

操作码/指令 Op/En 64/32位模式支持 CPUID 特征标志 描述
VEX.L1.0F.W0 41 /r KANDW k1, k2, k3 RVR V/V AVX512F 对 k2 和 k3 中的 16 位掩码进行按位与，并将结果放入 k1。
VEX.L1.66.0F.W0 41 /r KANDB k1, k2, k3 RVR V/V AVX512DQ 对 k2 和 k3 中的 8 位掩码进行按位与，并将结果放入 k1。
VEX.L1.0F.W1 41 /r KANDQ k1, k2, k3 RVR V/V AVX512BW 对 k2 和 k3 中的 64 位掩码进行按位与，并将结果放入 k1。
VEX.L1.66.0F.W1 41 /r KANDD k1, k2, k3 RVR V/V AVX512BW 对 k2 和 k3 中的 32 位掩码进行按位与，并将结果放入 k1。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3
RVR ModRM:reg (w) VEX.1vvv (r) ModRM:r/m (r, ModRM:[7:6] 必须是 11b)

描述¶

在向量掩码 k2 和向量掩码 k3 之间执行按位与操作，并将结果写入向量掩码 k1。

操作¶

KANDW¶

DEST[15:0] := SRC1[15:0] BITWISE AND SRC2[15:0]
DEST[MAX_KL-1:16] := 0

KANDB¶

DEST[7:0] := SRC1[7:0] BITWISE AND SRC2[7:0]
DEST[MAX_KL-1:8] := 0

KANDQ¶

DEST[63:0] := SRC1[63:0] BITWISE AND SRC2[63:0]
DEST[MAX_KL-1:64] := 0

KANDD¶

DEST[31:0] := SRC1[31:0] BITWISE AND SRC2[31:0]
DEST[MAX_KL-1:32] := 0
