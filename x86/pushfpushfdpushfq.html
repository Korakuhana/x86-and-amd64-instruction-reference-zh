PUSHF/PUSHFD/PUSHFQ — 将 EFLAGS 寄存器压入栈

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
9C PUSHF ZO Valid Valid 将 EFLAGS 的低 16 位压入栈中。
9C PUSHFD ZO N.E. Valid 将 EFLAGS 压入栈中。
9C PUSHFQ ZO Valid N.E. 将 RFLAGS 压入栈中。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将栈指针减 4（如果当前操作数大小属性为 32），并将整个 EFLAGS 寄存器的内容压入栈中；或者将栈指针减 2（如果操作数大小属性为 16），并将 EFLAGS 寄存器的低 16 位（即 FLAGS 寄存器）压入栈中。这些指令的作用与 POPF/POPFD 指令相反。

将整个 EFLAGS 寄存器复制到栈中时，不复制 VM 和 RF 标志（第 16 和 17 位）；相反，在存储在栈上的 EFLAGS 镜像中清除这些标志的值。有关 EFLAGS 寄存器的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 1 卷的第 3 章。

助记符 PUSHF（压入标志）和 PUSHFD（双字压入标志）引用相同的操作码。PUSHF 指令适用于操作数大小属性为 16 的情况，而 PUSHFD 指令适用于操作数大小属性为 32 的情况。某些汇编器可能会在下述情况下强制确定操作数大小：使用 PUSHF时为 16，使用 PUSHFD时为 32。其他汇编器可能将这些助记符视为同义词（PUSHF/PUSHFD），并使用操作数大小属性的当前设置来确定要从栈中压入的值的大小，而不考虑所使用的助记符。

在 64 位模式下，该指令的默认操作是将栈指针（RSP）减 8 并将 RFLAGS 压入栈。使用操作数大小覆盖前缀 66H 支持 16 位操作。在此模式下，无法编码 32 位操作数大小。将 RFLAGS 复制到栈中时，不复制 VM 和 RF 标志（第 16 和 17 位）；相反，在存储在栈上的 RFLAGS 镜像中清除这些标志的值。

在没有虚拟 8086 模式扩展（CR4.VME = 0）的情况下在虚拟 8086 模式（EFLAGS.VM = 1）下运行时，只有在 IOPL = 3 时才能使用 PUSHF/PUSHFD 指令；否则会发生一般保护性异常（#GP）。如果启用了虚拟 8086 模式扩展（CR4.VME = 1），则可以在 IOPL < 3 的虚拟 8086 模式下执行 PUSHF（但不能执行 PUSHFD）。

（通过设置 CR4.PVI 启用的保护模式虚拟中断功能与虚拟 8086 模式扩展以相同的方式影响 CLI 和 STI 指令。然而，PUSHF 不受 CR4.PVI 的影响。）

在实地址模式下，如果执行 PUSHF/PUSHFD 指令时 ESP 或 SP 寄存器为 1：将产生 #SS 异常，但不会传送（报告的栈错误会阻止 #SS 传送）。接下来，处理器产生 #DF 异常并进入关机状态，如《Intel® 64 和 IA-32 架构软件开发人员手册》第 3A 卷第 6 章中有关 #DF 的讨论所述。

操作¶

IF (PE = 0) or (PE = 1 and ((VM = 0) or (VM = 1 and IOPL = 3)))
(* Real-Address Mode, Protected mode, or Virtual-8086 mode with IOPL equal to 3 *)
    THEN
        IF OperandSize = 32
            THEN
                push (EFLAGS AND 00FCFFFFH);
                (* VM and RF bits are cleared in image stored on the stack *)
            ELSE
                push (EFLAGS); (* Lower 16 bits only *)
        FI;
    ELSE IF 64-bit MODE (* In 64-bit Mode *)
        IF OperandSize = 64
            THEN
                push (RFLAGS AND 00000000_00FCFFFFH);
                (* VM and RF bits are cleared in image stored on the stack; *)
            ELSE
                push (EFLAGS); (* Lower 16 bits only *)
        FI;
    ELSE (* In Virtual-8086 Mode with IOPL less than 3 *)
        IF (CR4.VME = 0) OR (OperandSize = 32)
            THEN #GP(0); (* Trap to virtual-8086 monitor *)
            ELSE
                tempFLAGS = EFLAGS[15:0];
                tempFLAGS[9] = tempFLAGS[19]; (* VIF replaces IF *)
                tempFlags[13:12]=3; (*IOPL is set to 3 in image stored on the stack*)
                push (tempFLAGS);
        FI;
FI;

影响的标志位¶

无。

保护模式异常¶

#SS(0) 如果 ESP 寄存器的新值超出栈段边界。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且在 CPL = 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#GP(0) 如果 I/O 特权级别小于 3。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#SS(0) 如果栈地址是非规范形式。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且在 CPL = 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。
