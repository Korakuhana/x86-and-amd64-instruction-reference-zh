描述¶

PMULHRSW 将目的操作数（第一个操作数）的每个有符号 16 位整数与源操作数（第二个操作数）的对应有符号 16 位整数垂直相乘，产生中间的、有符号的 32 位整数。每个中间 32 位整数被截断为 18 个最高有效位。始终通过向 18 位中间结果的最低有效位加 1 来执行舍入。最终结果是通过选择每个 18 位中间结果的最高有效位右侧紧邻的 16 位并打包到目的操作数中来获得的。

当源操作数是 128 位内存操作数时，该操作数必须对齐在 16 字节边界上，否则将产生通用保护异常（#GP）。

在 64 位模式下，如果未采用 VEX/EVEX 编码，请使用 REX 前缀访问 XMM8-XMM15 寄存器。

传统 SSE 版本 64 位操作数：两个操作数都可以是 MMX 寄存器。第二个源操作数是 MMX 寄存器或 64 位内存位置。

128位传统 SSE 版本：第一个源操作数和目的操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：第一个源操作数 and 目的操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。目的 YMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：第二个源操作数可以是 YMM 寄存器或 256 位内存位置。第一个源操作数和目的操作数是 YMM 寄存器。

EVEX 编码版本：第一个源操作数是 ZMM/YMM/XMM 寄存器。第二个源操作数可以是 ZMM/YMM/XMM 寄存器、512/256/128 位内存位置。目的操作数是 ZMM/YMM/XMM 寄存器，并根据写掩码 k1 条件性地更新。

VPMULHRSW（EVEX 编码版本）¶

(KL, VL) = (8, 128), (16, 256), (32, 512)
FOR j := 0 TO KL-1
    i := j * 16
    IF k1[j] OR *no writemask*
        THEN
            temp[31:0] := ((SRC1[i+15:i] * SRC2[i+15:i]) >>14) + 1
            DEST[i+15:i] := tmp[16:1]
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+15:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+15:i] := 0
            FI
    FI;
ENDFOR
DEST[MAXVL-1:VL] := 0
