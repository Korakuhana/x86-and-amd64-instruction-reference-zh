INC — 加 1

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
FE /0 INC r/m8 M Valid Valid 将 r/m 字节加 1。
REX + FE /0 INC r/m81 M Valid N.E. 将 r/m 字节加 1。
FF /0 INC r/m16 M Valid Valid 将 r/m 字加 1。
FF /0 INC r/m32 M Valid Valid 将 r/m 双字加 1。
REX.W + FF /0 INC r/m64 M Valid N.E. 将 r/m 四字加 1。
40+ rw2 INC r16 O N.E. Valid 将字寄存器加 1。
40+ rd INC r32 O N.E. Valid 将双字寄存器加 1。

1. 在 64 位模式下，如果使用了 REX 前缀，则不能通过编码 r/m8 来访问以下字节寄存器：AH、BH、CH、DH。

2. 在 64 位模式下，40H 至 47H 被用作 REX 前缀。

描述 ¶

将目的操作数加 1，同时保留 CF 标志的状态。目的操作数可以是一个寄存器或一个内存位置。该指令允许在不干扰 CF 标志的情况下更新循环计数器。（若要执行更新 CF 标志的加 1 操作，请使用立即数为 1 的 ADD 指令。）

该指令可以与 LOCK 前缀一起使用，以便以原子方式执行该指令。

在 64 位模式下，INC r16 和 INC r32 无法编码（因为操作码 40H 至 47H 被用作 REX 前缀）。否则，该指令在 64 位模式下的默认操作大小为 32 位。使用 REX.R 前缀允许访问额外的寄存器（R8-R15）。使用 REX.W 前缀可将操作提升至 64 位。

操作 ¶ 

DEST := DEST + 1;

受影响的标志位 ¶

CF 标志位不受影响。根据结果设置 OF、SF、ZF、AF 和 PF 标志位。

保护模式异常 ¶

#GP(0) 如果目的操作数位于不可写的段中。
如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段界限。
如果使用 DS、ES、FS 或 GS 寄存器访问内存且它包含 NULL段选择子。
#SS(0) 如果内存操作数的有效地址超出 SS 段界限。
#PF(错误码) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查，并且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀但目标操作数不是内存操作数。
