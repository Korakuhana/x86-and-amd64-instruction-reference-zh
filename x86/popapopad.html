POPA/POPAD — 将所有通用寄存器出栈

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
61 POPA ZO Invalid Valid 弹出 DI, SI, BP, BX, DX, CX, 以及 AX。
61 POPAD ZO Invalid Valid 弹出 EDI, ESI, EBP, EBX, EDX, ECX, 以及 EAX。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

从栈中弹出双字（POPAD）或字（POPA）到通用寄存器中。寄存器的加载顺序如下：EDI、ESI、EBP、EBX、EDX、ECX、EAX（如果操作数大小属性为 32）；以及 DI、SI、BP、BX、DX、CX、AX（如果操作数大小属性为 16）。（这些指令执行与 PUSHA/PUSHAD 指令相反的操作。）栈中用于 ESP 或 SP 寄存器的值被忽略。相反，每次加载寄存器后都会递增 ESP 或 SP 寄存器。

助记符 POPA（全部出栈）和 POPAD（全部双字出栈）指向同一个操作码。POPA 指令旨在操作数大小属性为 16 时使用，而 POPAD 指令旨在操作数大小属性为 32 时使用。某些汇编器可能会在下述情况下强制确定操作数大小：POPA 为 16，POPAD 为 32（必要时使用操作数大小覆盖前缀 [66H]）。其他汇编器可能将这些助记符视为同义词（POPA/POPAD），并使用操作数大小属性的当前设置来确定从栈中弹出的值的大小，而不管所使用的助记符如何。（当前代码段的段描述符中的 D 标志决定了操作数大小属性。）

此指令在非 64 位模式下的执行情况如上所述。它在 64 位模式下无效。

操作¶

IF 64-Bit Mode
    THEN
        #UD;
ELSE
    IF OperandSize = 32 (* Instruction = POPAD *)
    THEN
        EDI := Pop();
        ESI := Pop();
        EBP := Pop();
        Increment ESP by 4; (* Skip next 4 bytes of stack *)
        EBX := Pop();
        EDX := Pop();
        ECX := Pop();
        EAX := Pop();
    ELSE (* OperandSize = 16, instruction = POPA *)
        DI := Pop();
        SI := Pop();
        BP := Pop();
        Increment ESP by 2; (* Skip next 2 bytes of stack *)
        BX := Pop();
        DX := Pop();
        CX := Pop();
        AX := Pop();
    FI;
FI;

影响的标志位¶

无。

保护模式异常¶

#SS(0) 如果栈的起始或结束地址不在栈段边界内。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且在当前特权级别为 3 时进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

实地址模式异常¶

#SS 如果栈的起始或结束地址不在栈段边界内。
#UD 如果使用了 LOCK 前缀。

虚拟 8086 模式异常¶

#SS(0) 如果栈的起始或结束地址不在栈段边界内。
#PF(fault-code) 如果发生缺页异常。
#AC(0) 如果启用了对齐检查且进行了未对齐的内存引用。
#UD 如果使用了 LOCK 前缀。

兼容模式异常¶

与保护模式下的异常相同。

64位模式异常¶

#UD 如果处于 64 位模式下。
