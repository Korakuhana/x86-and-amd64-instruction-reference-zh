描述¶

将目的操作数（第一个操作数）的各个有符号字与源操作数（第二个操作数）的对应有符号字相乘，产生临时的有符号双字结果。然后将相邻的双字结果相加并存储在目的操作数中。例如，源操作数和目的操作数中对应的低位字 (15-0) 和 (31-16) 相互相乘，双字结果相加并存储在目的寄存器的低双字中 (31-0)。对其他相邻的字对执行相同的操作。（图 4-11 显示了使用 64 位操作数时的此操作）。

(V)PMADDWD 指令仅在一种情况下发生折绕（wrap around）：当一组中被操作的 2 对字都是 8000H 时。在这种情况下，结果折绕为 80000000H。

在 64 位模式下且不使用 VEX/EVEX 编码时，使用 REX.R 形式的 REX 前缀允许此指令访问其他寄存器（XMM8-XMM15）。

传统 SSE 版本：第一个源操作数和目的操作数是 MMX 寄存器。第二个源操作数是 MMX 寄存器或 64 位内存位置。

128位传统 SSE 版本：第一个源操作数和目的操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。对应 YMM 目的寄存器的位（MAXVL-1:128）保持不变。

VEX.128 编码版本：第一个源操作数和目的操作数是 XMM 寄存器。第二个源操作数是 XMM 寄存器或 128 位内存位置。目的 YMM 寄存器的位（MAXVL-1:128）清零。

VEX.256 编码版本：第二个源操作数可以是 YMM 寄存器或 256 位内存位置。第一个源操作数和目的操作数是 YMM 寄存器。

EVEX.512 编码版本：第二个源操作数可以是 ZMM 寄存器或 512 位内存位置。第一个源操作数和目的操作数是 ZMM 寄存器。

DEST[31:0] := (DEST[15:0] ∗ SRC[15:0]) + (DEST[31:16] ∗ SRC[31:16]);
DEST[63:32] := (DEST[47:32] ∗ SRC[47:32]) + (DEST[63:48] ∗ SRC[63:48]);

VPMADDWD（EVEX 编码版本）¶

(KL, VL) = (4, 128), (8, 256), (16, 512)
FOR j := 0 TO KL-1
    i := j * 32
    IF k1[j] OR *no writemask*
        THEN DEST[i+31:i] := (SRC2[i+31:i+16]* SRC1[i+31:i+16]) + (SRC2[i+15:i]*SRC1[i+15:i])
        ELSE
            IF *merging-masking* ; merging-masking
                THEN *DEST[i+31:i] remains unchanged*
                ELSE *zeroing-masking*
                        ; zeroing-masking
                    DEST[i+31:i] = 0
            FI
    FI;
ENDFOR;
DEST[MAXVL-1:VL] := 0
