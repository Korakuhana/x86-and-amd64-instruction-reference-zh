RSM — 从系统管理模式恢复

操作码 指令 Op/En 64位模式 兼容/传统模式 描述
0F AA RSM ZO Valid Valid 恢复被中断程序的执行。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
ZO N/A N/A N/A N/A

描述¶

将程序控制权从系统管理模式 (SMM) 返回到处理器接收到 SMM 中断时被中断的应用程序或操作系统例程。处理器状态将从进入 SMM 时创建的转储中恢复。如果处理器在状态恢复期间检测到无效的状态信息，它将进入关机（shutdown）状态。以下无效信息可能导致关机：

  • CR4 的任何保留位被设置为 1。
  • CR0 中任何非法的位组合，例如 (PG=1 且 PE=0) 或 (NW=1 且 CD=0)。
  • （仅限 Intel Pentium 和 Intel486™ 处理器。）存储在状态转储基址字段中的值不是 32-KByte 对齐的地址。

模型特定寄存器 (MSR) 的内容不受从 SMM 返回的影响。

RSM 使用的 SMM 状态图支持恢复非 64 位模式和 64 位模式下的处理器上下文。

关于 SMM 以及 RSM 指令行为的更多信息，请参阅《Intel® 64 和 IA-32 架构软件开发人员手册》第 3C 卷第 32 章“系统管理模式”。

操作¶

ReturnFromSMM;
IF (IA-32e mode supported) or (CPUID DisplayFamily_DisplayModel = 06H_0CH )
    THEN
        ProcessorState := Restore(SMMDump(IA-32e SMM STATE MAP));
    Else
        ProcessorState := Restore(SMMDump(Non-32-Bit-Mode SMM STATE MAP));
FI

受影响的标志位¶

全部。

如果使用了 LOCK 前缀。
