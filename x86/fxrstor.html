FXRSTOR — 恢复 x87 FPU、MMX、XMM 和 MXCSR 状态

操作码/指令 Op/En 64位模式 兼容/传统模式 描述
NP 0F AE /1 FXRSTOR m512byte M Valid Valid 从 m512byte 恢复 x87 FPU、MMX、XMM 和 MXCSR 寄存器状态。
NP REX.W + 0F AE /1 FXRSTOR64 m512byte M Valid N.E. 从 m512byte 恢复 x87 FPU、MMX、XMM 和 MXCSR 寄存器状态。

指令操作数编码¶

Op/En 操作数 1 操作数 2 操作数 3 操作数 4
M ModRM:r/m (r) N/A N/A N/A

描述¶

从源操作数中指定的 512 字节内存映像中重新加载 x87 FPU、MMX 技术、XMM 和 MXCSR 寄存器。该数据之前应该已经使用 FXSAVE 指令写入内存，并且格式与运行模式要求的格式相同。数据的第一个字节应该位于 16 字节边界上。FXSAVE 状态图有三种不同的布局：一种用于传统和兼容模式，第二种格式用于具有 REX.W=0 的 64 位模式 FXSAVE/FXRSTOR，第三种格式用于具有 FXSAVE64/FXRSTOR64 的 64 位模式。表 3-43 显示了内存中传统/兼容模式状态信息的布局，并描述了 FXRSTOR 和 FXSAVE 指令的内存映像中的字段。表 3-46 显示了设置 REX.W 时（FXSAVE64/FXRSTOR64）64 位模式状态信息的布局。表 3-47 显示了清除 REX.W 时（FXSAVE/FXRSTOR）64 位模式状态信息的布局。

用 FXRSTOR 指令引用的状态映像必须已使用 FXSAVE 指令保存，或具有与表 3-43、表 3-46 或表 3-47 所要求的相同格式。引用使用 FSAVE、FNSAVE 指令保存的状态映像或不兼容的字段布局将导致错误的状态恢复。

FXRSTOR 指令不会刷新待处理的 x87 FPU 异常。要在使用 FXRSTOR 指令加载 x87 FPU 状态信息时检查并引发异常，请在 FXRSTOR 指令之后使用 FWAIT 指令。

如果未设置控制寄存器 CR4 中的 OSFXSR 位，则 FXRSTOR 指令可能不会恢复 XMM 和 MXCSR 寄存器的状态。此行为取决于具体实现。

如果 MXCSR 状态包含未屏蔽的异常，且相应的状态标志也已设置，则使用 FXRSTOR 指令加载该寄存器不会导致产生 SIMD 浮点错误条件。只有下一次发生该未屏蔽异常时才会导致产生异常。

MXCSR 寄存器的第 16 到 32 位定义为保留位，且应设置为 0。尝试从保存的状态映像中向这些位中的任何一位写入 1 将导致产生一般保护异常 (#GP)。

FXSAVE 映像的 464:511 字节可供软件使用。FXRSTOR 会忽略 FXSAVE 状态映像中 464:511 字节的内容。

操作¶

IF 64-Bit Mode
    THEN
        (x87 FPU, MMX, XMM15-XMM0, MXCSR)
                Load(SRC);
    ELSE
            (x87 FPU, MMX, XMM7-XMM0, MXCSR) := Load(SRC);
FI;

x87 FPU 和 SIMD 浮点异常¶

无。

对于尝试在 MXCSR 中设置保留位的行为。

#NM 如果 CR0.TS[bit 3] = 1。
如果 CR0.EM[bit 2] = 1。
#UD 如果 CPUID.01H:EDX.FXSR[bit 24] = 0。
如果指令前有 LOCK 前缀。

如果 CR0.EM[bit 2] = 1。
#UD 如果 CPUID.01H:EDX.FXSR[bit 24] = 0。

虚拟-8086 模式异常¶

与实地址模式下的异常相同。

#PF(错误码) 如果发生缺页异常。

兼容模式异常¶

与保护模式下的异常相同。
