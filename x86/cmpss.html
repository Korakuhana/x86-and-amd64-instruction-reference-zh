CMPSS — 比较标量单精度浮点值

操作码/指令 Op / En 64/32位模式支持 CPUID 特征标志 描述
F3 0F C2 /r ib CMPSS xmm1, xmm2/m32, imm8 A V/V SSE 使用 imm8 的 bits 2:0 作为比较关系谓词，比较 xmm2/m32 和 xmm1 中的低位单精度浮点值。
VEX.LIG.F3.0F.WIG C2 /r ib VCMPSS xmm1, xmm2, xmm3/m32, imm8 B V/V AVX 使用 imm8 的 bits 4:0 作为比较关系谓词，比较 xmm3/m32 和 xmm2 中的低位单精度浮点值。
EVEX.LLIG.F3.0F.W0 C2 /r ib VCMPSS k1 {k2}, xmm2, xmm3/m32{sae}, imm8 C V/V AVX512F 使用 imm8 的 bits 4:0 作为比较关系谓词，在使用写掩码 k2 的情况下比较 xmm3/m32 和 xmm2 中的低位单精度浮点值，并将结果保存在掩码寄存器 k1 中。

描述¶

比较第二源操作数和第一源操作数中的低位单精度浮点值，并将比较结果返回给目的操作数。比较关系谓词操作数（即立即数操作数）指定所执行的比较类型。

128 位传统 SSE 版本：第一源与目的操作数（第一操作数）是 XMM 寄存器。第二源操作数（第二操作数）可以是 XMM 寄存器或 32 位内存位置。对应的 YMM 目的寄存器的 bits (MAXVL-1:32) 保持不变。比较结果是全 1 的双字掩码（比较为真）或全 0（比较为假）。

VEX.128 编码版本：第一源操作数（第二操作数）是 XMM 寄存器。第二源操作数（第三操作数）可以是 XMM 寄存器或 32 位内存位置。结果存储在目的操作数的低 32 位中；目的操作数的 bits 127:32 从第一源操作数中复制。目的 ZMM 寄存器的 bits (MAXVL-1:128) 清零。比较结果是全 1 的双字掩码（比较为真）或全 0（比较为假）。

EVEX 编码版本：第一源操作数（第二操作数）是 XMM 寄存器。第二源操作数可以是 XMM 寄存器或 32 位内存位置。目的操作数（第一操作数）是 opmask（操作掩码）寄存器。比较结果是单个掩码位，1 表示比较为真，0 表示比较为假，根据写掩码 k2 从最低有效位（LSB）开始写入目的地。目的寄存器的 bits (MAX_KL-1:128) 清零。

比较关系谓词操作数是 8 位立即数：

  • 对于使用 VEX 前缀编码的指令，bits 4:0 定义了要执行的比较类型（参见表 3-1）。立即数的 bits 5 到 7 保留。
  • 对于未使用 VEX 前缀的指令编码，bits 2:0 定义了要进行的比较类型（参见表 3-1 的前 8 行）。立即数的 bits 3 到 7 保留。

当被比较的两个源操作数中至少有一个是 NaN 时，无序（unordered）关系为真；当两个源操作数均不是 NaN 时，有序（ordered）关系为真。

后续使用目的操作数中的掩码结果作为输入操作数的计算指令不会产生异常，因为全 0 掩码对应浮点值 +0.0，而全 1 掩码对应 QNaN。

请注意，具有 “CPUID.1H:ECX.AVX = 0” 的处理器未实现 “大于”、“大于或等于”、“不大于” 和 “不大于或等于” 关系谓词。这些比较可以通过使用相反关系（即使用 “不小于或等于” 来进行 “大于” 比较）或通过软件模拟来完成。使用软件模拟时，程序必须交换操作数（必要时复制寄存器以保护现在将存于目的寄存器中的数据），然后使用不同的谓词执行比较。要用于这些模拟的谓词列在《Intel® 64 和 IA-32 架构软件开发人员手册》第 2A 卷中表 3-7（位于 cli.html 对应章节）的前 8 行中的 Emulation 标题下。

编译器和汇编器除了三操作数 CMPSS 指令之外，对于具有 “CPUID.1H:ECX.AVX = 0” 的处理器，还可以实现以下双操作数伪操作。参见表 3-8。编译器应将保留的 imm8 值视为非法语法。

处理器未实现的大于关系在软件中需要多于一条指令来模拟，因此不应被实现为伪操作。（对于这些关系，程序员应当反转相应小于关系的操作数，并使用移动指令确保掩码被移动到正确的目的寄存器，且源操作数保持完整。）

具有 “CPUID.1H:ECX.AVX = 1” 的处理器实现了表 3-7 中所示的全部 32 个谓词，不再需要软件模拟。编译器和汇编器除了四操作数 VCMPSS 指令之外，还可以实现以下三操作数伪操作。参见表 3-9，其中 reg1、reg2 和 reg3 记号表示 XMM 寄存器或 YMM 寄存器。编译器应将保留的 imm8 值视为非法语法。或者，内部函数可以将伪操作映射到预定义常量，以支持更简单的内部函数接口。编译器和汇编器可以类似地通过扩展表 3-9 中列出的语法，为 EVEX 编码的 VCMPSS 指令实现三操作数伪操作。

软件应确保 VCMPSS 编码为 VEX.L=0。用 VEX.L=1 编码 VCMPSS 可能会在不同处理器代次之间遇到不可预测的行为。

    5: OP3 := NLT_US; OP5 := NLT_US;

    9: OP5 := NGE_US;
    10: OP5 := NGT_US;
    11: OP5 := FALSE_OQ;
    12: OP5 := NEQ_OQ;
    13: OP5 := GE_OS;
    14: OP5 := GT_OS;
    15: OP5 := TRUE_UQ;
    16: OP5 := EQ_OS;
    17: OP5 := LT_OQ;
    18: OP5 := LE_OQ;
    19: OP5 := UNORD_S;
    20: OP5 := NEQ_US;
    21: OP5 := NLT_UQ;
    22: OP5 := NLE_UQ;

VCMPSS (EVEX Encoded Version)¶

CMP0 := SRC1[31:0] OP5 SRC2[31:0];
IF k2[0] or *no writemask*
    THEN IF CMP0 = TRUE
        THEN DEST[0] := 1;
        ELSE DEST[0] := 0; FI;
    ELSE DEST[0] := 0
            ; zeroing-masking only
FI;
DEST[MAX_KL-1:1] := 0

CMPSS (128-bit Legacy SSE Version)¶

CMP0 := DEST[31:0] OP3 SRC[31:0];
IF CMP0 = TRUE
THEN DEST[31:0] := FFFFFFFFH;
ELSE DEST[31:0] := 00000000H; FI;
DEST[MAXVL-1:32] (Unmodified)
