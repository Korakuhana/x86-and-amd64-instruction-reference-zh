描述¶

根据写掩码操作数确定的活动元素，将源操作数（第二操作数）中的最多 64 个字节值或 32 个字值压缩（存储）到目的操作数（第一操作数）中。注意：EVEX.vvvv 是保留的，必须为 1111b，否则指令会导致 #UD。

将源操作数（第二操作数）中的最多 512 位打包字节值移动到目的操作数（第一操作数）中。此指令用于在使用操作数写掩码中的活动元素时，将向量寄存器的部分内容存储到字节向量或单个内存位置中。

内存目的版本：只有连续的向量被写入目的内存位置。EVEX.z 必须为零。

寄存器目的版本：如果连续向量的向量长度小于源操作数中输入向量的向量长度，且未设置 EVEX.z，则目的寄存器的高位不会被修改，否则高位清零。

此指令支持内存故障抑制。

注意，压缩位移假设对应的预缩放比例（N）为单个元素的大小，而不是整个向量的大小。

操作¶

VPCOMPRESSB 存储形式¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
k := 0
FOR j := 0 TO KL-1:
    IF k1[j] OR *no writemask*:
        DEST.byte[k] := SRC.byte[j]
        k := k +1

VPCOMPRESSB 寄存器-寄存器形式¶

(KL, VL) = (16, 128), (32, 256), (64, 512)
k := 0
FOR j := 0 TO KL-1:
    IF k1[j] OR *no writemask*:
        DEST.byte[k] := SRC.byte[j]
        k := k + 1
IF *merging-masking*:
    *DEST[VL-1:k*8] remains unchanged*
    ELSE DEST[VL-1:k*8] := 0
DEST[MAX_VL-1:VL] := 0

VPCOMPRESSW 存储形式¶

(KL, VL) = (8, 128), (16, 256), (32, 512)
k := 0
FOR j := 0 TO KL-1:
    IF k1[j] OR *no writemask*:
        DEST.word[k] := SRC.word[j]
        k := k + 1

VPCOMPRESSW 寄存器-寄存器形式¶

(KL, VL) = (8, 128), (16, 256), (32, 512)
k := 0
FOR j := 0 TO KL-1:
    IF k1[j] OR *no writemask*:
